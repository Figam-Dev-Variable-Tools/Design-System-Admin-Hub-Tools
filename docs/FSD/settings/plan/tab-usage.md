# 플랜·이용 현황 — 이용현황 탭

## 1. 화면 개요

| 항목 | 내용 |
|---|---|
| 화면명 | 이용현황 (탭) |
| 화면 ID | `TAB-SETTINGS-PLAN-USAGE` |
| 메뉴 경로 | 시스템 설정 > 플랜·이용 현황 > 이용현황 |
| 작성자 | Admin Hub 문서화 |
| 작성일 | 2026-07-22 |
| 버전 | 1.0 |

**화면 목적** — 잠금을 만난 운영자의 두 질문에 답한다: **지금 무슨 플랜인가** · **그래서 무엇이 열리고 무엇이 잠겼나**.

**업무 배경** — 사이드바의 어떤 항목에는 잠금 꼬리표가 붙고, 어떤 항목은 아예 보이지 않는다. 그 차이가 무엇인지, 자기가 무엇을 더 사야 그것이 열리는지 화면 어딘가는 답해야 한다. 답이 없으면 운영자는 켜 줄 수 없는 관리자에게 권한을 요청한다.

**화면 설명** — 카드 둘이다. 위는 현재 플랜(플랜 이름 배지 · 플랜 등급 · 변경 적용 예정 · 사내 홈페이지 링크), 아래는 포함 기능 표(기능 · 종류 · 상태 · 사용량·수준 네 열)다. **이 탭은 읽기만 한다.**

**주요 사용자** — 최상위 관리자 · 이 화면의 조회 권한을 가진 운영자.

**사용 시나리오**

1. 쿠폰 메뉴에 자물쇠가 붙었다 → 표에서 `쿠폰` 행의 상태가 `프로 플랜부터` 인 것을 보고 사내 홈페이지로 나간다.
2. 상품이 '포함'인데 새 상품이 등록되지 않는다 → 표의 `종류` 열이 `한도` 이고 사용량이 `상품 관리 200/200` 인 것을 보고 이유를 안다.
3. 판매하지 않는 모듈은 표에 아예 없다 → 그것을 찾아 헤매지 않는다.

**선행 조건** — 로그인 · `/settings/plan` 의 조회 권한. 부모 화면(`SCR-SETTINGS-PLAN`)의 탭 하나다.

**후행 처리** — 없다. 이 탭은 아무것도 바꾸지 않는다.

**관련 화면** — 부모 [플랜·이용 현황](index.md) · 나머지 탭 [결제](tab-billing.md) · [결제내역](tab-history.md). 표의 각 행이 설명하는 대상은 **모듈이 지배하는 화면들**이다 — `shared/entitlements/module-resources.ts:29-98`.

**관련 메뉴** — 사이드바에 자기 항목이 없다. 부모 화면의 `?tab=usage` 상태다(기본 탭이라 주소에 쓰지 않는다).

**관련 기능** — 여기 표시되는 판정이 곧 사이드바의 잠금 꼬리표·잠금 화면·대시보드 되돌림을 정하는 판정이다. 같은 함수 하나를 읽는다.

**관련 API** — 목적 하나: 이 계정의 플랜·엔타이틀먼트 조회. **조회만 한다 — 쓰기 엔드포인트가 없다** — `shared/entitlements/plan.ts:30-34`. 지금은 브라우저 저장소 픽스처를 읽는다.

**참고 문서** — [ID 규약](../../id-convention.md) · [인벤토리](../../inventory.md) · [품질 기준](../../../reference/quality-bar.md)

**구현 파일** — `apps/admin/src/pages/settings/plan/components/UsagePanel.tsx` (판정·카탈로그 `shared/entitlements/plan.ts` · 쿼터 사용량 `shared/entitlements/entitlement-store.ts` · 공용 스타일 `../styles.ts`)

### 1.1 값의 형태가 셋이라 '종류' 열이 있다

카탈로그의 모듈 값은 세 형태다 — `plan.ts:82-96,173-257`. **셋을 한 낱말('포함')로 뭉치면 '포함인데 왜 등록이 안 되죠?'(한도 소진)를 화면이 설명하지 못한다** — `UsagePanel.tsx:10-17`.

| 형태 | 표의 `종류` | 뜻 | 잠기는가 |
|---|---|---|---|
| 켬/끔 | `포함 여부` | 티어에 있으면 열리고 없으면 잠긴다 | **잠긴다** |
| 한도 | `한도` | 몇 건까지 만들 수 있는가 | **잠기지 않는다** — 한도를 넘겨도 화면은 열리고 신규 등록만 막힌다 |
| 수준 | `수준` | 기본/고급 중 어느 수준인가. `미포함` 이면 꺼진 것으로 읽는다 | 미포함이면 잠긴다 |

낱말의 정본은 한 벌이다 — `UsagePanel.tsx:49-54`.

### 1.2 판정 결과가 셋이라 어떤 행은 표에 없다

| 결과 | 뜻 | 운영자가 할 수 있는 일 | 표에서 |
|---|---|---|---|
| `granted` | 열려 있다 | — | `포함` 배지(성공 톤) |
| `locked` | 상위 플랜에서 **살 수 있다** | 사내 홈페이지에서 플랜을 올린다 | `{티어} 플랜부터` 배지(중립 톤) |
| `absent` | 어떤 플랜에도 없다(판매하지 않는다) | **할 수 있는 일이 없다** | **행 자체를 그리지 않는다** |

**살 수 없는 것을 표에 올리면 그것이 곧 티저**이고, 운영자는 결제 페이지를 뒤지다 아무것도 찾지 못한다 — `UsagePanel.tsx:7-8` · `plan.ts:100-112`.

무엇이 `locked` 이고 무엇이 `absent` 인지는 카탈로그의 **최소 티어**가 정한다: 티어가 지정돼 있으면 잠금, `null` 이면 숨김이다. 지금 카탈로그에서 `null` 인 것은 SMS 발송 하나다 — 발신번호 등록과 발송사 계약이 따로 있어야 켤 수 있어 **플랜을 올려도 저절로 열리지 않기 때문**이다 — `plan.ts:234-241`.
