# 청구·입금 목록

## 1. 화면 개요

| 항목 | 내용 |
|---|---|
| 화면명 | 청구·입금 |
| 화면 ID | `SCR-SALES-BILLING` |
| 메뉴 경로 | 영업 관리 > 청구·입금 |
| 작성자 | Admin Hub 문서화 |
| 작성일 | 2026-07-22 |
| 버전 | 1.0 |

**화면 목적** — 나간 청구를 모아 **누가 아직 안 냈는지** 한 화면에서 답한다. 중심 열은 청구액이 아니라 **잔액과 입금 상태**다.

**업무 배경** — 결제대행(PG)을 쓰지 않는 운영에서 '결제완료'를 만드는 것은 시스템이 아니라 **판매자의 입금확인**이다. 문의 → 견적 → 수주까지는 화면이 있었는데 그 다음이 없었다 — 돈이 언제 얼마나 들어왔는지는 앱 밖(통장·엑셀)에만 있었다 — `apps/admin/src/pages/sales/billing/types.ts:4-14`. 이 목록이 그 자리를 채운다.

**화면 설명** — 좌측 필터 레일(입금 상태 + 안내 3~4줄)과 우측 10열 읽기 전용 표다. **등록·삭제·일괄 작업이 전부 없다** — 청구는 회계 기록이다.

**주요 사용자** — 최상위 관리자 · 영업/정산 담당 운영자.

**사용 시나리오**

1. 좌측에서 '미입금'을 고른다 → 그중 '안내 발송'이 `미발송` 인 행을 먼저 연다(가장 먼저 집을 행이다).
2. 좌측 안내의 미수금 합계로 지금 얼마가 안 들어왔는지 확인한다.
3. 고객이 견적번호를 대며 입금을 알린다 → 검색창에 견적번호를 넣어 그 청구를 찾고 상세에서 기록한다.

**선행 조건** — 로그인 · 읽기 권한 · `sales.pipeline` 모듈. **행이 생기려면 수주로 전환된 견적이 있어야 한다.**

**후행 처리** — 이 목록은 아무것도 바꾸지 않는다. 입금확인·안내 기록은 전부 [청구 상세](detail.md)의 일이다.

**관련 화면** — [청구 상세](detail.md) `SCR-SALES-BILLING-DETAIL` · [견적 상세](../quotes/detail.md)(청구가 태어나는 곳) · [거래처 상세](../accounts/detail.md) · 공통 규약은 [공통 확인 다이얼로그 · 토스트](../../_common/index.md).

**관련 메뉴** — 사이드바 `영업 관리 > 청구·입금`.

**관련 기능** — 청구는 `sales.pipeline` 모듈에 함께 묶여 있다. 견적이 플랜에 잠기는데 그 파생물이 안 잠기면 잠금이 반쪽이 되기 때문이다 — `apps/admin/src/shared/entitlements/module-resources.ts:59-61`.

**관련 API** — 청구 목록 조회 1건. **백엔드가 없다** — `TODO(backend): GET /api/sales/billing · GET/PUT /api/sales/billing/:id · POST /api/sales/billing/:id/payments (입금확인 수기 처리 — 되돌리는 전이는 없다) · POST /api/sales/billing/:id/notices (청구 안내 발송 기록)` — `apps/admin/src/pages/sales/billing/data-source.ts:183-185`.

**참고 문서** — [ID 규약](../../id-convention.md) · [인벤토리](../../inventory.md) · [품질 기준](../../../reference/quality-bar.md)

**구현 파일** — `apps/admin/src/pages/sales/billing/BillingListPage.tsx` (규칙 `types.ts` · 저장소 `data-source.ts`)

**등록·삭제 액션이 이 자원에 없다** — 청구를 만드는 문은 견적 상세 하나이고(그 버튼은 `sales-quotes` 의 `update` 로 게이팅된다), 삭제는 화면 어디에도 없다. 그래서 `create`·`remove` 권한은 이 화면에서 아무것도 바꾸지 않는다.

**권한이 없을 때 화면이 어떻게 되는가** — 읽기가 없으면 본문이 권한 화면이 된다. 수정 권한이 없으면 목록은 그대로 보이되 좌측이 그 사실을 미리 밝히고, 상세의 입력·버튼이 사라진다.
