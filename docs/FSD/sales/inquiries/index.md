# 영업 문의

## 1. 화면 개요

| 항목 | 내용 |
|---|---|
| 화면명 | 문의 |
| 화면 ID | `SCR-SALES-INQUIRIES` |
| 메뉴 경로 | 영업 관리 > 문의 |
| 작성자 | Admin Hub 문서화 |
| 작성일 | 2026-07-22 |
| 버전 | 1.0 |

**화면 목적** — 고객이 남긴 영업 문의를 한 표에 모아, 무엇을 먼저 처리할지 고르고 상세로 들어간다.

**업무 배경** — 이 어드민의 매출은 **문의 → 견적 → 계약 → 청구 → 프로젝트** 한 경로로만 들어온다. 그 사슬의 첫 칸이 이 화면이다. 여기서 걸러 낸 한 건이 견적이 되고, 그 견적이 계약과 청구를 낳는다. 문의를 놓치면 뒤의 네 화면에는 아무것도 생기지 않는다.

**화면 설명** — 읽기 전용 목록이다. 유형·채널·상태 세 필터와 검색이 툴바에 있고, 행을 누르면 문의 상세로 간다. 이미 견적이 발행된 문의는 마지막 열에서 그 견적으로 건너간다.

**주요 사용자** — 최상위 관리자 · 영업 담당 운영자.

**사용 시나리오**

1. 운영자가 상태를 '접수'로 걸러 미배정 문의만 본다 → 한 건을 열어 담당자를 넣고 처리 상태를 옮긴다.
2. 고객이 전화로 문의번호를 대며 진행을 묻는다 → 검색창에 문의번호를 넣어 그 행을 찾고 상세를 연다.
3. '견적 발행' 상태로 걸러 이미 견적이 나간 건만 보고, '견적 보기'로 그 견적서를 확인한다.

**선행 조건** — 로그인 · 이 화면의 읽기 권한 · 플랜에 `sales.pipeline` 모듈이 있어야 한다.

**후행 처리** — 이 화면 자체는 아무 값도 바꾸지 않는다. 변경은 전부 상세(`SCR-SALES-INQUIRIES-DETAIL`)에서 일어나고, 그중 '견적 발행' 전이가 `SCR-SALES-QUOTES` 에 새 견적을 만든다.

**관련 화면** — [문의 상세](detail.md) `SCR-SALES-INQUIRIES-DETAIL` · [견적 상세](../quotes/detail.md) `SCR-SALES-QUOTES-DETAIL` · 공통 규약은 [공통 확인 다이얼로그 · 토스트](../../_common/index.md).

**관련 메뉴** — 사이드바 `영업 관리 > 문의`. 조건부 노출이 아니다 — 상품·프로그램의 '문의' 잎과 달리 결제 설정과 무관하게 항상 보인다 — `apps/admin/src/shared/layout/nav-config.ts:238`.

**관련 기능** — 상세에서 처리 상태를 '견적 발행'으로 바꾸면 견적이 자동 생성된다. 상품 문의(`SCR-PRODUCTS-INQUIRIES`)·프로그램 문의(`SCR-PROGRAMS-INQUIRIES`)도 같은 견적 저장소에 닿지만, 그쪽은 공통 층의 발행기를 지나고 이 화면은 같은 페이지 안이라 저장소를 직접 부른다 — `apps/admin/src/shared/domain/quote-issue.ts:16-23`.

**관련 API** — 문의 목록 조회 1건. **백엔드가 없다** — 서버가 맡을 계약은 `TODO(backend): GET /api/sales/inquiries` 로 표시되어 있다(`apps/admin/src/pages/sales/inquiries/data-source.ts:160`).

**참고 문서** — [ID 규약](../../id-convention.md) · [인벤토리](../../inventory.md) · [품질 기준](../../../reference/quality-bar.md)

**구현 파일** — `apps/admin/src/pages/sales/inquiries/InquiryListPage.tsx` (도메인 규칙 `types.ts` · 데이터 `data-source.ts`)

**등록·삭제 액션이 이 화면에 없다** — 문의는 고객 채널이 만들고, 관리자 화면에는 생성 경로가 없다(`apps/admin/src/pages/sales/inquiries/data-source.ts:4`). 삭제 컨트롤도 없다. 그래서 `create`·`remove` 권한은 이 화면에서 아무것도 바꾸지 않는다.

**권한이 없을 때 화면이 어떻게 되는가** — 읽기 권한이 없으면 본문이 통째로 `접근 권한이 없어요` 화면으로 바뀐다(사이드바는 남는다). 쓰기 권한만 없는 경우 이 목록은 그대로 보이고, 막히는 것은 상세의 저장이다.
