# 거래처 목록

## 1. 화면 개요

| 항목 | 내용 |
|---|---|
| 화면명 | 거래처 |
| 화면 ID | `SCR-SALES-ACCOUNTS` |
| 메뉴 경로 | 영업 관리 > 거래처 |
| 작성자 | Admin Hub 문서화 |
| 작성일 | 2026-07-22 |
| 버전 | 1.0 |

**화면 목적** — 거래처 마스터를 관리한다. 사슬 네 칸(견적·계약·프로젝트·문의)이 전부 이 목록의 한 행을 가리킨다.

**업무 배경** — 예전에는 네 도메인 모두 거래처를 **자유 입력 문자열**로만 들었고, 이 마스터를 참조하는 코드가 앱 전체에 한 줄도 없었다. 그 결과가 셋이다 — ① 오타 하나가 한 거래처를 둘로 쪼갰다 ② 거래처를 개명하면 이미 저장된 계약·견적의 이름표가 그 자리에서 낡았다 ③ **거래처 → 그 거래처의 계약/견적/프로젝트 역방향 조회가 어디에도 없었다** — `apps/admin/src/pages/sales/_shared/account-reference.ts:3-8`.

**화면 설명** — 검색 + 거래유형 필터 툴바와 7열 표다. **영업 관리에서 유일하게 등록 버튼이 있는 목록이다** — 거래처는 사슬의 칸이 아니라 마스터 데이터이기 때문이다. 상태 열에서 거래중/중지를 바로 토글할 수 있다.

**주요 사용자** — 최상위 관리자 · 영업 담당 운영자.

**사용 시나리오**

1. 새 거래처와 첫 계약을 앞두고 `거래처 등록` 으로 사업자정보를 넣는다.
2. 사업자번호로 검색해 그 거래처를 찾고 행을 눌러 상세에서 계약·견적 이력을 확인한다.
3. 거래가 끝난 곳을 상태 토글로 '중지'로 내린다 — 마스터에서 지우지 않고 선택지에서 흐리게 남긴다.

**선행 조건** — 로그인 · 읽기 권한 · `sales.pipeline` 모듈.

**후행 처리** — 여기서 만든 거래처가 견적·계약·프로젝트 폼의 거래처 셀렉트에 나타난다. 상태를 '중지'로 내리면 그 셀렉트에서 `{이름} (거래중지)` 로 표기된다 — `apps/admin/src/pages/sales/_shared/AccountSelectField.tsx:160`.

**관련 화면** — [거래처 상세](detail.md) `SCR-SALES-ACCOUNTS-DETAIL` · [거래처 등록·수정](form.md) `SCR-SALES-ACCOUNTS-FORM` · 거래처를 참조하는 네 화면([견적](../quotes/index.md) · [계약](../contracts/index.md) · [프로젝트](../projects/index.md) · [청구](../billing/index.md)) · 공통 규약은 [공통 확인 다이얼로그 · 토스트](../../_common/index.md).

**관련 메뉴** — 사이드바 `영업 관리 > 거래처`.

**관련 기능** — 견적·계약·프로젝트 폼의 거래처 셀렉트가 이 목록을 읽는다. 그 폼들에서 '미등록 거래처'를 고르면 여기의 `거래처 등록` 으로 오는 링크가 함께 뜬다.

**관련 API** — 거래처 목록 조회 1건 · 인라인 갱신 1건(상태 토글) · 삭제 N건. **백엔드가 없다** — `TODO(backend)` 로 어댑터 본문만 교체하고 화면은 그대로 둔다 — `apps/admin/src/pages/sales/accounts/data-source.ts:3-4`.

**참고 문서** — [ID 규약](../../id-convention.md) · [인벤토리](../../inventory.md) · [품질 기준](../../../reference/quality-bar.md)

**구현 파일** — `apps/admin/src/pages/sales/accounts/AccountListPage.tsx` (규칙 `types.ts` · 데이터 `data-source.ts` · 사업자번호 검증 `../_shared/business.ts`)

## 4. 기능 명세

### 4.1 거래처 목록 조회

| 항목 | 내용 |
|---|---|
| 기능 목적 | 거래처 마스터를 훑는다 |
| 실행 조건 | 진입 · 조회 조건 변경 · 쓰기 성공 후 |
| 사용자 동작 | 없음(자동) |
| 시스템 처리 | 목록을 받아 **상호 가나다순**으로 정렬한다 — `apps/admin/src/pages/sales/accounts/types.ts:147-149` |
| 데이터 처리 | 읽기만 한다 |
| Validation | 해당 없음 |
| API 호출 여부 | Y — 거래처 전체 목록 1건 |
| 성공 처리 | 표가 그려진다. 토스트 없음. 보조기술에는 `거래처 {N}건을 찾았어요.` |
| 실패 처리 | 표 자리에 danger 배너 + `다시 시도` |
| 예외 처리 | 재조회 중에는 이전 행을 유지한다 |
| 화면 변경 사항 | 목록영역 |
| 후속 동작 | 없음 |
| 로그 기록 여부 | N |

**다른 목록과 정렬 축이 다르다** — 사슬의 네 목록은 전부 날짜순인데 거래처만 이름순이다. 마스터 데이터는 '언제'가 아니라 '무엇'으로 찾기 때문이다.

| 수정 실패 | 폼에서의 저장 실패는 [거래처 등록·수정](form.md) 의 일이다 | — | — |

| 네트워크 오류 | 위와 같다 | 자동 재시도를 하지 않는다 | 〃 |
| 권한 없음 | 읽기가 없으면 `접근 권한이 없어요` 화면. 등록 권한이 없으면 등록 버튼이, 수정 권한이 없으면 토글과 연필이, 삭제 권한이 없으면 체크박스와 휴지통이 **사라진다** | 사이드바는 남는다 | 관리자에게 권한 요청 |
