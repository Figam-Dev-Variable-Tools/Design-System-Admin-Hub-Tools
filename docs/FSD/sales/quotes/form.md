# 견적 등록·수정

## 1. 화면 개요

| 항목 | 내용 |
|---|---|
| 화면명 | 견적 수정 |
| 화면 ID | `SCR-SALES-QUOTES-FORM` |
| 메뉴 경로 | 영업 관리 > 견적 > (행 선택 · 수정) |
| 작성자 | Admin Hub 문서화 |
| 작성일 | 2026-07-22 |
| 버전 | 1.0 |

**화면 목적** — 발행된 견적의 내용(거래처 정보 · 품목 명세 · 기간 · 비고)을 채우고 고친다. **그리고 `/sales/quotes/new` 로 들어온 사람에게 견적을 어디서 만드는지 알려 준다.**

**업무 배경** — 문의에서 발행된 견적은 품목 이름만 있고 단가가 0 이다(문의는 금액을 모른다). '무엇에 대한 견적인지'는 문의가 알고 '얼마인지'는 운영자가 안다 — 그 나머지 절반을 채우는 화면이다.

**화면 설명** — 라우트가 둘(`/new` · `/:id/edit`)이지만 **같은 컴포넌트**이고, `:id` 가 없으면 폼 대신 차단 안내 화면이 열린다. 수정 모드는 좌우 2단이다: 왼쪽에 입력 카드 4개(견적 정보 · 품목 명세 · 원본 문의 · 비고), 오른쪽에 견적서 실시간 미리보기와 인쇄 버튼.

**주요 사용자** — 최상위 관리자 · 영업 담당 운영자.

**사용 시나리오**

1. 문의에서 발행된 견적을 연다 → 품목의 단가를 채우고 과세유형·유효기간을 확인한 뒤 저장한다.
2. 거래처가 마스터에 없어 '미등록'으로 발행된 견적을 연다 → 거래처 셀렉트에서 마스터의 그 회사를 고른다 → 사업자번호·대표자가 함께 채워진다.
3. 즐겨찾기에 남아 있던 `/sales/quotes/new` 를 연다 → **빈 폼이 아니라** '견적은 문의에서 발행돼요' 안내와 문의 목록으로 가는 링크가 뜬다.

**선행 조건** — 로그인 · `sales.pipeline` 모듈 · 수정 권한. **등록은 어떤 권한으로도 열리지 않는다.**

**후행 처리** — 저장하면 견적 목록으로 돌아간다. 바뀐 품목·과세유형은 합계금액을 바꾸고, 그 합계는 **앞으로 만들 ** 계약·청구의 승계값이 된다. 이미 만들어진 계약·청구의 금액은 스냅숏이라 **바뀌지 않는다**.

**관련 화면** — [견적 목록](index.md) · [견적 상세](detail.md) · [문의 상세](../inquiries/detail.md)(견적이 태어나는 곳) · [거래처 등록·수정](../accounts/form.md)(미등록 거래처를 만드는 곳) · 공통 규약은 [공통 확인 다이얼로그 · 토스트](../../_common/index.md).

**관련 메뉴** — 사이드바에 자기 자리가 없다.

**관련 기능** — 견적서 상단의 공급자(자사) 정보는 회사 정보 화면(`SCR-COMPANY-PROFILE`)에서 온다. **상태는 이 폼에서 고르지 않는다** — 예전에는 여기 상태 드롭다운이 있었고, 전이 규칙을 아무것도 읽지 않는 자유 선택이라 견적 상세가 막는 전이를 이 폼이 그냥 통과시켰다 — `apps/admin/src/pages/sales/quotes/QuoteFormPage.tsx:324-328`.

**관련 API** — 견적 상세 조회 1건 · 견적 갱신 1건. **백엔드가 없다** — `TODO(backend): GET/POST /api/sales/quotes · GET/PUT/DELETE /api/sales/quotes/:id` — `apps/admin/src/pages/sales/quotes/data-source.ts:201-202`.

**참고 문서** — [ID 규약](../../id-convention.md) · [인벤토리](../../inventory.md) · [품질 기준](../../../reference/quality-bar.md)

### 1.1 `/new` 가 폼을 열지 않는다 — 조용한 404 가 아니다

목록의 등록 CTA 는 지웠지만 `/sales/quotes/new` 주소는 라우트 표에 그대로 남아 있다. 주소창·즐겨찾기·옛 링크로 들어오면 빈 폼이 열리고, 거기서 저장한 견적은 **원본 문의가 없는 유령**이 된다 — `apps/admin/src/pages/sales/_shared/ChainOnlyCreateNotice.tsx:1-8`.

빈 화면이나 404 로 받지 않은 이유: **고장과 구분되지 않기 때문이다.** 운영자는 자기가 뭘 잘못 눌렀는지, 그럼 어디로 가야 하는지 알 수 없다. 그래서 '없다'가 아니라 **'여기서는 만들지 않는다, 만드는 곳은 저기다'** 라고 말하고 그 문까지 링크한다 — `apps/admin/src/pages/sales/quotes/ChainOnlyCreateNotice.tsx:10-14`.

**막은 것은 생성뿐이다.** `/:id/edit` 는 평소처럼 폼을 열고, 삭제도 목록에서 그대로 된다.

### 4.3 거래처 연결 (미등록 → 마스터)

| 항목 | 내용 |
|---|---|
| 기능 목적 | 문의에서 발행된 견적의 '이름만 있는 거래처'를 마스터의 실제 행에 잇는다 |
| 실행 조건 | 폼이 열려 있고 거래처 목록 조회가 끝났을 때 |
| 사용자 동작 | 거래처 셀렉트에서 마스터의 한 건을 고른다(또는 `미등록 거래처 — 이름 직접 입력` 을 고른다) |
| 시스템 처리 | 고르는 순간 `accountId` 와 `accountName` 이 함께 채워지고, **견적은 사업자번호·대표자까지 승계한다** — 운영자가 세 칸을 손으로 옮겨 적지 않게 한다 — `apps/admin/src/pages/sales/quotes/QuoteFormPage.tsx:375-382` |
| 데이터 처리 | 조회·링크의 정본은 `accountId` 다. `accountName` 은 견적서에 인쇄되는 비정규화 라벨이며, 거래처가 나중에 개명해도 **이미 보낸 견적서의 상호가 바뀌어서는 안 되므로** 일부러 함께 보관한다 — `apps/admin/src/pages/sales/quotes/types.ts:79-86` |
| Validation | `accountName` 은 필수(60자). `accountId` 는 **빈 값을 허용한다** — 여기서 요구하면 문의 → 견적 발행 경로가 통째로 막힌다 — `validation.ts` 견적 스키마 주석 |
| API 호출 여부 | Y — 거래처 목록 조회 1건(셀렉트 채우기) |
| 성공 처리 | 힌트가 `사업자 {번호} · 대표 {이름}` 으로 바뀐다 — `apps/admin/src/pages/sales/quotes/AccountSelectField.tsx:227` |
| 실패 처리 | 목록 조회 실패 시 danger 배너 `거래처 목록을 불러오지 못해 지금은 고를 수 없어요.` + `다시 시도` — `apps/admin/src/pages/sales/quotes/AccountSelectField.tsx:201` |
| 예외 처리 | **세 갈래를 구분한다**: ① 미등록 → warning 배너 `거래처 마스터에 연결되지 않아요 — 거래처 상세의 계약·견적·프로젝트 이력에 이 건이 나타나지 않아요.` + `거래처 등록` 링크 — `apps/admin/src/pages/sales/quotes/AccountSelectField.tsx:214-218` ② 저장된 id 가 마스터에 없음(연결 끊김) → warning `연결된 거래처를 찾을 수 없어요. 이미 삭제되었을 수 있으니 거래처를 다시 고르세요.` — `apps/admin/src/pages/sales/quotes/AccountSelectField.tsx:209` ③ **아직 못 읽음** → 단정하지 않고 `거래처 목록을 불러오는 중이에요…` — `apps/admin/src/pages/sales/quotes/AccountSelectField.tsx:226`. **'없다'와 '모른다'를 같은 문구로 말하지 않는다** |
| 화면 변경 사항 | 입력영역 · 미리보기 |
| 후속 동작 | 저장하면 그 견적이 거래처 상세의 '이 거래처의 견적'에 나타난다 |
| 로그 기록 여부 | N |

**승계 견적에서도 마스터 연결은 허용한다** — 이름은 사람이 고쳐 쓰지 못하지만(원본 문의와 어긋나면 어느 쪽이 진실인지 알 수 없다) 마스터에서 고르는 것은 막지 않는다. 그것은 승계 값을 바꾸는 일이 아니라 **문의가 애초에 갖지 못한 '어느 거래처인가'를 뒤늦게 채우는 일**이고, 이 연결이 없으면 그 견적은 거래처 상세의 견적 이력에서 영원히 빠진다 — `apps/admin/src/pages/sales/quotes/QuoteFormPage.tsx:355-359`.

**등록은 권한으로 열리지 않는다** — 전체 권한을 가진 최상위 관리자도 이 화면에서 견적을 만들 수 없다. 만드는 곳은 문의 상세뿐이다.

**권한이 없을 때 화면이 어떻게 되는가** — 읽기가 없으면 본문이 권한 화면이 된다. 수정 권한만 없을 때 이 폼은 열리지만 저장이 배너로 막힌다(`apps/admin/src/shared/crud/useCrudForm.ts:290-298`).
