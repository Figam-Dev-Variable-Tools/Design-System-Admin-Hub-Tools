# 연혁 등록 · 수정

> **이 문서는 기업 관리 폼 화면의 정본이다.** [인증서/특허](../certificates/form.md) ·
> [ESG](../esg/form.md) · [채용 공고](../careers/form.md)가 같은 공용 껍데기(`useCrudForm` +
> `FormPageShell`)를 쓰므로 그 문서들은 **차이만** 적는다. 이 골격은 콘텐츠 관리의
> [뉴스 등록·수정](../../content/news/form.md)과도 같다 — 그쪽이 콘텐츠 묶음의 유일한 예다.

## 1. 화면 개요

| 항목 | 내용 |
|---|---|
| 화면명 | 연혁 등록 · 수정 |
| 화면 ID | `SCR-COMPANY-HISTORY-FORM` |
| 메뉴 경로 | 기업 관리 > 연혁 > 연혁 등록 (또는 행의 수정) |
| 작성자 | Admin Hub 문서화 |
| 작성일 | 2026-07-22 |
| 버전 | 1.0 |

**화면 목적** — 연혁 한 건의 연도·월·내용을 정한다. **한 화면이 등록과 수정을 겸한다** — 주소에 id 가 있으면 수정, 없으면 등록이다.

**업무 배경** — 연혁 항목은 짧고 규칙적이다. 그래서 폼도 세 칸뿐이며, 순서를 정하는 칸이 없다(연·월이 곧 순서다).

**화면 설명** — 위쪽에 '목록으로', 그 아래 제목(`연혁 등록`/`연혁 수정`)과 안내문, 카드 하나에 (연도 · 월) 한 줄과 내용이 놓인다. 카드 안 오른쪽 아래에 취소·제출.

**주요 사용자** — 최상위 관리자 · 기업 정보를 다루는 운영자.

**사용 시나리오**

1. 목록에서 '연혁 등록'을 눌러 2024 · 3월 · 내용을 적고 등록한다.
2. 오래된 항목의 내용을 다듬고 저장한다.
3. 반쯤 쓰다 다른 메뉴를 누른다 → 저장하지 않은 변경 확인이 뜬다.

**선행 조건** — 로그인 · **등록이면 등록 권한, 수정이면 수정 권한**(없으면 화면이 열리지 않는다 — §9).

**후행 처리** — 등록·수정 모두 성공하면 목록으로 간다(상세가 없다). 히스토리를 덮어쓴다 — `apps/admin/src/shared/crud/useCrudForm.ts:250`.

**관련 화면** — [연혁](index.md)(`SCR-COMPANY-HISTORY`) · 공통 규약은 [공통 확인 다이얼로그 · 토스트](../../_common/index.md).

**관련 메뉴** — 사이드바에 자리가 없다. 권한 리소스는 잎 `/company/history` 가 덮는다.

**관련 기능** — 여기서 적은 연·월이 목록의 정렬 순서를 정한다([목록 문서](index.md) §4.2).

**관련 API** — 등록 1건 · 수정 1건 · (수정 진입 시) 상세 조회 1건. **백엔드가 없다** — `apps/admin/src/pages/company/history/data-source.ts:19`.

**참고 문서** — [ID 규약](../../id-convention.md) · [인벤토리](../../inventory.md)

**구현 파일** — `apps/admin/src/pages/company/history/HistoryFormPage.tsx` (껍데기 `apps/admin/src/shared/crud/FormPageShell.tsx` · 컨트롤러 `useCrudForm.ts` · 검증 `validation.ts` · 상수 `types.ts`)

### 4.3 저장 충돌

| 항목 | 내용 |
|---|---|
| 기능 목적 | 내가 폼을 여는 사이 다른 관리자가 같은 항목을 지웠거나 바꿨을 때 **덮어쓰지 않는다** |
| 실행 조건 | 저장이 충돌(409/412)로 거절됐을 때 — `apps/admin/src/shared/crud/useCrudForm.ts:191-206` |
| 사용자 동작 | 다이얼로그에서 '최신 내용 불러오기' 또는 '이어서 편집' |
| 시스템 처리 | 성공 토스트도, 목록 이동도 일어나지 않는다(유령 저장 금지). **입력은 그대로 살아 있다** |
| 데이터 처리 | '최신 내용 불러오기'는 서버 최신본으로 폼을 덮는다. '이어서 편집'은 아무것도 바꾸지 않는다 |
| Validation | 해당 없음 |
| API 호출 여부 | Y — 불러오기를 고르면 상세를 다시 조회한다 |
| 성공 처리 | 다이얼로그가 닫히고 고른 쪽에 따라 폼이 바뀌거나 그대로 남는다. **토스트를 띄우지 않는다** |
| 실패 처리 | 재조회가 실패하면 폼의 조회 실패 표시가 대신 뜬다 |
| 예외 처리 | 픽스처가 주는 사유는 `다른 사용자가 먼저 삭제한 항목이에요.` — `apps/admin/src/shared/crud/crud.ts:145`. 사유가 없으면 기본 문장을 쓴다 — `apps/admin/src/shared/crud/useCrudForm.ts:197` |
| 화면 변경 사항 | 다이얼로그 · 입력영역 |
| 후속 동작 | 폼 값 교체 또는 계속 편집 |
| 로그 기록 여부 | 확인하지 못했다 |

**확인 다이얼로그** — `DLG-COMMON-FORM-CONFLICT`([공통 문서](../../_common/index.md) §4.5).
