# ESG

> 이 화면은 [연혁](../history/index.md)(`SCR-COMPANY-HISTORY`)과 **같은 공용 목록 껍데기**를 쓴다 —
> 알림 영역 · 요약 · 선택 바 · 표 · 조회 실패 배너 · 공통 삭제 확인 둘 · 권한에 따라 사라지는 컨트롤.
> 아래 각 절은 **다른 것만** 적는다. 다른 것은 둘이다: **왼쪽 분류 필터**(건수 배지 포함) ·
> **필터 때문에 0행이 되면 빈 상태가 '필터 초기화'를 준다**.

## 1. 화면 개요

| 항목 | 내용 |
|---|---|
| 화면명 | ESG |
| 화면 ID | `SCR-COMPANY-ESG` |
| 메뉴 경로 | 기업 관리 > ESG |
| 작성자 | Admin Hub 문서화 |
| 작성일 | 2026-07-22 |
| 버전 | 1.0 |

**화면 목적** — 회사의 ESG 활동(환경·사회·지배구조)을 분류별로 모아 두고 등록·수정·삭제한다.

**업무 배경** — ESG 활동은 분류가 세 축으로 고정돼 있고, 그 축마다 몇 건이 쌓였는지가 곧 대외 설명의 근거가 된다. 그래서 목록은 왼쪽에 분류를 세우고 **건수를 배지로** 보여 준다.

**화면 설명** — 왼쪽에 분류 필터, 오른쪽에 '`ESG 활동` 등록' 하나뿐인 툴바와 요약 · (선택이 있으면) 선택 바 · 표가 온다. 검색도 페이지네이션도 없다.

**주요 사용자** — 최상위 관리자 · 기업 정보를 다루는 운영자.

**사용 시나리오**

1. 새 환경 활동이 생겨 'ESG 활동 등록'을 눌러 분류·일자·제목·내용·이미지를 넣는다.
2. '지배구조'만 걸어 그 축의 활동만 훑는다 → 건수 배지로 얼마나 쌓였는지 함께 본다.
3. 필터를 건 채 아무것도 없으면 빈 상태의 '필터 초기화'를 눌러 전체로 되돌린다.

**선행 조건** — 로그인. 기업 관리 잎은 채용 공고를 뺀 여덟이 플랜 모듈 매핑에 없어 플랜 축으로 잠기지 않는다 — `apps/admin/src/shared/entitlements/module-resources.ts:64-76`.

**후행 처리** — 저장된 활동은 홈페이지의 ESG 소개에 쓰인다. 홈페이지는 이 리포 밖이라 반영 경로를 코드에서 확인하지 못했다.

**관련 화면** — [ESG 활동 등록·수정](form.md)(`SCR-COMPANY-ESG-FORM`) · [연혁](../history/index.md)(같은 껍데기) · 공통 규약은 [공통 확인 다이얼로그 · 토스트](../../_common/index.md). **상세 화면이 없다.**

**관련 메뉴** — 기업 관리 가지의 여덟 번째 잎 — `apps/admin/src/shared/layout/nav-config.ts:162`.

**관련 기능** — 없음.

**관련 API** — 목록 조회 1건 · 단건 삭제 1건. **백엔드가 없다** — `apps/admin/src/pages/company/esg/data-source.ts:45`.

**참고 문서** — [ID 규약](../../id-convention.md) · [인벤토리](../../inventory.md)

**구현 파일** — `apps/admin/src/pages/company/esg/EsgListPage.tsx` (껍데기 `apps/admin/src/shared/crud/CrudListShell.tsx` · 컨트롤러 `apps/admin/src/shared/crud/useCrudList.tsx` · 순수 규칙 `apps/admin/src/pages/company/esg/types.ts` · 데이터 `apps/admin/src/pages/company/esg/data-source.ts`)
