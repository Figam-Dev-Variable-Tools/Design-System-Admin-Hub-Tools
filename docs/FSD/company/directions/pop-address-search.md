# 주소 검색 (팝업)

## 1. 화면 개요

| 항목 | 내용 |
|---|---|
| 화면명 | 주소 검색 |
| 화면 ID | `POP-COMPANY-DIRECTIONS-ADDRESS-SEARCH` |
| 메뉴 경로 | 기업 관리 > 오시는 길 · 기업 관리 > 회사 정보 (**두 화면이 같은 팝업을 쓴다**) |
| 작성자 | Admin Hub 문서화 |
| 작성일 | 2026-07-22 |
| 버전 | 1.0 |

**화면 목적** — 우편번호 체계에 실재하는 주소 한 건을 고르게 한다. 고른 값은 부모 화면의 주소 칸에 들어간다.

**업무 배경** — 주소를 자유 입력으로 받으면 우편번호 체계에 없는 주소가 저장될 수 있고, 그 값은 홈페이지 푸터·사업자정보·견적서로 그대로 나간다 — 화면 어디에도 틀렸다는 표시가 없는 채로. 그래서 주소는 **고르는 값**이고, 부모의 입력 칸은 읽기 전용이다 — `apps/admin/src/shared/address-search/AddressField.tsx:7-9`.

**화면 설명** — 우리 모달 안에 우편번호 서비스의 검색 위젯을 심는다. 위젯이 오는 동안에는 '여는 중' 안내가, 못 받으면 danger 배너와 '다시 시도'가 그 자리에 온다. 푸터에는 '닫기' 하나뿐이다.

**주요 사용자** — 최상위 관리자 · 기업 정보를 다루는 운영자.

**사용 시나리오**

1. 운영자가 주소 칸을 눌러 팝업을 연다 → 검색창에 도로명을 치고 결과에서 하나를 고른다 → 팝업이 닫히고 주소 칸이 채워진다.
2. 사내 프록시가 스크립트를 막아 위젯이 오지 않는다 → 배너가 원인을 두 가지로 함께 말하고 '다시 시도'를 준다.
3. 잘못 열었다 → Esc 로 닫는다. 부모의 주소 값은 그대로다.

**선행 조건** — 부모 화면(회사 정보 또는 오시는 길)이 열려 있고, 저장 중·최초 로딩 중이 아니어야 한다 — `apps/admin/src/shared/address-search/AddressField.tsx:69-72`.

**후행 처리** — 고른 주소가 부모의 주소 칸에 들어가고 그 자리에서 검증이 다시 돈다. **상세주소가 비어 있을 때만** 건물명이 상세주소에 제안된다(부모 화면의 규칙이다) — `apps/admin/src/pages/company/directions/DirectionsPage.tsx:94-98` · `apps/admin/src/pages/company/profile/CompanyProfilePage.tsx:88-92`.

**관련 화면** — [오시는 길](index.md)(`SCR-COMPANY-DIRECTIONS`) · [회사 정보](../profile/index.md)(`SCR-COMPANY-PROFILE`) · 공통 규약은 [공통 확인 다이얼로그 · 토스트](../../_common/index.md).

**관련 메뉴** — 해당 없음 — 메뉴 항목이 아니다.

**관련 기능** — 부모 화면의 상세주소 제안. 이 팝업 자체는 상세주소를 다루지 않는다.

**관련 API** — **없다.** 이 팝업도, 이 부품의 어느 파일도 우리 서버를 부르지 않는다 — `apps/admin/src/shared/address-search/contract.ts:24`. 검색 위젯은 브라우저가 **외부 스크립트를 내려받아** 돌며, **앱 키도 도메인 등록도 요구하지 않는다** — `apps/admin/src/shared/address-search/contract.ts:16-18`.

**참고 문서** — [ID 규약](../../id-convention.md) · [인벤토리](../../inventory.md)

**구현 파일** — `apps/admin/src/shared/address-search/AddressSearchModal.tsx` (부품 `AddressField.tsx` · 계약과 순수 변환 `contract.ts` · 어댑터 `adapter.ts`)

---

**⚠ 인벤토리와 다른 자리** — 인벤토리는 이 팝업의 구현을 `apps/admin/src/pages/company/directions/AddressSearchModal.tsx` 로 적고 있으나, 지금 코드에서는 **`apps/admin/src/shared/address-search/`** 아래에 있다. 소비자가 둘(오시는 길·회사 정보)이 되면서 옮겨졌고, 폴더 이름도 남은 표면(우편번호 서비스 하나)에 맞춰 `address-search` 다 — `apps/admin/src/shared/address-search/contract.ts:3-8`.

## 6. Validation

해당 없음 — **이 팝업에는 우리 입력 칸이 하나도 없다.** 검색어 입력은 외부 위젯의 것이고, 우리는 그 결과만 받는다. 받은 값의 형태를 믿지 않는 한 겹 검사는 검증이 아니라 방어이며 §4.2 에 적었다.
