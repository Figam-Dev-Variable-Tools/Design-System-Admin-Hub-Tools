# 회원 관리

## 1. 화면 개요

| 항목 | 내용 |
|---|---|
| 화면명 | 회원 관리 |
| 화면 ID | `SCR-USERS-MEMBERS` |
| 메뉴 경로 | 사용자 관리 > 회원 관리 |
| 작성자 | Admin Hub 문서화 |
| 작성일 | 2026-07-22 |
| 버전 | 1.0 |

**화면 목적** — 가입한 고객을 등급·그룹·검색어로 좁혀 찾고, 찾은 회원에게 알림을 보내거나 계정을 지우거나 명단을 CSV 로 받는다.

**업무 배경** — 고객 문의·주문 처리·마케팅 발송의 출발점이 '이 사람이 누구인가'다. 그 조회 창구가 여기 하나여야 각 화면이 회원 명부를 따로 들지 않는다. 동시에 이 화면은 **개인정보가 가장 많이 모이는 곳**이라, 내보내기·삭제·발송 세 액션이 권한으로 각각 갈린다.

**화면 설명** — 상단에 마케팅 안내 배너, 좌측에 등급 필터와 그룹 필터(둘은 다른 축이고 함께 고르면 AND), 우측에 검색·일괄 액션·내보내기 툴바 → 총 건수 → 표 → 페이지네이션이다. 조회 조건(등급·그룹·검색어·페이지)은 **주소 쿼리스트링이 소유**해서 공유·북마크·뒤로가기가 성립한다.

**주요 사용자** — 최상위 관리자 · 운영자 · 상담원. 무엇을 할 수 있는지는 역할의 `page:/users/members` 판정이 정한다.

**사용 시나리오**

1. VIP 등급만 걸고 명단을 CSV 로 받아 마케팅 대상자를 뽑는다.
2. 닉네임 일부로 검색해 행을 눌러 회원 상세로 들어간다.
3. 여러 명을 체크해 한 번에 알림을 보낸다 — 부분 실패는 건수로 알려준다.
4. '무료 배송 결제 그룹' 같은 묶음을 새로 만든다(회원을 만드는 것이 아니다).

**선행 조건** — 로그인 · 이 화면의 `read` 권한.

**후행 처리** — 삭제한 회원은 목록에서 사라지고, 만든 그룹은 좌측 그룹 필터에 나타난다(운영진 그룹으로 만들었다면 관리자 관리 화면의 그룹 목록에 나타난다). 알림 발송과 CSV 내보내기는 이 화면의 데이터를 바꾸지 않는다.

**관련 화면** — `SCR-USERS-MEMBERS-DETAIL`(행 클릭 · 메모 아이콘) · `SCR-USERS-ADMINS`(운영진 그룹으로 만든 묶음이 도착하는 곳) · `SCR-MARKETING-TEMPLATES`(상단 배너의 링크) · `SCR-USERS-SETTINGS`(등급의 정의와 승급 정책을 정하는 곳). 팝업 1건은 별도 문서다: [새 그룹 만들기](pop-create-group.md). 공통 규약은 [공통 확인 다이얼로그 · 토스트](../../_common/index.md).

**관련 메뉴** — 사이드바 `사용자 관리 > 회원 관리`(`/users/members`) — `apps/admin/src/shared/layout/nav-config.ts:128`. 조건부 노출이 아니다.

**관련 기능** — 등급 축의 정의는 고객 설정(`SCR-USERS-SETTINGS`)이 갖는다. 좌측 등급 필터는 **기본 제공 등급 3종만** 나열한다(§7 의 알려진 격차). 그룹 만들기에서 '운영진 그룹'을 고르면 그 그룹은 관리자 관리의 정본 저장소로 들어간다.

**관련 API** — 목적 일곱. ① 목록 조회(등급·그룹·검색어·페이지) ② 그룹 목록 조회 ③ 내보내기용 전체 조회 ④ 그룹 생성 ⑤ 회원 삭제 ⑥ 알림 발송 ⑦ (상세에서 쓰는) 상세 조회. **백엔드가 없다** — 전부 픽스처 어댑터가 응답하며, 연동 지점은 `TODO(backend)` 주석이 경계다: `GET /api/members` · `GET /api/member-groups` · `GET /api/members/export` · `POST /api/member-groups` · `DELETE /api/members/:id` · `POST /api/members/:id/notifications` — `apps/admin/src/pages/members/data-source.ts:153,176,200,216,318,325`. **회원을 만드는 API 는 계약에 없다** — 고객은 회원가입으로만 유입된다(`data-source.ts:11-12`).

**참고 문서** — [ID 규약](../../id-convention.md) · [인벤토리](../../inventory.md) · [품질 기준](../../../reference/quality-bar.md)

**구현 파일** — `apps/admin/src/pages/members/MembersPage.tsx` (툴바 `components/MembersToolbar.tsx` · 표 `components/MembersTable.tsx` · 팝업 `components/CreateGroupModal.tsx` · 조회 `queries.ts` · 어댑터 `data-source.ts` · 도메인 `apps/admin/src/shared/domain/member.ts`)

| 실행 조건 | 화면 진입 또는 조건 변경 |

| 데이터 처리 | 없음(읽기만) |

| API 호출 여부 | Y — 목록 조회 1건. 좌측 배지 숫자(등급별·그룹별)가 같은 응답에 함께 실려 온다 |

| 예외 처리 | ① 조건이 바뀌면 1페이지로 돌아가고 선택이 해제된다 ② 다른 관리자가 회원을 지워 총 페이지가 줄면 **마지막 페이지로 보정**하고 재조회한다 — `MembersPage.tsx:189-192` ③ 재조회 중에는 이전 행을 유지한다(스켈레톤은 최초 로드에만) |
| 화면 변경 사항 | 목록영역 · 요약 줄 · 좌측 배지 |
| 후속 동작 | 없음 |
| 로그 기록 여부 | N |

**그룹 목록은 별개의 조회다** — 필터·검색과 무관해 캐시 키가 다르다. 실패하면 좌측 그룹 축에만 실패 표시와 전용 재시도가 열린다(목록은 정상 표시된다) — `MembersPage.tsx:384-387`.

### 4.5 회원 삭제 (단건 · 일괄)

| 항목 | 내용 |
|---|---|
| 기능 목적 | 회원 계정을 지운다. 되돌릴 수 없다 |
| 실행 조건 | `remove` 권한이 있을 때. 단건은 행 ⋯, 일괄은 선택 후 툴바 |
| 사용자 동작 | 삭제 클릭 → 확인 다이얼로그에서 '회원 삭제' |
| 시스템 처리 | 대상 전원에게 삭제를 시도하고 실패 건수를 모은다. **다이얼로그를 닫으면 진행 중이던 요청도 취소된다** — `MembersPage.tsx:285-292` |
| 데이터 처리 | 성공한 건만 목록에서 사라진다. 부분 실패가 성립한다 |
| Validation | 대상이 비어 있으면 아무 요청도 만들지 않는다 — `MembersPage.tsx:295` |
| API 호출 여부 | Y — 대상 건수만큼. **전원 성공일 때만** 목록을 다시 조회한다 — `apps/admin/src/pages/members/queries.ts:104-114` |
| 성공 처리 | 다이얼로그가 닫히고 선택이 해제되며 목록이 갱신된다. 토스트 `MSG-USERS-MEMBERS-04`: 단건 `{닉네임} 회원을 삭제했어요.` · 다건 `회원 {N}명을 삭제했어요.` — `MembersPage.tsx:328-332` |
| 실패 처리 | **다이얼로그를 닫지 않는다.** 그 안에 danger 배너로 이유가 뜨고 확인 버튼이 되살아난다(재클릭 = 재시도). 단건 `회원을 삭제하지 못했어요. 잠시 후 다시 시도해 주세요.` · 다건 `회원 {N}명 중 {M}명을 삭제하지 못했어요. 잠시 후 다시 시도해 주세요.` — `MembersPage.tsx:320-321` |
| 예외 처리 | 취소·Esc·딤 클릭은 진행 중 요청을 중단하고 **실패로 치지 않는다**(배너도 토스트도 없다) — `MembersPage.tsx:285-292` |
| 화면 변경 사항 | 다이얼로그 · 목록영역 · 요약 줄 · 툴바 |
| 후속 동작 | 목록 재조회 · 선택 해제(전원 성공일 때만) |
| 로그 기록 여부 | 확인하지 못했다 |

**확인 다이얼로그** — 제목 `회원 삭제` · 확인 라벨 `회원 삭제` — `MembersPage.tsx:474,477`. 본문은 두 경우로 갈린다(`MembersPage.tsx:338-343`).

| 경우 | 본문 |
|---|---|
| 1명 | `{닉네임}({계정}) 회원을 삭제할까요? 되돌릴 수 없어요.` |
| 여러 명 | `선택한 회원 {N}명을 삭제할까요? 되돌릴 수 없어요.` |

| 검색어 | 제약 없음 | — | — | 한글 조합이 끝난 뒤에만 조회에 반영된다 |
| 페이지 | 총 페이지를 넘지 않을 것 | 응답을 받은 뒤 | (문구 없음) | 마지막 페이지로 보정 후 재조회 |

**`null`(모름) 과 0 을 가르는 자리** — 좌측 배지 숫자다. 등급·그룹 건수는 응답이 오기 전에는 `null` 로 넘어가고, 그때 배지는 **0 이 아니라 그리지 않는다**(`counts={data?.counts ?? null}` — `MembersPage.tsx:372,381`). '전체 그룹' 항목은 응답이 온 뒤에도 배지를 숨긴다 — 세는 대상이 없어 0 을 띄우면 '해당 그룹 0명'이라는 거짓이 된다.
