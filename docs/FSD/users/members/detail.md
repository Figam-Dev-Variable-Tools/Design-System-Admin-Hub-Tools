# 회원 상세

## 1. 화면 개요

| 항목 | 내용 |
|---|---|
| 화면명 | 회원 상세 |
| 화면 ID | `SCR-USERS-MEMBERS-DETAIL` |
| 메뉴 경로 | 사용자 관리 > 회원 관리 > (행 클릭) |
| 작성자 | Admin Hub 문서화 |
| 작성일 | 2026-07-22 |
| 버전 | 1.0 |

**화면 목적** — 한 회원에 대해 이 어드민이 아는 것을 한 화면에 모은다: 회원이 제출한 정보 · 동의 정보 · 활동 · 적립금 · 보유 쿠폰 · 관리자 메모.

**업무 배경** — 상담 전화가 오면 운영자는 '이 사람이 언제 가입했고, 얼마나 샀고, 적립금이 얼마고, 무엇에 동의했는지'를 한 번에 봐야 한다. 그 여섯 질문을 카드 여섯 장으로 나눠 답한다.

**화면 설명** — 상단에 '리스트로 돌아가기'와 우측 ⋯ 메뉴, 아래에 2단 카드 격자다. **좌측 열은 회원이 제출한 정보(회원 정보 · 동의 정보), 우측 열은 운영 기록(활동 정보 · 적립금 · 보유 쿠폰 · 관리자 메모)** 이다. 좁아지면 한 단으로 접힌다.

**주요 사용자** — 최상위 관리자 · 운영자 · 상담원.

**사용 시나리오**

1. 상담 중 회원의 최근 로그인 시각과 누적 구매금액을 확인한다.
2. 비밀번호를 잃어버린 회원을 위해 비밀번호를 바꿔 주고 별도 경로로 안내한다.
3. 이벤트 보상으로 적립금을 지급하고 사유를 남긴다.
4. 잘못 지급한 적립금 내역을 지워 잔액을 되돌린다.
5. 다음 상담자를 위해 관리자 메모를 남긴다.

**선행 조건** — 로그인 · `page:/users/members` 의 `read` 권한 · 유효한 회원 id.

**후행 처리** — 삭제하면 목록으로 되돌아가고(히스토리 대체) 목록 캐시가 무효화된다. 적립금 지급·차감과 내역 삭제는 목록의 '적립금' 열을 낡게 만들므로 목록만 다시 조회하게 한다.

**관련 화면** — `SCR-USERS-MEMBERS`(목록 · 돌아갈 곳) · `SCR-PRODUCTS-REVIEWS`(구매평 건수 링크) · `SCR-SUPPORT-TICKETS`(문의 건수 링크) · `SCR-PRODUCTS-COUPONS-FORM`(보유 쿠폰의 원본 정의). 팝업 1건은 별도 문서다: [비밀번호 변경](pop-password-change.md). 공통 규약은 [공통 확인 다이얼로그 · 토스트](../../_common/index.md).

**관련 메뉴** — 사이드바에 자기 자리가 없다. 권한은 잎 `/users/members` 가 덮는다(라우트가 그 잎에 감싸이기 때문이다).

**관련 기능** — 보유 쿠폰의 이름·혜택·만료일은 이 화면이 아니라 **쿠폰 카탈로그**에서 조인해 온다. 활동 정보의 구매평·문의 건수는 각 목록 화면의 검색어(`?q=`)로 이어진다.

**관련 API** — 목적 여섯. ① 상세 조회 ② 비밀번호 변경 ③ 적립금 지급·차감 ④ 적립금 내역 삭제 ⑤ 관리자 메모 저장 ⑥ 회원 삭제·알림 발송. **백엔드가 없다** — 연동 지점은 `TODO(backend)` 주석이 경계다: `GET /api/members/:id` · `PATCH /api/members/:id/password` · `POST /api/members/:id/points`(헤더 `Idempotency-Key`) · `DELETE /api/members/:id/points/:entryId` · `PUT /api/members/:id/memo` · `DELETE /api/members/:id` · `POST /api/members/:id/notifications` — `apps/admin/src/pages/members/data-source.ts:226,251,270-272,298,310,318,325`. **쓰기 계열은 지금 아무것도 저장하지 않는다** — resolve 만 하고 화면이 성공 안내를 띄운다(`data-source.ts:8-9`).

**참고 문서** — [ID 규약](../../id-convention.md) · [인벤토리](../../inventory.md) · [품질 기준](../../../reference/quality-bar.md)

**구현 파일** — `apps/admin/src/pages/members/MemberDetailPage.tsx` (카드 `components/MemberInfoCard.tsx` · `components/ConsentCard.tsx` · `components/ActivityCard.tsx` · `components/PointsCard.tsx` · `components/CouponsCard.tsx` · `components/MemoCard.tsx` · 팝업 `components/PasswordChangeModal.tsx`)

| 영역 | 영역 목적 | 표시 조건 | 노출 대상 | 접기·펼치기 | 기본 표시 상태 |
|---|---|---|---|---|---|
| 상단 줄 | '리스트로 돌아가기' + 우측 ⋯ 메뉴 | 항상 | ⋯ 는 쓰기 권한 보유자 | 불가 | 표시 |
| 오류영역 | 상세 조회 실패 안내 + 복구 버튼 둘 | 조회 실패 시 | 전체 | 불가 | 숨김 |
| 좌측 열 — 회원 정보 카드 | 회원이 제출한 값(전부 읽기 전용) + 비밀번호 변경 | 조회 성공 시 | 전체 | 불가 | 표시 |
| 좌측 열 — 동의 정보 카드 | 어떤 동의를 언제 했는가(읽기 전용) | 〃 | 전체 | 불가 | 표시 |
| 우측 열 — 활동 정보 카드 | 가입·로그인·작성 활동 | 〃 | 전체 | 불가 | 표시 |
| 우측 열 — 적립금 카드 | 잔액 + 지급/차감 폼 + 증감 내역 표 | 〃 | 폼은 `update`, 삭제 열은 `remove` | 불가(내역은 페이지네이션) | 표시 |
| 우측 열 — 보유 쿠폰 카드 | 발급받은 쿠폰 목록 | 〃 | 전체 | 불가(페이지네이션) | 표시 |
| 우측 열 — 관리자 메모 카드 | 내부 참고 메모 | 〃 | 편집은 `update` | 불가 | 표시 |
| 팝업(비밀번호 변경) | 새 비밀번호 입력 | '비밀번호 변경' 클릭 | `update` 보유자 | 해당 없음 | 숨김 |
| 회원 삭제 확인 다이얼로그 | 삭제 확인 | ⋯ '회원 삭제' 클릭 | `remove` 보유자 | 해당 없음 | 숨김 |
| 적립금 내역 삭제 확인 다이얼로그 | 내역 삭제 확인 | 내역 행의 휴지통 클릭 | `remove` 보유자 | 해당 없음 | 숨김 |

**회원 정보는 전부 읽기 전용이다** — 관리자가 바꿀 수 있는 유일한 값은 비밀번호뿐이다(계정 복구 목적). 관리자 메모와 적립금은 회원의 정보가 아니라 **운영 기록**이라 예외다. **운영진 그룹 섹션이 없다** — 운영진은 관리자 관리의 관심사다.

## 6. Validation

이 화면의 검증은 **카드 안에** 있다(폼 화면이 아니므로 공통 폼 컨트롤러를 쓰지 않는다). 비밀번호 검증은 팝업 문서 §6 이 갖는다.

| 대상 항목 | 검증 조건 | 검증 시점 | 오류 메시지 | 처리 방식 |
|---|---|---|---|---|
| 적립금 금액 | 비어 있지 않고 **1 이상의 정수**일 것 | 제출 시 | `금액은 1 이상의 정수로 입력하세요.` — `PointsCard.tsx:145` | 요청을 보내지 않고 폼 하단에 표시. 입력 테두리가 오류 상태가 된다 |
| 적립금 사유 | 앞뒤 공백을 턴 값이 비어 있지 않을 것 | 제출 시 | `사유를 입력하세요.` — `PointsCard.tsx:149` | 〃 |
| 적립금 금액(차감) | 보유 적립금보다 크지 않을 것 | 제출 시 | `보유 적립금보다 많이 차감할 수 없어요.` — `PointsCard.tsx:154` | 〃 |
| 관리자 메모 | 500자 이하 | 저장 클릭 시(입력 중에는 카운터와 테두리만 바뀐다) | `메모는 500자를 넘을 수 없어요.` — `MemoCard.tsx:64` | 요청을 보내지 않고 카드 안에 표시. **입력을 자르지 않는다** |

**검증 순서가 곧 우선순위다** — 금액 → 사유 → 잔액. 한 번에 한 문장만 뜬다.

| `update` | ⋯ 의 '알림 발송' · 비밀번호 변경 버튼 · 적립금 지급/차감 폼 · 메모 저장 | `MemberDetailPage.tsx:129,278,290,294` |
| `remove` | ⋯ 의 '회원 삭제' · 적립금 내역 삭제 열 | `MemberDetailPage.tsx:222,291` |
| `export` | 이 화면에 내보내기가 없다 | — |

등록·다운로드·업로드·승인·반려는 이 화면에 존재하지 않는 동작이다.

**권한이 없을 때 화면이 어떻게 되는가**

| 없는 것 | 결과 |
|---|---|
| 화면 읽기(`read`) | 본문이 `접근 권한이 없어요` 화면으로 대체된다 |
| `update` | 비밀번호 변경 버튼·적립금 폼·메모 저장 버튼이 **사라진다.** 메모 입력은 남지만 `readOnly` 다 — 읽는 것은 허용된 정보이기 때문이다(`disabled` 로 두면 흐려지고 일부 브라우저에서 복사도 막힌다) |
| `remove` | ⋯ 의 '회원 삭제'와 적립금 내역의 삭제 **열 자체**가 사라진다 |
| `update`·`remove` 둘 다 | 상단 ⋯ 버튼 자체를 그리지 않는다 |
| 플랜(엔타이틀먼트) | 해당 없음 — `/users/**` 는 어떤 모듈에도 매핑되어 있지 않다 |
