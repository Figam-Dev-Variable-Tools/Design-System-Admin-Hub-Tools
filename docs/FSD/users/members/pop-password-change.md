# 비밀번호 변경 (팝업)

## 1. 화면 개요

| 항목 | 내용 |
|---|---|
| 화면명 | 비밀번호 변경 |
| 화면 ID | `POP-USERS-MEMBERS-PASSWORD-CHANGE` |
| 메뉴 경로 | 사용자 관리 > 회원 관리 > 회원 상세 (팝업 — 사이드바에 자리가 없다) |
| 작성자 | Admin Hub 문서화 |
| 작성일 | 2026-07-22 |
| 버전 | 1.0 |

**화면 목적** — 회원의 비밀번호를 관리자가 새로 정한다. **회원 정보 중 관리자가 바꿀 수 있는 유일한 값**이다.

**업무 배경** — 계정 복구다. 회원이 비밀번호를 잃어 스스로 재설정하지 못할 때 상담 창구가 대신 정해 준다. 반대로 말하면 이 팝업은 **다른 어떤 회원 정보도 고치지 않는다** — 이름·연락처·주소·등급은 상세 화면에서 읽기 전용 텍스트로만 보인다.

**화면 설명** — 안내 한 줄 + 비밀번호 두 칸(새 비밀번호 · 확인) + (실패 시) 배너 + 버튼 둘이다. 필드 오류는 **입력 아래 인라인**, 서버 저장 실패는 **모달 안 배너**로 자리를 나눈다.

**주요 사용자** — 회원 상세에서 `update` 권한을 가진 관리자.

**사용 시나리오**

1. 상담 중 계정 복구 요청을 받아 새 비밀번호를 정하고, 회원에게 별도 경로로 알려 준다.
2. 두 칸이 다르면 팝업이 닫히지 않고 확인 칸 아래에 이유가 뜬다.
3. 한 칸만 채운 채 Esc 를 눌렀다 → 이탈 가드가 뜨고 '취소'를 고르면 입력이 그대로 남는다.

**선행 조건** — `SCR-USERS-MEMBERS-DETAIL` 이 열려 있고 `update` 권한이 있어 '비밀번호 변경' 버튼이 보일 때만 열린다.

**후행 처리** — 성공하면 팝업이 닫히고 상세 화면이 토스트로 알린다. **상세를 다시 조회하지 않는다** — 비밀번호는 화면에 뜨지 않으므로 갱신할 표시값이 없다. **바뀐 비밀번호를 회원에게 전달하는 것은 이 앱 밖의 일이다.**

**관련 화면** — `SCR-USERS-MEMBERS-DETAIL`(부모). 공통 규약은 [공통 확인 다이얼로그 · 토스트](../../_common/index.md).

**관련 메뉴** — 해당 없음 — 메뉴 항목이 아니다.

**관련 기능** — 회원 상세의 '회원 정보' 카드가 이 팝업의 유일한 진입점이다. 비밀번호 자리에는 언제나 `••••••••` 만 보이며 **실제 값은 서버에서 내려오지 않는다.**

**관련 API** — 목적 하나 — 비밀번호 변경. **백엔드가 없다**. 연동 지점은 `TODO(backend): PATCH /api/members/:id/password` — `apps/admin/src/pages/members/data-source.ts:251`. 지금 어댑터는 값을 버리고 resolve 만 한다(`data-source.ts:252-261`).

**참고 문서** — [ID 규약](../../id-convention.md) · [인벤토리](../../inventory.md) · [회원 상세](detail.md)

**구현 파일** — `apps/admin/src/pages/members/components/PasswordChangeModal.tsx` (검증 `apps/admin/src/pages/members/validation.ts:35-77` · 요청 `apps/admin/src/pages/members/queries.ts:182-187`)

## 4. 기능 명세

### 4.1 비밀번호 변경

| 항목 | 내용 |
|---|---|
| 기능 목적 | 이 회원의 비밀번호를 새 값으로 바꾼다 |
| 실행 조건 | 팝업이 열려 있고 저장 중이 아닐 때 |
| 사용자 동작 | 두 칸 입력 → '저장' 또는 Enter |
| 시스템 처리 | 새 제출 시도마다 **직전 서버 실패 배너와 필드 오류를 먼저 걷어낸다** — `PasswordChangeModal.tsx:106-108`. 검증을 통과하면 회원 id 와 새 비밀번호를 실어 요청한다 |
| 데이터 처리 | 서버가 비밀번호를 바꾼다. 화면의 다른 값은 하나도 바뀌지 않는다 |
| Validation | §6 |
| API 호출 여부 | Y — 비밀번호 변경 1건. **상세를 다시 조회하지 않는다**(갱신할 표시값이 없다) |
| 성공 처리 | 팝업이 닫히고 부모가 토스트를 띄운다: `비밀번호를 변경했어요.` — `apps/admin/src/pages/members/MemberDetailPage.tsx:305`(부모 문서의 `MSG-USERS-MEMBERS-DETAIL-01`) |
| 실패 처리 | **팝업이 닫히지 않는다.** 모달 안 danger 배너: `비밀번호를 변경하지 못했어요. 잠시 후 다시 시도해 주세요.` — `PasswordChangeModal.tsx:68`. 저장 버튼이 되살아나 재클릭이 곧 재시도다. 입력값은 그대로 남는다 |
| 예외 처리 | 팝업이 닫히면 진행 중이던 요청이 **중단된다** — `PasswordChangeModal.tsx:75`. 중단은 실패로 치지 않아 배너도 토스트도 뜨지 않는다 — `PasswordChangeModal.tsx:88-89` |
| 화면 변경 사항 | 팝업(닫힘 또는 배너) · 부모의 토스트 |
| 후속 동작 | 없음. **바뀐 비밀번호를 회원에게 알리는 일은 이 앱 밖에 있다** — 안내 문구가 그 사실을 직접 말한다 |
| 로그 기록 여부 | 확인하지 못했다 — 비밀번호 변경을 관리자 로그로 남기는 배선을 코드에서 찾지 못했다 |

| 새 비밀번호 | **8자 이상**(공백도 유효한 문자로 센다 — trim 하지 않는다) | 〃 | `비밀번호는 8자 이상이어야 해요.` — `apps/admin/src/pages/members/validation.ts:48` | 〃 |
| 새 비밀번호 | 영문과 숫자를 **모두** 포함할 것 | 〃 | `영문과 숫자를 모두 포함해야 해요.` — `apps/admin/src/pages/members/validation.ts:56` | 〃 |
| 새 비밀번호 확인 | 앞뒤 공백을 턴 값이 비어 있지 않을 것 | 〃 | `새 비밀번호를 한 번 더 입력하세요.` — `apps/admin/src/pages/members/validation.ts:62` | 〃 |
| 새 비밀번호 확인 | 새 비밀번호와 **정확히 같을 것** | 〃 | `비밀번호가 일치하지 않아요.` — `apps/admin/src/pages/members/validation.ts:73` | 팝업 유지 · **확인 칸** 아래 인라인 |

**비밀번호는 trim 하지 않는다** — 공백도 유효한 비밀번호 문자다. 다만 '입력했는가' 판정만 공백을 턴 값으로 한다(`validation.ts:38-39`).

**확인 칸이 비어 있으면 불일치를 함께 띄우지 않는다** — 한 칸에 두 문장이 겹치지 않게 한다.

**서버 측 정책은 아직 없다** — 8자 상한선·복잡도는 화면 상수이며, 서버 정책이 생기면 계약으로 내려오도록 표시되어 있다(`validation.ts:34`).
