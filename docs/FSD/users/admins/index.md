# 관리자 관리

## 1. 화면 개요

| 항목 | 내용 |
|---|---|
| 화면명 | 관리자 관리 |
| 화면 ID | `SCR-USERS-ADMINS` |
| 메뉴 경로 | 사용자 관리 > 관리자 관리 |
| 작성자 | Admin Hub 문서화 |
| 작성일 | 2026-07-22 |
| 버전 | 1.0 |

**화면 목적** — 이 어드민을 함께 쓰는 **운영자 명부**를 본다. 그룹·역할·검색어로 좁혀 찾고, 운영진 그룹을 만들고 지우며, 새 운영자를 등록하는 화면으로 들어간다.

**업무 배경** — 권한 관리(`SCR-USERS-ROLES`)가 '무엇을 할 수 있는가'를 정하고, 이 화면이 '누가 그 역할인가'를 정한다. 역할 모델은 스스로에게서 운영자 배정을 제거하면서 그 책임을 여기로 넘겼다. 또한 운영진 그룹은 마케팅의 **발신 프로필**과 같은 실체라, 그 실체를 만드는 자리도 여기다.

**화면 설명** — 좌측은 운영진 그룹 패널(필터 + 그룹 만들기·지우기 + 안내 3문단), 우측은 탭 줄 + 등록 CTA → 검색 카드(검색 + 역할 필터) → 표 카드 → 페이지네이션이다.

**주요 사용자** — 최상위 관리자 · 운영자.

**사용 시나리오**

1. 권한 관리에서 '이 역할 운영자 3명'을 눌러 이 화면에 그 역할 필터가 걸린 채 착지한다.
2. 새 운영자를 등록한다 → 등록 화면으로 이동한다.
3. 고객지원센터 그룹을 만든다 → 발신 프로필로 쓸 수 있게 켠다.
4. 쓰지 않는 그룹을 지우려 한다 → 그 그룹에 운영자가 남아 있거나 발신 프로필로 쓰이면 이유를 먼저 보여 준다.

**선행 조건** — 로그인 · 이 화면의 `read` 권한.

**후행 처리** — 만든 그룹은 좌측 목록과 **메시지 템플릿의 발신 프로필 목록**(발신 자격을 켰다면)에 나타난다. 지운 그룹은 두 곳에서 함께 사라진다.

**관련 화면** — `SCR-USERS-ADMINS-DETAIL`(행 클릭 · 메모 아이콘) · `SCR-USERS-ADMINS-FORM`(등록·수정) · `SCR-USERS-ROLES`(역할의 정본 · `?role=` 링크의 출발점) · `SCR-USERS-SETTINGS`·`SCR-SETTINGS-OAUTH`(좌측 안내문의 링크) · `SCR-MARKETING-TEMPLATES`(발신 프로필의 소비처). 팝업 1건은 별도 문서다: [운영진 그룹 만들기](pop-create-group.md). 공통 규약은 [공통 확인 다이얼로그 · 토스트](../../_common/index.md).

**관련 메뉴** — 사이드바 `사용자 관리 > 관리자 관리`(`/users/admins`) — `apps/admin/src/shared/layout/nav-config.ts:130`.

**관련 기능** — ① 역할 목록은 **권한 스토어**에서 그대로 읽는다(복제하지 않는다) ② 권한 관리의 역할 삭제 가드가 이 화면의 명부에 배정 인원을 되묻는다 ③ 회원 관리의 그룹 만들기에서 '운영진 그룹'을 고르면 그 그룹이 여기로 들어온다(발신 자격은 꺼진 채).

**관련 API** — 목적 다섯. ① 운영자 목록 조회(그룹·역할·검색어·페이지) ② 운영진 그룹 목록 조회 ③ 그룹 생성 ④ 그룹 참조 현황 조회 ⑤ 그룹 삭제. **백엔드가 없다** — 연동 지점은 `TODO(backend)` 주석이 경계다: `GET /api/admins` · `GET /api/admin-groups` · `GET /api/sender-phones` · `POST /api/admin-groups` · `GET /api/admin-groups/:id/usage` · `DELETE /api/admin-groups/:id` — `apps/admin/src/pages/admins/data-source.ts:148,170,183,196,229,249`.

**참고 문서** — [ID 규약](../../id-convention.md) · [인벤토리](../../inventory.md) · [품질 기준](../../../reference/quality-bar.md)

**구현 파일** — `apps/admin/src/pages/admins/AdminsPage.tsx` (좌측 `components/AdminGroupPanel.tsx` · 탭+CTA `components/AdminsToolbar.tsx` · 검색 `components/AdminsSearchCard.tsx` · 표 `components/AdminsTable.tsx` · 팝업 `components/CreateAdminGroupModal.tsx` · 조회 `queries.ts` · 어댑터 `data-source.ts` · 그룹 도메인 `apps/admin/src/shared/domain/admin-group.ts`)

## 4. 기능 명세

### 4.1 목록 조회 · 필터 · 검색

| 항목 | 내용 |
|---|---|
| 기능 목적 | 그룹 · 역할 · 검색어 · 페이지로 좁힌 운영자 명부를 본다 |
| 실행 조건 | 화면 진입 또는 조건 변경 |
| 사용자 동작 | 좌측 그룹 클릭 · 역할 셀렉트 · 검색어 입력 · 페이지 이동 |
| 시스템 처리 | 역할 필터만 **주소 쿼리스트링**이 소유한다(다른 화면이 링크로 가리키는 유일한 조건이기 때문) — `AdminsPage.tsx:53-60`. 그룹·검색어·페이지는 화면 안 상태다. 역할 필터는 서버 페이지네이션 **전에** 걸린다 — 받은 10건에서 거르면 페이지마다 건수가 들쭉날쭉해지고 총 건수도 거짓이 된다 |
| 데이터 처리 | 없음(읽기만) |
| Validation | **지금 존재하는 역할이 아니면 `전체` 로 되돌린다** — 손으로 고친 값이나 **삭제된 역할을 가리키는 옛 링크**가 목록을 0건으로 만들지 않게 한다 — `AdminsPage.tsx:147-150` |

### 4.4 운영진 그룹 삭제 — 두 단계다

| 항목 | 내용 |
|---|---|
| 기능 목적 | 쓰지 않는 그룹을 지우되 **그 그룹을 가리키는 것들을 고아로 만들지 않는다** |
| 실행 조건 | 좌측에서 그룹(‘전체 운영자’가 아닌 항목)을 고른 상태 |
| 사용자 동작 | '{그룹명} 그룹 삭제' → (막히지 않았다면) 확인 다이얼로그에서 '그룹 삭제' |
| 시스템 처리 | **① 참조 현황부터 조회한다**(그 그룹의 운영자 수 · 그 그룹을 발신 프로필로 쓰는 템플릿 이름들). 조회가 끝날 때까지 다이얼로그를 열지 않는다. **② 막을 이유가 있으면 다이얼로그 대신 화면 배너로 이유를 띄운다** — 확인 버튼을 잠글 수단이 없어, 막힌 그룹에 다이얼로그를 띄우면 눌러도 매번 같은 오류만 나오는 버튼을 보여 주게 된다 — `AdminsPage.tsx:220-260` |
| 데이터 처리 | 그룹이 사라진다. 발신 자격이 켜져 있었다면 발신번호·발신 이메일도 함께 사라진다 |
| Validation | 아래 차단 사유 표 |
| API 호출 여부 | Y — 참조 현황 조회 1건 + (통과 시) 삭제 1건. 삭제 성공하면 그룹 목록과 운영자 목록을 둘 다 무효화한다 |
| 성공 처리 | 다이얼로그가 닫히고 **그룹 필터가 '전체 운영자'로 되돌아간다**(지운 그룹이 필터에 걸린 채 남으면 목록이 영원히 0건이다). 토스트 `MSG-USERS-ADMINS-02`: `'{그룹명}' 그룹을 삭제했어요.` — `AdminsPage.tsx:285` |
| 실패 처리 | **다이얼로그를 닫지 않는다.** 그 안에 danger 배너로 이유가 뜬다 — 경합으로 어댑터 가드에 걸렸다면 **그 문장이 그대로** 뜨고, 그 밖에는 `그룹을 삭제하지 못했어요. 잠시 후 다시 시도해 주세요.` — `AdminsPage.tsx:291-295` |
| 예외 처리 | 취소·Esc·딤은 진행 중 요청을 중단하고 실패로 치지 않는다. **참조 현황 조회 자체가 실패하면 지울 수 있는지 알 수 없으므로 막는다** |
| 화면 변경 사항 | 배너 또는 다이얼로그 · 좌측 그룹 목록 · 표 |
| 후속 동작 | 그룹 목록·운영자 목록 재조회. 메시지 템플릿의 발신 프로필 목록에서도 사라진다 |
| 로그 기록 여부 | 확인하지 못했다 |

**삭제 차단 사유 — 다이얼로그가 아니라 화면 배너로 뜬다.** 문구의 정본은 도메인이 갖는다(`apps/admin/src/shared/domain/admin-group.ts:96-105`) — 화면이 미리 띄우는 안내와 어댑터가 거절하며 던지는 오류가 같은 문장이어야 하기 때문이다.

| 상태 | 문구 | 출처 |
|---|---|---|
| 그 그룹에 운영자가 남아 있다 | `'{그룹명}' 그룹에는 운영자 {N}명이 속해 있어 삭제할 수 없어요. 이 운영자들을 다른 그룹으로 옮긴 뒤 다시 삭제해 주세요.` | `admin-group.ts:98` |
| 발신 프로필로 쓰는 템플릿이 있다 | `'{그룹명}' 그룹을 발신 프로필로 쓰는 메시지 템플릿이 있어 삭제할 수 없어요 ({템플릿 이름들}). 해당 템플릿의 발신 프로필을 먼저 바꾼 뒤 다시 삭제해 주세요.` | `admin-group.ts:102` |
| 발신 사용처를 **확인할 수 없다** | 위 문장과 같은 모양이되 템플릿 이름 자리에 `'__사용처 확인 불가__'` 가 들어간다 — 모르는 것을 '없음'으로 읽으면 발신 프로필로 쓰이는 그룹이 조용히 지워지기 때문이다 | `apps/admin/src/pages/admins/data-source.ts:223-227` |
| 참조 현황 조회 자체가 실패 | `그룹의 사용 현황을 확인하지 못해 삭제할 수 없어요. 잠시 후 다시 시도해 주세요.` | `AdminsPage.tsx:257` |

**확인 다이얼로그는 막을 이유가 없을 때만 열린다.** 제목 `운영진 그룹 삭제` · 확인 라벨 `그룹 삭제` — `AdminsPage.tsx:414,420`. 본문은 발신 자격에 따라 갈린다(`AdminsPage.tsx:416-418`).

| 경우 | 본문 |
|---|---|
| 발신 프로필로 쓸 수 있는 그룹 | `'{그룹명}' 그룹을 삭제할까요? 이 그룹에 속한 운영자는 없으며, 등록된 발신번호·발신 이메일도 함께 사라져 메시지 템플릿의 발신 프로필 목록에서 빠져요.` |
| 그 밖 | `'{그룹명}' 그룹을 삭제할까요? 이 그룹에 속한 운영자는 없으며, 좌측 그룹 필터에서 사라져요.` |

## 6. Validation

| 대상 항목 | 검증 조건 | 검증 시점 | 오류 메시지 | 처리 방식 |
|---|---|---|---|---|

| API 오류 | 조회는 인라인 배너, 쓰기는 다이얼로그 안 배너·팝업 안 오류다. 서버 응답 원문을 문장으로 노출하지 않는다 | 조회 실패는 표를 그리지 않는다 | 배너의 `다시 시도` |

**권한이 없을 때 화면이 어떻게 되는가**

| 없는 것 | 결과 |
|---|---|

| `remove` | 화면이 달라지지 않는다(위 격차) |
| 플랜(엔타이틀먼트) | 해당 없음 — `/users/**` 는 어떤 모듈에도 매핑되어 있지 않다 |
