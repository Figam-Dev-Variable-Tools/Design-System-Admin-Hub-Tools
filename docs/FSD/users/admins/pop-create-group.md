# 새 운영진 그룹 만들기 (팝업)

## 1. 화면 개요

| 항목 | 내용 |
|---|---|
| 화면명 | 새 운영진 그룹 만들기 |
| 화면 ID | `POP-USERS-ADMINS-CREATE-GROUP` |
| 메뉴 경로 | 사용자 관리 > 관리자 관리 (팝업 — 사이드바에 자리가 없다) |
| 작성자 | Admin Hub 문서화 |
| 작성일 | 2026-07-22 |
| 버전 | 1.0 |

**화면 목적** — 운영진 그룹을 만든다. 그룹명 · **발신 프로필로 쓸지 여부** · 대표 발신번호 · 대표 발신 이메일 넷을 받는다.

**업무 배경** — 운영진 그룹과 메시지 템플릿의 '발신 프로필'은 같은 실체다. 실무에서 운영자가 그 둘을 같은 것으로 부르므로 모델을 하나로 합쳤고, 그 실체를 만드는 화면이 필요해졌다 — `apps/admin/src/shared/domain/admin-group.ts:4-14`. 다만 **목록을 합쳤다고 모두가 발신 프로필은 아니다** — 조회·권한 필터 용도로만 쓰는 그룹까지 발송 화면 드롭다운에 뜨면, 고르면 발신번호 후보가 0개라 저장도 못 하는 항목이 된다.

**화면 설명** — 그룹명 + 발신 자격 체크박스 + 발신번호 셀렉트 + 발신 이메일 입력 + (실패 시) 오류 한 줄 + 안내 한 줄 + 버튼 둘이다. 제출하면 곧바로 만들지 않고 **생성 확인 다이얼로그**를 한 번 더 세운다.

**주요 사용자** — 관리자 관리 화면에 들어올 수 있는 관리자.

**사용 시나리오**

1. `고객지원센터` 그룹을 만들고 발신 프로필로 켠 뒤, 사전등록된 번호와 대표 메일 주소를 고른다.
2. 권한 필터 용도로만 쓸 그룹은 발신 자격을 꺼서 만든다 — 그러면 번호·주소를 요구하지 않는다.
3. 이미 있는 이름을 쓰면 확인 다이얼로그가 닫히고 폼에 그 사유가 뜬다.

**선행 조건** — `SCR-USERS-ADMINS` 좌측의 '+ 새 그룹 만들기' 를 눌렀을 때만 열린다.

**후행 처리** — 성공하면 관리자 관리의 좌측 그룹 목록과 **메시지 템플릿의 발신 프로필 목록**(발신 자격을 켰다면)에 나타난다. 그룹을 만들어도 **운영자가 자동으로 들어가지는 않는다.**

**관련 화면** — `SCR-USERS-ADMINS`(부모) · `SCR-MARKETING-TEMPLATES`·`SCR-MARKETING-TEMPLATES-FORM`(발신 프로필의 소비처). 공통 규약은 [공통 확인 다이얼로그 · 토스트](../../_common/index.md).

**관련 메뉴** — 해당 없음 — 메뉴 항목이 아니다.

**관련 기능** — 회원 관리에도 그룹 만들기 팝업이 있으나 **다른 폼이다**([새 그룹 만들기](../members/pop-create-group.md)) — 그쪽은 그룹 유형과 배송비 혜택을 받는다. 두 팝업이 공유하는 것은 **그룹명 길이 규칙 · 모달 껍데기 · 이탈 가드 · 생성 확인 한 겹 · 정본 저장소**뿐이고, 폼 본문만 갈라져 있다 — `CreateAdminGroupModal.tsx:4-17`. 회원 관리에서 '운영진 그룹'으로 만든 그룹도 같은 저장소로 들어오지만 **발신 자격이 꺼진 채** 태어난다(그 팝업에는 번호·주소 칸이 없다).

**관련 API** — 목적 둘. ① 사전등록 발신번호 목록 조회 ② 그룹 생성. **백엔드가 없다** — 연동 지점은 `TODO(backend): GET /api/sender-phones` · `POST /api/admin-groups`(요청 `{ name, phoneNumbers, emails, usableAsSender }` / 201 `AdminGroup` / 409 이름 중복) — `apps/admin/src/pages/admins/data-source.ts:183,196-198`.

**참고 문서** — [ID 규약](../../id-convention.md) · [인벤토리](../../inventory.md) · [관리자 관리](index.md)

**구현 파일** — `apps/admin/src/pages/admins/components/CreateAdminGroupModal.tsx` (검증 `apps/admin/src/pages/admins/validation.ts:38-84` · 도메인 `apps/admin/src/shared/domain/admin-group.ts` · 어댑터 `apps/admin/src/pages/admins/data-source.ts:199-207`)

## 6. Validation

규칙의 정본은 스키마다 — `apps/admin/src/pages/admins/validation.ts:38-84`. 발신번호·발신 이메일의 필수 여부는 **발신 자격 체크박스가 정한다** — 필드 하나만으로는 판정할 수 없어 객체 수준 검사로 올라가 있다.

| 대상 항목 | 검증 조건 | 검증 시점 | 오류 메시지 | 처리 방식 |
|---|---|---|---|---|
| 그룹명 | 앞뒤 공백을 턴 값이 비어 있지 않을 것 | 제출 시 | `그룹명을 입력하세요.` — `apps/admin/src/pages/admins/validation.ts:41` | 팝업 유지 · 그 칸 아래 인라인 · 그룹명 칸으로 포커스 |
| 그룹명 | 앞뒤 공백을 턴 길이가 **30자 이하** | 제출 시 | `그룹명은 30자를 넘을 수 없어요.` — `apps/admin/src/pages/admins/validation.ts:43` | 〃 |
| 그룹명 | **중복이 아닐 것** | **생성 시점(저장소)** | `같은 이름의 그룹이 이미 있어요.` — `apps/admin/src/shared/fixtures/admin-groups.ts:116` | 확인 다이얼로그를 닫고 폼 하단에 그 문장을 그대로 |
| 대표 발신번호 | **발신 자격이 켜져 있으면** 골랐을 것 | 제출 시 | `발신 프로필로 쓰려면 대표 발신번호를 골라야 해요.` — `apps/admin/src/pages/admins/validation.ts:61` | 그 칸 아래 인라인 |
| 대표 발신 이메일 | **발신 자격이 켜져 있으면** 비어 있지 않을 것 | 제출 시 | `발신 프로필로 쓰려면 대표 발신 이메일을 입력해야 해요.` — `apps/admin/src/pages/admins/validation.ts:70` | 〃 |
| 대표 발신 이메일 | 이메일 형식일 것(최소한만 본다) | 제출 시 | `이메일 주소 형식이 올바르지 않아요.` — `apps/admin/src/pages/admins/validation.ts:80` | 〃 |

**발신 자격이 꺼져 있으면 두 칸은 선택 입력이다** — 조회·권한 필터 전용 그룹은 발신하지 않으므로 번호·주소를 요구할 이유가 없다 — `validation.ts:33-34`.

**발신번호를 자유 입력으로 두지 않는 이유** — 사전등록제라 미등록 번호로는 발송 자체가 되지 않는다. 화면이 셀렉트로 고르게 하고, 검증은 '골랐는가'만 본다 — 값 자체의 형식을 여기서 다시 판정하면 등록 풀과 정규식이 어긋날 수 있다 — `validation.ts:25-27`.

**대표값을 1개씩만 받는 이유** — 그룹은 발신번호와 발신 이메일을 여러 개 가질 수 있지만, 만드는 순간에 전부 채우게 하면 입력이 무거워진다. 나머지는 그룹이 생긴 뒤 덧붙인다 — `validation.ts:29-31`.

**30자 상한은 회원 그룹 만들기와 같은 상수를 본다** — `apps/admin/src/shared/domain/admin-group.ts:64`.

**제출 전에는 오류를 띄우지 않는다** — 새 제출 시도마다 직전 오류가 먼저 지워진다.

**권한이 없을 때 화면이 어떻게 되는가** — 이 팝업은 **자기 권한 판정을 갖지 않는다.** 그리고 이 팝업을 여는 부모의 '+ 새 그룹 만들기' 버튼에도 `canCreate` 게이팅이 없다(`apps/admin/src/pages/admins/AdminsPage.tsx:322`). 즉 관리자 관리 화면의 `read` 만 가진 역할도 운영진 그룹을 만들 수 있고, 그 그룹은 발신 자격을 켜면 **메시지 템플릿의 발신 프로필 목록에 나타난다.** 같은 화면의 '운영자 등록'이 `create` 로 게이팅된 것과 대비된다. 부모 화면의 `read` 가 없으면 본문이 `접근 권한이 없어요` 화면으로 대체되어 이 팝업에 도달할 수 없다.
