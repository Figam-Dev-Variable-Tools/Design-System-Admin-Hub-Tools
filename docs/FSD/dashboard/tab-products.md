# 대시보드 — 상품 탭

## 1. 화면 개요

| 항목 | 내용 |
|---|---|
| 화면명 | 대시보드 상품 탭 |
| 화면 ID | `TAB-DASHBOARD-PRODUCTS` |
| 메뉴 경로 | 일반 관리 > 대시보드 > (업무 영역) 상품 |
| 작성자 | Admin Hub 문서화 |
| 작성일 | 2026-07-22 |
| 버전 | 1.0 |

**화면 목적** — 상품·주문 업무를 맡은 운영자가 오늘 처리할 주문/클레임 건수와 최근 주문·구매평을 한눈에 본다.

**업무 배경** — 주문이 들어오면 처리 시한이 붙고, 취소·반품·교환은 늦어질수록 고객 불만이 커진다. 이 탭은 주문 목록과 클레임 목록을 열어 보지 않고도 '지금 몇 건인가'를 알려 주고, 숫자를 누르면 그 화면으로 바로 보낸다.

**화면 설명** — 대시보드의 업무 탭 중 하나다. 오늘의 할일 카드(4항목)와 리스트 카드 2장(`최근 주문` · `최근 구매평`)으로 이루어진다. 탭 줄과 통계는 부모 화면([대시보드](index.md))의 것이며 탭을 바꿔도 남는다.

**주요 사용자** — `dashboard.tab.products` 권한을 가진 역할.

**사용 시나리오**

1. 아침에 대시보드를 열면 기본으로 이 탭이 선택되어 있다 → 신규주문 건수를 보고 눌러 주문 목록으로 간다.
2. 취소관리 건수가 0 이 아니면 눌러 취소 조건이 걸린 클레임 목록으로 간다.

**선행 조건** — 로그인 · `dashboard.tab.products` 권한. 할일·리스트 카드는 각각 `dashboard.todo` · `dashboard.lists` 권한이 더 필요하다.

**후행 처리** — 없다. 읽고 이동할 뿐이다.

**관련 화면** — 부모 [대시보드](index.md) · 다른 탭 [문의](tab-inquiries.md) · [영업](tab-sales.md) · 이동 대상 `SCR-ORDERS` · `SCR-ORDERS-CLAIMS` · `SCR-PRODUCTS-REVIEWS`.

**관련 메뉴** — 사이드바 항목이 아니다. 대시보드 안의 탭이다.

**관련 기능** — 권한 관리 화면의 위젯 토글 `탭 — 상품`. 주문·클레임·리뷰 화면의 실제 건수가 이 탭이 말해야 할 값이다.

**관련 API** — 상품 탭의 할일·리스트 조회 1건(부모 화면의 조회와 같은 것으로, 활성 탭을 키로 삼는다). **백엔드가 없다** — 지금 보이는 건수와 행은 전부 고정 픽스처다 — `apps/admin/src/pages/dashboard/api.ts:65-121`.

**참고 문서** — [ID 규약](../id-convention.md) · [인벤토리](../inventory.md)

**구현 파일** — `apps/admin/src/pages/dashboard/components/DashboardTabPanel.tsx` (탭 정의 `apps/admin/src/pages/dashboard/types.ts:19` · 권한 매핑 `useDashboardTabs.ts:18` · 데이터 `api.ts`)

## 4. 기능 명세

### 4.1 상품 탭 활성화

| 항목 | 내용 |

## 6. Validation

해당 없음 — 이 탭에는 입력 칸이 없다. 사용자가 정하는 값은 '어느 탭을 보는가'뿐이고, 그 값은 보이는 탭 목록에서 되찾아 좁혀지므로 잘못된 값이 들어올 자리가 없다 — `DashboardPage.tsx:49-55`.

| 세션 만료 | 이 탭에 자기 문구가 없다. 401 이 관측되면 로그인 화면으로 보낸다 | 부모 화면과 같은 경로 | 재로그인 |
| 데이터 충돌 | 해당 없음 — 쓰기가 없다 | — | — |
| 중복 데이터 | 해당 없음 — 아무것도 만들지 않는다 | — | — |
| 파일 업로드 실패 | 해당 없음 — 파일 업로드가 없다 | — | — |

## 8. 화면 상태(State)

| 상태 | 화면 표시 | 버튼 활성화 여부 | Loading 여부 | 사용자 안내 메시지 |
|---|---|---|---|---|
| 최초 로딩 | 할일 카드는 골격 + 스켈레톤, 리스트는 자리표시 2장 | 클릭 × | Y | 없음 |
| 데이터 있음 | 할일 4항목 + 리스트 2장 | 클릭 ○ | N | 없음 |
| 데이터 없음 | 할일 건수 전부 0(흐림), 리스트 두 장 모두 `표시할 항목이 없어요.` | 클릭 ○ | N | 리스트 빈 안내 |
| 조회 실패 | 패널이 배너로 대체된다 | — | N | `대시보드 데이터를 불러오지 못했어요. 잠시 후 다시 시도해 주세요.` |
| 탭 전환 중 | 이전 탭 내용을 남기지 않고 스켈레톤 | 클릭 × | Y | 없음 |
| 저장 중 | 해당 없음 — 저장 기능이 없다 | — | N | — |
| 읽기 전용(권한) | 이 탭은 언제나 읽기 전용이다. 권한 차이는 '보이는가/안 보이는가'로만 나타난다 | — | N | — |

**건수는 0 일 수 있고, 그것은 '모름'이 아니다** — 이 탭의 할일 건수는 조회가 성공해 돌려준 값이므로 0 은 '없다'는 사실이다. 값을 모르는 상태를 `—` 로 그리는 위젯은 대시보드의 문의 현황뿐이다([대시보드](index.md) §4.6).

등록·수정·삭제·다운로드·업로드·승인·반려는 이 탭에 존재하지 않는 동작이다.

**권한이 없을 때 화면이 어떻게 되는가** — **탭이 사라진다.** 비활성이 아니라 목록에서 빠진다 — 누를 수 없는 탭을 남기지 않는다는 규칙이다 — `apps/admin/src/pages/dashboard/useDashboardTabs.ts:7-8`. 보고 있던 중에 권한이 꺼지면 남아 있는 첫 탭으로 흘러간다. 할일·리스트 카드는 각자의 위젯 권한으로 따로 사라진다.
