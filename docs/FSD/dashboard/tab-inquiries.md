# 대시보드 — 문의 탭

## 1. 화면 개요

| 항목 | 내용 |
|---|---|
| 화면명 | 대시보드 문의 탭 |
| 화면 ID | `TAB-DASHBOARD-INQUIRIES` |
| 메뉴 경로 | 일반 관리 > 대시보드 > (업무 영역) 문의 |
| 작성자 | Admin Hub 문서화 |
| 작성일 | 2026-07-22 |
| 버전 | 1.0 |

**화면 목적** — 고객센터 업무를 맡은 상담원·운영자가 답을 기다리는 1:1 문의 건수와 최근 문의를 한눈에 본다.

**업무 배경** — 문의는 들어온 순서보다 **기다린 시간**이 중요하다. 상담원이 티켓 목록을 열어 필터를 걸기 전에, 이 탭이 '지금 몇 건이 답을 기다리는가'를 먼저 말한다.

**화면 설명** — 대시보드의 업무 탭 중 하나다. 오늘의 할일 카드(3항목)와 리스트 카드 2장(`최근 문의` · `답변 대기`)으로 이루어진다. 탭 줄과 통계는 부모 화면([대시보드](index.md))의 것이다.

**주요 사용자** — `dashboard.tab.inquiries` 권한을 가진 역할(주로 상담원).

**사용 시나리오**

1. 상담원이 대시보드를 열고 문의 탭을 고른다 → 답변대기 건수를 눌러 접수 상태로 좁혀진 티켓 목록으로 간다.
2. `최근 문의` 카드에서 어떤 문의가 들어왔는지 훑고, 전체 목록으로 넘어간다.

**선행 조건** — 로그인 · `dashboard.tab.inquiries` 권한. 할일·리스트 카드는 각각 `dashboard.todo` · `dashboard.lists` 권한이 더 필요하다.

**후행 처리** — 없다. 읽고 이동할 뿐이다.

**관련 화면** — 부모 [대시보드](index.md) · 다른 탭 [상품](tab-products.md) · [영업](tab-sales.md) · 이동 대상 `SCR-SUPPORT-TICKETS`.

**관련 메뉴** — 사이드바 항목이 아니다. 대시보드 안의 탭이다.

**관련 기능** — 권한 관리 화면의 위젯 토글 `탭 — 문의`. 대시보드 아래쪽의 **문의 현황** 카드는 이 탭과 다른 것이다 — 그쪽은 상품·프로그램 문의를 세고 결제를 쓰지 않을 때만 뜬다([대시보드](index.md) §4.6).

**관련 API** — 문의 탭의 할일·리스트 조회 1건(활성 탭을 키로 삼는다). **백엔드가 없다** — 지금 보이는 건수와 행은 고정 픽스처다 — `apps/admin/src/pages/dashboard/api.ts:123-161`.

**참고 문서** — [ID 규약](../id-convention.md) · [인벤토리](../inventory.md)

**구현 파일** — `apps/admin/src/pages/dashboard/components/DashboardTabPanel.tsx` (탭 정의 `apps/admin/src/pages/dashboard/types.ts:20` · 권한 매핑 `useDashboardTabs.ts:19` · 데이터 `api.ts`)

## 4. 기능 명세

### 4.1 문의 탭 활성화

| 항목 | 내용 |
|---|---|
| 기능 목적 | 고객센터 업무 영역의 할일·목록을 본다 |
| 실행 조건 | `dashboard.tab.inquiries` 권한이 있고, 탭 줄에서 이 탭을 골랐을 때 |
| 사용자 동작 | 탭 클릭. 상품 탭 권한이 없는 역할은 진입과 동시에 이 탭이 활성이 된다 |

**0 은 '모름'이 아니다** — 이 탭의 건수는 조회가 성공해 돌려준 값이다. 값을 모르는 상태를 `—` 로 그리는 위젯은 대시보드의 문의 현황뿐이다([대시보드](index.md) §4.6). 두 위젯이 모두 '문의'를 말하지만 세는 대상이 다르다 — 이 탭은 1:1 문의(고객센터 티켓)를, 문의 현황은 상품·프로그램 문의를 센다.

**권한이 없을 때 화면이 어떻게 되는가** — **탭이 사라진다.** 비활성이 아니라 목록에서 빠진다 — `apps/admin/src/pages/dashboard/useDashboardTabs.ts:7-8`. 보고 있던 중에 권한이 꺼지면 남아 있는 첫 탭으로 흘러간다. 할일·리스트 카드는 각자의 위젯 권한으로 따로 사라진다.
