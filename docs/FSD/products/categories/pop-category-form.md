# 카테고리 추가 · 수정 (팝업)

## 1. 화면 개요

| 항목 | 내용 |
|---|---|
| 화면명 | 카테고리 추가 · 카테고리 수정 |
| 화면 ID | `POP-PRODUCTS-CATEGORIES-CATEGORY-FORM` |
| 메뉴 경로 | 상품 관리 > 카테고리 > `카테고리 추가` / `+` / 연필 |
| 작성자 | Admin Hub 문서화 |
| 작성일 | 2026-07-22 |
| 버전 | 1.0 |

**화면 목적** — 카테고리 하나의 **이름**과 **상위 소속**을 정한다.

**업무 배경** — 카테고리는 필드가 둘뿐이라 별도 화면을 가질 무게가 아니다. 대신 소속을 고르는 일에 규칙이 있다 — 2단계까지만 만들 수 있고, 하위를 가진 대분류는 다른 대분류 밑으로 옮길 수 없다. 그 규칙을 **선택지 자체에서** 지운다(고를 수 없게 만든다).

**화면 설명** — 이름 한 칸 + 상위 카테고리 셀렉트 + 버튼 둘. 하나의 모달이 등록과 수정을 겸한다.

**주요 사용자** — 최상위 관리자 · 운영자.

**사용 시나리오**

1. `카테고리 추가` → 이름에 '가방' 을 넣고 상위는 '없음 (대분류)' 그대로 → `추가`.
2. '아우터' 행의 `+` → 상위가 '아우터' 로 채워진 채 열린다 → 이름 '코트' → `추가`.
3. 하위가 있는 '아우터' 를 수정하려 한다 → 상위 셀렉트가 잠기고 그 이유가 힌트에 뜬다.

**선행 조건** — 부모 화면(카테고리 목록)이 열려 있어야 한다. 추가는 `create` 권한이 있어야 여는 버튼이 보인다.

**후행 처리** — 저장하면 목록이 갱신되고 성공 토스트가 뜬다. **이름을 고치면 그 카테고리를 쓰는 상품의 라벨도 함께 바뀐다.**

**관련 화면** — `SCR-PRODUCTS-CATEGORIES`(부모) · `SCR-PRODUCTS-FORM`(2단 셀렉트가 같은 목록을 읽는다) · `SCR-PRODUCTS-COUPONS-FORM`(연쇄 선택). 공통 규약은 [공통 확인 다이얼로그 · 토스트](../../_common/index.md).

**관련 메뉴** — 해당 없음 — 메뉴 항목이 아니다.

**관련 기능** — 상위 선택지는 **1Depth 만**이며, 수정 중이면 자기 자신이 빠진다. 하위를 가진 카테고리는 상위 셀렉트 자체가 잠긴다.

**관련 API** — 카테고리 생성 또는 갱신 1건. **백엔드가 없다** — `apps/admin/src/pages/products/categories/data-source.ts` 의 `TODO(backend)`.

**참고 문서** — [카테고리 목록](index.md) · [ID 규약](../../id-convention.md) · [인벤토리](../../inventory.md)

**구현 파일** — `apps/admin/src/pages/products/categories/components/ProductCategoryFormModal.tsx` (검증 `../validation.ts` · 계층 규칙 `../../_shared/store.ts`)

## 6. Validation

| 대상 항목 | 검증 조건 | 검증 시점 | 오류 메시지 | 처리 방식 |
|---|---|---|---|---|
| 카테고리 이름 | 필수(공백만 불가) | 제출 시 | 공통 필수 문구 — `apps/admin/src/pages/products/categories/validation.ts:8` 의 `requiredText('카테고리 이름', 40)` | 요청 차단 · 인라인 오류 · **이름 칸으로 포커스** |
| 카테고리 이름 | 40자 이하 | 〃 | 공통 길이 문구(같은 출처) | 〃 (입력 자체도 40자에서 잘린다) |
| 상위 카테고리 | 형식 검증 없음(빈 문자열은 '최상위') | — | — | 셀렉트라 목록 밖 값이 나올 수 없다 |
| 상위 카테고리 | 자기 자신 금지 | 저장 시(저장소) | `자기 자신을 상위 카테고리로 지정할 수 없어요.` | 선택지에서 이미 빠져 있다 · 저장 실패 배너 |
| 상위 카테고리 | 실재해야 한다 | 〃 | `상위 카테고리를 찾을 수 없어요.` | 저장 실패 배너 |
| 상위 카테고리 | 2Depth 를 상위로 지정 금지 | 〃 | `카테고리는 2단계까지만 만들 수 있어요.` | 선택지에 1Depth 만 있다 · 저장 실패 배너 |
| 상위 카테고리 | 하위를 가진 것을 옮기기 금지 | 〃 | `하위 카테고리가 있는 카테고리는 다른 카테고리 밑으로 옮길 수 없어요.` | 셀렉트가 아예 잠긴다 |

**규칙의 정본은 둘로 나뉜다** — 이름의 형식은 zod 스키마가(`validation.ts`), 계층 규칙은 저장소가 갖는다. 계층은 다른 카테고리들의 상태를 봐야 하는 판단이라 폼 값만 보는 스키마로는 표현되지 않는다.

**⚠ 이름 중복을 막는 규칙이 없다** — 같은 이름의 카테고리를 두 개 만들 수 있다.
