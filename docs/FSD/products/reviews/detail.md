# 리뷰 상세

## 1. 화면 개요

| 항목 | 내용 |
|---|---|
| 화면명 | 리뷰 상세 |
| 화면 ID | `SCR-PRODUCTS-REVIEWS-DETAIL` |
| 메뉴 경로 | 상품 관리 > 리뷰 > (행 클릭) |
| 작성자 | Admin Hub 문서화 |
| 작성일 | 2026-07-22 |
| 버전 | 1.0 |

**화면 목적** — 리뷰 한 건의 내용과 신고 사유를 확인하고, **노출 여부와 관리자 답변**을 정한 뒤 고객 화면에서 어떻게 보일지 함께 본다.

**업무 배경** — 답변은 고객에게 그대로 나가는 글이라, 쓰는 동안 그 결과를 볼 수 있어야 한다. 그래서 왼쪽에 편집, 오른쪽에 **고객 화면 렌더 미리보기**를 둔다.

**화면 설명** — 2단이다: 왼쪽 `리뷰 정보`(메타 + 신고 배너 + 노출 토글 + 답변 + 버튼 셋), 오른쪽 `고객 화면 미리보기`.

**주요 사용자** — 최상위 관리자 · 운영자 · 상담원.

**사용 시나리오**

1. 1점 리뷰를 연다 → 신고 사유를 읽고 답변을 단다 → 저장 → 미리보기에 답변이 붙는다.
2. 명백한 악성 리뷰를 숨긴다 → 노출 토글을 끄고 저장 → 미리보기가 숨김 상태로 바뀐다.
3. 삭제해야 할 리뷰를 지운다 → 확인 다이얼로그 → 목록으로 돌아간다.

**선행 조건** — 로그인 · `/products/reviews` 리소스 조회 권한.

**후행 처리** — 노출 여부와 답변이 고객 화면의 리뷰에 반영된다. 삭제하면 목록으로 되돌아간다(히스토리를 남기지 않는 이동).

**관련 화면** — `SCR-PRODUCTS-REVIEWS`(목록 · '목록으로'). 공통 규약은 [공통 확인 다이얼로그 · 토스트](../../_common/index.md).

**관련 메뉴** — 사이드바에 자기 자리가 없다.

**관련 기능** — **이 화면은 결제 잠금과 무관하다.**

**관련 API** — 상세 조회 1건 · 리뷰 갱신 1건 · 리뷰 삭제 1건. **백엔드가 없다** — `apps/admin/src/pages/products/reviews/data-source.ts` 의 `TODO(backend)`.

**참고 문서** — [ID 규약](../../id-convention.md) · [인벤토리](../../inventory.md) · [리뷰 비기능 명세서](../../../NFRS/products/reviews.md)

**구현 파일** — `apps/admin/src/pages/products/reviews/ReviewDetailPage.tsx` (미리보기 `components/ReviewPreview.tsx` · 도메인 `types.ts`)

| 항목 | 내용 |
|---|---|
