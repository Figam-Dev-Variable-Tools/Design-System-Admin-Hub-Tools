# 공지 상세

> **이 문서는 콘텐츠 관리 상세 화면의 정본이다.** FAQ 상세 · 약관 버전 상세 · 처리방침 버전 상세는
> 같은 골격(뒤로 · 상단 액션 둘 · 카드 하나 · 삭제 확인 다이얼로그)을 쓰므로 **차이만** 적는다.

## 1. 화면 개요

| 항목 | 내용 |
|---|---|
| 화면명 | 공지 상세 |
| 화면 ID | `SCR-CONTENT-NOTICES-DETAIL` |
| 메뉴 경로 | 콘텐츠 관리 > 공지사항 > (행 클릭) |
| 작성자 | Admin Hub 문서화 |
| 작성일 | 2026-07-22 |
| 버전 | 1.0 |

**화면 목적** — 공지 한 건의 본문 전문과 속성을 읽기 전용으로 보여 주고, 거기서 곧바로 수정·삭제로 갈 수 있게 한다.

**업무 배경** — 목록은 제목만 보여 준다. "이 공지 내용이 뭐였더라"를 확인하려면 본문을 열어야 하는데, 목록에서 전문을 펼치면 표가 읽을 수 없게 길어진다. 그래서 본문은 이 화면이 갖는다.

**화면 설명** — 위쪽에 '목록으로'와 (조회에 성공했을 때만) 수정·삭제 버튼이 마주 보고, 아래에 카드 하나가 제목·배지·속성 목록·본문을 담는다.

**주요 사용자** — 최상위 관리자 · 콘텐츠를 다루는 운영자.

**사용 시나리오**

1. 운영자가 목록에서 공지를 눌러 본문을 확인한다 → '목록으로'를 눌러 조건이 걸린 목록으로 돌아온다.
2. 운영자가 본문의 오타를 발견한다 → '수정'을 눌러 폼으로 간다.
3. 운영자가 잘못 올린 공지를 여기서 지운다 → 확인 후 목록으로 이동하며 성공 토스트가 뜬다.

**선행 조건** — 로그인 · 콘텐츠 관리 플랜 모듈 · 주소의 공지 id 가 실재해야 한다.

**후행 처리** — 삭제하면 목록으로 돌아가고, 그 이동은 히스토리를 덮어써 뒤로가기로 사라진 공지 화면에 되돌아오지 않는다 — `apps/admin/src/pages/content/notices/NoticeDetailPage.tsx:124`.

**관련 화면** — [공지사항 목록](index.md)(`SCR-CONTENT-NOTICES`) · [공지 등록·수정](form.md)(`SCR-CONTENT-NOTICES-FORM`) · 공통 규약은 [공통 확인 다이얼로그 · 토스트](../../_common/index.md).

**관련 메뉴** — 사이드바에 자리가 없다. 목록의 잎(`/content/notices`)이 이 화면의 권한 리소스를 덮는다.

**관련 기능** — 목록의 행 클릭이 이 화면의 유일한 진입 경로다(등록 폼은 저장 뒤 목록으로 가고, 수정 폼은 저장 뒤 이 화면으로 온다 — `apps/admin/src/pages/content/notices/NoticeFormPage.tsx:159`).

**관련 API** — 상세 조회 1건 · 삭제 1건. **백엔드가 없다** — `apps/admin/src/pages/content/notices/data-source.ts:171`(상세) · `:213`(삭제).

**참고 문서** — [ID 규약](../../id-convention.md) · [인벤토리](../../inventory.md)

**구현 파일** — `apps/admin/src/pages/content/notices/NoticeDetailPage.tsx`
