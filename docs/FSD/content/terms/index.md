# 약관 관리

> **이 문서는 버전 이력형 화면의 정본이다.** [개인정보 처리방침](../privacy/index.md)은 종류 축이
> 없다는 것 하나만 빼면 이 화면과 같아, 그 문서는 차이만 적는다.
>
> 이 묶음에서 이 화면이 어려운 이유는 목록의 모양이 아니라 **버전과 시행일**이다 —
> §4.5 가 지금 코드가 실제로 하는 판정을 적는다.

## 1. 화면 개요

| 항목 | 내용 |
|---|---|
| 화면명 | 약관 관리 |
| 화면 ID | `SCR-CONTENT-TERMS` |
| 메뉴 경로 | 콘텐츠 관리 > 약관 관리 |
| 작성자 | Admin Hub 문서화 |
| 작성일 | 2026-07-22 |
| 버전 | 1.0 |

**화면 목적** — 약관 종류(이용약관·전자금융거래·마케팅 수신 동의)를 고르고, 그 종류의 **버전 이력**을 시행일과 상태와 함께 훑고, 새 버전을 등록하거나 지운다.

**업무 배경** — 약관은 고쳐 쓰는 문서가 아니라 **버전을 쌓는 문서**다. 분쟁이 생기면 답해야 하는 질문은 "그때 이 사람이 동의한 문서가 무엇이었나"이고, 그 답은 버전이다 — `apps/admin/src/shared/domain/terms-version.ts:5-8`. 그래서 이 화면은 옛 버전을 지우는 곳이 아니라 **남겨 두는 곳**이다.

**화면 설명** — 왼쪽에 약관 종류 목록, 오른쪽에 검색 + '새 버전 등록' + 버전 이력 표가 온다. 현재 시행본에는 `현재` 배지가 붙고, 행(또는 버전 링크)을 누르면 전문을 보는 상세로 간다.

**주요 사용자** — 최상위 관리자 · 약관을 관리하는 운영자(법무·기획).

**사용 시나리오**

1. 운영자가 '이용약관'을 고르고 v2.0 을 등록한다 → 시행일을 미래로, 상태를 '시행예정'으로 둔다.
2. 시행일이 되면 v2.0 을 '시행중'으로, v1.1 을 '만료'로 바꾼다(⚠ 자동으로 바뀌지 않는다 — §4.5).
3. 잘못 등록한 버전을 지운다 → 확인 다이얼로그를 거친다.

**선행 조건** — 로그인 · 콘텐츠 관리 플랜 모듈(`cms.pages`) — `apps/admin/src/shared/entitlements/module-resources.ts:70`.

**후행 처리** — 시행 중인 버전은 동의 이력 화면(`SCR-USERS-CONSENTS`)이 **재동의 대상 산출**에 쓴다. 그 조회는 공통 층의 등록기를 지나며, 배선되지 않았을 때는 빈 배열이 아니라 `null`('모른다')을 돌려준다 — `apps/admin/src/shared/domain/terms-version.ts:19-22,56-64`.

**관련 화면** — [약관 버전 상세](detail.md)(`SCR-CONTENT-TERMS-DETAIL`) · [약관 버전 등록·수정](form.md)(`SCR-CONTENT-TERMS-FORM`) · [개인정보 처리방침](../privacy/index.md)(같은 형태) · 동의 이력 `SCR-USERS-CONSENTS` · 공통 규약은 [공통 확인 다이얼로그 · 토스트](../../_common/index.md).

**관련 메뉴** — 콘텐츠 관리 가지의 여섯 번째 잎 — `apps/admin/src/shared/layout/nav-config.ts:149`.

**관련 기능** — 동의 이력의 재동의 대상 산출이 이 화면의 '시행중' 버전과 시행일을 읽는다.

**관련 API** — 약관 종류 목록 조회 1건 · 그 종류의 버전 목록 조회 1건 · 삭제 1건. **백엔드가 없다** — `apps/admin/src/pages/content/terms/data-source.ts:60`(종류) · `:67`(버전 목록) · `:118`(삭제).

**참고 문서** — [ID 규약](../../id-convention.md) · [인벤토리](../../inventory.md)

**구현 파일** — `apps/admin/src/pages/content/terms/TermsPage.tsx` (표 `apps/admin/src/shared/ui/VersionHistoryTable.tsx` · 조회 `queries.ts` · 데이터 `data-source.ts` · 타입 `types.ts`)

## 4. 기능 명세

### 4.1 약관 종류 고르기

| 항목 | 내용 |
|---|---|
| 기능 목적 | 볼 약관 종류를 정한다 |
| 실행 조건 | 종류 목록이 도착했을 때 |
| 사용자 동작 | 왼쪽 목록에서 종류 클릭 |
| 시스템 처리 | 고른 값을 주소의 `type` 에 쓴다. **주소가 비어 있으면 첫 종류를 고른 것으로 계산한다** — 기본 화면의 주소를 둘로 가르지 않기 위해 주소에 써 넣지는 않는다 — `apps/admin/src/pages/content/terms/TermsPage.tsx:117-125` |
| 데이터 처리 | 없음 — 조회 조건만 바뀐다 |
| Validation | **목록에 없는 종류는 첫 종류로 접는다** — 지워진 종류나 오타(`?type=x`)로 빈 화면에 착지하지 않게 한다. 다만 종류 목록이 도착하기 전에는 주소 값을 그대로 믿는다(그래야 첫 조회가 옳은 종류로 나간다) — `apps/admin/src/pages/content/terms/TermsPage.tsx:120-131` |
| API 호출 여부 | Y — 그 종류의 버전 목록을 다시 조회한다 |
| 성공 처리 | 오른쪽 표가 그 종류의 이력으로 바뀐다. **표를 비우지 않고** 이전 종류의 행을 유지한 채 새 응답으로 바뀐다 — `apps/admin/src/pages/content/terms/queries.ts:49` |
| 실패 처리 | 표가 사라지고 배너: `약관을 불러오지 못했어요.` — `apps/admin/src/pages/content/terms/TermsPage.tsx:279` |
| 예외 처리 | 종류가 바뀌면 선택이 해제된다 — `apps/admin/src/pages/content/terms/TermsPage.tsx:147` |
| 화면 변경 사항 | 필터영역 · 목록영역 · 주소 |
| 후속 동작 | 상세·폼에서 돌아왔을 때 이 종류가 복원된다 |
| 로그 기록 여부 | N |

### 4.5 시행일과 상태 — 지금 코드가 하는 판정

| 항목 | 내용 |
|---|---|
| 기능 목적 | 어느 버전이 지금 시행본인지 화면에 표시한다 |
| 실행 조건 | 목록·상세를 그릴 때마다 |
| 사용자 동작 | 없음(표시) · 상태 자체는 폼에서 사람이 고른다 |
| 시스템 처리 | **`현재` 배지의 판정은 시행일이 아니라 저장된 상태다** — `isCurrent(version)` 는 `status === 'active'` 하나만 본다 — `apps/admin/src/pages/content/terms/types.ts:46-49`. 시행일(`effectiveDate`)은 표에 **그려지기만 하고 어떤 판정에도 쓰이지 않는다** |
| 데이터 처리 | 상태는 저장되는 값이다(`active`·`scheduled`·`archived`) — `apps/admin/src/pages/content/terms/types.ts:13-14` |
| Validation | 시행일은 형식만 검증한다(폼 §6). **시행일과 상태의 앞뒤가 맞는지는 검사하지 않는다** |
| API 호출 여부 | N — 화면에서 판정한다 |
| 성공 처리 | `시행중` 버전에 `현재` 배지가 붙는다 |
| 실패 처리 | 실패할 것이 없다 |
| 예외 처리 | ⚠ **여기서 세 가지가 성립한다.** ① 시행일이 지나도 상태는 저절로 바뀌지 않는다 — 사람이 폼에서 바꿔야 한다. ② 시행일이 미래인데 상태를 `시행중`으로 고르면 그 버전에 `현재` 배지가 붙는다. ③ **한 종류에 `시행중` 버전이 여럿 있어도 막지 않는다** — 배지가 여러 줄에 붙는다. 세 가지 모두 화면·검증·데이터 소스 어디에도 막는 규칙이 없다. (뉴스는 같은 문제를 '저장하지 않고 매번 계산한다'로 풀었다 — `apps/admin/src/shared/domain/publish-schedule.ts:10-15`) |
| 화면 변경 사항 | 목록영역의 상태 열과 `현재` 배지 · 상세의 배지 |
| 후속 동작 | 동의 이력의 재동의 대상 산출이 '시행중' 버전과 그 시행일을 읽는다 — `apps/admin/src/shared/domain/terms-version.ts:31-39` |
| 로그 기록 여부 | N |
