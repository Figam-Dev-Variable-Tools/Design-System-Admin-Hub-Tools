# FAQ

> 이 화면은 [공지사항 목록](../notices/index.md)(`SCR-CONTENT-NOTICES`)과 **같은 골격**이다 —
> 좌측 필터 · 검색 · 등록 · 선택 바 · 표 · 페이지네이션 · 조회 실패 배너 · 삭제 확인 둘.
> 아래 각 절은 그 정본과 **다른 것만** 적는다. 다른 것은 셋이다: **카테고리 관리 팝업** ·
> **목록에서 바로 켜고 끄는 노출 토글(일괄 포함)** · **드래그로 바꾸는 정렬 순서**.

## 1. 화면 개요

| 항목 | 내용 |
|---|---|
| 화면명 | FAQ |
| 화면 ID | `SCR-CONTENT-FAQ` |
| 메뉴 경로 | 콘텐츠 관리 > FAQ |
| 작성자 | Admin Hub 문서화 |
| 작성일 | 2026-07-22 |
| 버전 | 1.0 |

**화면 목적** — 홈페이지에 걸리는 자주 묻는 질문을 찾고, 노출 여부를 바로 켜고 끄고, 보이는 순서를 바꾸고, 지운다.

**업무 배경** — FAQ 는 "지금 문의가 몰리는 질문을 맨 위로 올린다"가 일상이다. 그래서 이 화면은 목록을 읽는 곳이 아니라 **순서와 노출을 만지는 곳**이다. 질문 하나를 잠시 내리는 일도 폼에 들어갔다 나오지 않고 목록에서 끝나야 한다.

**화면 설명** — 왼쪽에 카테고리·노출 여부 필터, 오른쪽에 검색 + '카테고리 관리' + 'FAQ 등록' + 표 + 페이지네이션이 선다. 표의 각 행에는 노출 토글이 있고, 조건이 걸리지 않은 상태에서는 행을 끌어 순서를 바꿀 수 있다.

**주요 사용자** — 최상위 관리자 · 콘텐츠를 다루는 운영자.

**사용 시나리오**

1. 운영자가 '결제' 카테고리의 '숨김' 만 걸어 한 건씩 손보고 상세에 갔다 돌아온다 → 조건이 그대로 살아 있다.
2. 문의가 몰리는 질문을 맨 위로 끌어 올린다 → 놓는 즉시 저장되고 성공 토스트가 뜬다.
3. 새 카테고리 '배송'을 만들려고 '카테고리 관리'를 연다 → 팝업에서 만들고 닫는다.

**선행 조건** — 로그인 · 콘텐츠 관리 플랜 모듈(`cms.pages`) — `apps/admin/src/shared/entitlements/module-resources.ts:67`.

**후행 처리** — 노출을 끄면 사용자 화면에서 숨는다 — `apps/admin/src/pages/content/faq/types.ts:31`. 홈페이지는 이 리포 밖이라 반영 경로를 코드에서 확인하지 못했다.

**관련 화면** — [FAQ 상세](detail.md) · [FAQ 등록·수정](form.md) · [FAQ 카테고리 관리](pop-manage-categories.md)(`POP-CONTENT-FAQ-MANAGE-CATEGORIES`) · 공통 규약은 [공통 확인 다이얼로그 · 토스트](../../_common/index.md).

**관련 메뉴** — 콘텐츠 관리 가지의 세 번째 잎 — `apps/admin/src/shared/layout/nav-config.ts:146`.

**관련 기능** — 카테고리 관리 팝업에서 만든 카테고리는 이 화면의 왼쪽 필터와 [FAQ 폼](form.md)의 분류 선택지에 함께 나타난다. 고객센터의 `자주 묻는 질문`(`SCR-SUPPORT-FAQ`)은 **다른 화면·다른 자료**다.

**관련 API** — 목록 조회 · 카테고리 목록 조회 · 단건 삭제 · 노출 토글 · 정렬 저장 다섯. **백엔드가 없다** — `apps/admin/src/pages/content/faq/data-source.ts:148`(목록) · `:141`(카테고리) · `:307`(삭제) · `:267`(노출) · `:250`(정렬).

**참고 문서** — [ID 규약](../../id-convention.md) · [인벤토리](../../inventory.md)

**구현 파일** — `apps/admin/src/pages/content/faq/FaqPage.tsx` (필터 `components/FaqFilters.tsx` · 표 `components/FaqTable.tsx` · 팝업 `components/ManageFaqCategoriesModal.tsx` · 조회 `queries.ts` · 데이터 `data-source.ts`)

---

**⚠ 지금 코드에서 확인되는 사실** — 등록·수정·삭제는 픽스처를 바꾸지 않고 성공만 돌려준다(`apps/admin/src/pages/content/faq/data-source.ts:292-312`). 반면 **정렬 저장과 노출 토글은 실제로 픽스처를 바꾼다**(`:257,275`) — 그래서 이 화면에서 순서와 노출만은 새로고침 전까지 눈에 남는다. 쓰기 권한 게이팅은 없다(§9).

## 4. 기능 명세

### 4.1 목록 조회 · 4.2 필터 · 4.3 검색

공지사항 목록의 같은 기능과 **동작이 같다**. 다른 것만 적는다.

| 항목 | 차이 |
|---|---|
| 주소 파라미터 | `category` · `visibility` · `q` · `page` — `apps/admin/src/pages/content/faq/FaqPage.tsx:47` |
| 카테고리 값의 검증 | **좁히지 않는다.** 카테고리는 서버가 만드는 값이라 컴파일 시점의 허용 목록이 없다 — `apps/admin/src/pages/content/faq/FaqPage.tsx:115-116`. 없는 카테고리를 주소에 적으면 0건 목록이 나온다 |
| 조회 실패 문구 | `FAQ 를 불러오지 못했어요.` — `apps/admin/src/pages/content/faq/FaqPage.tsx:420` |
| 빈 상태 문구 | `조회된 FAQ 가 없어요.` — `apps/admin/src/pages/content/faq/components/FaqTable.tsx:151` |
| 배지 건수 | 카테고리별·노출별 모두 **전체 기준**이다(검색·필터와 무관) — `apps/admin/src/pages/content/faq/data-source.ts:158-159` |

### 4.4 노출 여부 토글 (단건)

| 항목 | 내용 |
|---|---|
| 기능 목적 | 폼에 들어가지 않고 목록에서 바로 FAQ 를 노출하거나 숨긴다 |
| 실행 조건 | 그 행이 이미 토글 요청 중이 아닐 때 — `apps/admin/src/pages/content/faq/FaqPage.tsx:253` |
| 사용자 동작 | 행의 토글 클릭 |
| 시스템 처리 | 그 행의 토글만 잠그고 요청 1건을 보낸다. 다른 행은 계속 만질 수 있다 |
| 데이터 처리 | 그 FAQ 의 노출 여부만 바뀐다. 다른 값은 건드리지 않는다 |
| Validation | 해당 없음 |
| API 호출 여부 | Y — 노출 여부 변경 1건 |
| 성공 처리 | 토글이 새 상태로 남고 토스트가 뜬다. 노출로 바꿨으면 `MSG-CONTENT-FAQ-03` `'{질문}' 를 노출해요.`, 숨김이면 `MSG-CONTENT-FAQ-04` `'{질문}' 를 숨겼어요.` — `apps/admin/src/pages/content/faq/FaqPage.tsx:259` |
| 실패 처리 | 실패 토스트 `MSG-CONTENT-FAQ-E02`: `노출 상태를 변경하지 못했어요. 잠시 후 다시 시도해 주세요.` — `apps/admin/src/pages/content/faq/FaqPage.tsx:263`. 이 토스트에는 **'다시 시도'가 붙어 있고**, 누르면 같은 값으로 다시 시도한다 — `:264`. 실패 토스트는 자동으로 사라지지 않는다 |
| 예외 처리 | 요청이 중단되면 아무 표시도 하지 않는다 — `apps/admin/src/pages/content/faq/FaqPage.tsx:262`. 성공이든 실패든 그 행의 잠금은 반드시 풀린다 — `:267` |
| 화면 변경 사항 | 그 행의 토글 |
| 후속 동작 | 목록 조회가 무효화되어 다시 불린다(표는 비워지지 않는다) |
| 로그 기록 여부 | 확인하지 못했다 |

### 4.5 정렬 순서 바꾸기 (드래그 · 이동 버튼)

| 항목 | 내용 |
|---|---|
| 기능 목적 | 홈페이지에서 FAQ 가 보이는 순서를 바꾼다 |
