# 검색어 분석

> **조회 조건 축과 껍데기는 [방문자 통계](../visitors/index.md) 와 같다.**
> **이 문서는 그 한 벌을 다시 적지 않고 이 화면의 차이만 적는다.**
> 이 화면은 **플랜으로 잠기는 심화 리포트 3종** 중 하나이고, **검색 입력이 있는 유일한 통계 화면**이다.

## 1. 화면 개요

| 항목 | 내용 |
|---|---|
| 화면명 | 검색어 분석 |
| 화면 ID | `SCR-STATS-KEYWORDS` |
| 메뉴 경로 | 분석 · 운영 > 통계 > 검색어 분석 |
| 작성자 | Admin Hub 문서화 |
| 작성일 | 2026-07-22 |
| 버전 | 1.0 |

**화면 목적** — 손님이 무엇을 찾아 들어왔는지, 그중 우리 카탈로그가 받아내지 못한 검색어가 무엇인지 본다. 결과 없음으로 표시된 검색어는 상품을 추가하거나 동의어를 걸어 지금 메울 수 있는 자리다 — `apps/admin/src/pages/stats/keywords/KeywordStatsPage.tsx:242`.

**업무 배경** — 손님이 찾아 들어온 검색어인데 우리 카탈로그에 걸리는 상품이 0개라면, 그것은 운영자가 오늘 손댈 수 있는 매출 누수다 — `apps/admin/src/pages/stats/keywords/types.ts:12-18`.

**화면 설명** — 배치는 다른 통계 화면과 같되 **조회 조건 바에 검색 입력이 있다.** 세그먼트는 `검색어 유형`(전체·결과 있음·결과 없음)이고 **드릴다운 축이 하나뿐**이라 축 전환 컨트롤이 없다. 표가 이 화면의 본체이며 기본 정렬은 검색수 내림차순이다.

**주요 사용자** — 최상위 관리자 · 상품/MD 담당 운영자.

**사용 시나리오**

1. 세그먼트를 `결과 없음` 으로 좁혀 카탈로그가 받아내지 못한 검색어만 본다.
2. KPI 의 `검색결과 없음 검색어` 가 지난 기간보다 늘었는지 확인한다.
3. 특정 낱말이 궁금하면 검색 입력에 쳐서 그 검색어만 남긴다.
4. `구매전환율` 헤더를 눌러 전환이 좋은 검색어를 찾는다.
5. 조건 그대로 내보내 상품 등록·동의어 작업 목록으로 쓴다.

**선행 조건** — 로그인. 이 화면의 읽기 권한. `고급 통계` 모듈이 이 계정에 열려 있어야 한다(§9).

**후행 처리** — 아무것도 바꾸지 않는다. 다만 이 화면이 알려 준 검색어를 근거로 상품 관리(`SCR-PRODUCTS`)에서 상품이나 동의어를 손보게 된다 — 그 손질은 **다른 화면의 일이며 링크가 배선되어 있지 않다**.

**관련 화면** — 같은 껍데기: `SCR-STATS-VISITORS` · `SCR-STATS-MEMBERS` · `SCR-STATS-REVENUE` · `SCR-STATS-ORDERS` · `SCR-STATS-TRAFFIC`. 유입 분석이 같은 유입을 '어디서 왔나'로 쪼개는 화면이다.

**관련 메뉴** — 사이드바 `분석 · 운영 > 통계` 의 여섯 번째 항목 — `apps/admin/src/shared/layout/nav-config.ts:272`. **플랜으로 잠긴다** — `stats.advanced` 모듈이 이 경로를 지배한다 — `apps/admin/src/shared/entitlements/module-resources.ts:97-102`.

**관련 기능** — 상품 관리(`SCR-PRODUCTS`)가 이 화면이 가리키는 누수를 메우는 화면이다.

**관련 API** — 목적 하나: 조회 기간과 비교 기간의 검색어 집계(검색어별 합계 · 일자별)를 한 번에 받는다. 쓰기 목적의 호출이 없다. 경계는 `apps/admin/src/pages/stats/keywords/data-source.ts` 의 `TODO(backend)` 다.

**참고 문서** — [방문자 통계](../visitors/index.md)(껍데기 정본) · [ID 규약](../../id-convention.md) · [인벤토리](../../inventory.md)

**구현 파일** — `apps/admin/src/pages/stats/keywords/KeywordStatsPage.tsx` · `keywords/types.ts` · `keywords/data-source.ts`

### 4.3 '참조검색어 없음'은 결과 없음이 아니다

| 항목 | 내용 |
|---|---|
| 기능 목적 | **손댈 수 없는 것을 손댈 것으로 세지 않는다** |
| 실행 조건 | 항상 |
| 사용자 동작 | 없음 |
| 시스템 처리 | 참조검색어 없음 행은 별도 상태로 구분되며, 결과 없음 판정에서 제외된다 — `keywords/types.ts:47-57,104-108` |
| 데이터 처리 | 없음 |
| Validation | 해당 없음 |
| API 호출 여부 | N |
| 성공 처리 | 그 행의 `검색결과 수` 는 `—` 이고, `결과 없음` 배지가 붙지 않으며, KPI 의 `검색결과 없음 검색어` 에도 세지 않는다 |
| 실패 처리 | 실패 경로가 없다 |
| 예외 처리 | **세그먼트로 걸러도 그 행은 `전체` 에서만 보인다** — 검색어를 모르니 결과를 셀 수 없어 `결과 있음` 에도 `결과 없음` 에도 들지 않는다 — `keywords/types.ts:110-114` |
| 화면 변경 사항 | KPI · 드릴다운영역 |
| 후속 동작 | 없음 |
| 로그 기록 여부 | N |

**'참조검색어 없음'은 버그가 아니라 하나의 실제 행이다** — 유입 URL 에 검색어 파라미터가 없을 때 그대로 표시되는 값이며, 출처는 있는데 검색어를 알 수 없는 유입이다 — `keywords/types.ts:4-7`.

### 4.4 ⚠ '검색결과 없음' 지표는 업계 관례가 아니다

| 항목 | 내용 |
|---|---|
| 사실 | 이 화면이 1급 지표로 세운 `검색결과 없음 검색어` 는 국내 커머스 플랫폼의 통계 화면에 **없는 지표**다. 코드가 그 사실을 명시적으로 적고 있다 — `keywords/types.ts:11-18` |
| 왜 실었는가 | 의도한 차별점이다: 손님이 찾아 들어온 검색어인데 걸리는 상품이 0개라면 그것은 오늘 메울 수 있는 매출 누수다(상품을 넣거나 동의어를 걸면 메워진다) |
| 명세의 판단 | **결함이 아니라 의도된 확장**이다. 다만 뒤에 읽는 사람이 '다른 플랫폼에도 있는 지표겠거니' 하고 넘겨짚지 않도록 여기 적어 둔다 |
