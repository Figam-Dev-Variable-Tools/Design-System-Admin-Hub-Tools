# 방문자 통계

> **통계 6화면은 같은 조회 조건 축과 같은 껍데기를 공유한다.** 조회 조건 바(기간 프리셋 · 직접 입력 ·
> 비교 기준 · 세그먼트 · 검색 · 내보내기) · 조회 상태 4분기 · KPI 줄 · 추이 카드 · 드릴다운 표 ·
> 구성비 막대 · 빈 상태 · 권한 게이팅이 한 벌(`apps/admin/src/pages/stats/_shared/**`)이고,
> 각 화면이 주는 것은 **세그먼트 축 · 드릴다운 축 · 추이 지표 · KPI · 컬럼 · 기본 정렬** 여섯뿐이다.
> **그 한 벌의 전부를 이 문서가 갖는다.** [회원 통계](../members/index.md) · [매출 통계](../revenue/index.md) ·
> [주문 통계](../orders/index.md) · [유입 분석](../traffic/index.md) · [검색어 분석](../keywords/index.md)
> 다섯 문서는 이 문서를 가리키고 **자기 화면의 차이만** 적는다.

## 1. 화면 개요

| 항목 | 내용 |
|---|---|
| 화면명 | 방문자 통계 |
| 화면 ID | `SCR-STATS-VISITORS` |
| 메뉴 경로 | 분석 · 운영 > 통계 > 방문자 통계 |
| 작성자 | Admin Hub 문서화 |
| 작성일 | 2026-07-22 |
| 버전 | 1.0 |

**화면 목적** — 기간을 좁혀 비교하고, 방문자 유형과 시간대·요일까지 파고드는 방문 분석이다. 대시보드의 방문자 위젯이 '지금 몇 명'이라면 여기서는 '지난 기간과 견주어 어떻게 달라졌는지'를 본다 — `apps/admin/src/pages/stats/visitors/VisitorStatsPage.tsx:255`.

**업무 배경** — 대시보드 위젯은 요약이라 '왜 그런가'에 답하지 못한다. 방문이 줄었을 때 그것이 특정 시간대의 문제인지, 재방문이 마른 것인지, 지난주와 견주어 실제로 줄어든 것인지는 조건을 좁혀 봐야만 알 수 있다.

**화면 설명** — 위에서부터 화면 설명 한 줄 → 조회 조건 바 → KPI 카드 5장 → 방문 추이 차트 → 드릴다운 표(일자별·시간대별·요일별) 순이다. 조회 조건은 전부 주소에 실려 링크로 공유된다.

**주요 사용자** — 최상위 관리자 · 마케팅 담당 운영자 · 내보내기 권한이 있는 역할.

**사용 시나리오**

1. 기간을 `최근 30일`, 비교를 `직전 기간` 으로 두고 KPI 의 증감을 확인한다.
2. 방문이 줄었으면 드릴다운 축을 `시간대별` 로 바꿔 어느 시간대가 빠졌는지 본다.
3. 방문자 유형을 `다시 온 방문자` 로 좁혀 재방문이 마른 것인지 확인한다.
4. 조건을 그대로 둔 채 `엑셀 내보내기` 로 조건 전체를 받아 보고서에 붙인다.
5. 좁혀 놓은 조건 그대로 주소를 복사해 동료에게 보낸다.

**선행 조건** — 로그인. 이 화면의 읽기 권한. 방문 집계가 서버에 쌓여 있어야 한다(이 화면은 아무것도 만들지 않는다).

**후행 처리** — **이 화면은 아무것도 바꾸지 않는다.** 읽고, 파일로 받을 뿐이다.

**관련 화면** — 같은 껍데기를 쓰는 다섯 화면: `SCR-STATS-MEMBERS` · `SCR-STATS-REVENUE` · `SCR-STATS-ORDERS` · `SCR-STATS-TRAFFIC` · `SCR-STATS-KEYWORDS`. `SCR-DASHBOARD` 의 방문자 카드가 이 화면의 요약판이다. 공통 규약은 [공통 확인 다이얼로그 · 토스트](../../_common/index.md).

**관련 메뉴** — 사이드바 `분석 · 운영 > 통계` 의 첫 항목 — `apps/admin/src/shared/layout/nav-config.ts:267`. **이 화면은 플랜으로 잠기지 않는다** — 통계 묶음 자체는 늘 열리고, 상위 플랜의 것은 심화 리포트 세 화면(매출·유입·검색어)뿐이다 — `apps/admin/src/shared/entitlements/module-resources.ts:97-102`.

**관련 기능** — 권한 관리(`SCR-USERS-ROLES`)가 이 화면의 읽기·내보내기 권한을 정한다. 결제 설정(`SCR-SETTINGS-PAYMENT`)은 **이 화면에는 영향이 없다**(매출·주문 통계만 갈아 끼워진다).

**관련 API** — 목적 하나: 조회 기간과 비교 기간의 방문 집계(일자별·시간대별·요일별)를 한 번에 받는다. **쓰기 목적의 호출이 없다.** 백엔드는 아직 없고, 경계는 `apps/admin/src/pages/stats/visitors/data-source.ts:6-8` 의 `TODO(backend)` 세 줄이다.

**참고 문서** — [ID 규약](../../id-convention.md) · [인벤토리](../../inventory.md) · [품질 기준](../../../reference/quality-bar.md)

| Validation | §6 |
| API 호출 여부 | Y — 새 기간으로 재조회 |
