# 매출 통계

> **조회 조건 축과 껍데기는 [방문자 통계](../visitors/index.md) 와 같다.**
> **이 문서는 그 한 벌을 다시 적지 않고 이 화면의 차이만 적는다.**
> 이 화면은 **플랜으로 잠기는 심화 리포트 3종** 중 하나다(§9).

## 1. 화면 개요

| 항목 | 내용 |
|---|---|
| 화면명 | 매출 통계 |
| 화면 ID | `SCR-STATS-REVENUE` |
| 메뉴 경로 | 분석 · 운영 > 통계 > 매출 통계 |
| 작성자 | Admin Hub 문서화 |
| 작성일 | 2026-07-22 |
| 버전 | 1.0 |

**화면 목적** — 결제수단별로 결제·환불·순매출을 갈라 보고 비교 기간과 견준다. 순매출은 결제합계에서 환불합계를 뺀 금액이라, 결제합계만 세는 화면과는 다른 숫자를 낸다 — `apps/admin/src/pages/stats/revenue/RevenueStatsPage.tsx:300`.

**업무 배경** — 한 숫자로 '매출'을 말하면 환불이 얼마였는지 영영 보이지 않는다. '매출'을 결제합계로 부르는 화면과 순매출로 부르는 화면이 섞이면 같은 기간의 매출이 두 값으로 보고된다 — `apps/admin/src/pages/stats/revenue/types.ts:9-12`.

**화면 설명** — 배치는 다른 통계 화면과 같되 **조회 조건 바 아래에 부가세 고지가 상주한다.** 세그먼트는 `결제수단`, 드릴다운 축은 `일자별`·`결제수단별` 이며, 일자별 표에는 과세/면세/영세 세 칸이 더 붙는다.

**주요 사용자** — 최상위 관리자 · 정산 담당 · 경영 보고 담당.

**사용 시나리오**

1. 기간을 `지난 달` 로 두고 `순매출` 과 `환불합계` 를 비교 기간과 견준다.
2. 세그먼트를 `간편결제` 로 좁혀 그 수단의 객단가를 본다.
3. 드릴다운 축을 `결제수단별` 로 바꿔 어느 수단이 얼마를 차지하는지 본다.
4. 조건 그대로 내보내 정산 자료의 참고로 쓴다(제출용이 아니다 — §3 고지).

**선행 조건** — 로그인. 이 화면의 읽기 권한. `고급 통계` 모듈이 이 계정에 열려 있어야 한다(§9). **결제(PG)를 쓰고 있어야 이 화면이 매출을 보여 준다**(§4.6).

**후행 처리** — 아무것도 바꾸지 않는다.

**관련 화면** — 같은 껍데기: `SCR-STATS-VISITORS` · `SCR-STATS-MEMBERS` · `SCR-STATS-ORDERS` · `SCR-STATS-TRAFFIC` · `SCR-STATS-KEYWORDS`. 결제 사용 여부를 정하는 화면은 `SCR-SETTINGS-PAYMENT`. 결제를 쓰지 않을 때 대신 보이는 지표의 출처 화면: `SCR-PRODUCTS-INQUIRIES` · `SCR-PROGRAMS-INQUIRIES` · `SCR-SALES-QUOTES`.

**관련 메뉴** — 사이드바 `분석 · 운영 > 통계` 의 세 번째 항목 — `apps/admin/src/shared/layout/nav-config.ts:269`. **플랜으로 잠긴다** — `stats.advanced` 모듈이 이 경로를 지배한다 — `apps/admin/src/shared/entitlements/module-resources.ts:97-102`.

**관련 기능** — 결제 설정(`SCR-SETTINGS-PAYMENT`)이 이 화면을 통째로 갈아 끼운다(§4.6). 주문 통계(`SCR-STATS-ORDERS`)도 같은 규칙을 따른다.

**관련 API** — 목적 하나: 조회 기간과 비교 기간의 매출 집계(결제수단별 일자별 · 결제수단별 합계)를 한 번에 받는다. 쓰기 목적의 호출이 없다. 경계는 `apps/admin/src/pages/stats/revenue/data-source.ts` 의 `TODO(backend)` 다.

**참고 문서** — [방문자 통계](../visitors/index.md)(껍데기 정본) · [ID 규약](../../id-convention.md) · [인벤토리](../../inventory.md)

**구현 파일** — `apps/admin/src/pages/stats/revenue/RevenueStatsPage.tsx` · `revenue/types.ts` · `revenue/data-source.ts` · 대체 패널 `apps/admin/src/pages/stats/_shared/InquiryStatsPanel.tsx`
