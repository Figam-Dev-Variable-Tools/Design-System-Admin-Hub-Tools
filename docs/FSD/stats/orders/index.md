# 주문 통계

> **조회 조건 축과 껍데기는 [방문자 통계](../visitors/index.md) 와 같다.**
> **이 문서는 그 한 벌을 다시 적지 않고 이 화면의 차이만 적는다.**

## 1. 화면 개요

| 항목 | 내용 |
|---|---|
| 화면명 | 주문 통계 |
| 화면 ID | `SCR-STATS-ORDERS` |
| 메뉴 경로 | 분석 · 운영 > 통계 > 주문 통계 |
| 작성자 | Admin Hub 문서화 |
| 작성일 | 2026-07-22 |
| 버전 | 1.0 |

**화면 목적** — 주문이 어느 상태에 고여 있는지, 얼마나 취소·반품으로 되돌아오는지 본다. 개별 주문의 처리는 주문 관리에서 하고 여기서는 '지난 기간과 견주어 무엇이 달라졌는지'를 본다 — `apps/admin/src/pages/stats/orders/OrderStatsPage.tsx:278`.

**업무 배경** — 주문 목록은 한 건을 처리하는 화면이라 '배송보류가 왜 이번 주에 두 배인가' 같은 질문에 답하지 못한다 — `OrderStatsPage.tsx:3-8`.

**화면 설명** — 배치는 다른 통계 화면과 같다. 세그먼트는 `주문 상태`(전체 + 7종), 드릴다운 축은 `일자별`·`상태별` 이며 **상태별은 표가 아니라 구성비 막대**다.

**주요 사용자** — 최상위 관리자 · 주문/CS 담당 운영자.

**사용 시나리오**

1. 드릴다운 축을 `상태별` 로 바꿔 주문이 어느 단계에 고여 있는지 막대 길이로 본다.
2. KPI 의 `취소율`·`반품률` 을 비교 기간과 견준다.
3. 세그먼트를 `배송보류` 로 좁혀 그 상태의 추이만 본다.
4. 조건 그대로 내보낸다(축에 따라 다른 파일이 나간다).

**선행 조건** — 로그인. 이 화면의 읽기 권한. **결제(PG)를 쓰고 있어야 이 화면이 주문을 보여 준다**(§4.5).

**후행 처리** — 아무것도 바꾸지 않는다.

**관련 화면** — 같은 껍데기: `SCR-STATS-VISITORS` · `SCR-STATS-MEMBERS` · `SCR-STATS-REVENUE` · `SCR-STATS-TRAFFIC` · `SCR-STATS-KEYWORDS`. 개별 주문은 `SCR-ORDERS` · `SCR-ORDERS-DETAIL` 이 다루고, 취소/교환/반품 처리는 `SCR-ORDERS-CLAIMS` 가 갖는다. 결제 사용 여부는 `SCR-SETTINGS-PAYMENT` 가 정한다.

**관련 메뉴** — 사이드바 `분석 · 운영 > 통계` 의 네 번째 항목 — `apps/admin/src/shared/layout/nav-config.ts:270`. **플랜으로 잠기지 않는다** — 심화 리포트 세 화면(매출·유입·검색어)만 상위 플랜의 것이다 — `apps/admin/src/shared/entitlements/module-resources.ts:97-102`.

**관련 기능** — 주문 상태 어휘의 정본은 도메인 층이 갖고 이 화면은 그것을 재수출한다 — `apps/admin/src/pages/stats/orders/types.ts:20-26`. 어휘가 두 벌이면 통계의 '배송준비중 12건'과 주문 목록의 건수가 영원히 어긋난다.

**관련 API** — 목적 하나: 조회 기간과 비교 기간의 주문 집계(일자별 · 상태별 분포)를 한 번에 받는다. 쓰기 목적의 호출이 없다. 경계는 `apps/admin/src/pages/stats/orders/data-source.ts` 의 `TODO(backend)` 다.

**참고 문서** — [방문자 통계](../visitors/index.md)(껍데기 정본) · [ID 규약](../../id-convention.md) · [인벤토리](../../inventory.md)

**구현 파일** — `apps/admin/src/pages/stats/orders/OrderStatsPage.tsx` · `orders/types.ts` · `orders/data-source.ts` · 대체 패널 `apps/admin/src/pages/stats/_shared/InquiryStatsPanel.tsx`
