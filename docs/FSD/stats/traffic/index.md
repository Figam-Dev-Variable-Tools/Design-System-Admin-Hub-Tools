# 유입 분석

> **조회 조건 축과 껍데기는 [방문자 통계](../visitors/index.md) 와 같다.**
> **이 문서는 그 한 벌을 다시 적지 않고 이 화면의 차이만 적는다.**
> 이 화면은 **플랜으로 잠기는 심화 리포트 3종** 중 하나다(§9).

## 1. 화면 개요

| 항목 | 내용 |
|---|---|
| 화면명 | 유입 분석 |
| 화면 ID | `SCR-STATS-TRAFFIC` |
| 메뉴 경로 | 분석 · 운영 > 통계 > 유입 분석 |
| 작성자 | Admin Hub 문서화 |
| 작성일 | 2026-07-22 |
| 버전 | 1.0 |

**화면 목적** — 유입이 어디서 왔고 그중 무엇이 실제로 팔았는지 본다. 방문자 통계가 '몇 명이 왔나'라면 여기서는 채널마다 유입수 옆에 구매건수와 매출액을 붙여 '어느 유입이 매출로 이어졌는지'를 본다 — `apps/admin/src/pages/stats/traffic/TrafficStatsPage.tsx:267`.

**업무 배경** — 유입수 1위 채널이 매출 1위가 아닐 수 있다. '많이 온 곳'과 '파는 곳'을 가르는 것이 이 화면의 존재 이유다 — `TrafficStatsPage.tsx:7-8`.

**화면 설명** — 배치는 다른 통계 화면과 같다. 세그먼트는 `유입 채널`(전체 + 5종), 드릴다운 축은 `채널별`·`검색엔진별`·`랜딩페이지별` 이며 **채널별만 구성비 막대**이고 나머지 둘은 표다. 검색엔진별에는 고지가 하나 붙는다.

**주요 사용자** — 최상위 관리자 · 마케팅 담당 운영자.

**사용 시나리오**

1. 축을 `채널별` 로 두고 어느 채널이 많이 들어오는지 막대로 본다.
2. 축을 `랜딩페이지별` 로 바꾸고 `구매전환율` 헤더를 눌러 전환이 좋은 페이지를 찾는다.
3. 세그먼트를 `광고` 로 좁혀 광고 유입의 매출액과 전환율을 본다.
4. 조건 그대로 내보낸다(축에 따라 다른 파일이 나간다).

**선행 조건** — 로그인. 이 화면의 읽기 권한. `고급 통계` 모듈이 이 계정에 열려 있어야 한다(§9).

**후행 처리** — 아무것도 바꾸지 않는다.

**관련 화면** — 같은 껍데기: `SCR-STATS-VISITORS` · `SCR-STATS-MEMBERS` · `SCR-STATS-REVENUE` · `SCR-STATS-ORDERS` · `SCR-STATS-KEYWORDS`. 방문자 통계가 같은 방문을 '사람 수'로 세는 화면이다.

**관련 메뉴** — 사이드바 `분석 · 운영 > 통계` 의 다섯 번째 항목 — `apps/admin/src/shared/layout/nav-config.ts:271`. **플랜으로 잠긴다** — `stats.advanced` 모듈이 이 경로를 지배한다 — `apps/admin/src/shared/entitlements/module-resources.ts:97-102`.

**관련 기능** — ⚠ **유입 채널 어휘는 이 화면이 스스로 갖는다.** 다른 화면이 쓰는 '유입원' 모듈이 이 리포에 없으므로 이 화면의 다섯 채널이 그 어휘의 유일한 정의다 — `apps/admin/src/pages/stats/traffic/types.ts:18-31`. 없는 모듈을 참조하지 않는다.

**관련 API** — 목적 하나: 조회 기간과 비교 기간의 유입 집계(일자별 채널 분해 · 검색엔진별 · 랜딩페이지별)를 한 번에 받는다. 쓰기 목적의 호출이 없다. 경계는 `apps/admin/src/pages/stats/traffic/data-source.ts` 의 `TODO(backend)` 다.

**참고 문서** — [방문자 통계](../visitors/index.md)(껍데기 정본) · [ID 규약](../../id-convention.md) · [인벤토리](../../inventory.md)

**구현 파일** — `apps/admin/src/pages/stats/traffic/TrafficStatsPage.tsx` · `traffic/types.ts` · `traffic/data-source.ts`

### 4.3 ⚠ 유입 채널 어휘 — 이 화면이 스스로 갖는다

| 항목 | 내용 |
|---|---|
| 사실 | 유입 채널은 다섯이다: `검색엔진` · `북마크` · `외부사이트` · `SNS` · `광고` — `traffic/types.ts:25-31` |
| 어디에 사는가 | **이 화면의 타입 파일 하나**다. 이 어휘를 공유하는 다른 모듈이 리포에 없다 |
| `북마크` 가 담는 것 | 브라우저 즐겨찾기만이 아니라 **유입 경로가 남지 않은 방문 전부**다: 주소창 직접입력 · 앱 접속 · 리퍼러를 잃은 유입 — `traffic/types.ts:6-11` |
| 왜 이름을 고치지 않는가 | 뜻이 '직접 유입'에 가깝다고 해서 라벨을 바꾸지 않는다 — 운영자는 기존 통계 화면에서 `북마크` 로 보던 숫자를 여기서도 같은 이름으로 찾는다. 이름을 고치면 같은 값이 두 이름을 갖는다 — `traffic/types.ts:8-11` |
| 명세의 판단 | 화면이 그 오해를 스스로 막는다 — 세그먼트를 `북마크` 로 고르면 KPI 카드에 설명 힌트가 붙는다(§3 No.4) |
