# 카테고리

> 포트폴리오의 **분류를 만드는 화면**이다. [포트폴리오](../items/index.md) 목록·폼과 달리
> **공용 목록 껍데기를 쓰지 않는다** — 표가 아니라 카드 안의 줄 목록이고, 등록·수정은 팝업이다.

## 1. 화면 개요

| 항목 | 내용 |
|---|---|
| 화면명 | 카테고리 |
| 화면 ID | `SCR-PORTFOLIO-CATEGORIES` |
| 메뉴 경로 | 비즈니스 > 포트폴리오 관리 > 카테고리 — `apps/admin/src/shared/layout/nav-config.ts:245` |
| 작성자 | Admin Hub 문서화 |
| 작성일 | 2026-07-22 |
| 버전 | 1.0 |

**화면 목적** — 포트폴리오 분류를 **추가·수정·삭제**하고, 각 분류가 **몇 건에 쓰이는지** 보여 준다.

**업무 배경** — 분류는 포트폴리오 등록의 **필수 항목**이라 여기가 비어 있으면 포트폴리오를 만들 수 없다. 반대로 **쓰이고 있는 분류를 지우면 그 항목들이 갈 곳을 잃는다**(고아 항목). 그래서 이 화면의 중심은 목록이 아니라 **'사용 중이면 못 지운다'는 규칙**이다.

**화면 설명** — 위에 `전체 N개` + `카테고리 추가` 버튼, 아래 카드 안에 분류가 한 줄씩 놓인다. 각 줄은 `이름 | 사용 건수 배지 | 수정 | 삭제` 다. 카드 맨 아래에 규칙을 설명하는 안내 한 줄이 있다.

**주요 사용자** — 최상위 관리자 · 마케팅·콘텐츠 담당 운영자.

**사용 시나리오**

1. 새 분류가 필요하면 `카테고리 추가` 로 팝업을 연다.
2. 이름을 잘못 지었으면 줄의 연필 버튼으로 고친다 — 이름을 고치면 **그 분류를 쓰는 포트폴리오의 표시도 함께 바뀐다.**
3. 지우려는데 삭제가 잠겨 있으면, 건수 배지를 눌러 **그 분류를 쓰는 포트폴리오 목록**으로 간다.
4. 거기서 분류를 바꾸거나 항목을 지운 뒤 돌아와 삭제한다.

**선행 조건** — 로그인. 이 화면의 읽기 권한.

**후행 처리** — 여기서 만든 분류가 [포트폴리오 등록·수정](../items/form.md) 의 분류 셀렉트에 나타난다.

**관련 화면** — [포트폴리오](../items/index.md)(`SCR-PORTFOLIO-ITEMS`) · [포트폴리오 등록·수정](../items/form.md) · **이 화면이 여는 팝업** [`POP-PORTFOLIO-CATEGORIES-CATEGORY-FORM`](pop-category-form.md). 공통 규약은 [공통 확인 다이얼로그 · 토스트](../../_common/index.md).

**관련 메뉴** — 사이드바 `비즈니스 > 포트폴리오 관리 > 카테고리` — `nav-config.ts:245`.

**관련 기능** — **사용 중 판정은 도메인이 갖는다** — 카테고리와 포트폴리오 항목이 같은 저장소를 본다. 두 화면의 데이터 소스가 서로를 부르면 순환이 되므로 규칙을 한 곳에 모았다 — `apps/admin/src/pages/portfolio/_shared/store.ts:3-6`.

**관련 API** — 목적 넷: **사용 건수를 포함한 목록 조회** · 추가 · 수정 · 삭제(사용 중이면 거절). 백엔드가 없다 — 경계는 `apps/admin/src/pages/portfolio/categories/data-source.ts:20` 의 `TODO(backend)` 한 줄이며 **거기에 '사용 중이면 409' 가 적혀 있다.**

**참고 문서** — [포트폴리오](../items/index.md) · [ID 규약](../../id-convention.md) · [인벤토리](../../inventory.md)

**구현 파일** — `apps/admin/src/pages/portfolio/categories/PortfolioCategoriesPage.tsx` · 팝업 `categories/components/PortfolioCategoryFormModal.tsx` · 어댑터 `portfolio/categories/data-source.ts` · 문구 헬퍼 `portfolio/categories/types.ts` · 도메인 `apps/admin/src/pages/portfolio/_shared/store.ts`

### 4.4 건수 배지로 이동 — 막힌 곳에서 나갈 길

| 항목 | 내용 |

| Validation | 해당 없음 |
| API 호출 여부 | N(이동) |
| 성공 처리 | 포트폴리오 목록이 그 분류로 걸러진 채 열린다 |
| 실패 처리 | 실패 경로가 없다 |
| 예외 처리 | **0건이면 링크로 만들지 않는다** — 열어 봐야 빈 목록이라 클릭할 이유가 없다 — `PortfolioCategoriesPage.tsx:126-128` |

**이 링크가 성립하는 이유** — 포트폴리오 목록이 조회 조건을 **주소로** 갖기 때문이다([포트폴리오 §4.1](../items/index.md)). 조건이 화면 안 상태였다면 이 배지는 걸러지지 않은 목록으로만 갈 수 있었다.

## 6. Validation

**이 화면에는 입력이 없다.** 이름 검증은 전부 팝업이 갖는다 — [`POP-PORTFOLIO-CATEGORIES-CATEGORY-FORM`](pop-category-form.md) §6.

이 화면이 갖는 규칙은 **삭제 가능 여부** 하나다.

| 대상 항목 | 검증 조건 | 검증 시점 | 오류 메시지 | 처리 방식 |
|---|---|---|---|---|
| 삭제 대상 | 사용 건수가 0이어야 한다 | 목록을 그릴 때(1차) · 도메인 호출 시(2차) | 문구를 띄우지 않는다 — **버튼의 접근성 이름과 툴팁이 사유를 말한다** | 버튼을 잠근다 |

## 8. 화면 상태(State)

| 상태 | 화면 표시 | 버튼 활성화 여부 | Loading 여부 | 사용자 안내 메시지 |
|---|---|---|---|---|
| 최초 로딩 | 툴바에 `불러오는 중…`, 카드에도 `불러오는 중…` | 추가 ○ | Y | `불러오는 중…` |
| 백그라운드 재조회 | **줄을 유지한 채** 툴바에 ` · 새로고침 중…` | 전부 ○ | N | `새로고침 중…` |
| 정상(결과 있음) | 줄 목록 + 규칙 안내 | 사용 중이 아닌 줄만 삭제 ○ | N | 규칙 안내 |
| 0건 | `등록된 카테고리가 없어요.` + 규칙 안내 | 추가 ○ | N | 안내 |
| 조회 실패 | 배너 + `다시 시도` (**카드를 그리지 않는다**) | 다시 시도 ○ | N | `카테고리를 불러오지 못했어요.` |
| 사용 중인 줄 | 배지가 링크가 되고 **휴지통이 잠긴다** | 수정 ○ · 삭제 × | N | 잠긴 버튼의 이름·툴팁 |
| 미사용 줄 | 배지 `미사용`(링크 아님) | 전부 ○ | N | 없음 |
| 삭제 진행 중 | 다이얼로그의 확인이 진행 표시 | **모든 줄의 삭제 버튼이 잠긴다** — `PortfolioCategoriesPage.tsx:158` · **수정 버튼은 잠기지 않는다** | Y | 없음 |
| 삭제 실패 | 다이얼로그 안 오류 | 확인 ○ · 취소 ○ | N | `삭제하지 못했어요. …` |
| 팝업 열림 | 목록 위에 모달 | 목록 조작 불가(포커스 가둠) | N | 없음 |

**`null`(모름)과 `[]`(없음)이 구분된다** — 조회 실패는 배너로, 정말 0건인 것은 카드 안 안내로 각각 다르게 보인다. 이 점은 [포트폴리오 목록](../items/index.md)(분류 선택지가 두 경우를 구분하지 못한다)보다 낫다.

## 10. 화면 이동(Screen Flow)

| 현재 화면 | 이벤트 | 이동 화면 | 팝업 여부 | 이동 조건 | 전달 파라미터 | 복귀 화면 |
|---|---|---|---|---|---|---|
| 사이드바 | `카테고리` 클릭 | `SCR-PORTFOLIO-CATEGORIES` | N | 읽기 권한 | 없음 | — |
| `SCR-PORTFOLIO-CATEGORIES` | `카테고리 추가` | `POP-PORTFOLIO-CATEGORIES-CATEGORY-FORM` | **Y** | 언제나 | 없음(등록 모드) | 이 화면 |
| `SCR-PORTFOLIO-CATEGORIES` | 줄의 연필 | 〃 | **Y** | 언제나 | 그 분류(수정 모드) | 이 화면 |
| `SCR-PORTFOLIO-CATEGORIES` | 줄의 휴지통 | 삭제 확인 다이얼로그 | **Y** | 사용 건수 0 | 분류 이름 | 이 화면 |

| `SCR-PORTFOLIO-CATEGORIES` | 401 관측 | `SCR-LOGIN` | N | 세션 만료 | 원래 주소 · 만료 표식 | 재인증 후 이 화면 |

**등록·수정이 별도 라우트가 아닌 이유** — 입력이 **이름 한 칸**이다. 한 칸을 위해 화면을 옮기면 목록에서 나갔다 돌아오는 왕복이 생긴다. 대신 팝업은 **입력이 있는 채로 닫으려 하면 확인을 세운다**([팝업 §4](pop-category-form.md)).
