# 카테고리 추가·수정 팝업

> [카테고리](index.md) 화면이 여는 모달이다. **한 팝업이 추가와 수정을 겸한다** — 수정 대상이
> 있으면 수정, 없으면 추가다.

## 1. 화면 개요

| 항목 | 내용 |
|---|---|
| 화면명 | 카테고리 추가·수정 |
| 화면 ID | `POP-PORTFOLIO-CATEGORIES-CATEGORY-FORM` |
| 메뉴 경로 | 비즈니스 > 포트폴리오 관리 > 카테고리 > 추가·수정 |
| 작성자 | Admin Hub 문서화 |
| 작성일 | 2026-07-22 |
| 버전 | 1.0 |

**화면 목적** — 포트폴리오 분류의 **이름 한 칸**을 입력받아 만들거나 고친다.

**업무 배경** — 입력이 이름 하나뿐이라 화면을 옮길 이유가 없다. 목록을 보면서 그 자리에서 고치고 닫는 것이 가장 짧다.

**화면 설명** — 모달 하나에 입력 한 칸(`카테고리 이름`). 바닥에 `취소` 와 `추가`(수정이면 `저장`)가 있다.

**주요 사용자** — 최상위 관리자 · 마케팅·콘텐츠 담당 운영자.

**사용 시나리오**

1. `카테고리 추가` 를 누르면 빈 칸이 뜨고 **커서가 그 칸에 가 있다.**
2. 이름을 적고 `추가` 를 누른다.
3. 수정이면 기존 이름이 채워진 채로 열리고 버튼이 `저장` 이 된다.
4. 뭔가 적다가 닫으려 하면 **버릴지 묻는다.**

**선행 조건** — [카테고리](index.md) 화면에 있어야 한다.

**후행 처리** — 저장에 성공하면 팝업이 닫히고 **여는 쪽 화면이 토스트를 띄운다**([카테고리 §4.2](index.md)). 수정이면 그 분류를 쓰는 포트폴리오의 표시 이름도 함께 바뀐다.

**관련 화면** — [카테고리](index.md)(`SCR-PORTFOLIO-CATEGORIES`) · 이름이 바뀌면 영향을 받는 [포트폴리오](../items/index.md)

**관련 메뉴** — 없다(팝업).

**관련 기능** — 이름 검증의 정본은 `apps/admin/src/pages/portfolio/categories/validation.ts` 다. 상한값은 `apps/admin/src/pages/portfolio/categories/types.ts:11` 이 갖는다.

**관련 API** — 목적 둘: 추가 · 수정. 백엔드가 없다 — 경계는 `apps/admin/src/pages/portfolio/categories/data-source.ts:20` 의 `TODO(backend)` 다.

**참고 문서** — [카테고리](index.md) · [ID 규약](../../id-convention.md) · [인벤토리](../../inventory.md)

**구현 파일** — `apps/admin/src/pages/portfolio/categories/components/PortfolioCategoryFormModal.tsx` · 검증 `portfolio/categories/validation.ts` · 어댑터 `portfolio/categories/data-source.ts`

## 4. 기능 명세

### 4.1 한 팝업이 추가와 수정을 겸한다

| 항목 | 내용 |
|---|---|
| 기능 목적 | 두 모드를 한 화면으로 덮는다 |
| 실행 조건 | 여는 쪽이 열었을 때 |
| 사용자 동작 | 없음(자동) |
| 시스템 처리 | **수정 대상이 넘어왔는지**로 모드를 가른다 — `PortfolioCategoryFormModal.tsx:46`. 제목·기본값·버튼 문구·호출 대상이 함께 갈린다 |
| 데이터 처리 | 수정이면 기존 이름을 채운 채로 연다 — `PortfolioCategoryFormModal.tsx:55` |
| Validation | §6 |
| API 호출 여부 | Y — 추가 또는 수정 — `PortfolioCategoryFormModal.tsx:84-95` |
| 성공 처리 | 여는 쪽에 **이름과 모드를 돌려주고** 닫힌다 — `PortfolioCategoryFormModal.tsx:87,94`. 토스트는 여는 쪽이 띄운다 |
| 실패 처리 | 배너를 세우고 **닫지 않는다** — `PortfolioCategoryFormModal.tsx:79-82` |
| 예외 처리 | 취소로 끝난 요청은 실패로 알리지 않는다 — `PortfolioCategoryFormModal.tsx:80` |
| 화면 변경 사항 | 팝업 |
| 후속 동작 | 목록 재조회 + 토스트 |
| 로그 기록 여부 | 확인하지 못했다 |

**성공 통지를 팝업이 띄우지 않는 이유** — 팝업은 성공과 동시에 사라진다. 사라지는 화면이 띄운 통지는 함께 사라진다.
