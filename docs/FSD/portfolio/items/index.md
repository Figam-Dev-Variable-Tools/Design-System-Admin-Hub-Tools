# 포트폴리오

> **이 문서가 포트폴리오 두 목록의 정본이다.** [성공 사례](../case-studies/index.md) 는 이 화면과
> 뼈대가 같고 **거른다는 축과 열 두 개만 다르다** — 그 문서는 차이만 적는다.

## 1. 화면 개요

| 항목 | 내용 |
|---|---|
| 화면명 | 포트폴리오 |
| 화면 ID | `SCR-PORTFOLIO-ITEMS` |
| 메뉴 경로 | 비즈니스 > 포트폴리오 관리 > 포트폴리오 |
| 작성자 | Admin Hub 문서화 |
| 작성일 | 2026-07-22 |
| 버전 | 1.0 |

**화면 목적** — 홈페이지에 싣는 시공·프로젝트 사례를 **목록으로 관리한다**: 분류로 거르고, 노출 여부를 표에서 바로 켜고 끄고, 등록·수정·삭제로 넘어간다.

**업무 배경** — 포트폴리오는 손댈 일이 드물지만 한 번 손댈 때는 **여러 건의 노출을 한꺼번에 정리한다**(시즌이 지난 사례를 내리고 새 사례를 올린다). 그래서 목록에서 바로 켜고 끄는 토글이 표 안에 있다 — 매번 폼을 열게 하면 열 건을 내리는 데 열 번 저장해야 한다.

**화면 설명** — 위에 분류 셀렉트 + `포트폴리오 등록` 버튼, 아래에 5열 표(분류 · 제목 · 고객사 · 일자 · 노출)다. **이미지 열은 없다.**

**주요 사용자** — 최상위 관리자 · 마케팅·콘텐츠 담당 운영자.

**사용 시나리오**

1. 분류 셀렉트로 `오피스` 만 남기고 훑는다.
2. 시즌이 지난 사례의 노출 토글을 꺼 홈페이지에서 내린다.
3. 새 사례는 `포트폴리오 등록` 으로 만든다.
4. 여러 건을 한 번에 지울 때는 행을 골라 일괄 삭제한다.

**선행 조건** — 로그인. 이 화면의 읽기 권한.

**후행 처리** — 노출을 켠 항목이 홈페이지 포트폴리오 목록에 나타난다(백엔드 없음 — §7).

**관련 화면** — [포트폴리오 등록·수정](form.md)(`SCR-PORTFOLIO-ITEMS-FORM`) · [카테고리](../categories/index.md)(`SCR-PORTFOLIO-CATEGORIES`) · [성공 사례](../case-studies/index.md)(`SCR-PORTFOLIO-CASE-STUDIES`). 공통 규약은 [공통 확인 다이얼로그 · 토스트](../../_common/index.md).

**관련 메뉴** — 사이드바 `비즈니스 > 포트폴리오 관리 > 포트폴리오` — `apps/admin/src/shared/layout/nav-config.ts:243-244`. **이 묶음이 사이드바 맨 뒤인 이유** — 이 묶음의 순서는 알파벳도 도메인 계보도 아닌 **매일 여는 횟수**다. 포트폴리오는 분기에 한 번 손대는 화면이라 맨 뒤다 — `nav-config.ts:169-171`.

**관련 기능** — **분류는 운영자가 만드는 값이다** — 고정 목록이 아니라 [카테고리](../categories/index.md) 화면이 관리한다. 그래서 이 목록의 분류 셀렉트는 서버에서 받아 채운다.

**관련 API** — 목적 넷: 목록 조회 · 한 건 수정(노출 토글) · 삭제 · **분류 선택지 조회**. 백엔드가 없다 — 경계는 `apps/admin/src/pages/portfolio/items/data-source.ts:19,29` 의 `TODO(backend)` 두 줄이다.

**참고 문서** — [ID 규약](../../id-convention.md) · [인벤토리](../../inventory.md) · [공통 규약](../../_common/index.md)

**구현 파일** — `apps/admin/src/pages/portfolio/items/PortfolioListPage.tsx` · 어댑터 `portfolio/items/data-source.ts` · 뷰 헬퍼 `portfolio/items/types.ts` · 도메인 `apps/admin/src/pages/portfolio/_shared/store.ts` · 공유 열 `apps/admin/src/pages/portfolio/_shared/publishColumn.tsx`

## 4. 기능 명세

### 4.1 분류로 거르기 — 조건을 URL 이 소유한다

| 항목 | 내용 |
|---|---|
| 기능 목적 | 분류 하나로 목록을 좁힌다 |
| 실행 조건 | 항상 |
| 사용자 동작 | 셀렉트에서 분류 선택 |
| 시스템 처리 | 선택값을 **주소 쿼리 `?category=`** 에 쓴다 — `PortfolioListPage.tsx:64,120`. 조건을 화면 안 상태로 들고 있으면 한 건을 고치러 갔다 돌아왔을 때 조건이 풀린다 — `PortfolioListPage.tsx:6-9` |
| 데이터 처리 | 받아 둔 목록을 클라이언트에서 거른다 — `apps/admin/src/pages/portfolio/_shared/store.ts:135-141` |
| Validation | **허용값 대조를 하지 않는다**(§6) |
| API 호출 여부 | N — 거르는 것 자체는 서버를 다시 부르지 않는다 |
| 성공 처리 | 표가 좁혀지고 선택이 풀린다 — `PortfolioListPage.tsx:82-84` |
| 실패 처리 | 실패 경로가 없다 |
| 예외 처리 | 기본값(`전체 분류`)이면 주소에서 지운다 — 공유 링크를 짧게 — `PortfolioListPage.tsx:53-54` |
| 화면 변경 사항 | 목록 표 · 주소 |
| 후속 동작 | **이 주소를 그대로 공유할 수 있다.** 카테고리 관리 화면의 '사용 중' 배지도 이 파라미터로 이 목록을 연다 |
| 로그 기록 여부 | 확인하지 못했다 |

**분류를 바꾸면 선택이 풀리는 이유** — 고른 행이 화면에서 사라진 채로 선택만 남으면, 일괄 삭제가 **보이지 않는 것을 지운다.**

### 4.2 노출 토글 — 표에서 바로 켜고 끈다

| 항목 | 내용 |
|---|---|
| 기능 목적 | 폼을 열지 않고 홈페이지 노출을 바꾼다 |
| 실행 조건 | 수정 권한 보유 |
| 사용자 동작 | 행의 토글 전환 |
| 시스템 처리 | 그 항목의 현재 값 전체에 노출만 바꿔 수정으로 보낸다 — `PortfolioListPage.tsx:102-112`(입력 변환은 `apps/admin/src/pages/portfolio/items/types.ts:19-31`) |
| 데이터 처리 | 노출 여부만 달라진다 |
| Validation | 해당 없음 |
| API 호출 여부 | Y — 한 건 수정 |
| 성공 처리 | 토스트가 결과를 말한다 — `'{제목}'을(를) 게시했어요.` / `'{제목}'을(를) 숨겼어요.` — 조립은 `PortfolioListPage.tsx:107-109`(조사는 제목의 받침을 보고 고른다 — `apps/admin/src/shared/format.ts` 의 `objectParticle`) |
| 실패 처리 | 공통 목록 층의 실패 통지가 뜨고 **토글이 원래 값으로 돌아간다** |
| 예외 처리 | **수정 권한이 없으면 토글이 잠긴다** — 이 판정은 옵션이 아니라 필수 인자다. 기본값 `true` 를 주면 이 열을 쓰는 새 목록이 인자를 빠뜨렸을 때 **조용히 열린 채로** 통과한다 — `apps/admin/src/pages/portfolio/_shared/publishColumn.tsx:17-22` |
| 화면 변경 사항 | 그 행 · 토스트 |
| 후속 동작 | 목록 재조회 |
| 로그 기록 여부 | 확인하지 못했다 |

### 4.3 등록·수정·삭제로 넘어가기

| 항목 | 내용 |
|---|---|
| 기능 목적 | 목록에서 폼과 삭제로 잇는다 |
| 실행 조건 | 각 권한 보유 |
| 사용자 동작 | `포트폴리오 등록` · 행의 수정 · 행 선택 후 일괄 삭제 |
| 시스템 처리 | 등록은 `/portfolio/items/new`, 수정은 `/portfolio/items/{id}/edit` 로 이동한다 — `PortfolioListPage.tsx:133,154` |
| 데이터 처리 | 삭제는 공통 목록 층이 맡는다 |
| Validation | 해당 없음 |
| API 호출 여부 | Y(삭제) |
| 성공 처리 | 공통 성공 토스트 — `MSG-COMMON-03`(단건) · `MSG-COMMON-04`(일괄) |
| 실패 처리 | 공통 실패 통지. 확인 다이얼로그는 닫히지 않는다 |
| 예외 처리 | 확인 다이얼로그는 공통 층의 것이다 — `DLG-COMMON-DELETE-ONE` · `DLG-COMMON-DELETE-BULK` |
| 화면 변경 사항 | 화면 전체(이동) · 표(삭제) |
| 후속 동작 | 목록 재조회 |
| 로그 기록 여부 | 확인하지 못했다 |

**행 클릭이 상세로 가지 않는다** — 이 도메인에는 상세 화면이 없다. 행의 동작은 수정 폼으로 가는 것뿐이다 — `PortfolioListPage.tsx:154`.

## 6. Validation

**입력 폼이 없는 화면이다.** 검증할 사용자 입력은 분류 필터 하나뿐이며, 그것은 **막지 않고 흘려보낸다.**

| 대상 항목 | 검증 조건 | 검증 시점 | 오류 메시지 | 처리 방식 |
|---|---|---|---|---|
| 주소의 `?category=` | **허용값 대조를 하지 않는다** | — | 없다 | 모르는 id 는 빈 목록으로 흘려보낸다 — 조회가 깨지지 않는다 |

**왜 대조하지 않는가** — 분류 id 는 운영자가 만드는 값이라 **닫힌 목록이 없고**, 선택지는 비동기로 늦게 온다. 허용값과 대조하면 **도착 전의 정상 id 까지 '전체'로 되돌려 버린다** — `PortfolioListPage.tsx:61-63`. [성공 사례](../case-studies/index.md) 는 업종이 고정 목록이라 반대로 대조한다 — 두 화면이 다른 이유가 여기 있다.

## 7. 예외 처리

| 발생 상황 | 사용자 메시지 | 시스템 처리 | 복구 방법 |
|---|---|---|---|
| 조회 결과 없음(필터로 0건) | 공통 목록 층의 빈 안내 + 조건 초기화 | 표를 비운다 | `전체 분류` 로 되돌린다 |
| 조회 결과 없음(등록된 것이 0건) | 공통 목록 층의 빈 안내 | 〃 | 등록한다 |
| 저장 실패 · 수정 실패 | 노출 토글 실패는 공통 목록 층이 통지하고 값을 되돌린다 | 표를 유지한다 | 다시 토글 |
| 삭제 실패 | 확인 다이얼로그 안에서 실패를 알린다(공통) | **다이얼로그가 닫히지 않는다** | 다시 시도 |
| API 오류 · 서버 오류 · 네트워크 오류 | 공통 목록 층의 실패 영역. 자동 재시도 없음 | 짧은 참조 코드만 남긴다 | `다시 시도` |
| 권한 없음(읽기) | 공통 접근 거부 화면 | 라우트에서 막는다 | 관리자에게 권한 요청 |
| 권한 없음(등록) | **문구가 없다** — 버튼이 그려지지 않는다 | 누를 수 없는 것을 보여 주지 않는다 | 〃 |
| 권한 없음(수정) | **문구가 없다** — 노출 토글이 잠긴다 | 〃 | 〃 |
| 세션 만료 | 이 화면에 자기 문구가 없다 | 원래 주소를 보존한 채 로그인 화면으로 | 재로그인하면 **거른 조건까지 그대로 돌아온다**(주소가 갖고 있다) |
| 데이터 충돌 | 이 화면에는 폼이 없어 충돌 다이얼로그가 없다 | — | — |
| 중복 데이터 | **제목 중복을 막지 않는다** | — | — |
| 파일 업로드 실패 | 해당 없음 — 이 화면은 이미지를 다루지 않는다 | — | — |
| 분류 선택지 조회 실패 | ⚠ **문구가 없다** — 셀렉트에 `전체 분류` 만 남아, 운영자는 분류가 하나도 없는 것인지 못 불러온 것인지 구분할 수 없다 | 빈 배열로 뭉갠다 — `PortfolioListPage.tsx:80` | 새로고침 |

## 8. 화면 상태(State)

| 상태 | 화면 표시 | 버튼 활성화 여부 | Loading 여부 | 사용자 안내 메시지 |
|---|---|---|---|---|
| 최초 로딩 | 공통 목록 층의 스켈레톤 | 툴바 ○ | Y | 없음 |
| 백그라운드 재조회 | **이전 행을 유지한다** | 전부 ○ | N | 가벼운 인디케이터 |
| 정상(결과 있음) | 5열 표 | 전부 ○ | N | 없음 |
| 결과 0건(필터 때문) | 빈 안내 + 조건 초기화 | 조건 초기화 ○ | N | 공통 문구 |
| 결과 0건(등록된 것이 없음) | 빈 안내 | 등록 ○ | N | 〃 |
| 조회 실패 | 공통 실패 영역 | 다시 시도 ○ | N | 공통 문구 |
| **분류 선택지가 아직 안 옴** | 셀렉트에 `전체 분류` 만 있다 | ○ | N | 없음 — 도착하면 늘어난다 |
| 분류 선택지 조회 실패 | 〃 (같은 모습이다) | ○ | N | **없다 — 구분되지 않는다** |
| 한 행이 저장 중 | 그 행의 토글만 잠긴다 | 나머지 ○ | 그 행만 | 없음 |
| 수정 권한 없음 | 노출 토글이 잠긴 채로 보인다 | 토글 × | N | 없음 |
| 등록 권한 없음 | 등록 버튼이 **없다** | — | N | 없음 |

**`null`(모름)과 `[]`(없음)이 화면에서 구분되지 않는다** — 분류 선택지는 조회 실패도, 정말 하나도 없는 것도 같은 모습이다. 이 화면의 결함으로 §7 에 적었다.
