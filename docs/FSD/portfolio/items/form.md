# 포트폴리오 등록·수정

> **이 문서가 포트폴리오 두 폼의 정본이다.** [성공 사례 등록·수정](../case-studies/form.md) 은 뼈대가
> 같고 **입력 항목 구성만 다르다** — 그 문서는 차이만 적는다.

## 1. 화면 개요

| 항목 | 내용 |
|---|---|
| 화면명 | 포트폴리오 등록·수정 |
| 화면 ID | `SCR-PORTFOLIO-ITEMS-FORM` |
| 메뉴 경로 | 비즈니스 > 포트폴리오 관리 > 포트폴리오 > 등록·수정 (사이드바에 자리가 없다) |
| 작성자 | Admin Hub 문서화 |
| 작성일 | 2026-07-22 |
| 버전 | 1.0 |

**화면 목적** — 포트폴리오 한 건의 **분류·일자·제목·고객사·소개·이미지·노출 여부**를 입력해 만들거나 고친다.

**업무 배경** — 홈페이지에 실리는 글이라 **빈 칸으로 나가면 안 되는 항목**이 분명하다(대표 이미지가 없으면 목록 카드가 깨진다). 그래서 필수 항목이 많고, 저장 전에 전부 막는다.

**화면 설명** — 공용 폼 껍데기 위의 카드 하나다: 첫 줄에 분류·일자가 나란히, 그 아래 제목 · 고객사 · 소개(여러 줄) · 대표 이미지 · 본문 이미지 · 노출 토글이 세로로 온다.

**주요 사용자** — 최상위 관리자 · 마케팅·콘텐츠 담당 운영자.

**사용 시나리오**

1. 목록의 `포트폴리오 등록` 으로 들어온다.
2. 분류를 고르고 일자를 넣는다.
3. 제목·고객사·소개를 쓴다.
4. 대표 이미지를 올리고 본문 이미지를 최대 10장까지 더한다.
5. 바로 공개할지 노출 토글로 정하고 저장한다.

**선행 조건** — 로그인. 등록은 `create`, 수정은 `update` 권한. **분류가 하나 이상 있어야 한다** — 분류는 필수인데 [카테고리](../categories/index.md) 화면이 만든다.

**후행 처리** — 저장하면 목록으로 돌아간다. 노출을 켰으면 홈페이지에 실린다.

**관련 화면** — [포트폴리오](index.md)(`SCR-PORTFOLIO-ITEMS`) · [카테고리](../categories/index.md)(분류를 만드는 곳) · [성공 사례 등록·수정](../case-studies/form.md). 공통 규약은 [공통 확인 다이얼로그 · 토스트](../../_common/index.md).

**관련 메뉴** — 사이드바에 자기 자리가 없다. 권한 리소스는 잎 `/portfolio/items` 가 덮는다.

**관련 기능** — **분류 선택지는 목록 화면과 같은 조회를 쓴다**(같은 캐시 키) — `apps/admin/src/pages/portfolio/items/PortfolioFormPage.tsx:92-95`.

**관련 API** — 목적 넷: 단건 조회(수정) · 등록 · 수정 · **분류 선택지 조회**. 백엔드가 없다 — 경계는 `apps/admin/src/pages/portfolio/items/data-source.ts:19,29` 의 `TODO(backend)` 다.

**참고 문서** — [포트폴리오](index.md) · [ID 규약](../../id-convention.md) · [인벤토리](../../inventory.md)

**구현 파일** — `apps/admin/src/pages/portfolio/items/PortfolioFormPage.tsx` · 검증 `portfolio/items/validation.ts` · 공유 하단부 `apps/admin/src/pages/portfolio/_shared/PortfolioMediaFields.tsx` · 상한값 `apps/admin/src/pages/portfolio/_shared/store.ts:53-56`

## 4. 기능 명세

### 4.1 등록·수정 저장

| 항목 | 내용 |
|---|---|
| 기능 목적 | 한 건을 만들거나 고친다 |
| 실행 조건 | 등록·수정 권한 |
| 사용자 동작 | 입력 후 저장 |
| 시스템 처리 | 공용 폼 컨트롤러가 검증 → 변환 → 저장을 맡는다 — `apps/admin/src/pages/portfolio/items/PortfolioFormPage.tsx:46-51`. **저장 직전에 문자열의 앞뒤 공백을 턴다** — `PortfolioFormPage.tsx:62-71` |
| 데이터 처리 | 저장 시 **분류 이름이 항목에 함께 박힌다**(목록 배지가 바로 쓰도록) — `apps/admin/src/pages/portfolio/_shared/store.ts:174-186` |

| API 호출 여부 | Y — 등록 또는 수정 |

| 실패 처리 | 목록으로 이동하지 않고 입력을 유지한 채 폼 배너가 이유를 말한다(공통) |
| 예외 처리 | 검증에 걸리면 **첫 오류 항목으로 포커스가 옮겨 간다**(공통 폼 규칙) |

### 4.4 저장하지 않고 떠나기 · 저장 충돌

공통 층의 것이다 — `DLG-COMMON-LEAVE-FORM` · `DLG-COMMON-FORM-CONFLICT`. 이탈 안내 문구만 이 화면이 갖는다: `포트폴리오에 저장하지 않은 변경 사항이 있어요. 이 화면을 벗어나면 입력한 내용이 사라져요.` — `PortfolioFormPage.tsx:30-31`.
