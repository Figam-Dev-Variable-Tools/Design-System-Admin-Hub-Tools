# 대화 목록

## 1. 화면 개요

| 항목 | 내용 |
|---|---|
| 화면명 | 대화 목록 |
| 화면 ID | `SCR-AI-CONVERSATIONS` |
| 메뉴 경로 | AI > AI 에이전트 > 대화 목록 |
| 작성자 | Admin Hub 문서화 |
| 작성일 | 2026-07-22 |
| 버전 | 1.0 |

**화면 목적** — 지금까지의 대화를 최근 갱신 순으로 보여주고, 열거나 지운다 — `apps/admin/src/pages/ai/ConversationsPage.tsx:3`.

**업무 배경** — 새 채팅 화면의 좌측 레일도 대화를 보여 주지만 그곳은 좁고 오늘/이전 두 묶음뿐이며 **지우는 수단이 없다.** 대화를 정리하는 자리가 따로 필요하다.

**화면 설명** — 상단 오른쪽에 `새 채팅`, 그 아래 보관되지 않는다는 안내, 그 아래 대화 한 줄씩(제목 · 메시지 수 · 갱신 시각 · 삭제)이다.

**주요 사용자** — 최상위 관리자 · 운영자.

**사용 시나리오**

1. 대화 제목을 눌러 그 대화를 이어 묻는다.
2. 필요 없는 대화의 `삭제` 를 눌러 확인 후 지운다.
3. 대화가 하나도 없으면 빈 상태의 `새 채팅 시작` 으로 넘어간다.

**선행 조건** — 로그인. 이 화면의 읽기 권한.

**후행 처리** — 대화가 지워진다. 그 대화를 열어 두고 있던 새 채팅 화면은 다음 조회에서 삭제됨 경고를 만난다.

**관련 화면** — `SCR-AI-CHAT`(대화를 열고 이어 묻는 화면). 공통 규약은 [공통 확인 다이얼로그 · 토스트](../../_common/index.md).

**관련 메뉴** — 사이드바 `AI > AI 에이전트` 의 두 번째 항목 — `apps/admin/src/shared/layout/nav-config.ts:257`. **플랜으로 잠긴다** — `ai.agent` 모듈이 `/ai` 가지와 두 잎을 지배하며 최소 티어는 `엔터프라이즈` 다 — `apps/admin/src/shared/entitlements/module-resources.ts:81-85` · `apps/admin/src/shared/entitlements/plan.ts:243-248`.

**관련 기능** — 새 채팅 화면의 좌측 레일이 같은 목록을 다른 모양(오늘/이전 묶음)으로 보여 준다. 두 화면이 같은 조회를 공유한다 — `apps/admin/src/pages/ai/queries.ts:20-26`.

**관련 API** — 목적 둘. ① 대화 목록 조회 ② 대화 삭제. 삭제는 여러 건을 받을 수 있는 형태이나 **이 화면은 언제나 한 건만 보낸다** — `ConversationsPage.tsx:100`. 백엔드가 없다 — 경계는 `apps/admin/src/pages/ai/data-source.ts` 의 `TODO(backend)` 다.

**참고 문서** — [ID 규약](../../id-convention.md) · [인벤토리](../../inventory.md) · [새 채팅](../chat/index.md)

**구현 파일** — `apps/admin/src/pages/ai/ConversationsPage.tsx` · `apps/admin/src/pages/ai/queries.ts` · `apps/admin/src/pages/ai/data-source.ts` · 저장소 `apps/admin/src/pages/ai/_shared/conversations.ts`

### 4.3 대화 삭제

| 항목 | 내용 |
|---|---|
| 기능 목적 | 필요 없는 대화를 지운다 |
| 실행 조건 | 목록에 대화가 있을 때 |
| 사용자 동작 | 행의 `삭제` 클릭 → 확인 다이얼로그에서 확인 또는 취소 |
| 시스템 처리 | 진행 중이던 삭제 요청을 중단하고 그 한 건만 보낸다. **다이얼로그는 열려 있을 때만 렌더된다**(존재 자체가 열림이다) — `ConversationsPage.tsx:185-186` |
| 데이터 처리 | 그 대화와 메시지가 사라진다. **되돌릴 수 없다** |
| Validation | 해당 없음 — 입력 칸이 없다 |
| API 호출 여부 | Y — 대화 삭제 1건 |
| 성공 처리 | 다이얼로그가 닫히고 목록이 갱신된다. 성공 토스트: `대화를 삭제했어요.` — `ConversationsPage.tsx:107` |
| 실패 처리 | **다이얼로그를 닫고** 실패 토스트를 띄운다: `대화를 삭제하지 못했어요.` + `다시 시도` — `ConversationsPage.tsx:113`. 실패 토스트는 자동으로 사라지지 않는다 |
| 예외 처리 | 취소·중단은 실패로 치지 않는다 — 토스트를 만들지 않는다 — `ConversationsPage.tsx:110`. ⚠ **실패 시 다이얼로그가 닫히는 것은 공통 삭제 확인과 다르다** — 공통 층은 다이얼로그를 열어 둔 채 배너로 이유를 말한다([공통 확인 다이얼로그 · 토스트](../../_common/index.md)) |
| 화면 변경 사항 | 목록영역 · 확인 다이얼로그 · 토스트 |
| 후속 동작 | 대화 목록과 각 대화의 조회가 무효화된다 — `apps/admin/src/pages/ai/queries.ts:66-69`. **그 대화를 열어 두고 있던 새 채팅 화면**은 다음 조회에서 삭제됨 경고를 만난다 |
| 로그 기록 여부 | 확인하지 못했다 |

| 삭제 실패 | 응답 오류 | 다이얼로그를 닫고 실패 토스트(`다시 시도` 포함) | 토스트 | 목록(그대로) |
| 실패 토스트의 `다시 시도` | 토스트의 버튼 | 같은 삭제를 다시 시도한다 | 토스트 | 성공 또는 재실패 |
| 조회 실패 | 목록 조회 오류 | 목록 자리를 배너로 대체 | 오류영역 | 배너 |
| 빈 상태의 `새 채팅 시작` | 버튼 클릭 | 새 채팅 화면으로 이동 | 빈 상태 | `SCR-AI-CHAT` |
| 새로고침 | F5 | **대화가 전부 사라진다**(메모리 저장) | 화면 전체 | 빈 상태 |

**이 화면의 확인 다이얼로그는 화면 고유 1건**이다 — 인벤토리 §8.2 의 `SCR-AI-CONVERSATIONS` 행. 공통 삭제 확인을 쓰지 않는다.

## 6. Validation

**해당 없음 — 이 화면에는 입력 칸이 하나도 없다.** 사용자가 정하는 값이 없고, 누르는 것은 링크와 버튼뿐이다. 삭제 확인 다이얼로그에도 입력이 없다.
