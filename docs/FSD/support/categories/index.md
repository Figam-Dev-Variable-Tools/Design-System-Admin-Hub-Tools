# 문의 유형

## 1. 화면 개요

| 항목 | 내용 |
|---|---|
| 화면명 | 문의 유형 |
| 화면 ID | `SCR-SUPPORT-CATEGORIES` |
| 메뉴 경로 | 고객센터 > 문의 유형 |
| 작성자 | Admin Hub 문서화 |
| 작성일 | 2026-07-22 |
| 버전 | 1.0 |

**화면 목적** — 고객센터 문의를 분류하는 **유형**을 만들고 고치고, 쓰이지 않는 것을 지운다.

**업무 배경** — 유형은 두 곳이 참조한다 — **티켓**(그 유형으로 접수된다)과 **답변 템플릿**(그 유형 티켓에만 노출된다). 그래서 삭제 규칙이 '참조되면 하드 삭제하지 않는다'이고, 그 대신 **사용여부를 꺼서 신규 선택에서만 숨기는** 소프트 비활성이 있다 — `apps/admin/src/pages/support/_shared/domain.ts:12`.

**화면 설명** — 툴바(전체 건수 + `유형 추가`)와 카드 안 목록이다. 행마다 사용여부 배지와 **참조 건수 배지 둘**(티켓 · 템플릿), 수정·삭제 액션이 붙는다.

**주요 사용자** — 최상위 관리자 · 고객센터 운영자.

**사용 시나리오**

1. `유형 추가` 로 '주문/결제'를 만든다 → 티켓 접수와 템플릿 태깅에서 고를 수 있게 된다.
2. 더는 쓰지 않는 유형을 **삭제 대신 사용여부를 끈다** → 기존 티켓의 유형 표기는 그대로 남고 신규 선택에서만 사라진다.
3. 정말 지우려면 `티켓 N` 배지를 눌러 그 유형의 티켓 목록을 열고, 거기서 유형을 바꾼 뒤 돌아와 지운다.

**선행 조건** — 로그인 · 이 화면의 읽기 권한.

**후행 처리** — 만든 유형이 티켓 목록의 유형 필터와 답변 템플릿의 유형 태그 셀렉트에 나타난다. **사용여부를 끄면 신규 선택 목록에서 빠지되 기존 참조는 유지된다** — `apps/admin/src/pages/support/_shared/domain.ts:25-26`.

**관련 화면** — [문의 유형 추가·수정 팝업](pop-category-form.md) `POP-SUPPORT-CATEGORIES-CATEGORY-FORM` · [1:1 문의 목록](../tickets/index.md) `SCR-SUPPORT-TICKETS` · [문의 답변](../replies/index.md) `SCR-SUPPORT-REPLIES` · 공통 규약은 [공통 확인 다이얼로그 · 토스트](../../_common/index.md).

**관련 메뉴** — 사이드바 `고객센터 > 문의 유형`.

**관련 기능** — 세 화면(티켓·유형·템플릿)이 같은 도메인 모듈을 공유한다. 세 데이터 소스가 서로를 가져오면 결합이 되므로 도메인 타입·순수 규칙을 잎 모듈 하나에 모았다 — `apps/admin/src/pages/support/_shared/domain.ts:3-7`.

**관련 API** — 유형(사용량 포함) 목록 조회 1건 · 등록/수정 1건(팝업) · 삭제 1건. **백엔드가 없다** — 어댑터가 픽스처 위에 배선되어 있다.

**참고 문서** — [ID 규약](../../id-convention.md) · [인벤토리](../../inventory.md) · [품질 기준](../../../reference/quality-bar.md)

**구현 파일** — `apps/admin/src/pages/support/categories/CategoriesPage.tsx` (팝업 `components/CategoryFormModal.tsx` · 검증 `validation.ts` · 도메인 규칙 `../_shared/domain.ts` · 저장소 `../_shared/store.ts`)

**토스트 대조** — [인벤토리](../../inventory.md) §9.3 은 `pages/support/categories/CategoriesPage.tsx` 를 **성공 2 · 실패 0** 으로 센다. §4.2 가 한 건(추가/저장이 한 호출의 두 문구), §4.3 이 나머지 한 건이다. 삭제 실패는 토스트가 아니라 다이얼로그 안 배너다.

## 6. Validation

**입력 칸은 팝업에 있다.** 이 화면이 갖는 것은 **삭제 차단 규칙**이다.

| 대상 항목 | 검증 조건 | 검증 시점 | 오류 메시지 | 처리 방식 |
|---|---|---|---|---|
| 삭제 — 참조 있음 | 티켓 수 + 템플릿 수가 0이어야 한다 | 렌더 · 클릭 · 저장소 | 화면: 툴팁·aria-label 에 사용 표기(`티켓 {N} · 템플릿 {N}` — `apps/admin/src/pages/support/_shared/domain.ts:49`)가 사유로 실린다. 저장소: `사용 중인 문의 유형은 삭제할 수 없어요.` — `apps/admin/src/pages/support/_shared/store.ts:268` | 휴지통 비활성 + 사유 노출 |
| 없는 유형 조회 | 실재해야 한다 | 저장소 | `문의 유형을 찾을 수 없어요` — `apps/admin/src/pages/support/_shared/store.ts:233` | 조회 실패 배너 |
| 이름 중복 | ⚠ **검증하지 않는다** | — | — | 같은 이름의 유형을 여럿 만들 수 있다 |

## 7. 예외 처리

| 발생 상황 | 사용자 메시지 | 시스템 처리 | 복구 방법 |
|---|---|---|---|
| 조회 결과 없음 | `등록된 문의 유형이 없어요.` — `apps/admin/src/pages/support/categories/CategoriesPage.tsx:279`. **필터·검색이 없어 '조건에 맞는 것이 없다'는 갈래가 성립하지 않는다** | 목록 대신 한 줄 문구. 카드와 하단 안내는 남는다 | `유형 추가` |
| 저장 실패 | 팝업이 말한다: `저장하지 못했어요. 잠시 후 다시 시도해 주세요.` — `apps/admin/src/pages/support/categories/components/CategoryFormModal.tsx:93` | 팝업이 닫히지 않고 입력을 유지한다 | 팝업에서 다시 저장 |
| 수정 실패 | 저장 실패와 같은 경로다 — 팝업이 등록과 수정을 겸한다 | 〃 | 〃 |
| 삭제 실패 | `삭제하지 못했어요. 잠시 후 다시 시도해 주세요.` — `apps/admin/src/pages/support/categories/CategoriesPage.tsx:243` | **다이얼로그를 닫지 않고** 그 안 배너로 알린다 | 확인 재클릭이 재시도 |
| API 오류 | `문의 유형을 불러오지 못했어요.` — `apps/admin/src/pages/support/categories/CategoriesPage.tsx:268` | 카드 대신 배너 | `다시 시도` |
| 서버 오류 | 위와 같다 | 〃 | 〃 |
| 네트워크 오류 | 위와 같다 | **자동 재시도를 하지 않는다** — `apps/admin/src/shared/query/queryClient.ts:82` | 〃 |
| 권한 없음 | 읽기가 없으면 `접근 권한이 없어요` 화면 — `apps/admin/src/shared/errors/ErrorScreens.tsx:73`. ⚠ **쓰기 컨트롤에는 권한 판정이 붙어 있지 않다**(§9) | 사이드바는 남는다 | 관리자에게 권한 요청 |
| 세션 만료 | 로그인 화면이 `세션이 만료되었어요. 다시 로그인해 주세요.` 를 띄운다 — `apps/admin/src/pages/login/LoginPage.tsx:58` | 세션을 지우고 경로를 보존해 이동 | 재인증 후 이 화면 |
| 데이터 충돌 | 다른 관리자가 먼저 지웠거나 고쳤으면 그 자리(다이얼로그 또는 팝업)의 배너로 실패를 말한다. **전용 충돌 다이얼로그는 없다** | 덮어쓰지 않는다 | 목록을 다시 읽는다 |
| 중복 데이터 | ⚠ **막지 않는다** — 같은 이름의 유형을 여럿 만들 수 있다 | 그대로 저장된다 | 목록에서 눈으로 확인해 정리한다 |
| 파일 업로드 실패 | 해당 없음 — 이 화면에 파일 입력이 없다 | — | — |

**'0건'과 '모름'을 다르게 말한다** — 참조 배지는 **센 결과**다. 티켓·템플릿이 0이면 그 배지를 아예 그리지 않고 `사용 안 함` 하나만 남긴다. 아직 못 센 상태에서는 목록 자체가 `불러오는 중…` 이다.
