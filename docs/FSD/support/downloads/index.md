# 자료실 목록

## 1. 화면 개요

| 항목 | 내용 |
|---|---|
| 화면명 | 자료실 |
| 화면 ID | `SCR-SUPPORT-DOWNLOADS` |
| 메뉴 경로 | 고객센터 > 자료실 |
| 작성자 | Admin Hub 문서화 |
| 작성일 | 2026-07-22 |
| 버전 | 1.0 |

**화면 목적** — 고객에게 배포할 파일(카탈로그·매뉴얼·양식)을 등록해 두고, **노출 여부와 버전**을 관리한다.

**업무 배경** — 국내 자료실(게시판형 다운로드)의 축은 여섯이다: 제목·카테고리·첨부파일·**버전**·**다운로드수**·**노출여부**. 뒤의 셋이 일반 파일 목록과 자료실을 가른다 — `apps/admin/src/pages/support/downloads/types.ts:3-5`.

**화면 설명** — 검색 + 카테고리·노출 필터 툴바와 6열 표다. 노출 여부를 표에서 바로 토글할 수 있고, 등록·수정·삭제가 모두 있다.

**주요 사용자** — 최상위 관리자 · 고객센터 운영자 · 마케팅 담당.

**사용 시나리오**

1. `자료 등록` 으로 카탈로그 PDF 를 올리고 카테고리를 '카탈로그'로 둔다.
2. 개정본이 나오면 그 행을 열어 파일을 바꾸고 버전을 `v2.1` 로 올린다.
3. 아직 공개하면 안 되는 자료는 노출 토글을 꺼 둔다.

**선행 조건** — 로그인 · 이 화면의 읽기 권한.

**후행 처리** — 노출로 둔 자료가 고객센터 자료실에 보인다(고객 화면은 이 리포 밖이다).

**관련 화면** — [자료 등록·수정](form.md) `SCR-SUPPORT-DOWNLOADS-FORM` · 공통 규약은 [공통 확인 다이얼로그 · 토스트](../../_common/index.md).

**관련 메뉴** — 사이드바 `고객센터 > 자료실`.

**관련 기능** — 자료 카테고리는 **문의 유형과 별개다** — 자료실 고유의 고정 분류(카탈로그·제품 매뉴얼·양식/서식·브로슈어·기타)이고 운영자가 만들지 않는다 — `apps/admin/src/pages/support/downloads/types.ts:50-57`.

**관련 API** — 자료 목록 조회 1건 · 인라인 갱신 1건(노출 토글) · 삭제 N건. **백엔드가 없다** — 어댑터가 픽스처 위에 배선되어 있다.

**참고 문서** — [ID 규약](../../id-convention.md) · [인벤토리](../../inventory.md) · [품질 기준](../../../reference/quality-bar.md)

**구현 파일** — `apps/admin/src/pages/support/downloads/DownloadListPage.tsx` (도메인 규칙 `types.ts` · 어댑터 `data-source.ts`)

| 항목 | 내용 |

| 후속 동작 | 고객센터 자료실에서 사라진다 |

**'다운로드' 액션은 이 화면에 없다** — 관리자 화면에서 파일을 내려받는 컨트롤이 없다(등록한 파일을 다시 받을 길이 없다). 다운로드수는 고객 화면에서 늘어나는 값이다.

**⚠ 플랜(엔타이틀먼트) 매핑이 없다** — 모듈 표에 `/support/**` 가 없어 fail-open 으로 언제나 열린다 — `apps/admin/src/shared/entitlements/module-resources.ts`.

**권한이 없을 때 화면이 어떻게 되는가** — 읽기가 없으면 본문이 통째로 권한 화면이 된다. 쓰기 권한은 축별로 컨트롤을 지우고, **노출 토글만은 사라지지 않고 읽기 표시로 바뀐다**(값을 계속 볼 수 있어야 하기 때문이다).
