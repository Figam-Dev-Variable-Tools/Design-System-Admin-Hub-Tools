# 자료 등록·수정

## 1. 화면 개요

| 항목 | 내용 |
|---|---|
| 화면명 | 자료 등록 / 자료 수정 |
| 화면 ID | `SCR-SUPPORT-DOWNLOADS-FORM` |
| 메뉴 경로 | 고객센터 > 자료실 > (등록 · 수정) |
| 작성자 | Admin Hub 문서화 |
| 작성일 | 2026-07-22 |
| 버전 | 1.0 |

**화면 목적** — 배포할 자료 한 건의 **제목 · 카테고리 · 버전 · 노출 여부 · 첨부 파일**을 받아 만들거나 고친다.

**업무 배경** — 자료실의 첨부는 이미지만이 아니다(문서·압축본까지 받는다). 그래서 이미지 전용 업로드 규칙을 쓰지 않고 **자료실 전용 검증**을 따로 둔다 — `apps/admin/src/pages/support/downloads/types.ts:108-111`.

**화면 설명** — `/new` 와 `/:id/edit` 가 **같은 컴포넌트**다. 좌우 2단이다: 왼쪽 입력 카드(자료 정보 + 첨부), 오른쪽 자료 카드 미리보기.

**주요 사용자** — 최상위 관리자 · 고객센터 운영자 · 마케팅 담당.

**사용 시나리오**

1. `자료 등록` 으로 들어와 제목·카테고리를 정하고 PDF 를 끌어다 놓는다 → 저장.
2. 개정본을 올릴 때 버전을 `v2.1` 로 바꾼다.
3. 준비만 해 두고 아직 공개하지 않으려면 **노출 여부를 끄고** 저장한다.

**선행 조건** — 로그인 · 등록은 `create`, 수정은 `update` 권한.

**후행 처리** — 저장하면 자료실 목록으로 돌아간다. 노출로 둔 자료가 고객센터 자료실에 보인다.

**관련 화면** — [자료실 목록](index.md) `SCR-SUPPORT-DOWNLOADS` · 공통 규약은 [공통 확인 다이얼로그 · 토스트](../../_common/index.md).

**관련 메뉴** — 사이드바에 자기 자리가 없다.

**관련 기능** — 파일 종류(문서·이미지·압축·기타)는 **확장자에서 파생한다** — 저장 모델에 종류 필드가 있지만 사람이 고르지 않는다 — `apps/admin/src/pages/support/downloads/types.ts:76-86`.

**관련 API** — 자료 상세 조회 1건(수정 시) · 등록/갱신 1건. **백엔드가 없다** — 어댑터가 픽스처 위에 배선되어 있다.

**참고 문서** — [ID 규약](../../id-convention.md) · [인벤토리](../../inventory.md) · [품질 기준](../../../reference/quality-bar.md)

**구현 파일** — `apps/admin/src/pages/support/downloads/DownloadFormPage.tsx` (검증 `validation.ts` · 업로드 필드 `components/FileUploadField.tsx` · 미리보기 `components/DownloadPreview.tsx` · 도메인 규칙 `types.ts`)

## 4. 기능 명세

### 4.1 자료 등록

| 항목 | 내용 |
|---|---|
| 기능 목적 | 새 배포 파일을 올린다 |
| 실행 조건 | `/support/downloads/new` 진입 · `create` 권한 |
| 사용자 동작 | 값을 채우고 파일을 첨부한 뒤 `등록` 클릭 |
| 시스템 처리 | 공통 폼 컨트롤러가 검증 → 제출을 맡는다. **파일 종류는 파일명 확장자에서 파생해 함께 저장한다** |
| 데이터 처리 | 자료 1건이 생긴다. 다운로드수는 0에서 시작한다 |
| Validation | §6 전부 |
| API 호출 여부 | Y — 등록 1건. 재시도는 같은 멱등키를 재사용한다 — `apps/admin/src/shared/crud/useCrudForm.ts:122-150` |
| 성공 처리 | 목록으로 이동하고 공통 토스트 `MSG-COMMON-01`(success)이 뜬다 — 문구는 공통 폼 컨트롤러가 엔티티 이름으로 조립한다(엔티티에 `자료` 가 들어간다) — `apps/admin/src/shared/crud/useCrudForm.ts:249` |
| 실패 처리 | 폼 배너 `저장하지 못했어요. 잠시 후 다시 시도해 주세요.` — `apps/admin/src/shared/crud/useCrudForm.ts:221` + `오류 코드 {참조}` — `apps/admin/src/shared/crud/FormFeedback.tsx:45`. 이동하지 않고 입력을 유지한다 |
| 예외 처리 | 서버가 필드를 거절하면(422) 그 입력의 인라인 오류로 꽂히고 포커스가 옮겨간다 |
| 화면 변경 사항 | 안내영역 · 화면 전체(성공 시 이동) |
| 후속 동작 | 목록에 행이 생기고, 노출로 뒀으면 고객센터 자료실에 나타난다 |
| 로그 기록 여부 | 확인하지 못했다 |

### 4.2 자료 수정

| 항목 | 내용 |
|---|---|
| 기능 목적 | 제목·카테고리·버전·노출·파일을 고친다 |
| 실행 조건 | `/support/downloads/{id}/edit` 진입 · `update` 권한 |
| 사용자 동작 | 값을 고치고 `저장` 클릭 |
| 시스템 처리 | 상세를 조회해 폼을 채운 뒤 저장 시 한 벌을 통째로 갱신한다 |
| 데이터 처리 | **다운로드수는 이 폼이 손대지 않는다** — 자동 증가 값이라 입력 대상이 아니다 — `apps/admin/src/pages/support/downloads/types.ts:24-25` |
| Validation | §6 전부 |
| API 호출 여부 | Y — 조회 1건 + 갱신 1건 |
| 성공 처리 | 목록으로 이동하고 공통 토스트 `MSG-COMMON-01`(success) |
| 실패 처리 | §4.1 과 같다 |
| 예외 처리 | **파일을 바꿔도 다운로드수는 이어진다** — 같은 자료의 개정본으로 본다(버전 칸이 그 사실을 적는 자리다) |
| 화면 변경 사항 | 안내영역 · 화면 전체(성공 시 이동) |
| 후속 동작 | 목록의 그 행이 갱신된다 |
| 로그 기록 여부 | 확인하지 못했다 |

### 4.3 파일 첨부 (클릭 · 드래그)

| 항목 | 내용 |
|---|---|
| 기능 목적 | 배포할 파일을 붙인다 |
| 실행 조건 | 폼이 열려 있고 저장 중이 아닐 때 |
| 사용자 동작 | 드롭 영역 클릭(파일 선택창) 또는 **파일을 끌어다 놓기** |
| 시스템 처리 | 파일을 받으면 **자료실 전용 검증**을 먼저 돌리고, 통과하면 파일명과 크기를 폼 값으로 넣는다 — `apps/admin/src/pages/support/downloads/components/FileUploadField.tsx:150-151` |
| 데이터 처리 | 저장되는 것은 **파일명·크기·종류**다. 종류는 확장자에서 파생한다 |
| Validation | §6 의 파일 2규칙 |
| API 호출 여부 | N — **업로드 엔드포인트가 없다**(§7) |
| 성공 처리 | 드롭 영역이 첨부된 파일 표시로 바뀌고 미리보기에 반영된다 |
| 실패 처리 | 그 파일을 받지 않고 필드의 인라인 오류로 이유를 말한다(§6) |
| 예외 처리 | 끌고 있는 동안 드롭 영역 문구가 `여기에 놓으면 첨부돼요` 로 바뀐다. **드롭 영역이 `<button>` 이라 키보드로도 파일 선택창을 열 수 있다** |
| 화면 변경 사항 | 입력영역 · 미리보기 |
| 후속 동작 | 저장하면 목록의 '파일' 열에 `{파일명} · {종류} · {용량}` 으로 그려진다 |
| 로그 기록 여부 | N |

**⚠ 파일이 서버로 가지 않는다** — 이 앱에는 업로드 엔드포인트가 없다. 폼이 보관하는 것은 **파일명과 바이트 수**뿐이고 파일 자체는 어디에도 저장되지 않는다. 백엔드가 붙으면 이 이음매를 함께 만들어야 한다.

### 4.4 저장하지 않은 변경 보호

| 항목 | 내용 |
|---|---|
| 기능 목적 | 반쯤 채운 자료를 잃지 않게 한다 |
| 실행 조건 | 폼이 dirty 이고 저장 중이 아닐 때 |
| 사용자 동작 | 앱 안 링크 · 뒤로가기 · 탭 닫기 · `취소` |
| 시스템 처리 | 공통 이탈 가드 `DLG-COMMON-LEAVE-FORM` |

| Validation | 해당 없음 |

| 성공 처리 | 이동 또는 머무름. 토스트 없음 |

**이 화면 고유 문구** — `자료에 저장하지 않은 변경 사항이 있어요. 이 화면을 벗어나면 입력한 내용이 사라져요.` — `apps/admin/src/pages/support/downloads/DownloadFormPage.tsx:37`.

**이 화면에는 자기 토스트가 없다** — [인벤토리](../../inventory.md) §9.3 에 `pages/support/downloads/DownloadFormPage.tsx` 가 없다. 저장 통지는 공통 층이 낸다.

## 6. Validation

**규칙의 정본은 zod 스키마다** — `apps/admin/src/pages/support/downloads/validation.ts`. 파일 규칙만 순수 함수가 따로 갖는다(업로드 필드가 선택 즉시 부른다).

| 대상 항목 | 검증 조건 | 검증 시점 | 오류 메시지 | 처리 방식 |
|---|---|---|---|---|
| 제목 | 필수 · 100자 이하 | 제출 시 | 공통 필수 문구 생성기가 만든다(`requiredText('제목', 100)` — `apps/admin/src/pages/support/downloads/validation.ts:8`) | 제출 차단 · 그 필드로 포커스 |
| 카테고리 | 필수 · 30자 이하 | 〃 | `requiredText('카테고리', 30)` — `apps/admin/src/pages/support/downloads/validation.ts:9` | 〃 (셀렉트라 사실상 도달하지 않는다) |
| 버전 | 20자 이하 | 〃 | `버전은 20자를 넘을 수 없어요.` — `apps/admin/src/pages/support/downloads/validation.ts:12` | 〃 |
| 첨부 파일 — 필수 | 파일명이 있어야 한다 | 〃 | `requiredText('첨부 파일', 260)` — `apps/admin/src/pages/support/downloads/validation.ts:17` | 제출 차단 · 업로드 필드의 인라인 오류 |
| 첨부 파일 — 종류 | 문서·이미지·압축만 | **파일 선택·드롭 즉시** | `문서·이미지·압축 파일만 올릴 수 있어요.` — `apps/admin/src/pages/support/downloads/types.ts:115` | 그 파일을 받지 않는다 |
| 첨부 파일 — 용량 | 20MB 이하 | 〃 | `파일 용량은 20MB 를 넘을 수 없어요.` — `apps/admin/src/pages/support/downloads/types.ts:118` | 〃 |
| 노출 여부 | 검증하지 않는다(참/거짓뿐) | — | — | — |
| 파일 종류·크기 | 사람이 입력하지 않는다 | — | — | 파일명·파일 객체에서 파생한다 |
| 제목·파일명 중복 | ⚠ **검증하지 않는다** | — | — | 같은 제목·같은 파일명을 여럿 만들 수 있다 |

**허용 확장자** — 문서 `pdf`·`doc`·`docx`·`xls`·`xlsx`·`ppt`·`pptx`·`hwp`·`hwpx`·`txt`·`csv` · 이미지 `png`·`jpg`·`jpeg`·`gif`·`webp`·`svg` · 압축 `zip`·`rar`·`7z`·`tar`·`gz` — `apps/admin/src/pages/support/downloads/types.ts:60-74`. **확장자가 없거나 목록에 없으면 '기타'이고 그것은 거절된다.**

**이미지 전용 규칙과 다른 이유** — 공통 이미지 업로드는 이미지만 받고 5MB 상한이다. 자료실은 문서까지 허용하므로 **전용 규칙을 이 자원에 둔다** — `apps/admin/src/pages/support/downloads/types.ts:109-110`.

## 7. 예외 처리

| 발생 상황 | 사용자 메시지 | 시스템 처리 | 복구 방법 |
|---|---|---|---|
| 조회 결과 없음 | `자료를 찾을 수 없어요. 이미 삭제되었을 수 있어요.` — `apps/admin/src/pages/support/downloads/DownloadFormPage.tsx:167` | 폼 대신 배너 + `목록으로`. **재시도를 주지 않는다** | `목록으로` |
| 저장 실패 | `저장하지 못했어요. 잠시 후 다시 시도해 주세요.` + `오류 코드 {참조}` | 이동하지 않고 입력을 유지한다 | `등록`/`저장` 재클릭 · 계속 실패하면 오류 코드와 함께 신고 |
| 수정 실패 | 저장 실패와 같은 문구·같은 경로다 — 공통 폼이 등록과 수정을 한 훅으로 다룬다 | 〃 | 〃 |
| 삭제 실패 | 해당 없음 — 이 화면에는 삭제가 없다(목록의 일이다) | — | — |
| API 오류 | `자료를 불러오지 못했어요.` — `apps/admin/src/pages/support/downloads/DownloadFormPage.tsx:168`(조회) / 위 저장 실패 문구(쓰기) | 조회 실패는 `다시 시도` + `목록으로` | 〃 |
| 서버 오류 | 위와 같다 | 〃 | 〃 |
| 네트워크 오류 | 위와 같다 | **자동 재시도를 하지 않는다** — `apps/admin/src/shared/query/queryClient.ts:108` | 〃 |
| 권한 없음 | 등록이면 `이 항목을 등록할 권한이 없어요. 필요하다면 관리자에게 권한을 요청해 주세요.` · 수정이면 `이 항목을 수정할 권한이 없어요. …` — `apps/admin/src/shared/crud/useCrudForm.ts:294-295` | 검증도 요청도 시작하지 않고 폼 배너로 막는다 | 관리자에게 권한 요청 |

| 데이터 충돌 | `DLG-COMMON-FORM-CONFLICT` — 서버 사유가 없으면 `다른 사용자가 먼저 변경했어요. 최신 내용을 확인해 주세요.` — `apps/admin/src/shared/crud/useCrudForm.ts:197` | 성공 토스트도 이동도 없다(유령 저장 금지). 입력은 살아 있다 | '최신 내용 불러오기' 또는 '이어서 편집' |
| 중복 데이터 | ⚠ **같은 제목·같은 파일명을 막는 규칙이 없다** | 그대로 저장된다 | 목록에서 검색해 정리한다 |
