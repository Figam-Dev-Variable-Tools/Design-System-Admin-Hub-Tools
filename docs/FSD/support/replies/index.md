# 문의 답변(템플릿 목록)

## 1. 화면 개요

| 항목 | 내용 |
|---|---|
| 화면명 | 문의 답변 |
| 화면 ID | `SCR-SUPPORT-REPLIES` |
| 메뉴 경로 | 고객센터 > 문의 답변 |
| 작성자 | Admin Hub 문서화 |
| 작성일 | 2026-07-22 |
| 버전 | 1.0 |

**화면 목적** — 자주 쓰는 답변을 **템플릿**으로 모아 두고, 티켓 상세에서 골라 삽입할 수 있게 한다.

**업무 배경** — 상용구를 손으로 매번 쓰면 문장이 상담원마다 달라지고, 반대로 통째로 복사하면 로봇 같은 답변이 나간다. 그래서 본문에 **치환 변수**(`{{고객명}}` · `{{문의번호}}` · `{{담당자}}`)를 두고 티켓에 삽입할 때 실제 값으로 채운다 — `apps/admin/src/pages/support/replies/ReplyFormPage.tsx:4-5`.

**화면 설명** — 검색 툴바와 3열 표다. 선택 체크박스와 삭제가 있는 일반 CRUD 목록이고 **등록 버튼이 있다**. 행을 누르면 수정 폼이 열린다.

**주요 사용자** — 최상위 관리자 · 고객센터 운영자.

**사용 시나리오**

1. `템플릿 등록` 으로 '배송 지연 사과 안내'를 만들고 유형 태그를 '주문/결제'로 단다.
2. 티켓 상세에서 그 유형의 티켓을 열면 이 템플릿이 후보로 뜬다.
3. 더는 쓰지 않는 템플릿을 골라 지운다.

**선행 조건** — 로그인 · 이 화면의 읽기 권한.

**후행 처리** — 만든 템플릿이 [1:1 문의 상세](../tickets/detail.md)의 `답변 템플릿 삽입` 셀렉트에 나타난다. **유형 태그가 후보를 거른다** — 그 유형 티켓과 '전체' 공용만 온다.

**관련 화면** — [답변 템플릿 등록·수정](form.md) `SCR-SUPPORT-REPLIES-FORM` · [1:1 문의 상세](../tickets/detail.md) `SCR-SUPPORT-TICKETS-DETAIL`(템플릿을 쓰는 곳) · [문의 유형](../categories/index.md) `SCR-SUPPORT-CATEGORIES`(태그의 원천) · 공통 규약은 [공통 확인 다이얼로그 · 토스트](../../_common/index.md).

**관련 메뉴** — 사이드바 `고객센터 > 문의 답변`.

**관련 기능** — 문의 유형 화면의 `템플릿 {N}` 배지를 누르면 이 목록이 열린다. ⚠ **다만 유형 필터가 걸리지 않는다** — 이 화면에 그 축이 없기 때문이다.

**관련 API** — 템플릿 목록 조회 1건 · 삭제 N건. **백엔드가 없다** — 어댑터가 픽스처 위에 배선되어 있다.

**참고 문서** — [ID 규약](../../id-convention.md) · [인벤토리](../../inventory.md) · [품질 기준](../../../reference/quality-bar.md)

**구현 파일** — `apps/admin/src/pages/support/replies/RepliesPage.tsx` (도메인 규칙 `../_shared/domain.ts` · 저장소 `../_shared/store.ts` · 어댑터 `data-source.ts`)

| 선택 있음 | 선택 바에 건수 | 일괄 삭제 ○ | N | — |
| 공용 템플릿 | 유형 태그 배지가 `전체`(neutral) — **'태그가 없다'가 아니라 '모든 유형에 노출'이다** | 〃 | N | — |

**⚠ 플랜(엔타이틀먼트) 매핑이 없다** — 모듈 표에 `/support/**` 가 없어 fail-open 으로 언제나 열린다 — `apps/admin/src/shared/entitlements/module-resources.ts`.

**권한이 없을 때 화면이 어떻게 되는가** — 읽기가 없으면 본문이 통째로 권한 화면이 된다. 쓰기 권한은 축별로 컨트롤을 지운다(비활성이 아니다). 고객센터에서 **쓰기 권한을 실제로 읽는 유일한 목록 화면이다**(문의 유형 화면은 읽지 않는다).

## 10. 화면 이동(Screen Flow)

| 현재 화면 | 이벤트 | 이동 화면 | 팝업 여부 | 이동 조건 | 전달 파라미터 | 복귀 화면 |
|---|---|---|---|---|---|---|
| `SCR-SUPPORT-REPLIES` | 행 클릭 · 연필 | `SCR-SUPPORT-REPLIES-FORM` | N | 수정 권한 | 템플릿 id | 조회 조건이 보존된 이 목록 |
| `SCR-SUPPORT-REPLIES` | `템플릿 등록` | 〃 | N | 등록 권한 | 없음 | 이 목록 |
| `SCR-SUPPORT-REPLIES` | 행 삭제 | `DLG-COMMON-DELETE-ONE` | Y | 삭제 권한 | 대상 제목 | 이 목록 |
| `SCR-SUPPORT-REPLIES` | 일괄 삭제 | `DLG-COMMON-DELETE-BULK` | Y | 1건 이상 선택 | 선택 id 목록·건수 | 이 목록 |

**이 목록으로 오는 다른 문** — 문의 유형 화면의 `템플릿 {N}` 배지. ⚠ 그 링크는 **유형 필터 없이** 전체 목록을 연다.

**이 화면에는 자기 팝업이 없다** — 뜨는 것은 공통 삭제 확인 2종뿐이다.
