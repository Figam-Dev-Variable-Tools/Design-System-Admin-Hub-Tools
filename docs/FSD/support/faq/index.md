# 자주 묻는 질문(고객노출 큐레이션)

## 1. 화면 개요

| 항목 | 내용 |
|---|---|
| 화면명 | 자주 묻는 질문 |
| 화면 ID | `SCR-SUPPORT-FAQ` |
| 메뉴 경로 | 고객센터 > 자주 묻는 질문 |
| 작성자 | Admin Hub 문서화 |
| 작성일 | 2026-07-22 |
| 버전 | 1.0 |

**화면 목적** — 콘텐츠 관리에서 쓴 FAQ 를 **고객센터에 어떻게 보일지**만 정한다: 표시 순서 · 노출 여부 · BEST 고정.

**업무 배경** — 예전에는 이 화면이 **자기 FAQ 배열과 자기 카테고리 어휘**를 따로 들고 있었다. 콘텐츠 관리의 카테고리와 애초에 맞출 수 없었고, **콘텐츠 관리에서 쓴 FAQ 는 이 목록에 영영 뜨지 않았다** — 안내 문구만 두 화면이 이어져 있다고 말하고 있었다. 지금은 행이 콘텐츠 FAQ 에서 **파생**되고 이 모듈은 그 위에 얹는 큐레이션 값만 소유한다 — `apps/admin/src/pages/support/faq/types.ts:8-13`.

**화면 설명** — 상단 info 배너(콘텐츠 관리와의 역할 분담 + 링크), 건수 문구, 그리고 큐레이션 표 하나다. **등록·수정·삭제가 없다** — 작성은 콘텐츠 관리 소관이다.

**주요 사용자** — 최상위 관리자 · 고객센터 운영자.

**사용 시나리오**

1. 콘텐츠 관리에서 FAQ 를 쓴다 → 이 목록에 자동으로 올라온다(기본 노출).
2. 자주 묻는 두 건을 **BEST 로 고정**하고 순서를 위로 올린다.
3. 계절이 지난 FAQ 를 **숨김으로 내린다** — 콘텐츠 관리에서는 그대로 살아 있다.

**선행 조건** — 로그인 · 이 화면의 읽기 권한. **그리고 FAQ 원본 조회기가 배선되어 있어야 한다**(§7).

**후행 처리** — 순서·노출·고정이 고객센터 화면의 FAQ 표시에 반영된다(고객 화면은 이 리포 밖이다).

**관련 화면** — `SCR-CONTENT-FAQ`(작성·수정·삭제·카테고리를 맡는 곳) · 공통 규약은 [공통 확인 다이얼로그 · 토스트](../../_common/index.md).

**관련 메뉴** — 사이드바 `고객센터 > 자주 묻는 질문`.

**관련 기능** — 이 화면은 **콘텐츠 관리 모듈을 끝까지 모른다.** 원본은 공통 층의 조회기가 주고, 그 자리에 구현을 꽂는 것은 두 도메인을 아는 부팅 코드다 — `apps/admin/src/pages/support/faq/data-source.ts:3-5`.

**관련 API** — 큐레이션 목록 조회 1건 · 순서 저장 1건 · 노출 토글 1건 · 고정 토글 1건. **백엔드가 없다** — `TODO(backend): GET /api/support/faq (고객노출 순서·플래그) · PUT /api/support/faq/order · PATCH /api/support/faq/:id` — `apps/admin/src/pages/support/faq/data-source.ts:46`.

**참고 문서** — [ID 규약](../../id-convention.md) · [인벤토리](../../inventory.md) · [품질 기준](../../../reference/quality-bar.md)

**구현 파일** — `apps/admin/src/pages/support/faq/CustomerFaqPage.tsx` (표 `components/CustomerFaqTable.tsx` · 파생 규칙 `types.ts` · 데이터 `data-source.ts` · 원본 이음매 `apps/admin/src/shared/domain/faq-catalog.ts`)

### 1.1 두 축이 갈라져 있다 — 콘텐츠 노출 ≠ 고객센터 노출

| 축 | 소유 화면 | 무엇을 정하는가 |
|---|---|---|
| 질문·답변·카테고리 · 콘텐츠 노출 | `SCR-CONTENT-FAQ` | FAQ 자체가 존재하는가 |
| **표시 순서 · 고객센터 노출 · BEST 고정** | **이 화면** | 그 FAQ 가 고객센터에서 어떻게 보이는가 |

**큐레이션 레코드는 손댄 FAQ 에만 생긴다** — 콘텐츠 관리에서 FAQ 를 하나 쓰면 그것은 곧바로 이 목록에 뜨고(기본 노출), 운영자가 손대는 순간에만 레코드가 만들어진다. 모든 FAQ 에 대해 레코드를 미리 만들어 두면 원본이 늘 때마다 두 곳을 함께 고쳐야 하고, 그 동기화를 한 번 놓치면 **'콘텐츠에는 있는데 고객센터에는 없는 FAQ'가 다시 생긴다** — `apps/admin/src/pages/support/faq/types.ts:16-22`.

## 6. Validation

**이 화면에는 입력 칸이 하나도 없다** — 토글 둘과 순서 이동뿐이라 검증할 값이 없다. FAQ 의 질문·답변 검증은 `SCR-CONTENT-FAQ` 가 갖는다.

| 대상 항목 | 검증 조건 | 검증 시점 | 오류 메시지 | 처리 방식 |
|---|---|---|---|---|
| 노출·BEST 토글 | 참/거짓뿐이라 검증하지 않는다 | — | — | — |
| 순서 | 목록 안의 id 만 다시 배열한다 | 저장 시 | — | 알 수 없는 id 는 순서 계산에서 무시된다 |
