# 1:1 문의 목록

## 1. 화면 개요

| 항목 | 내용 |
|---|---|
| 화면명 | 1:1 문의 |
| 화면 ID | `SCR-SUPPORT-TICKETS` |
| 메뉴 경로 | 고객센터 > 1:1 문의 |
| 작성자 | Admin Hub 문서화 |
| 작성일 | 2026-07-22 |
| 버전 | 1.0 |

**화면 목적** — 들어온 고객 문의(티켓)를 **트리아지**한다. 상태·우선순위·채널·유형 네 축을 조합해 지금 볼 큐를 만들고, 한 건씩 열어 처리한다.

**업무 배경** — 고객센터는 마감시계가 보이지 않는 일이다. 그래서 우선순위별 **첫 응답 목표시간(SLA)** 을 환산해 `SLA 임박`·`SLA 초과` 배지로 바꾼다 — `apps/admin/src/pages/support/_shared/domain.ts:172-175`.

**화면 설명** — 툴바에 검색과 셀렉트 넷, 그 아래 9열 읽기 전용 표다. **등록·삭제·일괄 작업이 없다** — 문의는 고객 채널이 만들고 관리자는 처리·답변만 한다.

**주요 사용자** — 최상위 관리자 · 상담원 · 고객센터 운영자.

**사용 시나리오**

1. 상태 '접수' + 우선순위 '긴급'으로 걸러 큐를 만든다 → 그 주소를 그대로 둔 채 한 건씩 열고 닫는다(Back 을 눌러도 큐가 살아 있다).
2. SLA 열이 `SLA 초과`(붉은)인 행부터 집는다.
3. 고객이 문의번호를 대며 전화한다 → 검색창에 문의번호를 넣어 그 행을 연다.

**선행 조건** — 로그인 · 이 화면의 읽기 권한.

**후행 처리** — 이 목록은 아무것도 바꾸지 않는다. 담당 배정·상태 전이·답변은 전부 [상세](detail.md)의 일이다.

**관련 화면** — [1:1 문의 상세](detail.md) `SCR-SUPPORT-TICKETS-DETAIL` · [문의 유형](../categories/index.md) `SCR-SUPPORT-CATEGORIES`(유형 필터의 원천) · [문의 답변](../replies/index.md) `SCR-SUPPORT-REPLIES`(상세가 삽입하는 템플릿) · 공통 규약은 [공통 확인 다이얼로그 · 토스트](../../_common/index.md).

**관련 메뉴** — 사이드바 `고객센터 > 1:1 문의`.

**관련 기능** — 상태값과 그 순서는 **영업 문의와 공유한다**(`apps/admin/src/shared/domain/inquiry-status.ts`). 티켓이 더하는 상태는 없어 공통 유니온을 그대로 쓰고, **문구·색·전이 규칙만 고객센터의 것이다** — `apps/admin/src/pages/support/_shared/domain.ts:56-62`.

**관련 API** — 티켓 목록 조회 1건. **백엔드가 없다** — 어댑터가 픽스처 위에 배선되어 있다.

**참고 문서** — [ID 규약](../../id-convention.md) · [인벤토리](../../inventory.md) · [품질 기준](../../../reference/quality-bar.md)

**구현 파일** — `apps/admin/src/pages/support/tickets/TicketListPage.tsx` (도메인 규칙 `../_shared/domain.ts` · 저장소 `../_shared/store.ts` · 어댑터 `data-source.ts`)

**등록·삭제 액션이 이 자원에 없다** — 문의는 고객 채널이 만들고 관리자 화면에 생성·삭제 경로가 없다. 그래서 `create`·`remove` 권한은 이 화면에서 아무것도 바꾸지 않는다.

**⚠ 플랜(엔타이틀먼트) 매핑이 없다** — 모듈 표에 `/support/**` 가 한 줄도 없어(`apps/admin/src/shared/entitlements/module-resources.ts`) **fail-open** 으로 어떤 플랜에서도 이 화면이 잠기지 않는다. 상품(`commerce.products`)·영업(`sales.pipeline`)과 다른 자리다.

**미결(플랜 잠금)** — 고객센터를 어느 등급부터 열지는 상용 패키징 결정이라 **아직 정하지 않는다**(운영자 확정: 지금은 미결로 남긴다). 그때까지 `/support/**` 는 모든 플랜에서 열린다(fail-open). 패키징이 정해지면 `module-resources.ts` 에 매핑을 추가해 플랜별 잠금이 걸린다.

**권한이 없을 때 화면이 어떻게 되는가** — 읽기가 없으면 본문이 통째로 권한 화면이 된다. 쓰기 권한만 없는 경우 이 목록은 그대로 보이고, 막히는 것은 상세의 저장이다.
