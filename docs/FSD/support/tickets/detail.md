# 1:1 문의 처리(상세)

## 1. 화면 개요

| 항목 | 내용 |
|---|---|
| 화면명 | 문의 처리 |
| 화면 ID | `SCR-SUPPORT-TICKETS-DETAIL` |
| 메뉴 경로 | 고객센터 > 1:1 문의 > (행 선택) |
| 작성자 | Admin Hub 문서화 |
| 작성일 | 2026-07-22 |
| 버전 | 1.0 |

**화면 목적** — 티켓 한 건의 **담당 배정 · 상태 전이 · 답변/메모 작성**을 한 번의 저장으로 처리한다. 답변 템플릿을 골라 채우고, 마지막에 종결한다.

**업무 배경** — 고객센터의 상태는 **책임 이관 흐름**이다(접수 → 배정 → 처리중 → 답변완료 → 종결). 불가능한 전이를 막고, **담당 없이 처리중·답변완료로 갈 수 없게** 한다 — `apps/admin/src/pages/support/_shared/domain.ts:9-12,282-285`.

**화면 설명** — 좌우 2단이다. 왼쪽 카드에 티켓 정보·SLA 배지·담당/상태 입력·답변 작성칸(템플릿 선택 포함)이, 오른쪽에 처리 이력 타임라인이 있다. 저장 버튼 하나가 세 변경을 함께 커밋한다.

**주요 사용자** — 최상위 관리자 · 상담원 · 고객센터 운영자.

**사용 시나리오**

1. 접수된 티켓을 열어 담당자를 넣고 상태를 '처리중'으로 올린 뒤 내부메모를 남긴다 → `처리 저장`.
2. 답변 템플릿을 고른다 → **`{{고객명}}`·`{{문의번호}}`·`{{담당자}}` 가 실제 값으로 채워진 채** 작성칸에 들어온다 → 다듬어 저장한다.
3. 상태를 '종결'로 바꾸고 저장한다 → **확인 다이얼로그가 뜬다**(종결은 되돌릴 수 없다).

**선행 조건** — 로그인 · 읽기 권한. 저장하려면 수정 권한이 필요하다.

**후행 처리** — 담당·상태·타임라인이 저장되고 목록의 상태·담당·SLA 열에 반영된다.

**관련 화면** — [1:1 문의 목록](index.md) `SCR-SUPPORT-TICKETS` · [문의 답변](../replies/index.md) `SCR-SUPPORT-REPLIES`(템플릿의 원천) · [문의 유형](../categories/index.md) `SCR-SUPPORT-CATEGORIES`(템플릿을 거르는 축) · 공통 규약은 [공통 확인 다이얼로그 · 토스트](../../_common/index.md).

**관련 메뉴** — 사이드바에 자기 자리가 없다.

**관련 기능** — 템플릿은 **이 티켓의 유형에 맞는 것 + 전체 공용**만 후보로 온다 — `apps/admin/src/pages/support/_shared/domain.ts:414-422`. 치환은 **삽입 시점**에 일어난다(발송 시점이 아니다) — 마케팅 발송 템플릿의 `#{...}` 문법과 다른 이유가 그것이다.

**관련 API** — 티켓 상세 조회 1건 · 티켓 갱신 1건. **백엔드가 없다** — 어댑터가 픽스처 위에 배선되어 있다.

**참고 문서** — [ID 규약](../../id-convention.md) · [인벤토리](../../inventory.md) · [품질 기준](../../../reference/quality-bar.md)

**구현 파일** — `apps/admin/src/pages/support/tickets/TicketDetailPage.tsx` (본문 `components/TicketWorkspace.tsx` · 타임라인 `components/TicketTimeline.tsx` · 순수 규칙 `process.ts` · 도메인 규칙 `../_shared/domain.ts`)

### 4.4 문의 종결 (확인을 세우는 유일한 저장)

| 항목 | 내용 |
|---|---|
| 기능 목적 | 되돌릴 수 없는 종결에 확인을 한 번 세운다 |
| 실행 조건 | **이번 저장이 종결로 넘기는 경우에만**(이미 종결된 티켓은 해당하지 않는다) — `apps/admin/src/pages/support/tickets/TicketDetailPage.tsx:133` |
| 사용자 동작 | 상태를 '종결'로 바꾸고 `처리 저장` → 다이얼로그의 `종결` |
| 시스템 처리 | 저장을 곧장 커밋하지 않고 확인 다이얼로그를 세운다. 확인하면 §4.2 의 저장 경로를 탄다 |
| 데이터 처리 | 상태가 '종결'이 되고 타임라인에 상태 변경 이벤트가 남는다 |
| Validation | §6 의 전이 규칙 |
| API 호출 여부 | Y — 갱신 1건(확인 후) |
| 성공 처리 | 다이얼로그가 닫히고 §4.2 의 성공 토스트가 뜬다 |
| 실패 처리 | **다이얼로그가 닫히지 않고** 그 안 배너로 실패를 말한다. 확인 재클릭이 재시도다 — `apps/admin/src/pages/support/tickets/TicketDetailPage.tsx:246` |
| 예외 처리 | 취소하면 진행 중 요청을 중단하고 다이얼로그만 닫는다 — `apps/admin/src/pages/support/tickets/TicketDetailPage.tsx:248-251` |
| 화면 변경 사항 | 다이얼로그 · 상태 셀렉트(선택지가 비게 된다) · 이력영역 |
| 후속 동작 | 목록의 상태가 '종결'이 되고 SLA 가 `응답완료` 로 멈춘다 |
| 로그 기록 여부 | 확인하지 못했다 |

**되돌릴 수 있는 저장에는 확인을 붙이지 않는다** — 담당 배정·답변 등록·처리중 전환까지 확인을 세우면 확인이 무뎌진다. 종결만 **종착**이라(그 상태에서 갈 곳이 없다) 이 한 자리에만 세운다 — `apps/admin/src/pages/support/tickets/TicketDetailPage.tsx:129-132`.

## 6. Validation

**전이 규칙의 정본은 도메인 모듈이다** — `apps/admin/src/pages/support/_shared/domain.ts`. 셀렉트가 그리는 선택지와 저장의 허용 판정이 **같은 표**를 읽는다.

| 대상 항목 | 검증 조건 | 검증 시점 | 오류 메시지 | 처리 방식 |
|---|---|---|---|---|
| 상태 전이 — 허용된 칸 | 접수 → 배정·종결 · 배정 → 처리중·종결 · 처리중 → 답변완료·종결 · **답변완료 → 종결·처리중**(되돌아갈 수 있는 유일한 칸이다) · 종결 → **없음(종착)** — `apps/admin/src/pages/support/_shared/domain.ts:265-271` | 셀렉트 렌더 · 저장 시 | 선택지에 없어 고를 수 없다. 그래도 도달하면 `허용되지 않는 상태 전이예요.` — `apps/admin/src/pages/support/tickets/TicketDetailPage.tsx:140` | 셀렉트 후보에서 제외 + 저장 거절 |
| 담당 요건 | **처리중·답변완료는 담당 배정 없이 갈 수 없다** | 입력 중 · 저장 시 | `'{상태}' 상태는 담당 배정이 필요해요.` — `apps/admin/src/pages/support/tickets/process.ts:30` | 담당 칸의 인라인 오류 + **저장 버튼 비활성** |
| 작성칸 | 1,000자 이하 — `apps/admin/src/pages/support/_shared/domain.ts:100` | 입력 중 | 없음 — 입력이 상한에서 멈춘다 | 더 입력되지 않는다 |
| 담당자 이름 | 형식 검증이 없다. 저장 시 앞뒤 공백만 다듬는다 | 제출 시 | — | 빈 값이면 '미배정'으로 저장된다 |
| 변경 여부 | 담당·상태·작성칸 중 하나는 달라야 한다 | 렌더 | 없음 | 저장 버튼 비활성 |

**이 화면에는 zod 스키마가 없다** — 저장 대상이 세 값뿐이라 폼 라이브러리를 얹지 않았다. 규칙은 순수 함수 둘(`canSetStatus` · `assigneeError`)이 갖는다.

**등록·삭제 액션이 없다** — 문의는 고객이 만들고 이 화면에는 삭제가 없다.

**⚠ 이 화면은 쓰기 권한으로 컨트롤을 가리지 않는다** — `useRouteWritePermissions` 호출이 없어 입력칸과 저장 버튼이 언제나 그려진다. 형제 화면(프로그램 문의 상세·청구 상세)이 권한으로 쓰기 컨트롤을 감추는 것과 다른 자리다. 막는 것은 서버뿐이다.

**⚠ 플랜(엔타이틀먼트) 매핑이 없다** — 모듈 표에 `/support/**` 가 없어 fail-open 으로 언제나 열린다 — `apps/admin/src/shared/entitlements/module-resources.ts`.

**권한이 없을 때 화면이 어떻게 되는가** — 읽기가 없으면 본문이 통째로 권한 화면이 된다. 수정 권한만 없으면 화면은 그대로 열리고 저장이 실패한다.
