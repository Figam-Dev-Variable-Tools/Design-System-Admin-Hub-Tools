# 기능 명세서 템플릿

> **이 파일은 템플릿이다.** 복사해서 `docs/FSD/<폴더>/<잎>/index.md`(또는 `detail.md`·`form.md`·`tab-*.md`·`pop-*.md`)
> 로 저장하고 `〈…〉` 를 채운다. **인용부호(`>`)로 시작하는 안내 줄은 전부 지운다.**
>
> **절 이름과 순서를 바꾸지 않는다.** 열 절 전부를 갖는다 — 해당 없는 절은 지우지 말고 `해당 없음 — 〈사유〉` 를 적는다.
>
> 작성 원칙: 기획자 관점 · UI 중심 · 표 형식 우선 · 기능별 성공/실패/예외 필수 ·
> **API 는 호출 목적과 연계 여부만**(구현 로직 금지) · **DB 설계·SQL·알고리즘 금지** ·
> 개발자와 QA 가 같은 기준으로 구현·테스트할 수준의 상세함.
>
> 화면 ID · 폴더 · 파일 이름: [`id-convention.md`](id-convention.md) · 대상 목록: [`inventory.md`](inventory.md)
