# 이메일 발송

> **뼈대는 [SMS 발송](../sms/index.md) 과 같다** — 수신자 세그먼트 · 발신자 레지스트리 · 발송 상태 ·
> 예약 · 상태별 허용 액션 · 조회 조건의 주소 왕복 · 공용 목록 껍데기가 전부 한 벌에서 나온다.
> **이 문서는 그 한 벌을 다시 적지 않고 이 화면의 차이만 적는다.** 차이는 셋이다:
> **제목 열이 있고 · 유형(SMS/LMS/MMS) 축이 없으며 · 결과가 성공률이 아니라 오픈율/클릭율**이다.

## 1. 화면 개요

| 항목 | 내용 |
|---|---|
| 화면명 | 이메일 발송 |
| 화면 ID | `SCR-MARKETING-EMAIL` |
| 메뉴 경로 | 비즈니스 > 마케팅 관리 > 이메일 발송 |
| 작성자 | Admin Hub 문서화 |
| 작성일 | 2026-07-22 |
| 버전 | 1.0 |

**화면 목적** — 이메일 발송 캠페인을 한 목록에서 보고, 제목·대상 수·발송 상태·오픈율/클릭율을 확인하며 등록·수정·삭제한다.

**업무 배경** — 문자와 달리 이메일은 **열렸는지**가 성패를 가른다. 그래서 이 목록의 결과 열은 도달률이 아니라 오픈율과 클릭율이다.

**화면 설명** — 배치는 SMS 발송과 같다. 열은 발송명 · **제목** · 대상수 · 발송상태 · **오픈율/클릭율**이다.

**주요 사용자** — 최상위 관리자 · 마케팅 담당 운영자.

**사용 시나리오**

1. 상태 필터를 `예약` 으로 좁혀 아직 나가지 않은 발송만 본다.
2. 제목 열에서 수신함에 뜰 문장을 확인한다.
3. 발송완료 행의 오픈율/클릭율을 비교한다.
4. `이메일 발송 등록` 으로 새 캠페인을 만든다.

**선행 조건** — 로그인. 이 화면의 읽기 권한. **`이메일·뉴스레터 발송` 모듈이 이 계정에 있어야 한다**(§9). 발송하려면 도메인 인증이 끝난 발신자가 있어야 한다.

**후행 처리** — 등록·수정한 캠페인이 이 목록에 반영된다. **이 화면에서 메일이 나가지는 않는다.**

**관련 화면** — [이메일 발송 등록·수정](form.md)(`SCR-MARKETING-EMAIL-FORM`) · 같은 뼈대의 [SMS 발송](../sms/index.md) · [뉴스레터](../newsletters/index.md) · 본문을 채우는 [발송 템플릿 관리](../templates/index.md). 공통 규약은 [공통 확인 다이얼로그 · 토스트](../../_common/index.md).

**관련 메뉴** — 사이드바 `비즈니스 > 마케팅 관리` 의 다섯 번째 항목 — `apps/admin/src/shared/layout/nav-config.ts:225`. **플랜으로 잠긴다** — `marketing.email` 이 이 경로와 뉴스레터를 함께 지배하며 최소 티어는 `베이직` 이다 — `apps/admin/src/shared/entitlements/module-resources.ts:75-79` · `apps/admin/src/shared/entitlements/plan.ts:226-232`. **SMS 와는 다른 키다**(§9).

**관련 기능** — 시스템 설정의 사이트 기본 설정이 정하는 '메일·SMS 전용 사이트 이름'이 **메일 제목 앞에 붙는다**([이메일 발송 등록·수정](form.md)). 발송 템플릿 관리가 본문의 원천이다.

**관련 API** — [SMS 발송](../sms/index.md) 과 같은 다섯. 실제 전송은 하지 않는다 — `apps/admin/src/pages/marketing/email/data-source.ts` 의 `TODO(backend)` 주석뿐이다.

**참고 문서** — [SMS 발송](../sms/index.md)(뼈대 정본) · [ID 규약](../../id-convention.md) · [인벤토리](../../inventory.md)

**구현 파일** — `apps/admin/src/pages/marketing/email/EmailListPage.tsx` · `email/types.ts` · `email/data-source.ts` · 공용 모델 `apps/admin/src/pages/marketing/_shared/messaging.ts`

## 5. 이벤트 정의

| 이벤트 | 발생 조건 | 처리 내용 | 대상 컴포넌트 | 결과 화면 |
|---|---|---|---|---|

**이 화면 고유의 확인 다이얼로그는 없다.**
