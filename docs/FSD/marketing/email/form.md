# 이메일 발송 등록·수정

> **`/marketing/email/new` 와 `/marketing/email/:id/edit` 는 같은 화면이다.**
>
> **뼈대는 [SMS 발송 등록·수정](../sms/form.md) 과 같다** — 발송 정보 → 수신자 → 본문 → 발송 예약
> 카드 + 우측 미리보기, 세그먼트 선택기 · 발신자 검증 · 예약 검증 · 템플릿 불러오기 · 치환변수 ·
> 저장 · 이탈 가드 · 충돌이 전부 한 벌에서 온다.
> **이 문서는 그 한 벌을 다시 적지 않고 이 화면의 차이만 적는다.** 차이는 넷이다:
> **제목 칸이 있고 · 광고 요건이 본문이 아니라 제목에 걸리며 · 수신거부 링크 토글이 있고 ·
> 바이트·유형 축이 없다.**

## 1. 화면 개요

| 항목 | 내용 |
|---|---|
| 화면명 | 이메일 발송 등록·수정 |
| 화면 ID | `SCR-MARKETING-EMAIL-FORM` |
| 메뉴 경로 | 비즈니스 > 마케팅 관리 > 이메일 발송 > 등록·수정 (사이드바에 자리가 없다) |
| 작성자 | Admin Hub 문서화 |
| 작성일 | 2026-07-22 |
| 버전 | 1.0 |

**화면 목적** — 이메일 발송 캠페인의 발송명·제목·발신자·수신자 세그먼트·광고성 여부·본문·수신거부 링크·예약을 입력해 만들거나 고친다.

**업무 배경** — 메일에는 화면이 막아야 하는 경계가 셋 있다: ① 도메인 인증이 끝나지 않은 발신자(스팸함으로 간다) ② 광고 메일의 제목 `(광고)` 표기 ③ 수신거부 링크 누락. 셋 다 저장 전에 막는다(§6).

**화면 설명** — 배치는 SMS 발송 폼과 같다. 다른 것은 발송 정보 카드에 **제목** 칸이 하나 더 있고, 본문 카드에 **수신거부 링크 토글**이 있으며, 미리보기가 휴대폰이 아니라 메일 화면이라는 점이다.

**주요 사용자** — 최상위 관리자 · 마케팅 담당 운영자(등록·수정 권한 보유자).

**사용 시나리오**

1. 발송명과 제목을 적고 도메인 인증이 끝난 발신자를 고른다.
2. 수신자 세그먼트를 고르면 대상 수 합계가 뜬다.
3. 이메일 템플릿을 불러오면 **제목과 본문이 함께** 채워진다.
4. 광고성을 켜면 제목이 `(광고)` 로 시작하지 않는 한 경고가 뜬다.
5. `초안 저장` 또는 `예약 저장` 으로 저장한다 — **메일이 즉시 나가지는 않는다.**

**선행 조건** — 로그인. 등록은 `create`, 수정은 `update` 권한 + 상태 조건. **`이메일·뉴스레터 발송` 모듈**이 이 계정에 있어야 한다. 도메인 인증이 끝난 발신자가 있어야 저장할 수 있다.

**후행 처리** — 저장한 캠페인이 [이메일 발송 목록](index.md) 에 반영된다. **전송은 일어나지 않는다.**

**관련 화면** — [이메일 발송 목록](index.md)(`SCR-MARKETING-EMAIL`) · 본문의 원천 [발송 템플릿 관리](../templates/index.md) · 같은 뼈대의 [SMS 발송 등록·수정](../sms/form.md) · [뉴스레터 등록·수정](../newsletters/form.md). 공통 규약은 [공통 확인 다이얼로그 · 토스트](../../_common/index.md).

**관련 메뉴** — 사이드바에 자기 자리가 없다. 권한 리소스는 잎 `/marketing/email` 이 덮는다.

**관련 기능** — 발송 템플릿 관리의 발행된 이메일 템플릿이 '템플릿 불러오기'의 목록이다.

**관련 API** — 목적 셋(단건 조회 · 등록 · 수정). 실제 전송 호출은 없다 — `apps/admin/src/pages/marketing/email/data-source.ts` 의 `TODO(backend)` 주석뿐이다.

**참고 문서** — [SMS 발송 등록·수정](../sms/form.md)(뼈대 정본) · [ID 규약](../../id-convention.md) · [인벤토리](../../inventory.md)

**구현 파일** — `apps/admin/src/pages/marketing/email/EmailFormPage.tsx` · 본문 카드 `email/components/EmailBodyCard.tsx` · 검증 `email/validation.ts` · 미리보기 `apps/admin/src/pages/marketing/_shared/EmailPreview.tsx` · 공용 모델 `apps/admin/src/pages/marketing/_shared/messaging.ts`

## 4. 기능 명세

기능의 목록과 절차는 [SMS 발송 등록·수정](../sms/form.md) 과 같다: 폼 열기 · 세그먼트 선택 · 템플릿 불러오기 · 저장 · 이탈 가드 · 충돌. 아래는 이 화면에서만 달라지는 값이다.

### 4.1 템플릿 불러오기 — 제목까지 옮긴다

| 항목 | 내용 |
|---|---|
| 기능 목적 | 발행된 이메일 템플릿을 그대로 가져온다 |
| 실행 조건 | 발행되어 켜져 있는 이메일 템플릿이 1건 이상일 때 |
| 사용자 동작 | 템플릿 선택 |
| 시스템 처리 | **제목과 본문을 함께** 옮긴다 — SMS 는 본문과 이미지 여부를 옮긴다 |
| 데이터 처리 | 폼의 제목·본문 값이 바뀐다. 폼 상태를 우회하지 않아 변경 표시와 검증이 정상으로 따라온다 |
| Validation | 옮긴 뒤 곧바로 재검증된다 |
| API 호출 여부 | N |
| 성공 처리 | 제목과 본문이 채워지고 미리보기가 갱신된다 |
| 실패 처리 | 고른 템플릿이 이메일 종류가 아니면 아무 일도 하지 않는다 |
| 예외 처리 | ⚠ **이메일 템플릿의 본문은 블록 모델**이고 이 폼의 본문은 평문이다. 옮기는 과정에서 블록이 평문으로 바뀌며, **그 변환에서 잃는 것(레이아웃·버튼·이미지)은 화면이 알리지 않는다** |
| 화면 변경 사항 | 입력영역 · 미리보기영역 |
| 후속 동작 | 없음 |
| 로그 기록 여부 | N |

### 4.2 광고 요건 — 제목에 걸린다

| 항목 | 내용 |
|---|---|
| 기능 목적 | 광고 메일의 법 요건을 저장 전에 막는다 |
| 실행 조건 | 광고성 토글이 켜졌을 때 |
| 사용자 동작 | 광고성 토글 · 제목 작성 |
| 시스템 처리 | **제목**이 `(광고)` 로 시작하는지만 본다 — `apps/admin/src/pages/marketing/_shared/messaging.ts:355-357` |
| 데이터 처리 | 없음 |
| Validation | 저장을 막는다(§6) |
| API 호출 여부 | N |
| 성공 처리 | 요건을 갖추면 경고가 사라진다 |
| 실패 처리 | 입력 중에는 경고로, 저장 시에는 차단으로 알린다 |
| 예외 처리 | ⚠ **SMS 와 다르다** — 문자는 본문의 `(광고)` 표기 **와** 무료수신거부 문구를 함께 보고 야간 예약까지 막지만, 이메일은 **제목의 표기 하나만** 본다. 무료수신거부 문구는 검사하지 않고 대신 **수신거부 링크 토글**로 대체한다(§4.3) |
| 화면 변경 사항 | 입력영역 · 광고 제목 경고 |
| 후속 동작 | 없음 |
| 로그 기록 여부 | N |

### 4.4 저장

[SMS 발송 등록·수정](../sms/form.md) 과 **같다.** 다른 것은 성공 토스트의 엔티티 이름(`이메일 발송` — `EmailFormPage.tsx:48`)과 이탈 가드에 넘기는 문장이다: `이메일 발송에 저장하지 않은 변경 사항이 있어요. 이 화면을 벗어나면 입력한 내용이 사라져요.` — `EmailFormPage.tsx:50-51`.

## 5. 이벤트 정의

[SMS 발송 등록·수정](../sms/form.md) 의 이벤트 표와 **같다.** 다른 것은 셋이다.

| 이벤트 | 발생 조건 | 처리 내용 | 대상 컴포넌트 | 결과 화면 |
|---|---|---|---|---|
| 제목 입력 | 타이핑 | 미리보기의 수신함 줄이 갱신되고, 광고성이면 요건 경고가 다시 판정된다 | 입력영역 · 미리보기 | 폼 |
| 템플릿 선택 | 선택 변경 | **제목과 본문**을 함께 옮긴다 | 본문 카드 | 폼 |
| 수신거부 토글 | 전환 | 꺼지면 오류, 켜지면 도움말 | 본문 카드 | 폼 |

## 8. 화면 상태(State)

[SMS 발송 등록·수정](../sms/form.md) 의 상태 표와 **같다.** 문구의 엔티티 이름만 바뀐다. 이 화면만의 상태는 둘이다.
