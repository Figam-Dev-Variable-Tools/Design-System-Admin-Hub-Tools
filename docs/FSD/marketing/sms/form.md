# SMS 발송 등록·수정

> **`/marketing/sms/new` 와 `/marketing/sms/:id/edit` 는 같은 화면이다** — 같은 컴포넌트가 `:id`
> 유무로 등록과 수정을 겸한다.
>
> **발송 폼 3화면(SMS · 이메일 · 뉴스레터)은 같은 뼈대를 쓴다** — 발송 정보 카드 → 수신자 카드 →
> 본문 카드 → 발송 예약 카드 + 우측 미리보기. 세그먼트 선택기 · 발신자 검증 · 예약 검증 ·
> 템플릿 불러오기 · 치환변수 삽입이 한 벌에서 온다.
> **그 뼈대의 전부를 이 문서가 갖고**, [이메일 발송 등록·수정](../email/form.md) ·
> [뉴스레터 등록·수정](../newsletters/form.md) 은 차이만 적는다.

## 1. 화면 개요

| 항목 | 내용 |
|---|---|
| 화면명 | SMS 발송 등록·수정 |
| 화면 ID | `SCR-MARKETING-SMS-FORM` |
| 메뉴 경로 | 비즈니스 > 마케팅 관리 > SMS 발송 > 등록·수정 (사이드바에 자리가 없다) |
| 작성자 | Admin Hub 문서화 |
| 작성일 | 2026-07-22 |
| 버전 | 1.0 |

**화면 목적** — 문자 발송 캠페인의 발송명·발신번호·수신자 세그먼트·광고성 여부·본문·이미지 첨부·예약을 입력해 만들거나 고친다.

**업무 배경** — 문자에는 화면이 막아 주지 않으면 **법을 어기거나 돈을 더 내게 되는** 경계가 넷 있다: ① 사전등록되지 않은 발신번호 ② 광고 문자의 `(광고)` 표기·무료수신거부 누락 ③ 야간(21~08시) 광고 예약 ④ 90byte 를 넘겨 조용히 LMS 로 승격되는 본문. 이 화면은 넷을 전부 입력 시점에 알린다.

**화면 설명** — 좌측 2단에 발송 정보 → 수신자 → 메시지 → 발송 예약 카드가 쌓이고, 우측에 휴대폰 말풍선 미리보기가 붙는다.

**주요 사용자** — 최상위 관리자 · 마케팅 담당 운영자(등록·수정 권한 보유자).

**사용 시나리오**

1. 발송명을 적고 사전등록된 발신번호를 고른다.
2. 수신자 세그먼트를 하나 이상 고르면 **대상 수 합계**가 아래에 뜬다.
3. 템플릿을 불러와 본문을 채우고, `치환변수` 에서 변수를 꽂는다.
4. 바이트 안내가 `SMS` 인지 `LMS` 인지 알려 준다 — 90byte 를 넘으면 그 자리에서 승격 사유를 말한다.
5. 광고성을 켜면 본문 요건 경고가 뜬다. 야간 예약을 잡으면 붉은 경고가 뜬다.
6. `초안 저장` 또는 `예약 저장` 으로 저장한다 — **이 화면에서 문자가 나가지는 않는다.**

**선행 조건** — 로그인. 등록은 `create`, 수정은 `update` 권한 + 상태 조건(초안·예약만). **`SMS 발송` 모듈**이 이 계정에 있어야 한다. 사전등록(검증)된 발신번호가 있어야 저장할 수 있다.

**후행 처리** — 저장한 캠페인이 [SMS 발송 목록](index.md) 에 반영된다. **전송은 일어나지 않는다** — 저장은 예약일 뿐이다.

**관련 화면** — [SMS 발송 목록](index.md)(`SCR-MARKETING-SMS`) · 본문의 원천 [발송 템플릿 관리](../templates/index.md) · 발신 표시 이름을 정하는 `SCR-SETTINGS-SITE` · 같은 뼈대의 [이메일 발송 등록·수정](../email/form.md) · [뉴스레터 등록·수정](../newsletters/form.md). 공통 규약은 [공통 확인 다이얼로그 · 토스트](../../_common/index.md).

**관련 메뉴** — 사이드바에 자기 자리가 없다. 권한 리소스는 잎 `/marketing/sms` 가 덮는다.

**관련 기능** — 사이트 기본 설정의 '메일·SMS 전용 사이트 이름'이 **본문 앞에 붙고 그만큼 바이트를 먹는다**(§4.4). 발송 템플릿 관리의 발행된 문자 템플릿이 '템플릿 불러오기'의 목록이다.

**관련 API** — 목적 셋(단건 조회 · 등록 · 수정). **실제 전송 호출은 없다** — 발송 자리는 어댑터의 `TODO(backend)` 주석뿐이다 — `apps/admin/src/pages/marketing/sms/data-source.ts:7`.

**참고 문서** — [ID 규약](../../id-convention.md) · [인벤토리](../../inventory.md) · [품질 기준](../../../reference/quality-bar.md)

| 예약 일시 | **광고성이면 21시~익일 8시 금지** | 〃 | `광고성 메시지는 21시~익일 8시에 예약할 수 없어요(야간 광고 전송 제한).` — `sms/validation.ts:112` | 〃 |

**세 예약 검사는 순서를 갖는다** — 필수 → 과거 → 야간 순으로 보고 앞 검사가 걸리면 뒤를 보지 않는다 — `sms/validation.ts:86-115`.

**바이트 검사는 글자 수 검사와 별개다** — 2,000**자** 상한(공용 조립기)과 2,000**byte** 상한(이 화면)이 둘 다 걸린다.
