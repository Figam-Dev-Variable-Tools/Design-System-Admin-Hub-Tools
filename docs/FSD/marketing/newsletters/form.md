# 뉴스레터 등록·수정

> **`/marketing/newsletters/new` 와 `/marketing/newsletters/:id/edit` 는 같은 화면이다.**
>
> **뼈대는 [SMS 발송 등록·수정](../sms/form.md) 과 같다** — 발송 정보 → 수신자 → 본문 → 발송 예약
> 카드 + 우측 미리보기가 한 벌에서 온다.
> **이 문서는 그 한 벌을 다시 적지 않고 이 화면의 차이만 적는다.** 차이는 넷이다:
> **발송명이 없고(제목이 겸한다) · 회차 번호가 자동 부여되며 · 광고성 축이 없고 ·
> 수신거부 링크가 토글이 아니라 언제나 포함이다.**

## 1. 화면 개요

| 항목 | 내용 |
|---|---|
| 화면명 | 뉴스레터 등록·수정 |
| 화면 ID | `SCR-MARKETING-NEWSLETTERS-FORM` |
| 메뉴 경로 | 비즈니스 > 마케팅 관리 > 뉴스레터 > 등록·수정 (사이드바에 자리가 없다) |
| 작성자 | Admin Hub 문서화 |
| 작성일 | 2026-07-22 |
| 버전 | 1.0 |

**화면 목적** — 뉴스레터 발송회차의 제목·발신자·구독자 세그먼트·본문·예약을 입력해 만들거나 고친다.

**업무 배경** — 뉴스레터는 구독형이라 광고 규제 축(`(광고)` 표기·야간 제한)이 걸리지 않는 대신, **수신거부 링크는 언제나 들어간다.** 그래서 이 폼에는 그 토글 자체가 없다 — 켜고 끌 수 있는 것이 아니기 때문이다.

**화면 설명** — 배치는 SMS·이메일 발송 폼과 같다. 다른 것은 발송명 칸이 없고(제목 하나가 이름을 겸한다), 광고성 토글과 수신거부 토글이 없다는 점이다.

**주요 사용자** — 최상위 관리자 · 마케팅/콘텐츠 담당 운영자(등록·수정 권한 보유자).

**사용 시나리오**

1. 제목을 적고 도메인 인증이 끝난 발신자를 고른다.
2. 구독자 세그먼트를 하나 이상 고른다.
3. 이메일 템플릿을 불러와 제목·본문을 채우고 치환변수를 꽂는다.
4. `초안 저장` 또는 `예약 저장` 으로 저장한다 — 회차 번호는 이때 자동으로 매겨진다.

**선행 조건** — 로그인. 등록은 `create`, 수정은 `update` 권한 + 상태 조건. **`이메일·뉴스레터 발송` 모듈**이 이 계정에 있어야 한다.

**후행 처리** — 저장한 회차가 [뉴스레터 목록](index.md) 에 반영된다. **전송은 일어나지 않는다.**

**관련 화면** — [뉴스레터 목록](index.md)(`SCR-MARKETING-NEWSLETTERS`) · 본문의 원천 [발송 템플릿 관리](../templates/index.md) · 같은 뼈대의 [SMS 발송 등록·수정](../sms/form.md) · [이메일 발송 등록·수정](../email/form.md). 공통 규약은 [공통 확인 다이얼로그 · 토스트](../../_common/index.md).

**관련 메뉴** — 사이드바에 자기 자리가 없다. 권한 리소스는 잎 `/marketing/newsletters` 가 덮는다.

**관련 기능** — 발송 템플릿 관리의 이메일 템플릿이 '템플릿 불러오기'의 목록이다.

**관련 API** — 목적 셋(단건 조회 · 등록 · 수정). 실제 전송 호출은 없다 — `apps/admin/src/pages/marketing/newsletters/data-source.ts` 의 `TODO(backend)` 주석뿐이다.

**참고 문서** — [SMS 발송 등록·수정](../sms/form.md)(뼈대 정본) · [ID 규약](../../id-convention.md) · [인벤토리](../../inventory.md)

**구현 파일** — `apps/admin/src/pages/marketing/newsletters/NewsletterFormPage.tsx` · 검증 `newsletters/validation.ts` · 도메인 `newsletters/types.ts` · 미리보기 `apps/admin/src/pages/marketing/_shared/EmailPreview.tsx`

## 4. 기능 명세

기능의 목록과 절차는 [SMS 발송 등록·수정](../sms/form.md) 과 같다. 아래는 이 화면에서만 달라지는 값이다.

## 5. 이벤트 정의

[SMS 발송 등록·수정](../sms/form.md) 의 이벤트 표와 **같다.** 다른 것은 아래 둘이고, **광고성 토글·야간 예약·이미지 첨부 이벤트는 성립하지 않는다.**

| 이벤트 | 발생 조건 | 처리 내용 | 대상 컴포넌트 | 결과 화면 |
|---|---|---|---|---|
| 제목 입력 | 타이핑 | 미리보기의 수신함 줄이 갱신된다 | 입력영역 · 미리보기 | 폼 |
| 템플릿 선택 | 선택 변경 | 제목과 본문을 함께 옮긴다 | 본문 카드 | 폼 |

**이 화면 고유의 확인 다이얼로그는 없다.**
