# 새 템플릿 만들기 팝업

> [발송 템플릿 관리](index.md) 의 `새 템플릿` 이 여는 팝업이다. **편집기로 가기 전에 종류를 먼저 묻는다.**

## 1. 화면 개요

| 항목 | 내용 |
|---|---|
| 화면명 | 새 템플릿 만들기 |
| 화면 ID | `POP-MARKETING-TEMPLATES-NEW-KIND` |
| 메뉴 경로 | 비즈니스 > 마케팅 관리 > 발송 템플릿 관리 > 새 템플릿 |
| 작성자 | Admin Hub 문서화 |
| 작성일 | 2026-07-22 |
| 버전 | 1.0 |

**화면 목적** — 만들 템플릿의 **종류 하나**를 고르게 하고, 그 종류의 편집기로 곧바로 보낸다.

**업무 배경** — 네 종류는 편집기가 통째로 다르다(블록 빌더 · 단일 본문 · 카카오 두 종류). 편집기에 들어간 뒤 종류를 바꾸게 두면 **이미 쓴 내용을 버려야 한다** — 되돌릴 것이 가장 많아진 시점에 묻는 셈이다. 아직 아무것도 쓰지 않은 지금 묻는 것이 가장 싸다 — `apps/admin/src/pages/marketing/message-templates/components/NewTemplateKindDialog.tsx:3-5`.

**화면 설명** — 모달 하나에 세로로 놓인 큰 선택지 네 개. 각 선택지는 **종류 이름 + 한 줄 설명**이다. 바닥에 `취소` 만 있다.

**주요 사용자** — 발송 템플릿 등록 권한을 가진 마케팅 담당 운영자.

**사용 시나리오**

1. 목록에서 `새 템플릿` 을 누른다.
2. 네 선택지의 설명을 읽고 하나를 고른다 — 규제·요금·발송 시간이 종류마다 다르므로 여기서 판단한다.
3. 고르는 즉시 그 종류의 편집기(등록)로 들어간다.
4. 잘못 골랐으면 편집기에서 `취소` 로 나와 다시 연다.

**선행 조건** — 로그인. **등록 권한**이 있어야 `새 템플릿` 버튼 자체가 존재한다.

**후행 처리** — `/marketing/templates/new?kind={종류}` 로 이동한다. 이 시점에는 아무것도 저장되지 않는다.

**관련 화면** — [발송 템플릿 관리](index.md)(`SCR-MARKETING-TEMPLATES`) · [발송 템플릿 등록·수정](form.md)(`SCR-MARKETING-TEMPLATES-FORM`)

**관련 메뉴** — 없다(팝업).

**관련 기능** — 종류 이름의 정본은 `apps/admin/src/pages/marketing/message-templates/types.ts:71-76`. 선택지의 설명 문구는 이 팝업이 따로 갖는다.

**관련 API** — **없다.** 서버를 부르지 않는다 — 네 선택지가 코드 안의 상수다.

**참고 문서** — [발송 템플릿 관리](index.md)(모델 정본) · [ID 규약](../../id-convention.md) · [인벤토리](../../inventory.md)

**구현 파일** — `apps/admin/src/pages/marketing/message-templates/components/NewTemplateKindDialog.tsx` · 여는 쪽 `apps/admin/src/pages/marketing/message-templates/MessageTemplateListPage.tsx:234-238,263-268`

## 6. Validation

**입력이 없다 — 검증할 것이 없다.** 선택지가 넷으로 고정되어 있어 잘못된 값이 만들어질 수 없다.

다만 **주소로 직접 들어오는 경로는 다르다** — `?kind=` 를 손으로 고칠 수 있으므로, 넷 중 하나가 아니면 편집기가 **문자로 되돌린다** — `apps/admin/src/pages/marketing/message-templates/MessageTemplateEditorPage.tsx:31-33`. 그 판정은 이 팝업이 아니라 편집기가 갖는다.
