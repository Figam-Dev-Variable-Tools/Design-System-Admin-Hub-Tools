# 발송 템플릿 상세

> **템플릿 4종이 이 화면 하나를 공유한다** — 좌(발신 프로필) / 중앙(상태 이력 표) / 우(미리보기) 3단이
> 같고, **중앙 표의 줄 구성과 우측 미리보기만 종류마다 달라진다.**
> 종류가 공유하는 모델의 설명은 [발송 템플릿 관리](index.md) 가 갖는다.

## 1. 화면 개요

| 항목 | 내용 |
|---|---|
| 화면명 | 발송 템플릿 상세 |
| 화면 ID | `SCR-MARKETING-TEMPLATES-DETAIL` |
| 메뉴 경로 | 비즈니스 > 마케팅 관리 > 발송 템플릿 관리 > 상세 (사이드바에 자리가 없다) |
| 작성자 | Admin Hub 문서화 |
| 작성일 | 2026-07-22 |
| 버전 | 1.0 |

**화면 목적** — 템플릿 한 건을 읽기 전용으로 보여 주고, **사용 여부를 켜고 끄거나 · 발행하거나 · 수정으로 들어가거나 · 지운다.**

**업무 배경** — 이 도메인에서 가장 흔한 조작은 '켜고 끄기'다. 편집기로 곧장 보내면 켜기만 하려던 운영자가 매번 편집기를 열고 나오게 되므로, 목록의 행 클릭이 이 화면으로 온다 — `apps/admin/src/pages/marketing/message-templates/MessageTemplateListPage.tsx:6-7`.

**화면 설명** — 위에 `목록으로` · 종류 눈썹 · 제목 · 상태별 액션이 있고, 아래 3단이다: 좌측 발신 프로필(잠김) · 중앙 상태 이력 표 · 우측 미리보기.

**주요 사용자** — 최상위 관리자 · 마케팅 담당 운영자.

**사용 시나리오**

1. 목록에서 행을 눌러 들어와 미리보기로 문구를 확인한다.
2. 사용 여부 토글로 발송 화면에서 고를 수 있게 켜거나 끈다.
3. 초안이면 `발행` 을 눌러 켠 채로 내보낸다.
4. 알림톡이면 심사 상태·반려 사유를 확인한다.
5. 필요 없어지면 `삭제` 한다.

**선행 조건** — 로그인. 이 화면의 읽기 권한.

**후행 처리** — 사용 여부·발행 상태가 바뀌면 **발송 화면의 '템플릿 불러오기' 목록이 함께 바뀐다**(발행되어 켜진 것만 나타난다).

**관련 화면** — [발송 템플릿 관리](index.md)(`SCR-MARKETING-TEMPLATES`) · [템플릿 등록·수정](form.md)(`SCR-MARKETING-TEMPLATES-FORM`) · 이 템플릿을 쓰는 [SMS 발송 등록·수정](../sms/form.md) · [이메일 발송 등록·수정](../email/form.md) · [뉴스레터 등록·수정](../newsletters/form.md). 공통 규약은 [공통 확인 다이얼로그 · 토스트](../../_common/index.md).

**관련 메뉴** — 사이드바에 자기 자리가 없다. 권한 리소스는 잎 `/marketing/templates` 가 덮는다.

**관련 기능** — 발행 상태 전이 규칙(무엇을 켜고 끌 수 있는가)의 정본은 `apps/admin/src/pages/marketing/message-templates/status.ts` 다. 화면은 그 결과를 그릴 뿐 상태를 스스로 세지 않는다 — `apps/admin/src/pages/marketing/message-templates/MessageTemplateDetailPage.tsx:5-6`.

**관련 API** — 목적 셋: 단건 조회 · **상태만 바꾸기** · 삭제. 상태 변경이 일반 저장과 다른 호출인 이유는 §4.2 다. 백엔드가 없다 — 경계는 `apps/admin/src/pages/marketing/message-templates/data-source.ts:39-40` 의 `TODO(backend)` 다.

**참고 문서** — [발송 템플릿 관리](index.md)(모델 정본) · [ID 규약](../../id-convention.md) · [인벤토리](../../inventory.md)

### 4.2 사용 여부 토글 · 발행

| 항목 | 내용 |
|---|---|
| 기능 목적 | 발송 화면이 이 템플릿을 고를 수 있는지 정한다 |
| 실행 조건 | 발행된 템플릿(토글) · 초안(발행) |
| 사용자 동작 | 토글 전환 · `발행` 클릭 |
| 시스템 처리 | **어댑터의 일반 저장을 쓰지 않고 상태 전용 호출을 쓴다** — `MessageTemplateDetailPage.tsx:233-246`. 이 화면에는 편집 폼이 없어, 토글 하나를 켜려고 본문 전체를 되보내면 **그 사이 다른 관리자가 고친 본문을 화면이 들고 있던 옛 값으로 덮어쓴다** — `MessageTemplateDetailPage.tsx:8-11` |
| 데이터 처리 | 발행 상태만 바뀐다. `발행` 은 초안을 `사용중` 으로 만든다 — 발행하고도 꺼져 있으면 아무 데서도 고를 수 없어 발행이 아무 일도 하지 않은 것이 된다 — `message-templates/status.ts:24-32` |
| Validation | 해당 없음 |
| API 호출 여부 | Y — 상태만 바꾸는 호출 |
| 성공 처리 | 이 템플릿의 조회가 무효화되어 화면이 새 상태로 다시 그려진다 — `MessageTemplateDetailPage.tsx:243-245`. **토스트를 띄우지 않는다** |
| 실패 처리 | ⚠ **실패를 알리는 표시가 없다** — 상태 변경 호출의 실패 처리가 코드에 없다. 화면은 바뀌지 않은 채로 남는다 |
