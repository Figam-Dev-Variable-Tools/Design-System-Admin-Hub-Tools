# 블록 추가 팝업

> [발송 템플릿 등록·수정](form.md) 의 **이메일 편집기에서만** 열린다. 문자·알림톡·브랜드 메시지
> 편집기에는 이 팝업이 없다.

## 1. 화면 개요

| 항목 | 내용 |
|---|---|
| 화면명 | 블록 추가 |
| 화면 ID | `POP-MARKETING-TEMPLATES-FORM-BLOCK-PICKER` |
| 메뉴 경로 | 비즈니스 > 마케팅 관리 > 발송 템플릿 관리 > 등록·수정 > 블록 추가 |
| 작성자 | Admin Hub 문서화 |
| 작성일 | 2026-07-22 |
| 버전 | 1.0 |

**화면 목적** — 이메일 본문에 넣을 **블록 한 종류**를 고르거나, **여러 블록으로 이뤄진 구성 하나**를 통째로 넣는다.

**업무 배경** — 블록이 14종이고 구성 묶음이 4종이다. 캔버스 옆에 전부 늘어놓으면 편집 화면이 목록에 잠식된다. 넣을 때만 열리는 격자로 뺀다.

**화면 설명** — 모달 하나에 두 구획이 있다: 위가 `블록 종류`(아이콘 격자), 아래가 `구성 — 여러 블록을 한 번에`(이름 + 설명 목록). 바닥에 `취소` 만 있다.

**주요 사용자** — 이메일 템플릿을 만드는 마케팅 담당 운영자.

**사용 시나리오**

1. 캔버스의 `+`(끝에 추가) 또는 블록 사이의 `+`(그 뒤에 추가)를 누른다.
2. `블록 종류` 에서 하나를 고르면 그 자리에 빈 블록이 들어가고 팝업이 닫힌다.
3. 여러 블록이 필요하면 `구성` 에서 골라 한 번에 넣는다.
4. 컬럼(칸) 안의 `+` 로 열었으면 목록이 줄어 있다.

**선행 조건** — 이메일 편집기에 있어야 한다. 편집 모드여야 한다(미리보기 모드에는 `+` 가 없다).

**후행 처리** — 고른 블록이 캔버스에 들어가고 **되돌리기 이력에 한 단위로 쌓인다.**

**관련 화면** — [발송 템플릿 등록·수정](form.md)(`SCR-MARKETING-TEMPLATES-FORM`)

**관련 메뉴** — 없다(팝업).

**관련 기능** — 블록 이름·순서의 정본은 `apps/admin/src/pages/marketing/message-templates/email/blocks.ts`, 구성 묶음은 `apps/admin/src/pages/marketing/message-templates/email/groups.ts` 다.

**관련 API** — **없다.** 서버를 부르지 않는다 — 목록이 전부 코드 안의 상수다.

**참고 문서** — [발송 템플릿 관리](index.md) · [ID 규약](../../id-convention.md) · [인벤토리](../../inventory.md)

**구현 파일** — `apps/admin/src/pages/marketing/message-templates/email/BlockPicker.tsx` · 목록 `email/blocks.ts` · 구성 `email/groups.ts` · 여는 쪽 `email/EmailBuilder.tsx`

## 6. Validation

**입력이 없다 — 검증할 것이 없다.** 이 팝업은 값을 받지 않고 고르기만 한다. 넣을 수 없는 조합은 검증이 아니라 **목록에서 빼는 방식**으로 막는다(§4.2) — 고르고 나서 거절하는 화면을 만들지 않는다.
