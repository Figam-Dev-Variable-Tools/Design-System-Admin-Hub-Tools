# 이벤트

> **이벤트와 프로모션은 같은 뼈대를 쓴다** — 노출기간 · 상태(예정/진행/종료) · 대상 · 참조 연동 ·
> 설명이 한 모델(`apps/admin/src/pages/marketing/_shared/campaign.ts`)에서 온다.
> **그 뼈대의 전부를 이 문서가 갖고**, [프로모션](../promotions/index.md) 은 자기 차이(할인 축 ·
> 쿠폰 연동)만 적는다.

## 1. 화면 개요

| 항목 | 내용 |
|---|---|
| 화면명 | 이벤트 |
| 화면 ID | `SCR-MARKETING-EVENTS` |
| 메뉴 경로 | 비즈니스 > 마케팅 관리 > 이벤트 |
| 작성자 | Admin Hub 문서화 |
| 작성일 | 2026-07-22 |
| 버전 | 1.0 |

**화면 목적** — 진행 중이거나 예정된 마케팅 이벤트를 한 목록에서 보고, 기간·대상·혜택·연동 배너를 확인하며, 등록·수정·삭제한다.

**업무 배경** — 이벤트는 기간이 지나면 스스로 끝나야 하는데, 상태를 사람이 손으로 바꾼다. 관리자가 지정한 상태와 실제 기간이 어긋나면 이미 끝난 이벤트가 '진행'으로 남는다. 이 화면은 그 어긋남을 목록에서 **눈에 보이게** 만든다(§4.3).

**화면 설명** — 툴바(검색 · 상태 필터 · 등록 버튼) 아래 표가 있고, 열은 이벤트명 · 기간 · 대상 · 혜택 · 연동 배너 · 상태다. 행을 누르면 수정 화면으로 간다.

**주요 사용자** — 최상위 관리자 · 마케팅 담당 운영자.

**사용 시나리오**

1. 상태 필터를 `진행` 으로 좁혀 지금 돌고 있는 이벤트만 본다.
2. 상태 열에 `기간상 종료` 경고 배지가 붙은 행을 찾아 상태를 바로잡는다.
3. `연동 배너` 열의 링크를 눌러 그 배너의 편집 화면으로 건너간다.
4. `이벤트 등록` 으로 새 이벤트를 만든다.
5. 좁혀 놓은 조건 그대로 주소를 복사해 동료에게 보낸다.

**선행 조건** — 로그인. 이 화면의 읽기 권한. 배너를 연동하려면 콘텐츠 관리 배너에 등록된 배너가 있어야 한다.

**후행 처리** — 등록·수정한 이벤트가 이 목록에 반영된다. **연동한 배너 쪽은 바뀌지 않는다** — 이벤트가 배너를 참조할 뿐이다.

**관련 화면** — [이벤트 등록·수정](form.md)(`SCR-MARKETING-EVENTS-FORM`) · 같은 뼈대의 [프로모션](../promotions/index.md)(`SCR-MARKETING-PROMOTIONS`) · 연동 대상 `SCR-CONTENT-BANNERS-FORM`. 공통 규약은 [공통 확인 다이얼로그 · 토스트](../../_common/index.md).

**관련 메뉴** — 사이드바 `비즈니스 > 마케팅 관리` 의 첫 항목 — `apps/admin/src/shared/layout/nav-config.ts:221`. **플랜으로 잠기지 않는다** — 마케팅 모듈 매핑에는 발송 채널 둘(`/marketing/newsletters`·`/marketing/email`·`/marketing/sms`)만 들어 있고 이벤트는 없다 — `apps/admin/src/shared/entitlements/module-resources.ts:75-80`.

**관련 기능** — 콘텐츠 관리 배너(`SCR-CONTENT-BANNERS`)가 연동 대상 목록의 정본이다. 이 화면은 그 화면을 직접 참조하지 않고 공통 층의 카탈로그 조회기가 주는 것만 고른다 — `apps/admin/src/pages/marketing/events/types.ts:6-8`.

**관련 API** — 목적 다섯: 목록 조회 · 단건 조회 · 등록 · 수정 · 삭제. 공통 CRUD 어댑터가 배선한다. **백엔드가 없다** — 경계는 `apps/admin/src/pages/marketing/events/data-source.ts` 의 `TODO(backend)` 다.

**참고 문서** — [ID 규약](../../id-convention.md) · [인벤토리](../../inventory.md) · [품질 기준](../../../reference/quality-bar.md)
