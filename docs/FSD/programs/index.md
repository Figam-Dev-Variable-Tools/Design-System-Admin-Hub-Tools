# 프로그램(펀딩 현황 목록)

## 1. 화면 개요

| 항목 | 내용 |
|---|---|
| 화면명 | 프로그램 |
| 화면 ID | `SCR-PROGRAMS` |
| 메뉴 경로 | 프로그램 관리 > 프로그램 |
| 작성자 | Admin Hub 문서화 |
| 작성일 | 2026-07-22 |
| 버전 | 1.0 |

**화면 목적** — 후원형 펀딩 프로그램의 **현황판**이다. '무엇이 지금 얼마나 모였고, 언제 끝나는가'에 한 표로 답한다.

**업무 배경** — 프로그램은 상품과 같은 결이지만 축이 셋 더 있다: **목표 금액 · 기간 · 리워드**. 창작자가 목표와 기간을 걸고 열면 후원자가 리워드를 골라 후원하고, **기간이 끝나는 순간 목표 달성 여부로 성공/실패가 갈린다** — `apps/admin/src/pages/programs/_shared/store.ts:3-4`. 그래서 이 목록의 중심 열은 이름이 아니라 모금액/달성률과 남은 일수다.

**화면 설명** — 좌측 필터 레일(진행 상태 + 카테고리 + 안내 2줄)과 우측 9열 표다. 행을 누르면 **현황 상세**로 가고(수정 폼이 아니다), 수정은 연필 액션의 몫이다.

**주요 사용자** — 최상위 관리자 · 콘텐츠/펀딩 담당 운영자.

**사용 시나리오**

1. '진행 중'만 걸러 남은 일수가 적은 건부터 확인한다.
2. 좌측에서 대분류 '테크·가전'을 고른다 → **그 아래 중분류(음향기기·모바일 액세서리)의 프로그램까지 함께 보인다.**
3. 한 건을 열어 리워드 구성과 달성률을 확인하고, 필요하면 상세에서 바로 수정으로 넘어간다.

**선행 조건** — 로그인 · 이 화면의 읽기 권한. **플랜(엔타이틀먼트) 매핑이 없어 모듈 잠금이 걸리지 않는다**(§9).

**후행 처리** — 이 목록에서 일어나는 변경은 삭제뿐이다. 등록·수정은 폼의 일이다.

**관련 화면** — [프로그램 상세](detail.md) `SCR-PROGRAMS-DETAIL` · [프로그램 등록·수정](form.md) `SCR-PROGRAMS-FORM` · [카테고리](categories/index.md) `SCR-PROGRAMS-CATEGORIES` · [프로그램 문의](inquiries/index.md) `SCR-PROGRAMS-INQUIRIES` · 공통 규약은 [공통 확인 다이얼로그 · 토스트](../_common/index.md).

**관련 메뉴** — 사이드바 `프로그램 관리 > 프로그램`. **등록·수정은 메뉴에 없다** — 메뉴에 '등록'을 따로 걸면 목록을 거치지 않는 우회로가 생겨 수정 경로와 갈라진다 — `apps/admin/src/shared/layout/nav-config.ts:198-203`.

**관련 기능** — 카테고리 필터는 카테고리 관리 화면(`SCR-PROGRAMS-CATEGORIES`)이 만드는 2단계 분류를 읽는다. 결제(PG) 설정은 이 목록에 영향을 주지 않는다 — **금액 표시가 잠기는 것은 상세와 폼의 리워드 금액이고, 이 목록의 목표금액·모금액은 그대로 보인다**.

**관련 API** — 프로그램 목록 조회 1건 · 카테고리 목록 조회 1건 · 삭제 N건. **백엔드가 없다** — `TODO(backend): GET/POST /api/programs · PUT/DELETE /api/programs/:id` — `apps/admin/src/pages/programs/data-source.ts:27`.

**참고 문서** — [ID 규약](../id-convention.md) · [인벤토리](../inventory.md) · [품질 기준](../../reference/quality-bar.md)

**구현 파일** — `apps/admin/src/pages/programs/ProgramListPage.tsx` (표시 규칙 `types.ts` · 저장소 `_shared/store.ts` · 어댑터 `data-source.ts`)
