# 프로그램 상세(펀딩 진행 현황)

## 1. 화면 개요

| 항목 | 내용 |
|---|---|
| 화면명 | 프로그램 상세 |
| 화면 ID | `SCR-PROGRAMS-DETAIL` |
| 메뉴 경로 | 프로그램 관리 > 프로그램 > (행 선택) |
| 작성자 | Admin Hub 문서화 |
| 작성일 | 2026-07-22 |
| 버전 | 1.0 |

**화면 목적** — 프로그램 한 건의 **펀딩 현황**(달성률·모금액·후원자수·남은 일수)과 **리워드 구성**을 읽는다. 고객이 볼 후원 버튼이 지금 무엇인지도 여기서 확인한다.

**업무 배경** — 목록에서 한 건을 열어 '지금 얼마나 모였고 언제 끝나는가'를 확인하는 화면이다. 값을 고치는 것은 폼의 일이라 여기에는 입력 필드가 없다 — `apps/admin/src/pages/programs/ProgramDetailPage.tsx:3-4`.

**화면 설명** — 제목 줄(제목 + 상태 배지 + `수정`), 그 아래 카드 3장: **펀딩 현황**(진행바 + 정의목록 + 후원 CTA), **프로그램 정보**(창작자·카테고리·소개·스토리), **리워드**(잠금 안내 + 리워드 표).

**주요 사용자** — 최상위 관리자 · 콘텐츠/펀딩 담당 운영자.

**사용 시나리오**

1. 목록에서 한 건을 연다 → 진행바와 달성률로 상황을 파악한다.
2. 결제 연동이 없는 상태에서 리워드 표를 본다 → **금액이 전부 `가격문의` 로 대체되어 있고 왜 그런지가 표 위에 적혀 있다.**
3. 카드의 안내에서 `결제 설정 열기` 로 넘어가 연동을 마친다 → 되돌아오면 표가 원래 숫자를 되찾는다.

**선행 조건** — 로그인 · 읽기 권한.

**후행 처리** — 이 화면은 아무것도 바꾸지 않는다. 수정은 `수정` 버튼으로 폼에 가서 한다.

**관련 화면** — [프로그램 목록](index.md) · [프로그램 등록·수정](form.md) · [프로그램 문의](inquiries/index.md)(결제가 없는 동안 후원 대신 들어오는 것) · `SCR-SETTINGS-PAYMENT`(금액 표시를 정하는 곳) · 공통 규약은 [공통 확인 다이얼로그 · 토스트](../_common/index.md).

**관련 메뉴** — 사이드바에 자기 자리가 없다.

**관련 기능** — **결제(PG) 설정이 이 화면의 두 자리를 바꾼다**: ① 후원 CTA 가 `후원하기` 또는 `문의하기` 가 된다 ② 리워드 표의 금액이 전부 `가격문의` 로 대체된다. 판정은 공통 층의 규칙 함수 둘(`checkoutCta` · `resolvePriceDisplay`)이 한다 — 상품 화면도 **같은 함수**를 부른다.

**관련 API** — 프로그램 상세 조회 1건. **백엔드가 없다** — `TODO(backend): GET/POST /api/programs · PUT/DELETE /api/programs/:id` — `apps/admin/src/pages/programs/data-source.ts:27`.

**참고 문서** — [ID 규약](../id-convention.md) · [인벤토리](../inventory.md) · [품질 기준](../../reference/quality-bar.md)

**구현 파일** — `apps/admin/src/pages/programs/ProgramDetailPage.tsx` (파생값 `_shared/store.ts` · 표시 규칙 `types.ts` · 잠금 규칙 `apps/admin/src/shared/commerce/price-display.ts` · CTA 규칙 `apps/admin/src/shared/commerce/payment-settings.ts` · 잠금 안내 `apps/admin/src/shared/commerce/PgLockNotice.tsx`)

### 1.1 ⚠ 가격 표시가 결제 연동에서 파생된다

**운영 요구가 축 하나를 없앴다.** 예전에는 축이 둘이었다 — 사이트 전역의 '결제창을 열 수 있는가'와, **상품마다 사람이 고르는** '이 상품의 금액을 노출하는가'. 두 번째 축은 상품 폼의 라디오('금액 노출' / '가격문의로 대체')로 저장됐다.

지금은 **그 라디오가 없다.** 운영 지시: *"이렇게 선택하는 게 아니라, 결제 연동이 안 돼 있으면 전부 가격 문의로 대체"* — 금액이 이미 설정돼 있어도 그렇고, **프로그램도 같다** — `apps/admin/src/shared/commerce/price-display.ts:3-19`.

그래서 이 화면의 리워드 금액은 **고르는 값이 아니라 연동 상태의 결과**다. 판정 근거는 술어 하나(`pgSellable`)이고 그것이 세 가지를 접고 있다:

| 갈래 | 무엇 | 이 화면에 무엇이 되는가 |
|---|---|---|
| ⓐ | PG 를 쓰지 않도록 설정됨(`usePg === false`) | 리워드 금액 → `가격문의` · CTA → `문의하기` |
| ⓑ | 저장된 자격증명이 PG 에서 **거절됨** | 〃 (**백엔드만 아는 사실이다** — 지금은 배선이 없어 언제나 '확인한 적 없음'이다) |
| ⓒ | 필수 자격증명이 아직 안 채워짐 | 〃 |

**저장값은 지워지지 않는다** — 잠금은 표시와 입력만 막고, 연동을 마치면 저장된 후원 금액이 그대로 살아난다 — `apps/admin/src/shared/commerce/price-display.ts:54-57`.

## 4. 기능 명세

### 4.1 프로그램 상세 조회
