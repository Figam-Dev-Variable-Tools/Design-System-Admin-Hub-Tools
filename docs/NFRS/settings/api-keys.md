| 항목 | 내용 |
|---|---|
| 자원명 | API Key 설정 (연동 카탈로그) |
| 화면 ID | `SCR-SETTINGS-API-KEYS` · `SCR-SETTINGS-API-KEYS-DETAIL` |
| 메뉴 경로 | 시스템 설정 > API Key 설정 |
| 작성자 | Admin Hub 문서화 |
| 작성일 | 2026-07-22 |
| 버전 | 1.0 |
| 관련 기능 명세서 | [연동 목록](../../FSD/settings/api-keys/index.md) · [연동 상세](../../FSD/settings/api-keys/detail.md) · [저장 충돌(설정)](../../FSD/_common/pop-settings-conflict.md) |

**이 자원의 성격** — **문서 한 벌을 두 화면이 나눠 본다.** 목록은 읽기만 하고, 상세는 그 연동 자리 하나만 쓴다. 그래서 이 자원의 비기능 특성은 넷이다: ① **평문 자격증명을 갖지 않는 규율**(비밀 여부의 정본이 칸 이름 하나다) ② **확인하지 못한 것을 확인한 척하지 않는 규율**(연결 검증 없음 · `null` 을 `없음` 으로 뭉개지 않음) ③ 연동마다 **요구가 다른 것**을 카탈로그 하나가 흡수해 화면에 분기를 두지 않음 ④ 개정 토큰으로 동시 편집을 막되 각 화면이 자기 범위만 저장. 아래 각 절은 [공통 기준](../_common.md) 대비 **이 자원만의 차이**를 적는다.

**⚠ 이 자원은 AI 전용이 아니다** — 분류 축이 `model` · `foundation` · `shipping` 셋이고 배송 연동(CJ대한통운)이 같은 진열대에 서 있다. 화면 문구는 그것을 반영해 정리됐으나 **파일·심볼 이름은 아직 `Ai` 접두**다 — 그 어긋남은 §7 에 적는다.

## 3. 보안

**이 절이 이 자원의 중심이다.**

| 항목 | 요구 | 이 자원의 값 |
|---|---|---|
| 인증 | 공통 준수 | 셸 안쪽이라 세션이 없으면 렌더되지 않는다 |
| 권한 | 공통 준수 | 리소스 키는 잎 `/settings/api-keys` 하나이고 **상세도 그것이 덮는다** — `apps/admin/src/App.tsx:205-208`. 목록은 조회 권한만으로 완전히 동작한다(저장 컨트롤이 없다). 상세는 수정 권한이 없으면 저장 버튼을 **렌더하지 않는다** |
| **평문 자격증명을 저장하지 않는다** | — | 저장 요청에만 평문이 실리고 **문서에는 실리지 않는다** — 비밀 칸은 이름만 남고 평문은 버려진다. 그래서 이 모듈이 든 문서를 통째로 출력해도 키는 없다 — `apps/admin/src/pages/settings/api-keys/data-source.ts:14-21` |
| **비밀 여부의 정본이 하나다** | — | 어느 칸이 비밀인지는 **칸 이름 하나가 한 번** 답한다. 카탈로그 항목에는 그것을 적을 자리가 **타입에 없다** — 항목마다 적을 수 있으면 같은 칸이 한 연동에서는 비밀이고 다른 연동에서는 공개가 될 수 있고, 그 갈라짐은 **그 연동에서만 키가 평문으로 저장되는** 형태로 나타난다. 라벨이 갈리면 눈에 보이지만 이것은 조용하다 — `apps/admin/src/pages/settings/api-keys/ai-connections.ts:109-130` |
| 새 칸이 조용히 공개가 되지 않는다 | — | 비밀 여부 표가 `Record` 라 **칸을 더하면 반드시 답해야 하고**, 빠뜨리면 컴파일이 막힌다. 기본값을 두지 않는 이유: 두면 새 칸이 조용히 공개가 되고 그 방향의 실수가 더 비싸다 — `apps/admin/src/pages/settings/api-keys/ai-connections.ts:128-129` |
| 마스킹의 정밀도 | — | 저장된 비밀은 **고정 길이 글리프 하나**로만 표시한다 — 자릿수도 마지막 4자도 남기지 않는다. 그 글리프는 원본을 가린 표시가 아니라 **우리가 가진 정보의 전부**다 — `apps/admin/src/pages/settings/_shared/secret.ts:3-14` |
| 입력 요소의 부재 | — | 저장된 비밀이 있고 변경 중이 아니면 **입력 요소를 아예 렌더하지 않는다.** 평문이 들어갈 자리를 두지 않는 것이 방어다 — `apps/admin/src/pages/settings/api-keys/components/AiCredentialFields.tsx:14-22` |
| 새로 넣을 값의 계약 | — | 빈 문자열 = 기존 값 유지 · 값이 있음 = 교체. 화면에 평문이 살지 않으므로 **'기존 값을 다시 보내는' 경로 자체가 없다** — `apps/admin/src/pages/settings/api-keys/validation.ts:3-11` |
| 비밀과 공개를 한 자루에 담지 않는다 | — | 되읽을 수 있는 값(주소·배포명·리전·고객코드·조직/프로젝트 ID)과 저장 여부만 아는 값(키)을 **다른 자리에** 둔다 — 비밀이 들어갈 자리가 공개 값 쪽에 **구조적으로 없다** — `apps/admin/src/pages/settings/api-keys/ai-connections.ts:196-220` |
| 비밀이 아닌 값을 비밀처럼 다루지 않는다 | — | 조직·프로젝트 ID 는 그것만으로 아무 호출도 인증되지 않고, 고객코드는 '어느 계약으로 부르는가'를 가리키는 값이다. 되읽지 못하면 폼이 쓸모없어진다 — 배포명을 매번 다시 입력해야 하고 한 글자 틀리면 호출이 404 다 |
| 빈 값을 저장하지 않는다 | — | 선택 칸을 빈 문자열로 저장해 두면 호출부가 그것을 **보낼 값으로 읽는다** — 빈 헤더를 실제로 보내면 상대가 거절한다. '비어 있음'과 '없음'을 **없음으로 수렴시킨다** — `apps/admin/src/pages/settings/api-keys/data-source.ts:147-153` |

## 8. 제약사항

| 항목 | 내용 |
|---|---|
| 외부 시스템 연계 | 7종의 콘솔·포털은 전부 **이 리포 밖**이다. 키 발급·계약·승인은 그쪽에서 하고, 이 화면은 발급 위치와 함정을 안내할 뿐이다. **자격증명을 발급하지 않는다** — 진열대는 '무엇을 붙일 수 있는가'를 말하는 곳이고 키를 만들고 폐기하는 일은 다른 관심사다 — `apps/admin/src/pages/settings/api-keys/ApiKeysPage.tsx:12-13` |
| API 제약 | **백엔드가 없다.** 조회·저장은 픽스처이고, 검증과 모델 목록 프록시는 **계약만 적혀 있고 이 화면이 부르지 않는다** — `apps/admin/src/pages/settings/api-keys/data-source.ts:53-93` |
| **연결 검증이 없다** | ⚠ 자격증명이 실제로 통하는지 확인할 수단이 화면에 없고, 마지막 검증 시각은 언제나 비어 있다. 브라우저에서 부르지 않기로 한 결정의 결과이므로 **서버가 붙기 전에는 해소되지 않는다** |
| **부를 수 없는 API 를 부를 수 있게 보이지 않게 한다** | CJ대한통운의 API 묶음 7종은 **목록으로만** 그린다 — 버튼도, API 별 상태 배지도, 마지막 호출 시각도 없다. 이 앱에는 그 API 를 부르는 코드가 한 줄도 없다 — `apps/admin/src/pages/settings/api-keys/components/ApiPackageCard.tsx:9-17` |
| **확인하지 못한 것을 칸으로 만들지 않았다** | ⚠ CJ대한통운의 인증키를 어느 헤더로 보내는지, 테스트 환경이 따로 있는지, 우리 쪽 수신 주소를 등록해야 하는지는 **[미확인]** 이다(포털이 해시 라우트로 도는 단일 페이지 앱이라 문서 본문이 서버에서 오지 않는다). 추측한 헤더를 적으면 운영자는 그것이 확인된 사실이라고 믿는다 — 그래서 적지 않았고, 화면이 그 사실을 말한다 — `apps/admin/src/pages/settings/api-keys/integrations.ts:770-778` |
| 사용량·요금·한도가 없다 | 값을 들이지 않고 **갈 곳만** 가리킨다(§3). 한도 안내 주소는 4종만 갖고, 확인하지 못한 3종은 링크를 지어내지 않고 그 사실을 문장으로 남긴다 |
| 길이 상한 | 모든 자격증명 칸 **500자**. 이것은 형식 판정이 아니라 **붙여넣기 사고로 문서 한 편이 들어오는 것**을 막는 값이다 — `apps/admin/src/pages/settings/api-keys/validation.ts:46-52` |
| 형식 제약 | 없다 — 길이·문자셋·접두어를 검사하지 않는다. 어느 제공자의 키 형식도 문서로 보장받지 못했다. 의심스러운 값은 경고로만 되묻는다 |
| 지원하지 않는 인증 경로 | Amazon Bedrock 은 **베어러 토큰 경로만** 받는다(서명 키 한 쌍 경로는 자리는 있으나 이 화면이 받지 않는다) — `apps/admin/src/pages/settings/api-keys/integrations.ts:732-733`. Google Gemini 는 Vertex AI 와 별개 연동이라 이 항목으로 붙지 않는다 |
| 카탈로그 밖 레코드 | 프론트가 지우지 않는다. 폐기할지 남길지는 운영 정책이고 서버가 붙은 뒤 **사람이 고른다** |
| 파일 용량 제한 | 해당 없음 — 이 자원에 파일 입력이 없다 |
| 업로드 가능 확장자 | 해당 없음 — 같은 이유 |
| 다국어 | 공통 준수 — 측정하지 않음. 자격증명 라벨 일부는 **상대 콘솔의 표기를 그대로** 쓰므로 한국어로 번역하지 않는다 |
