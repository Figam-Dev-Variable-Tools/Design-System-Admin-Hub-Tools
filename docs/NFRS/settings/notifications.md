# 알림 설정 — 비기능 명세서

| 항목 | 내용 |
|---|---|
| 자원명 | 알림 설정 |
| 화면 ID | `SCR-SETTINGS-NOTIFICATIONS` |
| 메뉴 경로 | 시스템 설정 > 알림 설정 |
| 작성자 | Admin Hub 문서화 |
| 작성일 | 2026-07-22 |
| 버전 | 1.0 |
| 관련 기능 명세서 | [알림 설정](../../FSD/settings/notifications/index.md) |

**이 자원의 성격** — 값이 **켬/끔 여덟 개**뿐인 가장 단순한 설정 화면인데, 그 단순함 때문에 다른 비기능 특성이 생겼다: ① 문서를 **알림 층이 소유**하고 이 화면은 편집 표면 하나일 뿐이다 ② 개정 토큰이 없어 **동시 편집을 막지 않는다** ③ 대신 **읽기 실패가 쓰기 사고가 되는 경로를 막는다**. 아래 각 절은 [공통 기준](../_common.md) 대비 **이 자원만의 차이**를 적는다.

| 무엇을 알림으로 만드는가 | — | 기준이 코드에 적혀 있다: **운영자가 모르면 손해가 나는 것만.** 자기가 방금 한 일을 다시 알려 주는 것은 소음이고, 소음이 쌓이면 정말 중요한 알림도 함께 무시된다 — `catalog.ts:37-43` |
| 차단 술어의 정본 하나 | — | 저장을 막는 판정을 한 함수가 갖고 **네 곳이 같은 함수를 읽는다**(토글 잠금 · 버튼 잠금 · 배너 문구 · 클릭 시 재확인) — `preferences.ts:70-72` |
| 없는 계약을 흉내 내지 않는다 | — | 부분 저장을 하려면 '이 키만 바꿔라'는 계약이 필요한데 저장소는 문서를 통째로 치환한다. 없는 계약을 화면이 흉내 내면 **서버가 붙는 날 그 흉내와 실제 의미가 갈린다.** 그래서 목적만 지키고 수단은 가장 정직한 것(차단)을 고른다 — `preferences.ts:62-66` |
| 읽음 여부의 자리 | — | 지금은 모듈 메모리다. **서버가 가져야 하는 사실**이라(기기가 바뀌어도 따라와야 한다) 브라우저 저장소로 흉내 내지 않는다 — 그러면 '이 브라우저에서만 읽음'이라는, 서버 도입 시 되레 지워야 할 상태가 생긴다 — `store.ts:15-17` |
| 알림 층이 다른 저장소를 훑지 않는다 | — | 픽스처의 대상 id 는 실재하는 값을 **문자열로 적어 둔 것**이지 가져온 것이 아니다. 클라이언트가 다른 모듈의 저장소를 훑어 알림을 지어내면 그것은 알림이 아니라 '목록을 대신 새로고침해 주는 코드'이고, **그 착시가 있는 한 아무도 실시간 수신을 붙이지 않는다** — `store.ts:48-56` |
| 화면 확장성 | — | 종류가 늘어도 줄이 하나 늘 뿐이다. ⚠ 여덟을 크게 넘기면 그룹핑·검색이 필요해지는데 지금은 그 축이 없다 |
| 공통 컴포넌트 사용 여부 | 공통 준수 | 손수 조립한 UI 가 없다. 다만 **공용 설정 폼 껍데기를 쓰지 않는다** — 그 껍데기가 전제하는 개정 토큰·저장 확인·충돌 처리가 이 화면에 없기 때문이다 |

## 8. 제약사항

| 항목 | 내용 |
|---|---|
| 외부 시스템 연계 | 없다. 이 화면의 값은 이 앱 안에서만 쓰인다 |
| API 제약 | **백엔드가 없다.** 조회·저장이 픽스처다. 계약은 `preferences.ts:74` 의 `TODO(backend)` 가 갖는다 — **사람에 매인 설정이라 서버가 갖는다** |
| **실시간 수신 경로가 없다** | ⚠ 알림 목록은 서버 이벤트로 오는 것이 계약인데 지금은 픽스처를 **한 번** 읽는다. 폴링으로 흉내 내지 않는 것은 의도다 — `store.ts:10-14` |
| **동시 편집 보호가 없다** | ⚠ 같은 섹션의 다른 네 화면은 개정 토큰으로 충돌을 막는데 이 화면만 마지막 저장이 이긴다 |

| **저장 확인이 없다** | 되돌리기 쉬운 값이라 의도된 것이다. 다만 이 때문에 저장 실패를 얹을 배너 자리가 없어 토스트로 알린다 |
| 종류의 수 | **고정 여덟.** 운영자가 만들거나 지울 수 없다 — 종류는 코드가 아는 사실이다 |
| 읽음 여부의 수명 | 새로고침하면 초기화된다(모듈 메모리). 서버가 가져야 하는 사실이라 브라우저 저장소로 흉내 내지 않는다 |

| 업로드 가능 확장자 | 해당 없음 — 같은 이유 |
| 다국어 | 공통 준수 — 측정하지 않음 |
