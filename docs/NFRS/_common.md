**이 문서의 자리** — 자원별 비기능 명세서는 각 절에서 `공통 준수` 한 줄과 **그 자원만의 차이**를 적는다([README](README.md) 규칙 N-1). `공통 준수` 가 가리키는 것이 이 문서다. 판정 기준의 정본은 [품질 기준](../reference/quality-bar.md)이며, 여기서 기준을 새로 정의하지 않는다.

**재지 않은 값은 적지 않는다**([README](README.md) 규칙 N-3). 아래 표에서 `측정하지 않음` 은 통과가 아니라 **아직 모른다**는 뜻이다.

## 3. 보안

| 항목 | 요구 | 이 자원의 값 |
|---|---|---|
| 인증 | 세션 없이 셸 안 화면을 렌더하지 않는다 | 인증 가드가 셸 **바깥**에 있어 세션이 없으면 사이드바도 그리지 않고 로그인 화면으로 보낸다 — `apps/admin/src/App.tsx:556-563`. 렌더 중에 판정하므로 보호된 화면이 한 프레임도 그려지지 않는다 — `apps/admin/src/shared/auth/RequireAuth.tsx:56-60` |

## 8. 제약사항

| 항목 | 내용 |
|---|---|
| 외부 시스템 연계 | 고객이 보는 홈페이지는 **이 리포 밖**에 있다. 어드민의 값이 그 화면에서 무엇이 되는지는 [고객 화면 대응](../reference/user-view.md)과 [사이트 연동 계약](../reference/site-connect-contract.md)이 갖는다 — 후자는 대응 코드가 리포에 하나도 없어 그 문서가 유일한 원본이다. 구독·결제·계약은 사내 홈페이지 소관이다 |
| API 제약 | **백엔드가 없다.** 실제 네트워크 전송은 0건이며, HTTP 클라이언트에 픽스처 전송기가 꽂혀 있다 — 그 한 줄을 지우는 순간 실제 전송이 시작된다 — `apps/admin/src/shared/api/client.ts:225-227`. 서버가 맡을 일은 각 기능 명세의 API 절이 계약으로 적는다. 스키마 원본은 `openapi/openapi.yaml` 이다 |
| 레이트 리밋 | **측정하지 않음** — 서버가 없어 적용된 값이 없다 |
| 페이지 크기 상한 | 로그 섹션 기준 100행(§1). 다른 섹션은 각 자원 문서 |
| 파일 업로드 | **업로드 엔드포인트가 없다.** 이미지 필드는 파일을 서버로 보내지 않고 브라우저 안 미리보기만 만든다 — 그래서 폼을 떠나면 썸네일이 깨진다. 검증을 조여도 해결되지 않으며(조이면 저장 자체가 불가능해진다) 고칠 곳은 업로드 이음매다 — `apps/admin/src/shared/crud/validation.ts:41-62` |
