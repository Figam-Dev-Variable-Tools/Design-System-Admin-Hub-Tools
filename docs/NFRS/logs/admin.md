# 관리자 로그 — 비기능 명세서

| 항목 | 내용 |
|---|---|
| 자원명 | 관리자 로그 |
| 화면 ID | `SCR-LOGS-ADMIN` |
| 메뉴 경로 | 분석 · 운영 > 로그 관리 > 관리자 로그 |
| 작성자 | Admin Hub 문서화 |
| 작성일 | 2026-07-22 |
| 버전 | 1.0 |
| 관련 기능 명세서 | [관리자 로그](../../FSD/logs/admin/index.md) · 공유 팝업 [로그 원문 보기](../../FSD/_common/pop-log-payload.md) |

**이 자원의 성격** — **읽기 전용 감사(audit) 로그**다. 저장·수정·삭제가 하나도 없어 그 축의 요구가 성립하지 않는 대신, ① 불변성 ② 개인정보·비밀 노출 ③ 대용량 조회와 반출 ④ 조회 조건의 재현성 넷이 이 자원의 비기능 요구 전부다.

**로그 4화면이 같은 껍데기를 공유한다** — 성능·사용성·안정성·호환성·유지보수성의 값이 거의 전부 같다. **이 문서가 그 한 벌을 갖고**, [회원 활동 로그](member-activity.md) · [API 로그](api.md) · [오류 로그](errors.md) 는 자기 자원의 차이(보존기간 · 컬럼 · 데이터 성격 · 부피)만 적는다.

아래 각 절은 [공통 기준](../_common.md) 대비 **이 자원만의 차이**를 적는다.

## 3. 보안

| 항목 | 요구 | 이 자원의 값 |
|---|---|---|
| 인증 | 공통 준수 | 셸 안쪽이라 세션이 없으면 렌더되지 않는다 |
| 권한 | 공통 준수 | 리소스 키는 사이드바 잎 `/logs/admin` 에서 파생된다. **의미 있는 액션은 둘뿐**이다: `read`(화면 자체) · `export`(내보내기). 등록·수정·삭제 액션은 이 화면에 대응하는 동작이 없다 |
| 권한 게이팅의 위치 | — | 읽기 게이팅은 셸이 모든 라우트를 한 번에 덮으므로 이 화면의 일이 아니다. 화면이 직접 판정하는 것은 내보내기 버튼 하나이며, 권한이 없으면 **버튼을 렌더하지 않는다**(비활성이 아니다) — `LogListShell.tsx:114-116` · `LogToolbar.tsx:54-55` |

| 청구 상태(미납·정지) | 공통 준수 | 영향이 없다 — 쓰기가 없어 읽기 전용으로 내려앉을 것이 없다 |

| 마스킹의 층 | 공통 준수 | 클라이언트 마스킹은 **두 번째 방어선**이다 — `masking.ts:20-25`. 백엔드가 붙으면 서버가 저장 시점에 가려야 한다 |

| 반출 데이터의 통제 | — | CSV 에 **원문 열을 싣지 않는다.** 원문에는 비밀번호·토큰이 들어 있고 CSV 는 마스킹을 거치지 않은 채 메일로 오간다 — `apps/admin/src/pages/logs/admin/data-source.ts:88-92`. 실패는 성공 톤으로 옮겨 적지 않는다(색은 파일로 옮겨지지 않으므로 문자열이 스스로 말한다) — `data-source.ts:85-87` |

| 입력값 검증 | 공통 준수 | 사용자가 손으로 치는 값은 **기간 직접 지정 두 칸과 검색어**뿐이다. 기간 규칙의 정본은 스키마 하나이며 화면은 판정하지 않는다 — `apps/admin/src/pages/logs/validation.ts:5-16`. 주소에서 읽는 값(축·정렬·페이지·크기)은 전부 허용 목록으로 좁힌다 — `list-state.ts` 의 `readAxes`·`readSort`·`readPageSize` |
