# 이벤트 — 비기능 명세서

| 항목 | 내용 |
|---|---|
| 자원명 | 이벤트 |
| 화면 ID | `SCR-MARKETING-EVENTS` · `SCR-MARKETING-EVENTS-FORM` |
| 메뉴 경로 | 비즈니스 > 마케팅 관리 > 이벤트 |
| 작성자 | Admin Hub 문서화 |
| 작성일 | 2026-07-22 |
| 버전 | 1.0 |
| 관련 기능 명세서 | [목록](../../FSD/marketing/events/index.md) · [등록·수정](../../FSD/marketing/events/form.md) |

**이 자원의 성격** — 마케팅 묶음의 **캠페인 쌍**(이벤트·프로모션) 중 하나다. 두 자원은 기간·상태·혜택 모델을 공유하며, **이 문서가 그 쌍의 정본**이다. [프로모션](promotions.md) 은 차이만 적는다.

## 1. 성능

| 항목 | 요구 | 이 자원의 값 | 측정 방법 |
|---|---|---|---|
| 응답 시간 | 공통 준수 | **측정하지 않음** — 백엔드가 없다 | 백엔드 연동 후 재작성 |
| 조회 성능 | 공통 준수(자동 재시도 없음) | 목록 1건 · 폼(수정) 1건. **곁다리 조회가 없다** — 캠페인은 다른 자원의 선택지를 받아 오지 않는다 | 코드 확인 |
| 동시 사용자 | 공통 준수 | **측정하지 않음** | — |
| 대용량 데이터 처리 | 가상화 없이 대량 행을 그리지 않는다 | ⚠ **페이지네이션도 가상화도 없다** — 받은 전량을 한 번에 그린다. 캠페인은 쌓이는 속도가 느린 자원이라 당장의 위험은 낮지만, 상한이 없다는 사실은 같다 | 코드 확인 |
| 검색·필터 비용 | — | 검색과 상태 필터를 **받아 둔 목록 위에서** 건다. 조건을 바꿔도 서버를 다시 부르지 않는다 | 코드 확인 |

## 2. 사용성

| 항목 | 요구 | 이 자원의 값 |
|---|---|---|
| UI 일관성 | 공통 준수 | 목록은 공용 목록 껍데기, 폼은 공용 폼 껍데기를 쓴다. **손수 조립한 UI 가 없다** |
| 접근성 | 공통 준수 | 툴바의 상태 셀렉트는 보이는 라벨 없이 접근성 이름으로 이름을 갖는다. 검증 오류는 `aria-invalid`·`aria-describedby` 로 입력에 붙는다(공용 폼 규칙) |
| 입력 편의성 | 공통 준수 | 검색은 **한글 조합 중에 끊기지 않는다**(공용 검색 훅). 기간은 시작·종료 두 칸이며 브라우저 날짜 입력을 그대로 쓴다 |
| 상태를 세지 않고 읽기 | 같은 판정을 화면마다 다시 쓰지 않는다 | 진행 상태(예정·진행중·종료)는 **기간에서 파생된다** — 운영자가 직접 고르는 값이 아니다. 그래서 '종료일이 지났는데 진행중으로 보이는' 어긋남이 생기지 않는다 — `apps/admin/src/pages/marketing/_shared/campaign.ts` |
| 조회 상태 보존 | 목록이 Back·새로고침을 견딘다 | 검색어·상태 필터를 주소 쿼리스트링이 소유한다. 한 건을 고치러 갔다 돌아오면 조건이 살아 있다 |
| 사용자 피드백 | 공통 준수 | 이 자원의 토스트는 **전부 공통 층의 것**이다([인벤토리](../../FSD/inventory.md) §9.3 에 이 폴더의 파일이 없다). 저장 실패는 토스트가 아니라 폼 배너다 |

## 3. 보안

| 항목 | 요구 | 이 자원의 값 |
|---|---|---|
| 인증 | 공통 준수 | — |
| 권한 | 공통 준수 | 리소스 키는 잎 `/marketing/events`. 등록 버튼은 `create` 가 없으면 **그려지지 않고**, 저장은 공용 폼 컨트롤러가 요청 전에 막는다 |
| 플랜(엔타이틀먼트) | 공통 준수 | **걸리지 않는다** — 모듈→리소스 대응표에 `/marketing/events` 가 없다 — `apps/admin/src/shared/entitlements/module-resources.ts:80-84`. 같은 묶음의 이메일·SMS 와 결정적으로 다른 점이다 |
| 개인정보 마스킹 | 공통 준수 | **해당 없음** — 이 자원은 개인을 담지 않는다. 캠페인 이름·기간·혜택뿐이며 참여자 명단이 없다 |
| 입력값 검증 | 화면 검증과 서버 검증을 구분한다 | 화면 검증만 있다(스키마 · 기간 역전 금지 · 혜택 값 범위). **서버 검증은 없다 — 백엔드가 없다** |
| HTTPS | 공통 준수 | — |

## 4. 안정성

| 항목 | 요구 | 이 자원의 값 |
|---|---|---|
| 예외 처리 | 공통 준수 | 목록 조회 실패 · 폼 조회 실패 · 저장 실패 · 삭제 실패가 각각 다른 자리에서 알려진다([FSD §7](../../FSD/marketing/events/index.md)) |
| 장애 대응 | 조회 실패 시 화면이 무엇을 남기는가 | 목록은 공용 껍데기의 실패 영역 + `다시 시도`. 폼은 입력을 **유지한 채** 배너만 세운다 — 실패했다고 입력을 지우지 않는다 |
| 재시도 | 자동 재시도 여부 | **자동 재시도 없음**(공통). 사용자 재시도 수단은 목록의 `다시 시도` 와 폼의 저장 재시도 |
| 덮어쓰기 방지 | 두 사람이 같은 건을 고칠 때 | 공통 충돌 다이얼로그(`DLG-COMMON-FORM-CONFLICT`) — 덮어쓰지 않고 사용자가 고른다 |
| 로그 기록 | 어떤 동작이 어느 로그 화면에 남는가 | **확인하지 못했다** — 이 자원의 코드에 로그 적재 지점이 없다 |

## 5. 호환성

| 항목 | 요구 | 이 자원의 값 |
|---|---|---|
| 지원 브라우저 | 공통 준수 | **확인하지 못했다** — 이 저장소에 지원 범위 선언이 없다 |
| 모바일 지원 | 공통 준수 | 관리자 화면 전제(데스크톱 우선). 별도 대응 없음 |
| 해상도 | 넓은 표가 있으면 최소 폭·가로 스크롤 | 목록 열이 적어(이름·기간·상태·혜택 수준) 가로 스크롤이 잘 생기지 않는다 |
| 날짜 입력 | — | 브라우저의 날짜 입력을 그대로 쓴다 — **표시 형식이 브라우저·로캘을 따른다** |

## 6. 운영성

| 항목 | 요구 | 이 자원의 값 |
|---|---|---|
| 로그 관리 | 공통 준수 | 위 §4 와 같다 — 확인하지 못했다 |
| 모니터링 | 공통 준수 | **없다** — 프론트 단독이라 지표 수집 지점이 없다 |
| 재현 손잡이 | 공통 준수 | 공용 CRUD 의 개발 손잡이(지연·실패·빈 목록 주입)를 그대로 쓴다 — `apps/admin/src/shared/crud/dev.ts` |
| 배포 영향 | 이 자원을 고치면 어디가 함께 바뀌는가 | **캠페인 공유 모듈을 고치면 프로모션이 함께 바뀐다** — `apps/admin/src/pages/marketing/_shared/campaign.ts`. 공용 CRUD 를 고치면 마케팅 6화면 전부가 함께 바뀐다 |
| 설정 관리 | 이 자원의 동작을 바꾸는 설정 | **없다** — 플랜에도 걸리지 않고 설정 화면의 값도 참조하지 않는다 |

## 7. 유지보수성

| 항목 | 요구 | 이 자원의 값 |
|---|---|---|
| 코드 확장성 | — | 상태·혜택 어휘가 **공유 모듈 한 곳**에 있어 프로모션과 어긋날 수 없다. 통계 화면에서 회원 등급 어휘가 두 벌로 갈린 것과 반대 사례다 |
| 화면 확장성 | 열·필터·상태값이 늘 때 | 목록이 공용 껍데기라 열 추가는 열 정의 한 줄이다. **다만 행이 늘어나는 것에는 대비가 없다**(§1) |
| 공통 컴포넌트 사용 여부 | 손수 조립한 UI 가 있으면 적는다 | **없다** — 목록·폼 모두 껍데기 위에 얹혀 있다 |
| 두 자원의 중복 | — | 이벤트와 프로모션은 **화면 파일이 따로다.** 공유는 도메인 모듈 수준에서만 이뤄지고, 목록·폼 컴포넌트는 각자 갖는다 |

## 8. 제약사항

| 항목 | 내용 |
|---|---|
| 외부 시스템 연계 | 없다 |
| API 제약 | **백엔드가 없다** — 경계는 `apps/admin/src/pages/marketing/events/data-source.ts` 의 `TODO(backend)` 다. 레이트 리밋·페이지 크기 상한은 정해진 것이 없다 |
| 페이지 크기 상한 | 없다(§1) — 전량을 그린다 |
| 파일 용량 제한 · 업로드 확장자 | 해당 없음 — 이 자원의 폼에 파일 입력이 없다 |
| 참여자·성과 | 이 화면에서 볼 수 없다 — 캠페인의 **정의**만 다루고 실적은 통계 자원의 것이다 |
| 자동 종료 | 종료일이 지나도 데이터가 지워지거나 잠기지 않는다 — 상태 표시만 바뀐다 |
