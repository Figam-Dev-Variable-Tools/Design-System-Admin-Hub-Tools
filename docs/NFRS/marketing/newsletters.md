# 뉴스레터 — 비기능 명세서

| 항목 | 내용 |
|---|---|
| 자원명 | 뉴스레터 |
| 화면 ID | `SCR-MARKETING-NEWSLETTERS` · `SCR-MARKETING-NEWSLETTERS-FORM` |
| 메뉴 경로 | 비즈니스 > 마케팅 관리 > 뉴스레터 |
| 작성자 | Admin Hub 문서화 |
| 작성일 | 2026-07-22 |
| 버전 | 1.0 |
| 관련 기능 명세서 | [목록](../../FSD/marketing/newsletters/index.md) · [등록·수정](../../FSD/marketing/newsletters/form.md) |

**이 자원의 성격** — [SMS 발송](sms.md) 이 정본인 **발송 3종**의 하나이며, [이메일 발송](email.md) 과 **같은 플랜 키를 공유한다.** 이 문서는 **차이만** 적는다.

## 1. 성능

| 항목 | 요구 | 이 자원의 값 | 측정 방법 |
|---|---|---|---|
| 전 항목 | [SMS 발송](sms.md) 준수 | 같다 — **페이지네이션도 가상화도 없다** | 코드 확인 |
| 쌓이는 속도 | — | 발송 3종 중 **가장 느리게 쌓인다** — 뉴스레터는 주기 발행이라 건수가 적다. 행 상한이 없다는 사실의 위험이 셋 중 가장 낮다 | 코드 확인 |
| 바이트 등급 계산 | — | **없다** — 이메일 계열이다 |

## 2. 사용성

| 항목 | 요구 | 이 자원의 값 |
|---|---|---|
| 전 항목 | [SMS 발송](sms.md) 준수 | 같다 — 세그먼트 선택기 · 변수 삽입 바를 공유한다 |
| 발행 주기 | — | 뉴스레터는 **정기 발행물**이라 목록이 회차의 이력처럼 읽힌다. 이메일 캠페인 목록이 단발 발송의 이력인 것과 성격이 다르다 |
| 성과 지표 | — | 이메일 발송과 같은 열람·클릭 축을 쓴다 — 두 자원이 같은 계산을 공유한다 |
| 사용자 피드백 | 공통 준수 | 이 자원의 토스트는 **전부 공통 층의 것**이다 |

## 3. 보안

| 항목 | 요구 | 이 자원의 값 |
|---|---|---|
| 권한 | [SMS 발송](sms.md) 준수 | **리소스 키만 다르다** — 잎 `/marketing/newsletters` |
| 플랜(엔타이틀먼트) | 공통 준수 | ⚠ **[이메일 발송](email.md) 과 같은 스위치다** — `marketing.email`(최소 티어 `basic`)이 이 화면과 이메일 발송 **둘 다**를 덮는다 — `apps/admin/src/shared/entitlements/module-resources.ts:80-83`. 하위 플랜에서는 **잠금 화면**으로 뜬다(숨김이 아니다) |
| 개인정보 마스킹 | — | **해당 없음** — 구독자를 세그먼트와 인원수로만 다룬다. 구독자 개별 목록이 이 화면에 없다 |
| 입력값 검증 | [SMS 발송](sms.md) 준수 | 같다. 광고 표기·야간 발송 제약은 걸리지 않는다 |

## 8. 제약사항

| 항목 | 내용 |
|---|---|
| 전 항목 | [SMS 발송](sms.md) · [이메일 발송](email.md) 준수 — 백엔드 없음 · 페이지 상한 없음 · 수신 거부 규칙 없음 · 성과 지표의 집계 주체 없음 |
| 구독 관리 | **구독·구독 해지를 이 화면에서 다루지 않는다** — 구독자 목록·해지 처리 화면이 없다. 세그먼트로만 대상을 고른다 |
| 발행 예약 | 발송 상태 모델이 예약을 표현하되, **실제 예약 실행은 서버의 몫**이며 지금은 없다 |
| 파일 용량 제한 · 업로드 확장자 | 이 화면에 파일 입력이 없다 |
