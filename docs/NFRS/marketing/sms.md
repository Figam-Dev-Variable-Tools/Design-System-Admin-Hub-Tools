# SMS 발송 — 비기능 명세서

| 항목 | 내용 |
|---|---|
| 자원명 | SMS 발송 |
| 화면 ID | `SCR-MARKETING-SMS` · `SCR-MARKETING-SMS-FORM` |
| 메뉴 경로 | 비즈니스 > 마케팅 관리 > SMS 발송 |
| 작성자 | Admin Hub 문서화 |
| 작성일 | 2026-07-22 |
| 버전 | 1.0 |
| 관련 기능 명세서 | [목록](../../FSD/marketing/sms/index.md) · [등록·수정](../../FSD/marketing/sms/form.md) |

**이 자원의 성격** — 마케팅 묶음의 **발송 3종**(SMS·이메일·뉴스레터) 중 하나이며, **이 문서가 그 셋의 정본**이다. [이메일 발송](email.md) 과 [뉴스레터](newsletters.md) 는 차이만 적는다.

**이 자원만의 무게** — 발송은 **되돌릴 수 없다.** 나간 문자는 회수되지 않고 수신자 수만큼 곱해진다. 그래서 이 자원의 비기능 요구는 성능이 아니라 **'잘못 나가지 않게 하는 것'** 에 몰려 있다.

## 1. 성능

| 항목 | 요구 | 이 자원의 값 | 측정 방법 |
|---|---|---|---|
| 응답 시간 | 공통 준수 | **측정하지 않음** — 백엔드가 없다 | 백엔드 연동 후 재작성 |
| 조회 성능 | 공통 준수(자동 재시도 없음) | 목록 1건 · 폼(수정) 1건에 더해 **세그먼트·발신 프로필·템플릿 후보**를 함께 읽는다 — 곁다리가 캠페인 쌍보다 많다 | 코드 확인 |
| 동시 사용자 | 공통 준수 | **측정하지 않음** | — |
| 대용량 데이터 처리 | 가상화 없이 대량 행을 그리지 않는다 | ⚠ **페이지네이션도 가상화도 없다.** 발송 이력은 **가장 빨리 쌓이는 자원**이다 — 캠페인 정의와 달리 보낼 때마다 한 줄이 는다 | 코드 확인 |
| 본문 길이 계산 | 타이핑마다 도는 계산이 화면을 붙잡지 않는다 | 문자 등급 판정은 **본문 길이에 비례하는 바이트 계산**이며 입력마다 돈다. 본문 상한이 작아(수천 바이트) 체감 비용은 낮다 | 코드 확인 |
| 예상 비용·건수 | — | 세그먼트 수신자 수 합계는 **고른 세그먼트 수만큼의 덧셈**이다 — `apps/admin/src/pages/marketing/_shared/messaging.ts:551-559` | 코드 확인 |

## 2. 사용성

| 항목 | 요구 | 이 자원의 값 |
|---|---|---|
| UI 일관성 | 공통 준수 | 목록은 공용 목록 껍데기, 폼은 공용 폼 껍데기. **세그먼트 선택기와 변수 삽입 바는 이 묶음이 손수 만든 조립물**이다 — `apps/admin/src/pages/marketing/_shared/SegmentPicker.tsx` · `_shared/VariableInsertBar.tsx` |
| 접근성 | 공통 준수 | 검증 오류가 입력에 붙고 첫 오류로 포커스가 간다(공용 폼 규칙). 변수 삽입은 **포커스를 뺏지 않는다** — 커서 자리를 잃지 않기 위해서다 |
| 입력 편의성 | 공통 준수 | 치환변수를 패널에서 골라 커서 자리에 꽂는다. 템플릿을 불러오면 본문이 채워진다 |
| 되돌릴 수 없는 동작 | 실행 전에 판단할 재료를 준다 | 발송 전에 **등급·바이트·예상 건수·수신자 수**를 화면에서 보여 준다. 다 보내고 나서 명세서를 확인하는 순서가 되지 않게 한다 |
| 광고 표기 | 법에서 요구하는 것을 화면이 강제한다 | 광고성이면 본문이 `(광고)` 로 시작하고 무료수신거부 방법이 본문에 있어야 저장된다 — `apps/admin/src/pages/marketing/_shared/messaging.ts` |
| 야간 발송 | — | 발송 가능 시간대 밖은 화면이 먼저 알린다 — 발송사에서 거절당해 실패 목록으로 알게 되는 것보다 낫다 |
| 사용자 피드백 | 공통 준수 | 이 자원의 토스트는 **전부 공통 층의 것**이다([인벤토리](../../FSD/inventory.md) §9.3) |
| 조회 상태 보존 | — | 검색어·상태 필터를 주소가 소유한다 |

## 3. 보안

| 항목 | 요구 | 이 자원의 값 |
|---|---|---|
| 인증 | 공통 준수 | — |
| 권한 | 공통 준수 | 리소스 키는 잎 `/marketing/sms`. 등록 버튼은 `create` 가 없으면 그려지지 않고, 저장은 공용 폼 컨트롤러가 요청 전에 막는다 |
| 플랜(엔타이틀먼트) | 공통 준수 | ⚠ **완전 숨김이다** — `marketing.sms` 의 최소 티어가 `null` 이라, 계약이 없으면 **잠금 화면이 아니라 메뉴에서 아예 사라진다** — `apps/admin/src/shared/entitlements/plan.ts:233-241`. 이유가 코드에 적혀 있다: 발신번호 등록과 발송사 계약이 따로 있어야 켤 수 있어 **플랜을 올려도 저절로 열리지 않는다.** [이메일 발송](email.md)(`marketing.email`, 최소 티어 `basic` ⇒ 잠금 화면)과 **다른 축이다** |
| 개인정보 마스킹 | 어떤 필드를 어떤 규칙으로 가리는가 | **개인 식별 필드가 화면에 없다** — 수신자는 개인이 아니라 **세그먼트(그룹)와 인원수**로만 다뤄진다 — `apps/admin/src/pages/marketing/_shared/messaging.ts:412-417`. 마스킹할 전화번호 열 자체가 없다. **발신번호는 회사 번호라 개인정보가 아니다** |
| 입력값 검증 | 화면 검증과 서버 검증을 구분한다 | 화면 검증만 있다(본문·발신·세그먼트·광고 요건·시간대·치환변수). **서버 검증은 없다 — 백엔드가 없다.** 광고 표기·야간 발송처럼 법적 요구가 걸린 규칙이 지금은 **프론트에만** 있다 |
| HTTPS | 공통 준수 | — |

## 4. 안정성

| 항목 | 요구 | 이 자원의 값 |
|---|---|---|
| 예외 처리 | 공통 준수 | 목록 조회 실패 · 폼 조회 실패 · 저장 실패 · 삭제 실패가 각각 다른 자리에서 알려진다 |
| 장애 대응 | 조회 실패 시 무엇을 남기는가 | 폼은 입력을 유지한 채 배너만 세운다 — 본문을 다 써 놓고 저장에 실패했을 때 그것이 사라지면 피해가 크다 |
| 재시도 | 자동 재시도 여부 | **자동 재시도 없음**(공통). ⚠ **발송에 자동 재시도가 붙으면 안 되는 자원**이다 — 같은 문자가 두 번 나간다 |
| 덮어쓰기 방지 | — | 공통 충돌 다이얼로그 |
| 발송된 것 보호 | — | 이미 나간 발송은 본문을 고칠 수 없다 — 수신자가 받은 것과 관리자가 보는 것이 달라지면 안 된다 — `apps/admin/src/pages/marketing/_shared/messaging.ts:483` |
| 로그 기록 | 어떤 동작이 어느 로그 화면에 남는가 | **확인하지 못했다** — 이 자원의 코드에 로그 적재 지점이 없다. **발송은 감사 로그가 가장 필요한 동작**이므로 연동 시 우선 확인 대상이다 |

## 7. 유지보수성

| 항목 | 요구 | 이 자원의 값 |
|---|---|---|
| 코드 확장성 | — | 세그먼트·발신·등급 판정·광고 요건이 **공유 모듈 한 곳**에 있다. 발송 3종이 같은 규칙을 각자 들고 있지 않다 |
| 화면 확장성 | 열·필터·상태값이 늘 때 | 발송 상태가 늘면 상태별 가능 동작 표 한 곳만 고치면 된다 — `messaging.ts` 의 발송 상태·동작 판정 |
| 공통 컴포넌트 사용 여부 | 손수 조립한 UI | **둘 있다** — 세그먼트 선택기 · 변수 삽입 바. 둘 다 마케팅 묶음 안의 `_shared` 라 세 발송 화면이 나눠 쓴다 |
| 규칙이 프론트에만 있는 위험 | — | ⚠ 광고 표기·야간 발송·바이트 등급이 전부 화면 코드에 있다. **연동 시 서버가 같은 규칙을 갖지 않으면**, 화면을 우회한 요청이 규제를 어긴 문자를 보낼 수 있다 |

## 8. 제약사항

| 항목 | 내용 |
|---|---|
| 외부 시스템 연계 | 문자 발송사 · 카카오 비즈니스. **둘 다 연동되어 있지 않다** — 화면은 규칙만 갖고 있다 |
| API 제약 | **백엔드가 없다** — 경계는 `apps/admin/src/pages/marketing/sms/data-source.ts` 의 `TODO(backend)` 다. 발송사 레이트 리밋·건당 단가는 서버가 붙어야 정해진다 |
| 페이지 크기 상한 | 없다(§1) — 전량을 그린다 |
| 발송 시간 | 브랜드 메시지·광고성은 발송 가능 시간대가 제한된다(법적 요구) |
| 단가 | 화면의 예상 비용은 **참고값**이다 — 실단가는 연동 시 서버가 준다 — `messaging.ts:551` |
| 파일 용량 제한 · 업로드 확장자 | 이미지 첨부는 **발송 템플릿 자원**의 것이다([발송 템플릿](templates.md)). 이 화면 자체에는 파일 입력이 없다 |
| 수신 거부 | 수신 거부자 제외 규칙이 화면에 없다 — **서버가 맡아야 할 자리**이며 지금은 없다 |
| 발송 취소 | 예약 발송을 취소하는 동작은 상태별 가능 동작 표가 정한다. **이미 나간 것은 되돌릴 수 없다** |
