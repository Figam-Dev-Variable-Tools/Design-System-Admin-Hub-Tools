## 2. 사용성

| 항목 | 요구 | 이 자원의 값 |
|---|---|---|
| UI 일관성 | 공통 준수 | 골격은 **상품 카테고리 화면과 같다** — 같은 2단계 규칙과 같은 삭제 차단을 쓰기 때문이다. 규칙이 같은 화면이 서로 다른 골격을 갖기 시작하면 한쪽만 고쳐지는 날이 온다 — `apps/admin/src/pages/programs/categories/ProgramCategoriesPage.tsx:3-6` |
| 접근성 | 공통 준수 | 펼침 토글이 `aria-expanded`·`aria-controls` 와 `{카테고리명} 하위 카테고리 접기/펼치기 ({N}개)` 라벨을 갖는다. **토글이 없는 행도 라벨 시작점을 맞춘다**(들쭉날쭉하지 않게) — `apps/admin/src/pages/programs/categories/ProgramCategoriesPage.tsx:234,241-242` |
| 막힌 컨트롤은 이유를 말한다 | 비활성에 침묵을 두지 않는다 | 삭제가 막히면 **사유가 툴팁(`title`)과 `aria-label` 에 함께 실린다** — 마우스와 스크린리더가 같은 문장을 받는다 — `apps/admin/src/pages/programs/categories/ProgramCategoriesPage.tsx:290-295` |
| 근거를 확인할 길 | 막은 이유를 열어 볼 수 있게 한다 | 사용 건수 배지가 **그 카테고리로 걸러진 프로그램 목록**으로 가는 링크다. **미사용(0건)은 링크로 만들지 않는다** — 열어 봐야 빈 목록이다 — `apps/admin/src/pages/programs/categories/ProgramCategoriesPage.tsx:246-258` |
| 사용자 피드백 | 공통 준수 | 이 자원의 토스트는 **성공 2건**이다([인벤토리](../../FSD/inventory.md) §9.3 과 일치): 추가/저장 한 건, 삭제 한 건. **삭제 실패는 토스트가 아니라 다이얼로그 안 배너**다 |
| 상태 유지 | 재조회가 보고 있던 것을 지우지 않는다 | `불러오는 중…` 은 **최초 로드에만** 쓴다 — 재조회에도 쓰면 이미 보고 있던 목록이 덮인다 — `apps/admin/src/pages/programs/categories/ProgramCategoriesPage.tsx:335-336` |

## 3. 보안

| 항목 | 요구 | 이 자원의 값 |
|---|---|---|
| 인증 | 공통 준수 | — |
| ⚠ 권한 | 공통 준수 | 리소스 키는 `program-categories`. **화면이 가리는 것은 `create` 하나뿐이다** — `카테고리 추가` 와 하위 `+` 만 감춘다. **연필과 휴지통에는 권한 판정이 붙어 있지 않아** 읽기 권한만으로도 보이고 눌린다 — `apps/admin/src/pages/programs/categories/ProgramCategoriesPage.tsx:414`. 막는 것은 서버뿐이다 |
| ⚠ 플랜(엔타이틀먼트) | 공통 준수(fail-open) | **모듈 표에 `/programs/categories` 가 없다** — `apps/admin/src/shared/entitlements/module-resources.ts`. fail-open 이라 어떤 플랜에서도 잠기지 않는다 |
| 개인정보 마스킹 | 필드 이름으로 적는다 | **이 자원에 개인 식별 정보가 없다** — 이름과 상위 참조뿐이다 |
| 입력값 검증 | 공통 준수(스키마 하나가 정본) | 팝업의 zod 스키마 — `apps/admin/src/pages/programs/validation.ts:105-109`. **구조 규칙(3단계 금지·자기 자신 금지·하위 보유 이동 금지)은 저장소가 갖는다** — `apps/admin/src/pages/programs/_shared/store.ts:471-500` |
| 화면 잠금과 저장소 거절의 일치 | 고를 수 있는데 저장이 실패하지 않는다 | 팝업이 **대분류만 후보로 주고 자기 자신을 뺀다.** 하위를 가진 대분류는 상위 셀렉트를 미리 잠근다 — 저장소의 세 거절 문구는 **다른 관리자가 그 사이에 구조를 바꾼 경우에만** 화면에 닿는다 |
| 중복 제출 | 공통 준수 | 팝업의 저장 버튼이 저장 중 잠긴다 |
| HTTPS | 공통 준수 | **측정하지 않음** |

## 4. 안정성

| 항목 | 요구 | 이 자원의 값 |
|---|---|---|
| 예외 처리 | 공통 준수 | 조회 실패는 목록 대신 배너 + `다시 시도`. **삭제 실패는 다이얼로그를 닫지 않고** 그 안 배너로 말한다(확인 재클릭이 재시도다) |
| 삭제 차단의 이중화 | 화면과 저장소가 같은 사실을 본다 | 휴지통의 비활성 조건과 저장소의 거절 조건이 **같은 두 사실**(쓰는 프로그램이 있는가 · 하위가 있는가)을 본다 — '눌리는데 실패하는 버튼'이 생기지 않는다 — `apps/admin/src/pages/programs/_shared/store.ts:512-520` |
| 이름 변경의 파급 | 비정규화 라벨이 낡지 않는다 | 카테고리 이름을 고치면 **그 카테고리를 쓰는 프로그램의 라벨도 함께 갱신된다** — `apps/admin/src/pages/programs/_shared/store.ts:506-508` |
| 재시도 | 공통 준수(자동 재시도 없음) | 조회 배너의 `다시 시도` · 다이얼로그 확인 재클릭 · 팝업 저장 재클릭 |
| 동시 저장 | 공통 준수(last-write-wins) | **전용 충돌 다이얼로그가 없다** — 다른 관리자가 먼저 지웠거나 고쳤어도 화면은 일반 실패 문구를 쓴다 |
| 로그 기록 | 공통 준수 | **확인하지 못했다** |

## 7. 유지보수성

| 항목 | 요구 | 이 자원의 값 |
|---|---|---|
| 코드 확장성 | 공통 준수 | 사용 여부 필터는 표시 목록만 화면이 갖고 **거르는 규칙은 표시 규칙 모듈**이 갖는다. 필터 축을 그 모듈로 올리지 않은 이유는 소비자가 이 화면 하나이기 때문이다 — `apps/admin/src/pages/programs/categories/ProgramCategoriesPage.tsx:45-48` |
| 화면 확장성 | 열·필터·상태값이 늘 때 견디는가 | 고른 필터를 주소가 소유한다 — 필터를 건 화면을 그대로 공유할 수 있다 |
| 공통 컴포넌트 사용 여부 | 페이지가 UI 를 손수 조립하지 않는다 | 팝업 껍데기(포커스 트랩·Esc·딤·복귀)와 이탈 가드는 공통 층이다. **목록은 트리 구조라 공통 목록 껍데기를 쓰지 않고 직접 조립한다** — 그래서 공통 삭제 확인이 아니라 **화면 고유 확인 다이얼로그**를 갖는다 |
| 규칙의 정본 하나 | 같은 판정을 두 곳에 적지 않는다 | 3단계 금지·자기 참조 금지·하위 보유 이동 금지가 저장소의 한 함수에 모여 있다 — `apps/admin/src/pages/programs/_shared/store.ts:471-481` |

## 8. 제약사항

| 항목 | 내용 |
|---|---|
| 외부 시스템 연계 | 없음 |
| API 제약 | **백엔드가 없다.** 서버 계약은 §6 이 갖는다(사용 중이면 409) |
| 계층 | **2단계까지만.** 중분류 아래에는 하위 추가 버튼이 없다 |
| 삭제 차단 | 쓰는 프로그램이 있거나 하위가 달려 있으면 지울 수 없다. **우회로는 하나뿐이다** — 그 프로그램들의 카테고리를 옮기거나 하위를 먼저 정리한다 |
| 사용여부(소프트 비활성) | **없다** — 고객센터 문의 유형과 달리 이 자원에는 '사용 안 함'으로 내리는 축이 없다. 지우거나 그대로 두거나 둘뿐이다 |
| 문자 수 상한 | 카테고리 이름 40자 — `apps/admin/src/pages/programs/_shared/store.ts:132` |
| 중복 방지 | **없다** — 같은 이름의 카테고리를 여럿 만들 수 있다(대분류든 중분류든) |
| 정렬 | 저장된 순서 그대로다 — **표시 순서를 바꾸는 축이 없다** |
| 파일 업로드 | 해당 없음 — 이 자원에 파일 입력이 없다 |
