**이 자원의 성격** — 포트폴리오 묶음의 정본이다. [성공 사례](case-studies.md) 는 이 자원과 같은 껍데기·같은 공유 하단부를 쓰며 차이만 적는다. **이 묶음은 사이드바 맨 뒤에 있다** — 순서가 매일 여는 횟수로 정해져 있고, 포트폴리오는 분기에 한 번 손대는 화면이다 — `apps/admin/src/shared/layout/nav-config.ts:169-171`.

| 이미지 | 목록이 이미지 때문에 무거워지지 않는다 | **목록에 이미지 열을 넣지 않는다**(전 섹션 규칙). 이미지는 폼에서만 다룬다 — `apps/admin/src/pages/portfolio/items/PortfolioListPage.tsx:4` | 코드 확인 |

## 2. 사용성

| 항목 | 요구 | 이 자원의 값 |
|---|---|---|
| UI 일관성 | 공통 준수 | 목록은 공용 목록 껍데기, 폼은 공용 폼 껍데기. **손수 조립한 UI 는 두 폼이 나눠 쓰는 하단부 하나**다 — `apps/admin/src/pages/portfolio/_shared/PortfolioMediaFields.tsx:1-4` |
| 접근성 | 공통 준수 | 분류 셀렉트는 보이는 라벨 없이 접근성 이름을 갖는다. 행의 노출 토글도 **`{제목} 노출 여부`** 로 무엇을 켜는지 이름이 말한다 — `apps/admin/src/pages/portfolio/_shared/publishColumn.tsx:35` |
| 여러 건을 한 번에 정리하기 | 반복 작업을 왕복시키지 않는다 | 노출을 **표에서 바로** 켜고 끈다. 폼을 열게 하면 열 건을 내리는 데 열 번 저장해야 한다 |
| 조회 상태 보존 | 목록이 Back·새로고침을 견딘다 | 분류 필터를 **주소 쿼리스트링**이 소유한다 — 한 건을 고치러 갔다 돌아오면 조건이 살아 있고, 그 주소를 그대로 공유할 수 있다 — `PortfolioListPage.tsx:6-9` |
| 필터를 바꾸면 선택이 풀린다 | 보이지 않는 것을 지우지 않는다 | 분류를 바꾸면 행 선택이 해제된다 — `PortfolioListPage.tsx:82-84` |
| 사용자 피드백 | 공통 준수 | 노출 토글에만 **이 자원의 문구**가 붙는다(`게시했어요` / `숨겼어요`). 나머지는 공통 토스트다 |
| 입력 편의성 | 공통 준수 | 여러 줄 입력(소개)은 **타이핑 중 검증하지 않는다** — 아직 짧다고 붉게 알리는 것은 방해다 — `PortfolioFormPage.tsx:196-198` |

## 3. 보안

| 항목 | 요구 | 이 자원의 값 |
|---|---|---|
| 인증 | 공통 준수 | — |
| 권한 | 공통 준수 | 리소스 키는 잎 `/portfolio/items`. 등록 버튼은 `create` 가 없으면 그려지지 않고 — `PortfolioListPage.tsx:131-132` — **노출 토글은 `update` 가 없으면 잠긴다.** 그 판정은 옵션이 아니라 **필수 인자**다: 기본값을 주면 이 열을 쓰는 새 목록이 인자를 빠뜨렸을 때 조용히 열린 채로 통과한다 — `publishColumn.tsx:17-22` |
| 플랜(엔타이틀먼트) | 공통 준수 | **걸리지 않는다** — 포트폴리오 묶음은 모듈→리소스 대응표에 없다. 어떤 플랜에서도 잠기지 않는다 |
| 개인정보 마스킹 | 어떤 필드를 어떤 규칙으로 가리는가 | **해당 없음** — 이 자원이 담는 사람 관련 값은 `고객사`(법인명)뿐이며 개인 식별 정보가 아니다. 픽스처의 고객사는 **가상 표기**다 — `apps/admin/src/pages/portfolio/_shared/store.ts:29` |
| 입력값 검증 | 화면 검증과 서버 검증을 구분한다 | 화면 검증만 있다(스키마). **서버 검증은 없다 — 백엔드가 없다.** 대표 이미지는 **등록 여부만** 보고 형식을 강제하지 않는다 — 업로드가 없어 값이 `blob:` 뿐이라 형식을 요구하면 영영 제출할 수 없게 된다 — `apps/admin/src/shared/crud/validation.ts:38-54` |
| 공개되는 화면 | — | 이 자원은 **홈페이지에 실리는 내용**을 만든다. 노출 토글이 곧 공개 스위치이며, 그래서 **`update` 권한이 곧 공개 권한**이다 |
| HTTPS | 공통 준수 | — |

## 4. 안정성

| 항목 | 요구 | 이 자원의 값 |
|---|---|---|
| 예외 처리 | 공통 준수 | 목록 조회 실패 · 폼 조회 실패 · 저장 실패 · 삭제 실패가 각각 다른 자리에서 알려진다 |
| ⚠ 구분되지 않는 실패 | `null`(모름)과 `[]`(없음)을 섞지 않는다 | **분류 선택지의 조회 실패를 알리지 않는다** — 빈 배열로 뭉개서 `전체 분류`/`분류 선택` 하나만 남는다. 운영자는 **분류가 없는 것인지 못 불러온 것인지 구분할 수 없고**, 폼에서는 필수 항목을 채울 수 없어 저장 자체가 불가능해진다 — `PortfolioListPage.tsx:80` · `PortfolioFormPage.tsx:96` |
| 장애 대응 | 조회 실패 시 무엇을 남기는가 | 목록은 공용 껍데기의 실패 영역 + `다시 시도`. 폼은 입력을 유지한 채 배너만 세운다 |
| 재시도 | 자동 재시도 여부 | **자동 재시도 없음**(공통) |
| 덮어쓰기 방지 | — | 공통 충돌 다이얼로그(`DLG-COMMON-FORM-CONFLICT`) |
| 이름 변경의 파급 | 표시가 어긋나지 않는다 | 분류 이름을 고치면 **그 분류를 쓰는 항목의 표시 이름도 함께 바뀐다** — `portfolio/_shared/store.ts:222-225`. 백엔드가 붙으면 정합성은 서버의 몫이다 |
| 로그 기록 | — | **확인하지 못했다** |
