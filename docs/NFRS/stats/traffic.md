# 유입 분석 — 비기능 명세서

| 항목 | 내용 |
|---|---|
| 자원명 | 유입 분석 |
| 화면 ID | `SCR-STATS-TRAFFIC` |
| 메뉴 경로 | 분석 · 운영 > 통계 > 유입 분석 |
| 작성자 | Admin Hub 문서화 |
| 작성일 | 2026-07-22 |
| 버전 | 1.0 |
| 관련 기능 명세서 | [유입 분석](../../FSD/stats/traffic/index.md) |

**이 자원의 성격** — 통계 6화면이 공유하는 껍데기 위의 읽기 전용 분석 화면이다. 성능·사용성·안정성·호환성·유지보수성의 값은 [방문자 통계](visitors.md) 와 **같다.** 이 문서는 **이 자원만의 차이**를 적는다. 차이의 뿌리는 둘이다: **플랜으로 잠기고, 유입 채널 어휘를 이 화면이 홀로 갖는다.**

## 1. 성능

| 항목 | 요구 | 이 자원의 값 | 측정 방법 |

| 전 항목 | [방문자 통계](visitors.md) 준수 | 같다 | 코드 확인 |

| 랜딩페이지 행 수 | — | ⚠ 사이트 페이지 수만큼 늘 수 있는 유일한 축이다. **행 수 상한이 코드에 없고** 페이지 상한(100행)과 내보내기가 유일한 완충이다 | 코드 확인 |

## 8. 제약사항

| 항목 | 내용 |
|---|---|
| 전 항목 | [방문자 통계](visitors.md) 준수 — 백엔드 없음 · 쓰기 API 없음 · 조회 기간 상한 없음 · 미래 조회를 막지 않음 · 페이지 100행 · CSV 단일 형식 · 뒤로가기로 조건을 되돌릴 수 없음 |
| 플랜 | `고급 통계` 모듈(최소 `프로` 티어)이 있어야 열린다 |
| 유입 채널 | 다섯 고정. `북마크` 는 이름과 담기는 내용이 어긋나 있으나 의도된 유지다 |
| 검색어 수집의 한계 | 구글 유입은 검색어가 전달되지 않아 '참조검색어 없음'으로 잡힌다 — **우리가 고칠 수 없는 제약**이다 |
| 랜딩페이지 이동 | 표의 랜딩페이지에서 그 페이지로 가는 링크가 없다 — 고객이 보는 홈페이지는 이 리포 밖이다 |
| 결제 의존 | 결제를 쓰지 않아도 화면이 갈리지 않는다 — 구매·매출 지표가 0 으로 그려진다 |
| 파일 용량 제한 · 업로드 확장자 | 해당 없음 — 파일 입력이 없다 |
