## 2. 사용성

| 항목 | 요구 | 이 자원의 값 |
|---|---|---|
| UI 일관성 | 공통 준수 | 목록은 공통 목록 껍데기, **폼은 공통 폼 껍데기**를 쓴다 — 제목·설명·조회 실패 화면·서버 오류 배너·충돌 다이얼로그·이탈 가드·버튼 줄이 전부 껍데기의 것이고 이 화면은 필드 셋만 주입한다 — `apps/admin/src/pages/support/replies/ReplyFormPage.tsx:3-4` |
| 접근성 | 공통 준수 | 폼의 인라인 오류가 `aria-invalid`·`aria-describedby` 로 연결된다. 치환 변수 안내가 `<code>` 로 표기되어 문장과 구분된다 |
| 입력 편의성 | 공통 준수 | 유형 태그 셀렉트에 **사용 안 함으로 내린 유형까지 담고** `{유형명} (미사용)` 로 표기한다 — 편집 중 값이 조용히 사라지지 않게 한 자리다 — `apps/admin/src/pages/support/replies/ReplyFormPage.tsx:75,122` |
| 사용자 피드백 | 공통 준수 | 이 자원에는 **자기 토스트가 없다**([인벤토리](../../FSD/inventory.md) §9.3 에 두 화면 모두 없다). 저장·삭제 통지는 공통 층(`MSG-COMMON-01`~`03`)이 낸다 |
| 실수를 저장 전에 잡는다 | 사고를 사후에 알리지 않는다 | 마케팅 발송 문법(`#{...}`)이 섞이면 **저장을 막고 문제의 토큰을 문장에 실어** 보여 준다 — 놓치면 그 글자가 고객에게 그대로 간다 — `apps/admin/src/pages/support/replies/validation.ts:42` |
| 빈 상태의 갈래 | 왜 비었는지에 따라 복구 수단이 다르다 | **필터 축이 없어 '필터 때문에 비었다'는 갈래가 성립하지 않는다** — 검색 갈래와 진짜 0건 갈래 둘뿐이다 — `apps/admin/src/pages/support/replies/RepliesPage.tsx:123-126` |

## 3. 보안

| 항목 | 요구 | 이 자원의 값 |
|---|---|---|
| 인증 | 공통 준수 | — |
| 권한 | 공통 준수 | 리소스 키는 `support-replies`. **고객센터에서 쓰기 권한을 실제로 읽는 유일한 목록 화면이다** — 등록 버튼을 `create` 로 감추고, 행 액션·선택 체크박스는 공통 껍데기가 권한으로 가린다. 문의 유형·자주 묻는 질문 화면은 권한을 읽지 않는다 |
| ⚠ 플랜(엔타이틀먼트) | 공통 준수(fail-open) | **모듈 표에 `/support/**` 가 없다** — `apps/admin/src/shared/entitlements/module-resources.ts`. fail-open 이라 잠기지 않는다 |
| 개인정보 마스킹 | 필드 이름으로 적는다 | **이 자원 자체에는 개인 식별 정보가 없다** — 상용구 문장과 유형 태그뿐이다. 다만 **본문에 개인정보를 적어 넣는 것을 막는 규칙도 없다**(치환 변수를 쓰라는 안내만 있다) |
| 입력값 검증 | 공통 준수(스키마 하나가 정본) | `apps/admin/src/pages/support/replies/validation.ts`. 제목·본문 필수와 길이, 그리고 **마케팅 문법 혼입 차단** |
| 문법 경계 | 두 치환 문법이 섞이지 않게 한다 | 이 화면은 `#{...}` 를 막고, **반대 방향(마케팅 본문의 `{{…}}`)은 마케팅 쪽 검증이 막는다** — `apps/admin/src/pages/support/replies/validation.ts:29` |
| 중복 제출 | 공통 준수 | 폼은 동기 잠금 + 제출 시도 단위 멱등키 |
| HTTPS | 공통 준수 | **측정하지 않음** |

## 4. 안정성

| 항목 | 요구 | 이 자원의 값 |
|---|---|---|
| 예외 처리 | 공통 준수 | 폼의 조회 실패는 **공통 껍데기가 없음/오류를 갈라** 그린다(없는 항목에는 재시도를 주지 않는다) — `apps/admin/src/shared/crud/FormPageShell.tsx:147` |
| 삭제의 파급 | 지운 뒤 남는 참조를 감춰서는 안 된다 | **참조 차단이 없어도 된다** — 티켓에는 삽입된 **글자**만 남고 템플릿을 가리키는 참조가 없다. 그래서 템플릿을 지워도 이미 저장된 답변 글은 그대로다 |
| 수정의 파급 | 이미 나간 것이 사후에 바뀌지 않는다 | 같은 이유로 **템플릿을 고쳐도 이미 티켓에 삽입해 저장한 답변은 바뀌지 않는다** — 삽입은 복사이지 참조가 아니다 |
| 재시도 | 공통 준수(자동 재시도 없음) | 목록 배너의 `다시 시도` · 폼 저장 재클릭 |
| 동시 저장 | 공통 준수(last-write-wins + 충돌 다이얼로그) | 폼은 공통 충돌 다이얼로그를 쓴다 |
| 로그 기록 | 공통 준수 | **확인하지 못했다** |

## 6. 운영성

| 항목 | 요구 | 이 자원의 값 |
|---|---|---|
| 로그 관리 | 공통 준수 | **확인하지 못했다** |
| 모니터링 | 공통 준수 | **없다** — 어느 템플릿이 얼마나 쓰였는지 세는 축이 없다 |
| 배포 영향 | 이 자원을 고치면 어느 화면이 함께 영향을 받는가 | [1:1 문의 상세](tickets.md)의 `답변 템플릿 삽입` 후보가 이 자원을 읽는다. 치환 변수 이름을 바꾸면 **삽입 시점 치환 함수와 검증 문구를 함께** 고쳐야 한다 — `apps/admin/src/pages/support/_shared/domain.ts:404-412` |
| 설정 관리 | 이 자원의 동작을 바꾸는 설정값이 어디 있는가 | 없다. 유형 태그의 원천은 [문의 유형](categories.md)이다 |
| ⚠ 두 치환 문법의 공존 | 통일하지 않은 것이 결정이다 | 고객센터는 `{{고객명}}`(우리 앱이 **삽입 시점**에 치환), 마케팅은 `#{member.name}`(발송 대행사가 **발송 시점**에 치환). **표기를 통일하면 차이가 사라지는 것이 아니라 보이지 않게 된다** — 같은 생김새가 화면에 따라 다른 뜻을 갖는다 — `apps/admin/src/pages/support/replies/validation.ts:3-24` |
| 백엔드 없음의 이음매 | 서버가 맡을 일이 코드에 표시되어 있다 | 이 자원의 어댑터에는 `TODO(backend)` 주석이 없다 — **확인하지 못했다** |

## 7. 유지보수성

| 항목 | 요구 | 이 자원의 값 |
|---|---|---|
| 코드 확장성 | 공통 준수 | 치환 변수 셋이 한 함수에 모여 있어 축을 늘리려면 그 함수와 안내 문구·검증 문구를 함께 고친다 |
| 세 화면이 도메인을 공유한다 | 서로를 import 하지 않는다 | 티켓·문의 유형·답변 템플릿이 한 잎 모듈의 타입·순수 규칙을 함께 쓴다 — `apps/admin/src/pages/support/_shared/domain.ts:3-7` |
| 화면 확장성 | 열·필터·상태값이 늘 때 견디는가 | ⚠ **유형 태그로 거르는 필터가 없다** — 문의 유형 화면의 `템플릿 {N}` 링크가 전체 목록을 여는 것도 그 때문이다. 템플릿이 늘면 검색에 의존해야 한다 |
| 공통 컴포넌트 사용 여부 | 페이지가 UI 를 손수 조립하지 않는다 | 목록과 폼 모두 공통 껍데기를 쓴다 — **이 묶음에서 손수 조립한 곳이 가장 적은 자원이다** |
| 결정의 근거를 코드에 남긴다 | 왜 통일하지 않았는지 적어 둔다 | 두 치환 문법의 차이(치환 주체·시점·값의 출처)가 검증 파일 머리말에 표로 남아 있다 — `apps/admin/src/pages/support/replies/validation.ts:3-24` |

## 8. 제약사항

| 항목 | 내용 |
|---|---|
| 외부 시스템 연계 | **없다.** 이 자원은 상용구를 보관할 뿐이고 발송은 하지 않는다 |
| API 제약 | **백엔드가 없다.** ⚠ 서버 계약이 코드에 적혀 있지 않다(§6) |
| 치환 변수 | `{{고객명}}` · `{{문의번호}}` · `{{담당자}}` 셋뿐이다. **담당자가 비어 있으면 `담당자` 라는 낱말로 채운다** — `apps/admin/src/pages/support/_shared/domain.ts:411` |
| 치환 시점 | **삽입 시점**이다 — 저장된 본문에는 토큰이 남고, 티켓 작성칸에 들어갈 때 값이 박힌다 |
| 마케팅 문법 | `#{...}` 는 **저장이 거절된다** |
| 유형 태그 | 비우면 공용(`전체`)이다 — 모든 유형 티켓에 노출된다 |
| 문자 수 상한 | 제목 60자 · 본문 1,000자 — `apps/admin/src/pages/support/_shared/domain.ts:393-394` |
| 중복 방지 | **없다** — 같은 제목의 템플릿을 여럿 만들 수 있다 |
| 사용 이력 | **없다** — 어느 템플릿이 언제 몇 번 삽입됐는지 기록하지 않는다 |
| 파일 업로드 | 해당 없음 — 이 자원에 파일 입력이 없다 |
