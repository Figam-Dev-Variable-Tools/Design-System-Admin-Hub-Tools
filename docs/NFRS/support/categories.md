## 1. 성능

| 항목 | 요구 | 이 자원의 값 | 측정 방법 |
|---|---|---|---|
| 응답 시간 | 공통 준수 | **측정하지 않음** — 백엔드가 없다 | 백엔드 연동 후 재작성 |
| 조회 성능 | 공통 준수 | 요청 1건이다. **참조 건수(티켓 수·템플릿 수)를 목록과 함께 받는다** — 삭제 차단 판단과 배지가 같은 값을 쓰므로 추가 조회가 없다 — `apps/admin/src/pages/support/_shared/domain.ts:29-33` | 코드 확인 |
| 동시 사용자 | 공통 준수 | **측정하지 않음** | — |
| 대용량 데이터 처리 | 가상화 없이 대량 행을 그리지 않는다 | **페이지네이션도 필터도 검색도 없다** — 유형은 수가 적어 한 화면에 다 들어간다는 전제다. 프로그램 카테고리 화면이 사용 여부 필터를 두는 것과 다른 자리다 | 코드 확인 |

## 2. 사용성

| 항목 | 요구 | 이 자원의 값 |
|---|---|---|
| UI 일관성 | 공통 준수 | 팝업 껍데기(포커스 트랩·Esc·딤·복귀)와 이탈 가드는 공통 층이다. **목록은 트리가 아닌 단일 목록이라 공통 목록 껍데기를 쓰지 않고 직접 조립하며**, 그래서 공통 삭제 확인이 아니라 화면 고유 확인 다이얼로그를 갖는다 |
| 막힌 컨트롤은 이유를 말한다 | 비활성에 침묵을 두지 않는다 | 삭제가 막히면 **사유가 툴팁과 `aria-label` 에 함께 실린다** — `apps/admin/src/pages/support/categories/CategoriesPage.tsx:177-179` |
| 근거를 확인할 길 | 막은 이유를 열어 볼 수 있게 한다 | 참조 배지가 링크다. **티켓과 템플릿을 둘로 나눈다** — 사는 곳이 다르므로 합계 하나로는 어디로 가야 할지 알 수 없다. **0건인 쪽은 아예 그리지 않는다** — 열어 봐야 빈 목록이다 — `apps/admin/src/pages/support/categories/CategoriesPage.tsx:137-139` |
| 두 갈래 해법을 함께 말한다 | 막고 끝내지 않는다 | 하단 안내가 ① 참조를 옮기거나 ② **사용여부를 꺼서 신규 선택에서만 숨기는** 두 길을 모두 적는다 — `apps/admin/src/pages/support/categories/CategoriesPage.tsx:296-299` |
| 사용자 피드백 | 공통 준수 | 이 자원의 토스트는 **성공 2건**이다([인벤토리](../../FSD/inventory.md) §9.3 과 일치): 추가/저장 한 건, 삭제 한 건. **삭제 실패는 토스트가 아니라 다이얼로그 안 배너**다 |
| 상태 유지 | 재조회가 보고 있던 것을 지우지 않는다 | 재조회 중에는 이전 목록을 유지하고 건수 문구 뒤에 `· 새로고침 중…` 만 덧붙인다 — `apps/admin/src/pages/support/categories/CategoriesPage.tsx:257-258` |
| 입력 편의성 | 공통 준수 | 팝업이 열리면 이름 칸에 포커스가 간다. 사용여부는 **추가 시 기본이 켜짐**이다 — `apps/admin/src/pages/support/categories/components/CategoryFormModal.tsx:56` |

## 3. 보안

| 항목 | 요구 | 이 자원의 값 |
|---|---|---|
| 인증 | 공통 준수 | — |
| ⚠ 권한 | 공통 준수 | 리소스 키는 `support-categories`. **이 화면에는 쓰기 권한 판정이 하나도 없다** — `useRouteWritePermissions` 호출이 없어 `유형 추가`·연필·휴지통이 읽기 권한만으로도 보이고 눌린다. 형제 화면(프로그램 카테고리)이 최소한 `create` 는 가리는 것과도 다르다. 막는 것은 서버뿐이다 |
| ⚠ 플랜(엔타이틀먼트) | 공통 준수(fail-open) | **모듈 표에 `/support/**` 가 없다** — `apps/admin/src/shared/entitlements/module-resources.ts`. fail-open 이라 잠기지 않는다 |
| 개인정보 마스킹 | 필드 이름으로 적는다 | **이 자원에 개인 식별 정보가 없다** — 이름과 사용여부뿐이다 |
| 입력값 검증 | 공통 준수(스키마 하나가 정본) | `apps/admin/src/pages/support/categories/validation.ts`. 이름 필수·30자 상한 · 사용여부는 참/거짓이라 검증 대상이 아니다 |
| 중복 제출 | 공통 준수 | 팝업의 저장 버튼이 저장 중 잠긴다 |
| HTTPS | 공통 준수 | **측정하지 않음** |

## 4. 안정성

| 항목 | 요구 | 이 자원의 값 |
|---|---|---|
| 예외 처리 | 공통 준수 | 조회 실패는 카드 대신 배너 + `다시 시도`. **삭제 실패는 다이얼로그를 닫지 않고** 그 안 배너로 말한다 |
| 참조 차단의 이중화 | 화면과 저장소가 같은 사실을 본다 | 휴지통의 비활성 조건과 저장소의 거절 조건이 **같은 사실**(티켓 수 + 템플릿 수 > 0)을 본다 — `apps/admin/src/pages/support/_shared/domain.ts:40-43` · `apps/admin/src/pages/support/_shared/store.ts:265-268` |
| 소프트 비활성 | 참조를 끊지 않고 신규 선택만 막는다 | 사용여부를 꺼도 **기존 티켓·템플릿의 참조는 유지된다** — `apps/admin/src/pages/support/_shared/domain.ts:25-26` |
| ⚠ 이름 변경의 파급 | 비정규화 라벨이 낡지 않아야 한다 | **확인하지 못했다** — 티켓은 조회 시점 유형 라벨을 비정규화로 들고 있는데(`apps/admin/src/pages/support/_shared/domain.ts:82-83`) 유형 이름을 고쳤을 때 그 라벨이 함께 갱신되는지 코드에서 확인하지 못했다. 프로그램 카테고리는 그 갱신을 저장소가 명시적으로 한다 |
| 재시도 | 공통 준수(자동 재시도 없음) | 조회 배너의 `다시 시도` · 다이얼로그 확인 재클릭 · 팝업 저장 재클릭 |
| 동시 저장 | 공통 준수(last-write-wins) | **전용 충돌 다이얼로그가 없다** — 다른 관리자가 먼저 지웠거나 고쳤어도 화면은 일반 실패 문구를 쓴다 |
| 로그 기록 | 공통 준수 | **확인하지 못했다** |

## 6. 운영성

| 항목 | 요구 | 이 자원의 값 |
|---|---|---|
| 로그 관리 | 공통 준수 | **확인하지 못했다** |
| 모니터링 | 공통 준수 | **없다** |
| 배포 영향 | 이 자원을 고치면 어느 화면이 함께 영향을 받는가 | [1:1 문의](tickets.md)의 유형 필터와 [문의 답변](replies.md)의 유형 태그 셀렉트가 이 자원을 읽는다. **템플릿의 후보를 거르는 축**이기도 해서, 유형을 지우면 그 유형에 태그된 템플릿이 어느 티켓에도 뜨지 않게 된다 |
| 설정 관리 | 이 자원의 동작을 바꾸는 설정값이 어디 있는가 | 없다 |
| 백엔드 없음의 이음매 | 서버가 맡을 일이 코드에 표시되어 있다 | 이 자원의 어댑터에는 `TODO(backend)` 주석이 없다 — **확인하지 못했다.** 저장소의 삭제 차단은 `사용 중인 문의 유형은 삭제할 수 없어요.` 로 강제되므로 서버도 같은 규칙을 걸어야 한다 — `apps/admin/src/pages/support/_shared/store.ts:268` |

## 7. 유지보수성

| 항목 | 요구 | 이 자원의 값 |
|---|---|---|
| 코드 확장성 | 공통 준수 | 참조 건수와 그 표기(`사용 안 함` / `티켓 {N} · 템플릿 {N}`)가 도메인 모듈의 순수 함수 둘이다 — `apps/admin/src/pages/support/_shared/domain.ts:40-50` |
| 세 화면이 도메인을 공유한다 | 서로를 import 하지 않는다 | 티켓·문의 유형·답변 템플릿이 한 잎 모듈의 타입·순수 규칙을 함께 쓴다 — `apps/admin/src/pages/support/_shared/domain.ts:3-7` |
| 화면 확장성 | 열·필터·상태값이 늘 때 견디는가 | ⚠ **필터·검색 축이 없어** 유형이 늘면 목록이 그대로 길어진다. 프로그램 카테고리는 사용 여부 필터를 갖는다 |
| 공통 컴포넌트 사용 여부 | 페이지가 UI 를 손수 조립하지 않는다 | 목록·행·배지 배치를 직접 조립한다. 팝업 껍데기와 이탈 가드만 공통 층이다 |
| 계층 없음 | 프로그램 카테고리와 다른 자리 | **이 자원에는 상위/하위 개념이 없다** — 평평한 목록이다. 그래서 3단계 금지·자기 참조 금지 같은 구조 규칙도 없다 |

## 8. 제약사항

| 항목 | 내용 |
|---|---|
| 외부 시스템 연계 | 없음 |
| API 제약 | **백엔드가 없다.** ⚠ 서버 계약이 코드에 적혀 있지 않다(§6) |
| 삭제 차단 | 티켓 또는 템플릿이 1건이라도 참조하면 지울 수 없다 |
| 소프트 비활성 | **사용여부**가 삭제를 대신하는 축이다 — 꺼도 기존 참조는 유지되고 신규 선택에서만 숨는다 |
| 문자 수 상한 | 유형 이름 30자 — `apps/admin/src/pages/support/_shared/domain.ts:395` |
| 중복 방지 | **없다** — 같은 이름의 유형을 여럿 만들 수 있다 |
| 정렬 | 저장된 순서 그대로다 — 표시 순서를 바꾸는 축이 없다 |
| 템플릿 링크의 한계 | `템플릿 {N}` 배지가 여는 목록에는 **유형 필터가 걸리지 않는다** — 답변 템플릿 목록에 그 축이 없기 때문이다 |
| 파일 업로드 | 해당 없음 — 이 자원에 파일 입력이 없다 |
