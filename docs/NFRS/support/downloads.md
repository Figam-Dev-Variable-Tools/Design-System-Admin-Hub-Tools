## 1. 성능

| 항목 | 요구 | 이 자원의 값 | 측정 방법 |
|---|---|---|---|
| 응답 시간 | 공통 준수 | **측정하지 않음** — 백엔드가 없다 | 백엔드 연동 후 재작성 |
| 조회 성능 | 공통 준수(신선도 30초 · 자동 재시도 없음) | 목록·폼 각각 요청 1건. **카테고리는 코드 상수라 조회가 없다** — `apps/admin/src/pages/support/downloads/types.ts:51-56` | 코드 확인 |
| 동시 사용자 | 공통 준수 | **측정하지 않음** | — |
| 대용량 데이터 처리 | 가상화 없이 대량 행을 그리지 않는다 | ⚠ **페이지네이션도 가상화도 없다** — 받은 전량을 한 번에 그린다 | 코드 확인 |
| 파일 전송 | 업로드가 화면을 멈추지 않는다 | **전송 자체가 없다** — 파일은 서버로 가지 않고 폼이 파일명과 바이트 수만 보관한다(§8) | 코드 확인 |
| 파생 표기 | 저장된 파생값을 두지 않는다 | 파일 종류는 확장자에서, 용량 표기는 바이트에서 매번 계산한다 — `apps/admin/src/pages/support/downloads/types.ts:76-105` | 코드 확인 |

## 2. 사용성

| 항목 | 요구 | 이 자원의 값 |
|---|---|---|
| UI 일관성 | 공통 준수 | 목록은 공통 목록 껍데기를 쓴다. **폼은 다중 카드 + 미리보기라 공통 폼 껍데기 대신 직접 배치한다** |
| 접근성 | 공통 준수 | 드롭 영역이 **`<button>`** 이라 키보드로도 파일 선택창을 열 수 있고, `{라벨} 업로드 — 클릭하거나 파일을 끌어다 놓으세요` 라벨을 갖는다 — `apps/admin/src/pages/support/downloads/components/FileUploadField.tsx:231`. 파일 오류는 `role="alert"` 로 즉시 읽힌다 — `apps/admin/src/pages/support/downloads/components/FileUploadField.tsx:261` |
| 입력 편의성 | 공통 준수 | 클릭과 **드래그 앤 드롭** 둘 다 받는다. 끌고 있는 동안 문구가 `여기에 놓으면 첨부돼요` 로 바뀐다 — `apps/admin/src/pages/support/downloads/components/FileUploadField.tsx:242` |
| 제약을 미리 말한다 | 거절을 사후에 알리지 않는다 | 드롭 영역 안에 `문서 · 이미지 · 압축 · 최대 20MB` 가 상시 표시된다 — `apps/admin/src/pages/support/downloads/components/FileUploadField.tsx:245` |
| 즉시 검증 | 잘못된 파일을 폼에 담지 않는다 | 종류·용량 위반은 **선택·드롭 즉시** 거절하고 그 자리에서 이유를 말한다 — `apps/admin/src/pages/support/downloads/components/FileUploadField.tsx:150-151` |
| 사용자 피드백 | 공통 준수 | 이 자원에는 **자기 토스트가 없다**([인벤토리](../../FSD/inventory.md) §9.3 에 두 화면 모두 없다). 노출 토글·삭제 통지는 공통 층(`MSG-COMMON-02`~`05`)이 내고 **문구만 이 화면이 준다** |
| 권한 있는 읽기 표시 | 값을 계속 볼 수 있게 한다 | 수정 권한이 없으면 노출 토글이 사라지지 않고 **읽기 표시로 바뀐다** — `apps/admin/src/pages/support/downloads/DownloadListPage.tsx:159` |

## 3. 보안

| 항목 | 요구 | 이 자원의 값 |
|---|---|---|
| 인증 | 공통 준수 | — |
| 권한 | 공통 준수 | 리소스 키는 `support-downloads`. 등록 버튼은 `create`, 행 액션·선택은 공통 껍데기가 권한으로 가리고, 노출 토글은 `update` 로 읽기 표시와 갈린다 |
| ⚠ 플랜(엔타이틀먼트) | 공통 준수(fail-open) | **모듈 표에 `/support/**` 가 없다** — `apps/admin/src/shared/entitlements/module-resources.ts`. fail-open 이라 잠기지 않는다 |
| 개인정보 마스킹 | 필드 이름으로 적는다 | **이 자원에 개인 식별 정보가 없다** — 배포용 파일의 메타데이터뿐이다. ⚠ 다만 **올린 파일의 내용은 이 화면이 검사하지 않는다**(문서 안에 무엇이 들었는지 모른다) |
| 입력값 검증 | 공통 준수(스키마 하나가 정본) | `apps/admin/src/pages/support/downloads/validation.ts`. **파일 규칙만 순수 함수가 따로 갖는다** — 업로드 필드가 선택 즉시 부르기 때문이다 — `apps/admin/src/pages/support/downloads/types.ts:113-121` |
| 허용 종류를 좁게 유지 | 아무 파일이나 받지 않는다 | 문서·이미지·압축 확장자 목록에 없으면 **'기타'로 판정되어 거절된다** — `apps/admin/src/pages/support/downloads/types.ts:60-86` |
| ⚠ 확장자 기반 판정의 한계 | 판정 근거를 밝힌다 | 종류를 **파일명 확장자로만** 판정한다(내용 검사가 없다) — 확장자를 바꾼 실행 파일을 이 규칙만으로 걸러 낼 수 없다. 서버가 붙으면 그쪽에서 다시 판정해야 한다 |
| 중복 제출 | 공통 준수 | 폼은 동기 잠금 + 제출 시도 단위 멱등키 |
| HTTPS | 공통 준수 | **측정하지 않음** |

## 4. 안정성

| 항목 | 요구 | 이 자원의 값 |
|---|---|---|
| 예외 처리 | 공통 준수 | 폼의 조회 실패를 **없음과 오류로 가른다** — `apps/admin/src/pages/support/downloads/DownloadFormPage.tsx:163-179` |
| 이미지 규칙과 분리 | 비슷하다고 합치지 않는다 | 공통 이미지 업로드는 이미지만 받고 5MB 상한이다. 자료실은 문서까지 허용하므로 **전용 규칙을 이 자원에 둔다** — `apps/admin/src/pages/support/downloads/types.ts:109-110` |
| 재시도 | 공통 준수(자동 재시도 없음) | 목록 배너의 `다시 시도` · 폼 저장 재클릭 · 토글 재클릭 |
| 동시 저장 | 공통 준수(last-write-wins + 충돌 다이얼로그) | 폼은 공통 충돌 다이얼로그를 쓴다 |
| ⚠ 파일의 소실 | 저장했는데 실제로는 없는 상태를 만들지 않는다 | **업로드 엔드포인트가 없어 파일이 어디에도 저장되지 않는다.** 목록·미리보기는 파일명과 용량을 그리지만 그 파일을 다시 받을 길이 없다. 백엔드 연동 시 이 이음매를 함께 만들어야 한다 |
| 삭제의 파급 | 지운 뒤 남는 것을 밝힌다 | 자료를 지우면 **다운로드수 기록도 함께 사라진다.** ⚠ 다운로드수가 남아 있어도 삭제를 막지 않는다 |
| 로그 기록 | 공통 준수 | **확인하지 못했다** |

## 5. 호환성

| 항목 | 요구 | 이 자원의 값 |
|---|---|---|
| 지원 브라우저 | 공통 준수 | **리포에 정의되어 있지 않다** |
| 모바일 지원 | 공통 준수 | **측정하지 않음** — ⚠ 드래그 앤 드롭은 터치 환경에서 성립하지 않지만 **드롭 영역이 클릭으로도 열려** 파일 선택은 가능하다 |
| 해상도 | 넓은 표는 최소 폭과 가로 스크롤 방식을 갖는다 | 목록은 6열이다. 파일 열이 `{파일명} · {종류} · {용량}` 한 줄이라 파일명이 길면 그 칸이 넓어진다 |

## 6. 운영성

| 항목 | 요구 | 이 자원의 값 |
|---|---|---|
| 로그 관리 | 공통 준수 | **확인하지 못했다** |
| 모니터링 | 공통 준수 | **없다** — 다운로드수는 자료마다 보이지만 집계·추이를 보는 화면이 없다 |
| 배포 영향 | 이 자원을 고치면 어느 화면이 함께 영향을 받는가 | **다른 화면과 이어지지 않는다** — 고객센터 세 자원(티켓·유형·템플릿)이 도메인을 공유하는 것과 달리 자료실은 자기 타입·자기 저장소를 갖는다 |
| 설정 관리 | 이 자원의 동작을 바꾸는 설정값이 어디 있는가 | 없다. **카테고리도 코드 상수**라 운영자가 늘릴 수 없다 |
| 용량 상한 | 코드 상수다 | **20MB** — `apps/admin/src/pages/support/downloads/types.ts:43`. 공통 이미지 업로드(5MB)와 다른 값이다 |
| 다운로드수 | 관리자가 손대지 않는다 | 자동 증가 값이라 목록·폼 모두 읽기만 한다 — `apps/admin/src/pages/support/downloads/types.ts:24-25`. ⚠ **어디서 증가하는지는 이 리포에 없다**(고객 화면의 몫이다) |
| 백엔드 없음의 이음매 | 서버가 맡을 일이 코드에 표시되어 있다 | 이 자원의 어댑터에는 `TODO(backend)` 주석이 없다 — **확인하지 못했다.** 특히 **업로드 엔드포인트**가 정의되어 있지 않다 |

## 7. 유지보수성

| 항목 | 요구 | 이 자원의 값 |
|---|---|---|
| 코드 확장성 | 공통 준수 | 확장자 목록 셋(문서·이미지·압축)과 카테고리가 데이터 배열이라 늘리기 쉽다 |
| 순수 규칙의 분리 | 컴포넌트가 규칙을 갖지 않는다 | 종류 판정·용량 표기·파일 검증이 전부 순수 함수이고 **테스트가 그것을 직접 부른다** — `apps/admin/src/pages/support/downloads/types.ts:109-112` |
| 화면 확장성 | 열·필터·상태값이 늘 때 견디는가 | 두 필터와 검색어를 주소 쿼리스트링이 소유한다 |
| 공통 컴포넌트 사용 여부 | 페이지가 UI 를 손수 조립하지 않는다 | 목록은 껍데기를 쓴다. **업로드 필드는 이 자원 전용 컴포넌트다**(드롭존·미리보기·전용 검증) — 소비자가 이 폼 하나라 화면 폴더 안에 둔다 |
| 카테고리를 고정한 근거 | 운영자가 만드는 값과 구분한다 | 자료 카테고리는 **자료실 고유의 고정 분류**이고 문의 유형과 별개다 — `apps/admin/src/pages/support/downloads/types.ts:50`. 운영자가 만들 수 없다는 것이 이 자원의 결정이다 |

## 8. 제약사항

| 항목 | 내용 |
|---|---|
| 외부 시스템 연계 | 고객이 내려받는 자료실 페이지는 **이 리포 밖**이다 |
| API 제약 | **백엔드가 없다.** ⚠ 서버 계약이 코드에 적혀 있지 않다(§6) |
| ⚠ 파일 업로드 | **업로드 엔드포인트가 없다.** 폼이 보관하는 것은 **파일명과 바이트 수**뿐이고 파일 자체는 어디에도 저장되지 않는다 |
| 파일 용량 제한 | **20MB** — `apps/admin/src/pages/support/downloads/types.ts:43` |
| 업로드 가능 확장자 | 문서 `pdf`·`doc`·`docx`·`xls`·`xlsx`·`ppt`·`pptx`·`hwp`·`hwpx`·`txt`·`csv` · 이미지 `png`·`jpg`·`jpeg`·`gif`·`webp`·`svg` · 압축 `zip`·`rar`·`7z`·`tar`·`gz` — `apps/admin/src/pages/support/downloads/types.ts:60-74`. **확장자가 없거나 목록에 없으면 거절된다** |
| 종류 판정 | **확장자로만** 한다(내용 검사가 없다) |
| 카테고리 | 고정 5분류(카탈로그 · 제품 매뉴얼 · 양식/서식 · 브로슈어 · 기타) — **운영자가 늘릴 수 없다** |
| 문자 수 상한 | 제목 100자 · 버전 20자 — `apps/admin/src/pages/support/downloads/types.ts:40-41` |
| 버전 | 자유 문자열이다 — 형식(`v1.0` 등)을 강제하지 않고 비워 둘 수 있다 |
| 다운로드수 | 이 화면에서 초기화하거나 고칠 수 없다 |
| 관리자 다운로드 | **없다** — 등록한 파일을 관리자 화면에서 다시 받는 컨트롤이 없다 |
| 중복 방지 | **없다** — 같은 제목·같은 파일명을 여럿 만들 수 있다 |
