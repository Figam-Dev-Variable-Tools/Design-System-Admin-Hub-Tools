| 항목 | 요구 | 이 자원의 값 |

| 두 축의 분리 | 같은 사실을 두 곳이 소유하지 않는다 | 질문·답변·카테고리·콘텐츠 노출은 콘텐츠 관리가, **표시 순서·고객센터 노출·BEST 고정은 이 화면**이 갖는다. 질문을 여기서 만들지 않으므로 어긋날 자리가 없다 — `apps/admin/src/pages/support/faq/types.ts:8-13` |

## 8. 제약사항

| 항목 | 내용 |
|---|---|
| 외부 시스템 연계 | 고객이 보는 FAQ 페이지는 **이 리포 밖**이다. 이 자원이 정하는 것은 거기 실릴 순서·노출·강조뿐이다 |
| API 제약 | **백엔드가 없다.** 서버 계약은 §6 의 두 `TODO(backend)` 가 갖는다 |
| 작성·삭제 | **없다.** 질문·답변·카테고리는 콘텐츠 관리 FAQ 소관이다 |
| 원본 의존 | **원본 조회기가 배선되지 않으면 이 화면은 오류 화면이 된다**(빈 목록이 아니다) |
| BEST 고정 | 개수 상한이 **없다** — 전부 고정해도 막지 않는다 |
| 노출 축의 관계 | 콘텐츠 관리의 노출과 **별개 축**이다 — 콘텐츠에서 노출 중인데 고객센터에서만 내려 둔 상태가 정상이다 |
| 검색·필터 | **없다** — 순서가 의미인 표라 자르거나 거르지 않는다 |
| 동시 편집 | 보호 장치가 **없다**(last-write-wins) |
| 파일 업로드 | 해당 없음 — 이 자원에 파일 입력이 없다 |
