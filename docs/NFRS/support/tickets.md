| 입력값 검증 | 공통 준수 | **이 자원에는 zod 스키마가 없다** — 저장 대상이 세 값뿐이라 폼 라이브러리를 얹지 않았다. 규칙은 순수 함수 둘(전이 판정·담당 요건)이 갖는다 — `apps/admin/src/pages/support/tickets/process.ts:28-33` |
| 중복 제출 | 금액·생성·발송에서 두 번째 요청을 만들지 않는다 | 저장 중에는 입력·버튼이 잠긴다. 종결은 확인 다이얼로그를 한 번 더 거친다 |
| HTTPS | 공통 준수 | **측정하지 않음** |

## 4. 안정성

| 항목 | 요구 | 이 자원의 값 |
|---|---|---|
| 예외 처리 | 공통 준수 | ⚠ **상세의 조회 실패가 '없음'과 '오류'를 가르지 않는다** — 둘 다 같은 문구로 말하고 재시도도 주지 않는다. 형제 화면(견적·청구·프로그램 문의 상세)은 전부 가른다 — `apps/admin/src/pages/support/tickets/TicketDetailPage.tsx:177-188` |
| 원자성 | 두 사실이 갈라지지 않는다 | 담당·상태·타임라인이 **한 요청**으로 커밋된다 — '담당은 바뀌었는데 상태는 그대로'인 순간을 만들지 않는다 |
| 불가능한 전이 차단 | 책임 이관 흐름을 지킨다 | 전이표가 갈 수 있는 칸을 정하고 **셀렉트가 그 표를 그대로 그린다.** 그래서 '고를 수 있는데 저장이 거절되는' 경로가 사실상 없다 — `apps/admin/src/pages/support/_shared/domain.ts:265-271` |
| 담당 요건 | 사람 없는 처리 상태를 만들지 않는다 | **처리중·답변완료는 담당 배정 없이 갈 수 없다** — 그 위반은 저장 버튼을 미리 잠그고 담당 칸에 인라인으로 말한다 — `apps/admin/src/pages/support/_shared/domain.ts:282-291` |
| 종결에 확인을 세운다 | 되돌릴 수 없는 동작은 확인을 거친다 | 종결은 **종착**이라(그 상태에서 갈 곳이 없다) 확인 다이얼로그를 세운다. **되돌릴 수 있는 저장(담당 배정·답변 등록·처리중 전환)에는 붙이지 않는다** — 전부 붙이면 확인이 무뎌진다 — `apps/admin/src/pages/support/tickets/TicketDetailPage.tsx:129-132` |
| ⚠ 동시 저장 | 공통 준수(last-write-wins) | **충돌 다이얼로그가 없다.** 이 저장은 타임라인을 포함한 문서 한 벌을 통째로 보내므로, 내가 화면을 연 뒤 다른 상담원이 남긴 이력이 사라질 수 있다. 청구 자원은 같은 위험에 원장 소실 가드를 세웠지만 이 자원에는 없다 |
| 취소는 실패가 아니다 | 공통 준수 | 화면을 떠나 요청이 중단되면 성공·실패 어느 표시도 하지 않는다 |
| 재시도 | 공통 준수(자동 재시도 없음) | 저장 실패는 배너로 남고 버튼 재클릭이 재시도다. **종결 확인 중 실패하면 다이얼로그가 닫히지 않고** 확인 재클릭이 재시도가 된다 |
| 로그 기록 | 공통 준수 | **확인하지 못했다** — 다만 **티켓 자신의 타임라인**에 배정·상태변경·답변·메모가 남는다(append-only) |

## 8. 제약사항

| 항목 | 내용 |
|---|---|
| 외부 시스템 연계 | 티켓을 만드는 것은 **고객 채널**(웹·카카오톡·네이버톡톡·전화·이메일)이고 이 리포 밖이다. 관리자 화면에는 생성 경로가 없다 |
| API 제약 | **백엔드가 없다.** ⚠ 서버 계약이 코드에 적혀 있지 않다(§6) |
| 생성·삭제 | **둘 다 없다.** 관리자는 처리·답변만 한다 |
| 상태 전이 | 접수 → 배정·종결 / 배정 → 처리중·종결 / 처리중 → 답변완료·종결 / **답변완료 → 종결·처리중**(되돌아갈 수 있는 유일한 칸) / 종결 → **없음** — `apps/admin/src/pages/support/_shared/domain.ts:265-271` |
| 담당 요건 | 처리중·답변완료는 담당 배정이 필수다 |
| SLA | **첫 응답만 모델링한다** — 해결 시간·재응답 SLA 축은 없다 — `apps/admin/src/pages/support/_shared/domain.ts:174-175` |
| SLA 목표시간 | 코드 상수다(화면에서 바꿀 수 없다) |
| 문자 수 상한 | 답변·메모 1,000자 — `apps/admin/src/pages/support/_shared/domain.ts:100` |
| 타임라인 | append-only. 수정·삭제 경로가 없고 **상한도 없다** |
| 답변 회신 | **앱이 메시지를 보내지 않는다** — 작성한 답변은 이력에 남고 실제 회신은 앱 밖에서 이뤄진다 |
| 첨부 | **없다** — 티켓 모델에 첨부 필드가 없다(고객이 보낸 파일을 다루지 않는다) |
| 파일 업로드 | 해당 없음 — 이 자원에 파일 입력이 없다 |
