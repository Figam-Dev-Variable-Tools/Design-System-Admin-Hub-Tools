# 카테고리 (상품) — 비기능 명세서

| 항목 | 내용 |
|---|---|
| 자원명 | 상품 카테고리 |
| 화면 ID | `SCR-PRODUCTS-CATEGORIES` · `POP-PRODUCTS-CATEGORIES-CATEGORY-FORM` |
| 메뉴 경로 | 상품 관리 > 카테고리 |
| 작성자 | Admin Hub 문서화 |
| 작성일 | 2026-07-22 |
| 버전 | 1.0 |
| 관련 기능 명세서 | [목록](../../FSD/products/categories/index.md) · [추가·수정 팝업](../../FSD/products/categories/pop-category-form.md) |

**이 자원의 성격** — 다른 자원이 **참조하는** 값이다(상품·쿠폰 대상·쿠폰 연쇄 선택). 그래서 비기능 요구가 **삭제 차단**과 **계층 무결성**(최대 2단계)에 몰린다. 아래 각 절은 [공통 기준](../_common.md) 대비 이 자원만의 차이를 적는다.

## 1. 성능

| 항목 | 요구 | 이 자원의 값 | 측정 방법 |
|---|---|---|---|
| 응답 시간 | 공통 준수 | 측정하지 않음 — 백엔드가 없다 | 백엔드 연동 후 재작성 |
| 조회 성능 | 공통 준수 | 조회 1건에 **사용 중 상품 수와 하위 보유 여부가 함께** 온다 — 삭제 차단 판정을 위해 화면이 상품 목록을 따로 부르지 않는다 | 코드 확인 |
| 대용량 데이터 처리 | 가상화 없이 1,000행을 그리지 않는다 | ⚠ **페이지네이션도 가상화도 없다.** 다만 이 자원은 본질적으로 작다(픽스처 11개: 대분류 5 + 중분류 6) | 코드 확인 |
| 트리 구성 비용 | — | 필터 결과에서 트리를 매 렌더 다시 만든다(대분류 × 중분류). 카테고리 수에 비례하며 메모이제이션돼 있다 | 코드 확인 |
| 재조회 중 표시 | 공통 준수 | **건수 줄을 지우지 않는다** — 재조회 중에도 이전 사실을 유지한다 | `ProductCategoriesPage.tsx:408-411` |
| 건수 집계 | — | 좌측 3갈래 건수는 조회된 전체 집합에서 매 렌더 다시 센다. 못 셌으면 `null` 이고 화면은 `—` 를 그린다 | 코드 확인 |
| 동시 사용자 | 공통 준수 | 측정하지 않음 | — |

## 2. 사용성

| 항목 | 요구 | 이 자원의 값 |
|---|---|---|
| UI 일관성 | 공통 준수 | ⚠ **표가 아니라 손수 조립한 목록(`<ul>`/`<li>`)이다.** 2단 트리와 행마다 다른 액션 구성(대분류만 `+`)을 공통 표 컴포넌트로 표현하지 못한다. 모달·확인 다이얼로그·배지·빈 상태는 디자인 시스템 것이다 |
| 필터 축의 선택 | — | 축이 '사용 여부' 인 이유는 이 화면의 핵심 제약이 '사용 중이면 삭제할 수 없다' 이기 때문이다 — 정리하려는 운영자가 가장 먼저 하려는 일이 **지울 수 있는 것만 보기** 다 — `types.ts:23-28` |
| 계층이 곧 정보다 | — | 필터에 걸린 중분류의 **부모는 화면에 남긴다** — 부모가 사라지면 자식이 어디 소속인지 알 수 없다 |
| 기본은 전부 펼침 | — | 숨겨진 정보 없이 시작한다. 접기는 사용자의 선택이다 |
| 막힌 버튼은 이유를 말한다 | — | 삭제가 잠기면 그 사유가 **버튼 이름과 툴팁**에 함께 실린다(`{이름} — {N}개 상품이라 삭제할 수 없어요`) |
| 다음 행동을 준다 | — | 사용 건수 배지가 **곧 상품 목록 링크**다(그 카테고리로 이미 걸러진 목록이 열린다). 미사용(0건)은 링크로 만들지 않는다 — 열어 봐야 빈 목록이다 |
| 규칙을 선택지에서 지운다 | — | 3단계를 만들 수 없게 **상위 선택지에 1Depth 만** 둔다. 하위를 가진 카테고리를 수정할 때는 상위 셀렉트 자체를 잠근다 |
| 값이 없는 것과 모르는 것 | 공통 규약 5 준수 | 좌측 건수는 못 셌으면 `—` |
| 결과 통지 | 공통 준수 | 추가·저장·삭제 3문구 |
| 접근성 — 펼침 토글 | — | 토글에 `aria-expanded`·`aria-controls` 와 개수를 포함한 이름이 붙는다. 토글이 없는 행도 **라벨 시작점을 맞춘다**(들쭉날쭉하지 않게) |
| 접근성 — 검증 실패 | 공통 준수 | 모달의 검증 실패는 이름 칸으로 포커스를 옮긴다 |
| 미저장 보호 | 공통 준수 | 모달이 네 경로(Esc·딤·×·취소)를 한 요청으로 모아 확인을 세운다 |

## 3. 보안

| 항목 | 요구 | 이 자원의 값 |
|---|---|---|
| 인증 | 공통 준수 | 셸 안쪽 |
| 권한 | 공통 준수 | 리소스 키는 사이드바 잎 `/products/categories`. 화면이 읽는 것은 `create` 와 `remove` 둘이다 |
| 권한 실패 방향 | 공통 준수 | `create` 가 없으면 추가·하위 추가 버튼이 **사라진다** |
| ⚠ 수정 게이팅 없음 | — | **연필 버튼에는 권한 게이팅이 없다** — 조회할 수 있으면 누구에게나 보이고 모달이 열린다. 저장을 막는 층은 서버뿐이며 지금은 백엔드가 없어 실제로 막히지 않는다 |
| 개인정보 마스킹 | 공통 준수 | 해당 없음 — 이 자원에는 개인정보가 없다 |
| 입력값 검증 | 공통 준수 | 이름의 형식은 zod 스키마가, **계층 규칙은 저장소**가 갖는다 — 계층은 다른 카테고리들의 상태를 봐야 하는 판단이라 폼 값만 보는 스키마로는 표현되지 않는다 |
| 중복 제출 | 공통 준수 | 모달의 제출 버튼이 진행 중 잠긴다 |
| HTTPS | 공통 준수 | 측정하지 않음 |
