# 적립금 (정책) — 비기능 명세서

| 항목 | 내용 |
|---|---|
| 자원명 | 적립금 정책 |
| 화면 ID | `SCR-PRODUCTS-POINTS` |
| 메뉴 경로 | 상품 관리 > 적립금 |
| 작성자 | Admin Hub 문서화 |
| 작성일 | 2026-07-22 |
| 버전 | 1.0 |
| 관련 기능 명세서 | [적립금](../../FSD/products/points/index.md) |

**이 자원의 성격** — **문서 1건짜리 정책 화면**이다. 목록도 상세도 없고 조회 1 · 저장 1 이 전부다. 특이한 것은 하나 — **결제가 없으면 입력이 잠긴다**(같은 상품 섹션의 배송 정책 화면은 잠기지 않는다). 아래 각 절은 [공통 기준](../_common.md) 대비 이 자원만의 차이를 적는다.

## 1. 성능

| 항목 | 요구 | 이 자원의 값 | 측정 방법 |
|---|---|---|---|
| 응답 시간 | 공통 준수 | 측정하지 않음 — 백엔드가 없다 | 백엔드 연동 후 재작성 |
| 조회 성능 | 공통 준수 | 조회 1건(문서). 저장 1건. 그 밖의 요청이 없다 | 코드 확인 |
| 대용량 데이터 처리 | 공통 준수 | 해당 없음 — 목록도 페이지네이션도 없다. 필드가 7개인 폼 하나다 | — |
| 재조회 중 표시 | 공통 준수 | 단일 문서 껍데기의 규약을 따른다 | 코드 확인 |
| 동시 사용자 | 공통 준수 | 측정하지 않음 | — |

## 2. 사용성

| 항목 | 요구 | 이 자원의 값 |
|---|---|---|
| UI 일관성 | 공통 준수 | 단일 문서 껍데기 + 디자인 시스템 필드뿐이다. **손수 조립한 UI 가 없다** |
| 경계를 한 줄로 말한다 | — | 화면 안내가 이 화면과 상품 폼의 경계를 먼저 말한다: `적립률은 상품별로 설정하며, 여기서는 새 상품의 기본 적립률과 적립금 사용·소멸 규칙을 정해요.` |
| 기본값의 의미를 필드가 말한다 | — | '기본 적립률' 힌트가 `새 상품의 초기 적립률이에요. 상품별 적립 설정이 이 값을 덮어써요.` 라고 밝힌다 — 이 값을 바꿔도 **이미 등록된 상품은 움직이지 않는다** |
| 정책은 열되 입력은 잠근다 | — | 결제가 없어도 **조회는 되고 값이 화면에 보인다**. 잠기는 것은 입력뿐이며, 입력이 잠기면 폼이 dirty 가 되지 않아 저장 버튼도 함께 잠긴다 |
| 잠금 안내의 모양 | 공통 준수 | 잠금 배너는 다른 화면과 **한 벌의 같은 모습**이다(사유 + 결제 설정 링크) |
| 문의 링크를 붙이지 않는다 | — | 문의 링크는 '결제가 없는 동안 고객 요청이 실제로 쌓이는 곳' 을 가리키라고 있는 것이다. **이 화면은 정책 문서 한 건이라 대응하는 문의 대기열이 없다** — 상품 목록·쿠폰 목록처럼 '지금 무엇을 봐야 하는가' 의 답이 문의함인 자리가 아니다 — `PointsPolicyPage.tsx:137-140` |
| 결과 통지 | 공통 준수 | 저장 성공 1문구 |
| 접근성 — 검증 실패 | 공통 준수 | 첫 위반 필드로 포커스가 옮겨 간다 |
| 미저장 보호 | 공통 준수 | 이탈 가드 1건 |

## 3. 보안

| 항목 | 요구 | 이 자원의 값 |
|---|---|---|
| 인증 | 공통 준수 | 셸 안쪽 |
| 권한 | 공통 준수 | 리소스 키는 사이드바 잎 `/products/points`. 의미 있는 액션은 `read` 와 `update` 뿐 — 문서 1건이라 만들고 지울 것이 없다 |
| ⚠ 조회 전용 역할이 들어올 수 없다 | — | 단일 문서 껍데기가 **`update` 권한이 없으면 화면 전체를 접근 거부**로 바꾼다. 적립금 정책을 '보기만' 하는 경로가 없다(배송 정책 화면과 같은 한계다) |
| 개인정보 마스킹 | 공통 준수 | 해당 없음 — 이 자원에는 개인정보가 없다 |
| 입력값 검증 | 공통 준수 | 규칙의 정본은 zod 스키마 하나. 일곱 필드가 세 종류의 생성기(정수 · 백분율 · 1 이상 정수)로 만들어져 문구가 기계적으로 일관된다 |
| 잠금과 검증의 관계 | — | 상품 폼과 달리 **스키마가 결제 설정을 읽지 않는다** — 여기서는 화면이 입력을 잠그는 것으로 충분하다(잠긴 값이 검증을 막는 상황이 생기지 않는다) |
| 중복 제출 | 공통 준수 | 저장 중에는 입력과 버튼이 잠긴다 |
| HTTPS | 공통 준수 | 측정하지 않음 |

## 8. 제약사항

| 항목 | 내용 |
|---|---|
| 외부 시스템 연계 | 없다 |
| API 제약 | **백엔드가 없다.** 연동 지점은 `apps/admin/src/pages/products/points/data-source.ts` 의 `TODO(backend)` 다 |
| 조회 전용 접근 | ⚠ **없다** — `update` 권한이 없으면 화면 전체가 접근 거부다(§3) |
| 결제 잠금 | 결제가 없으면 **입력과 저장이 잠긴다**(조회는 열린다) |
| 값의 범위 | 적립률·1회 사용 한도 0~100% · 회원가입 적립금·최소 사용 포인트 0 이상 · 사용 단위·유효기간 **1 이상** |
| 적립률의 소유 | 상품마다 다른 적립은 **상품 폼**의 것이다. 여기 값은 새 상품의 초기값일 뿐이다 |
| 소급 적용 | **없다** — 기본 적립률을 바꿔도 이미 등록된 상품의 적립 설정은 움직이지 않는다 |
| 유효기간의 실행 | 소멸 규칙은 저장되지만 **실제로 소멸시키는 배치가 이 리포에 없다** — 서버의 몫이다 |
| 이력 | 정책 변경 이력을 남기지 않는다 |
| 파일 용량 제한 | 해당 없음 — 파일 입력이 없다 |
| 업로드 가능 확장자 | 해당 없음 — 같은 이유 |
