# 쿠폰 발급 현황 — 비기능 명세서

| 항목 | 내용 |
|---|---|
| 자원명 | 쿠폰 발급 현황 |
| 화면 ID | `SCR-PRODUCTS-COUPONS-ISSUANCE` |
| 메뉴 경로 | 상품 관리 > 쿠폰 > `발급 현황` — **사이드바에 잎이 없다** |
| 작성자 | Admin Hub 문서화 |
| 작성일 | 2026-07-22 |
| 버전 | 1.0 |
| 관련 기능 명세서 | [발급 현황](../../FSD/products/coupons/issuance.md) · (부모) [쿠폰 목록](../../FSD/products/coupons/index.md) |

**이 자원의 성격** — **잎이 없는 보조 목록**이자 완전한 읽기 전용 화면이다. 자기 리소스 키도 사이드바 자리도 없고 권한과 진입 경로를 부모(`/products/coupons`)에서 물려받는다. 그래서 비기능 요구가 ① 부모와의 조회 공유 ② **두 숫자가 무엇을 세는지 밝히는 일** ③ 잎 없는 라우트의 좌표 문제에 몰린다. 아래 각 절은 [공통 기준](../_common.md) 대비 이 자원만의 차이를 적는다.

| 요약 표의 크기 | — | 쿠폰 수만큼 행이 선다(발급 수와 무관) | 코드 확인 |

## 2. 사용성

| 항목 | 요구 | 이 자원의 값 |
|---|---|---|
| UI 일관성 | 공통 준수 | 이력 목록은 읽기 전용 공통 껍데기. ⚠ **요약 표는 손수 조립한 `<table>`** 이다(7열 · 캡션 포함) |
| 두 숫자가 무엇을 세는지 밝힌다 | — | 요약의 `발급`·`사용`·`사용률` 은 **이력 표본**에서 센 값이고 `소진율` 은 쿠폰의 **운영 누계**다. 픽스처 단계라 둘이 다르므로 캡션이 그 사실을 적는다 — 숫자가 어긋나 보이는 것을 버그로 오해하지 않게 — `CouponIssuanceListPage.tsx:8-10` |
| 사용률의 색이 판단을 돕는다 | — | 발급이 0이면 정보, **사용 0% 면 경고**(걸어 둔 의미가 없다), 50% 이상이면 성공. 색과 함께 숫자가 배지에 실린다 |
| 같은 열 이름, 다른 표기 | — | 요약의 '발급 기준' 은 **파라미터까지** 싣고(`VIP 승급 시`), 이력의 '발급 기준' 은 **종류 이름만** 싣는다. 요약은 정책을 구분해야 하고 이력은 유입 경로만 알면 된다 |
| 값이 없는 것과 모르는 것 | 공통 규약 5 준수 | 쿠폰을 못 찾으면 만료일이 `—` 다 — **날짜를 지어내지 않는다**. 발급이 0건인 쿠폰은 배지 대신 `발급 없음` |
| 삭제된 쿠폰을 감추지 않는다 | — | 쿠폰 이름 자리에 `삭제된 쿠폰 ({id})` 이 오고 **그 행은 사라지지 않는다** — 발급 사실은 남아야 한다 |
| 잎 없는 화면의 좌표 | — | 사이드바에 이 화면이 없어 **제목과 복귀 링크가 유일한 좌표**다. 그래서 화면이 제목을 스스로 말하고, 아래에 `쿠폰 목록으로 돌아가기` 를 둔다 |
| 결과 통지 | 공통 준수 | **없다** — 쓰기 동작이 없어 통지할 결과가 없다 |
| 접근성 — 요약 표 | — | 열 머리에 범위가 지정돼 있고 캡션이 두 숫자의 출처를 설명한다 |
| 미저장 보호 | 공통 준수 | 해당 없음 — 입력이 없다 |
