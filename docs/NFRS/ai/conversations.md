# 대화 목록 — 비기능 명세서

| 항목 | 내용 |
|---|---|
| 자원명 | 대화 목록 |
| 화면 ID | `SCR-AI-CONVERSATIONS` |
| 메뉴 경로 | AI > AI 에이전트 > 대화 목록 |
| 작성자 | Admin Hub 문서화 |
| 작성일 | 2026-07-22 |
| 버전 | 1.0 |
| 관련 기능 명세서 | [대화 목록](../../FSD/ai/conversations/index.md) |

**이 자원의 성격** — AI 에이전트 2화면 중 **유일하게 쓰기(삭제)를 갖는 화면**이다. 조회·저장소·플랜 축은 [새 채팅](chat.md) 과 같으므로 이 문서는 **이 자원만의 차이**를 적는다. 차이는 셋이다: **삭제가 있다 · 보관되지 않는다는 고지를 이 화면만 갖는다 · 조회 실패에 재시도 수단이 없다.**

## 1. 성능

| 항목 | 요구 | 이 자원의 값 | 측정 방법 |
|---|---|---|---|
| 조회 성능 | [새 채팅](chat.md) 준수 | 조회는 **하나**(대화 목록)이며 새 채팅 화면의 좌측 레일과 **같은 캐시를 공유한다** — 두 화면을 오가도 30초 안에는 요청이 생기지 않는다 — `apps/admin/src/pages/ai/queries.ts:20-26` | 코드 확인 |
| 목록 상한 | — | **페이지네이션이 없다.** 대화를 전부 한 화면에 그린다 — `apps/admin/src/pages/ai/ConversationsPage.tsx:159-179`. 메모리 저장이라 지금은 문제가 되지 않지만, 백엔드가 붙어 대화가 보관되면 상한이 필요해지는 자리다 | 코드 확인 |
| 삭제 요청 | — | 어댑터는 여러 건을 받을 수 있으나 화면은 **언제나 한 건만** 보낸다 — `ConversationsPage.tsx:100`. 연달아 누르면 앞 요청이 중단된다 — `ConversationsPage.tsx:96-98` | 코드 확인 |
| 응답 시간 | 공통 준수 | 측정하지 않음 — 백엔드가 없다 | 백엔드 연동 후 재작성 |

## 2. 사용성

| 항목 | 요구 | 이 자원의 값 |
|---|---|---|
| 전 항목 | [새 채팅](chat.md) 준수 | 같다 |
| 보관되지 않는다는 고지 | — | 목록 위에 상주 안내를 둔다: `대화는 브라우저 메모리에만 있어요 — 새로고침하면 사라져요. 보관은 백엔드 연동 후 가능해요.` — `apps/admin/src/pages/ai/ConversationsPage.tsx:137`. 밝히지 않으면 운영자는 대화가 보관된다고 믿고 **다음 날 없어진 것을 결함으로 본다** — `ConversationsPage.tsx:5-8` |
| ⚠ 고지의 자리 | — | **그 안내가 이 화면에만 있다.** 새 채팅 화면에서 한참 대화한 사람은 그 사실을 모른 채 새로고침으로 전부 잃을 수 있다 |
| 빈 상태의 생성 CTA | — | 통계·로그와 달리 **만들 수 있는 것**이라 빈 상태에 `새 채팅 시작` 버튼이 붙는다 — `ConversationsPage.tsx:147-155` |
| 파괴적 동작의 확인 | 공통 준수 | 삭제 전에 확인을 거치고, 되돌릴 수 없다는 사실을 본문이 말한다 — `ConversationsPage.tsx:187-188`. 이 다이얼로그는 **화면 고유**이며 공통 삭제 확인을 쓰지 않는다 |
| ⚠ 실패 시 다이얼로그의 거동 | 공통은 '닫지 않고 배너로' | **이 자원은 공통 규칙을 따르지 않는다** — 삭제에 실패하면 다이얼로그를 **닫고** 실패 토스트를 띄운다 — `ConversationsPage.tsx:111-114`. 공통 층은 다이얼로그를 열어 둔 채 배너로 이유를 말하고 확인 재클릭이 곧 재시도가 되게 한다([공통 FSD](../../FSD/_common/index.md)). 대신 이 화면은 실패 토스트에 재시도 동작을 붙여 그 경로를 메운다 |
| ⚠ 조회 실패의 복구 수단 | 공통은 '명시적 다시 시도 컨트롤' | **갖추지 못했다.** 목록 조회 실패 배너에 `다시 시도` 가 없어, 화면을 새로 고치는 것이 유일한 수단이다 — `ConversationsPage.tsx:141-142`. [품질 기준](../reference/quality-bar.md) STATE-02 미충족 |
| 결과 통지 | 공통 준수 | 토스트 2건(삭제 성공 1 · 실패 1). 실패 토스트는 자동으로 사라지지 않으며 재시도 동작을 함께 갖는다 — `ConversationsPage.tsx:113` |
| 취소는 실패가 아니다 | 공통 준수 | 취소·중단은 토스트를 만들지 않는다 — `ConversationsPage.tsx:110` |
| 시각 표기 | 공통은 서울 고정 | ⚠ **이 자원도 접속 기기의 현지 시각을 쓴다.** 저장값은 협정 세계시라 문자열을 잘라 쓰면 한국 시간 오전에 만든 대화가 전날 밤으로 보인다 — 그래서 파싱해 현지 값으로 조립한다 — `ConversationsPage.tsx:63-77` |

## 8. 제약사항

| 항목 | 내용 |
|---|---|
| 전 항목 | [새 채팅](chat.md) 준수 — 백엔드 없음 · 언어 모델 미연동 · 대화가 새로고침으로 사라짐 · 플랜(`AI 에이전트`, 최소 `엔터프라이즈` 티어) |
| 목록 기능 | **검색·필터·정렬·페이지네이션이 없다.** 정렬은 최근 갱신 순 고정이며 바꿀 수 없다. (대화 검색은 새 채팅 화면의 좌측 레일에만 있다) |
| 일괄 삭제 | 없다 — 체크박스도 선택 바도 없고 한 건씩만 지운다 |
| 삭제 권한 | **판정하지 않는다**(§3) |
| 소유자 구분 | 없다(§3) |
| 내보내기 | 없다 |
| 파일 용량 제한 · 업로드 확장자 | 해당 없음 — 파일 입력이 없다 |
