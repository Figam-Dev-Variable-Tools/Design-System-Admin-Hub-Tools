## 2. 사용성

| 항목 | 요구 | 이 자원의 값 |
|---|---|---|
| UI 일관성 | 공통 준수 | 목록은 공통 껍데기, 폼은 공통 컨트롤러. 파이프라인 스텝퍼는 디자인 시스템의 `Stepper` 를 쓴다 |
| 접근성 | 공통 준수 | **진행바는 장식이다**(`aria-hidden`) — 값은 바로 옆 숫자가 말한다. 색만으로 단계를 전달하지 않고 배지에 문구가 함께 실린다 — `apps/admin/src/pages/sales/projects/ProjectListPage.tsx:152-159` |
| 입력 편의성 | 공통 준수 | **단계를 바꾸면 확률이 그 단계 기본값으로 자동으로 채워진다**(사용자가 다시 조정할 수 있다) — `apps/admin/src/pages/sales/projects/ProjectFormPage.tsx:254-258`. 산출물은 줄바꿈으로 목록을 만든다 |

| 옛 데이터 고지 | 규칙 이전 데이터를 정상인 척하지 않는다 | 계약 없이 생긴 프로젝트에는 warning 배너가 뜬다. **지우지도 않는다** — 실제로 진행 중인 일이다 — `apps/admin/src/pages/sales/projects/ProjectFormPage.tsx:332-339` |

## 7. 유지보수성

| 항목 | 요구 | 이 자원의 값 |
|---|---|---|
| 코드 확장성 | 공통 준수 | 단계 메타(순서·기본확률·정상흐름 여부·색)를 데이터로 들고 있어 칸을 늘리기 쉽다. 스텝퍼는 그 데이터에서 파생된다(`PIPELINE_FLOW`) |
| 판정의 소유자 | 같은 판정을 두 곳에 적지 않는다 | '무엇이 계약 완료인가'는 **계약 도메인**이 갖는다(`isConcludedContract`) — 프로젝트가 계약 상태 유니온을 자기 방식으로 다시 읽으면 두 화면이 다른 '완료'를 갖게 된다 — `apps/admin/src/pages/sales/projects/types.ts:180-185` |
| 필요한 칸만 받는다 | 다른 도메인의 전부를 알지 않는다 | 계약에서 받아 오는 값을 `ProjectSourceContract` 로 좁혔다 — `Contract` 를 통째로 받으면 이 파일이 계약의 상태·서명·첨부·조항까지 알게 되고, 계약이 필드를 늘릴 때마다 프로젝트가 흔들린다 — `apps/admin/src/pages/sales/projects/types.ts:231-247` |
| 화면 확장성 | 열·필터·상태값이 늘 때 견디는가 | 조회 조건은 주소가 소유한다 |
| 공통 컴포넌트 사용 여부 | 페이지가 UI 를 손수 조립하지 않는다 | 목록의 **진행바는 디자인 시스템 컴포넌트가 아니다** — 트랙·채움을 토큰으로 조립한다(프로그램 상세도 같은 조합을 쓴다). 디자인 시스템에 진행바가 생기면 두 화면이 함께 옮겨 가야 한다 — `apps/admin/src/pages/programs/ProgramDetailPage.tsx:123-126` |
| 쓰지 않는 파생값 | 만들어 두고 안 쓰는 계산을 적어 둔다 | 마일스톤 완료율(`milestoneProgress`)이 순수 함수로 있으나 **목록도 폼도 그 값을 그리지 않는다.** 화면에 보이는 진척은 손으로 넣는 `진척률` 이다 — `apps/admin/src/pages/sales/projects/types.ts:117-122` |

## 8. 제약사항

| 항목 | 내용 |
|---|---|
| 외부 시스템 연계 | 없음. 일정·이슈 트래커 연동이 없다 |
| API 제약 | **백엔드가 없다.** 서버 계약은 §6 이 갖는다 |
| 등록 경로 | **없다.** `/sales/projects/new` 는 차단 안내 화면이며 어떤 권한으로도 폼이 열리지 않는다 |
| 상세 화면 | **없다.** 읽기 전용 표면이 없어 목록의 행 클릭이 수정 폼으로 간다(거래처 상세의 프로젝트 이력 링크도 같다) |
| 단계 전이 | **제약이 없다.** 리드에서 곧장 수주로, 수주에서 리드로도 갈 수 있다 |
| 마일스톤 수 상한 | 12개 — `apps/admin/src/pages/sales/projects/types.ts:61` |
| 문자 수 상한 | 프로젝트명 80자 · 산출물 1,000자 · 비고 500자 |
| 백분율 처리 | 확률·진척률은 저장 시 **100 에서 자른다**(음수는 숫자 추출 단계에서 성립하지 않는다) — `apps/admin/src/pages/sales/projects/ProjectFormPage.tsx:163` |
| 계약 연결 | `contractId` 가 빈 값이면 '계약 없이 생긴 옛 프로젝트'다. 사람이 채우는 칸이 아니라 폼이 값을 보존만 한다 |
| 파일 업로드 | 해당 없음 — 이 자원에 파일 입력이 없다 |
