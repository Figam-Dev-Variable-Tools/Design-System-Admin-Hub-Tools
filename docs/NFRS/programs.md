## 2. 사용성

| 항목 | 요구 | 이 자원의 값 |
|---|---|---|
| UI 일관성 | 공통 준수 — 디자인 시스템 컴포넌트만 사용 | 목록은 공통 목록 껍데기, 폼은 공통 폼 컨트롤러를 쓴다. ⚠ **상세와 폼의 진행바는 디자인 시스템 컴포넌트가 아니다** — 트랙·채움을 토큰으로 조립한다(영업 프로젝트 목록도 같은 조합을 쓴다) — `apps/admin/src/pages/programs/ProgramDetailPage.tsx:123-126` |
| 접근성 | 공통 준수 | 진행바가 `role="progressbar"` 와 값 문구를 갖되 **같은 사실이 바로 위에 글자로도 있다**. 폼의 구획 목차는 오류가 남은 구획에 붉은 점을 달아 '어디를 봐야 하는지'를 색 아닌 위치로도 말한다 |
| 색만으로 말하지 않기 | 공통 준수 | 목록의 달성 배지는 색과 함께 `{달성률}% · {모금액}원` 을 문구로 싣는다 — `apps/admin/src/pages/programs/types.ts:167-169` |
| 입력 편의성 | 공통 준수 | 폼에서 **단계가 아니라 구획**을 오간다(목차 클릭 → 스크롤). 카테고리는 대분류를 고르면 중분류 후보가 좁혀지고, 중분류가 없으면 그 셀렉트가 잠기며 이유가 힌트로 뜬다 |
| 사용자 피드백 | 공통 준수 | 이 자원에는 **자기 토스트가 없다**([인벤토리](../FSD/inventory.md) §9.3 에 세 화면 모두 없다). 저장·삭제 통지는 공통 층(`MSG-COMMON-01`~`03`)이 낸다 |
| 잠금 안내의 톤 | 잠금은 오류가 아니다 | 결제 미연동으로 금액이 잠길 때 **info 톤**을 쓴다 — danger 로 칠하면 운영자가 고장으로 읽는다 — `apps/admin/src/shared/commerce/PgLockNotice.tsx:30-32` |
| 미입력을 정보로 다룬다 | 빈 칸을 빈 칸으로 두지 않는다 | 폼 미리보기가 `프로그램명 미입력` · `한 줄 소개 미입력` · `—` 을 그린다 — **'무엇이 아직 안 채워졌는지'가 곧 정보다** — `apps/admin/src/pages/programs/ProgramFormPage.tsx:1355` |

| 입력값 검증 | 공통 준수(스키마 하나가 정본) | `apps/admin/src/pages/programs/validation.ts`. **리치텍스트 상한을 마크업이 아니라 평문 길이에 건다** — 값의 길이로 재면 '굵게' 한 번에 태그가 붙어 쓰지도 않은 글자 수로 제출이 막힌다. 카운터와 같은 함수로 판정해야 화면 표기와 어긋나지 않는다 — `apps/admin/src/pages/programs/validation.ts:43-45` |

## 4. 안정성

| 항목 | 요구 | 이 자원의 값 |
|---|---|---|
| 예외 처리 | 공통 준수 | 상세·폼 모두 조회 실패를 **없음(404)과 오류로 가른다** — 삭제된 프로그램에 '다시 시도'를 권하지 않는다 — `apps/admin/src/pages/programs/ProgramDetailPage.tsx:303-325` |
| 부분 실패 | 한 조회의 실패가 화면 전체를 무너뜨리지 않는다 | 목록에서 **카테고리 조회만 실패하면 그 필터 패널에만 재시도가 붙고** 표는 그대로 그려진다 — `apps/admin/src/pages/programs/ProgramListPage.tsx:350-351` |
| 파생 잠금의 즉시 반영 | 설정 변경이 다음 렌더에 반영된다 | 금액 잠금과 후원 CTA 를 **저장하지 않는다.** 프로그램마다 들고 있으면 설정 스위치를 내리는 순간 이미 등록된 수백 건이 전부 낡은 값이 된다 — `apps/admin/src/shared/commerce/price-display.ts:27-29` |
| ⚠ 잠금이 값을 지우지 않는다 | 되돌릴 수 없는 손실을 만들지 않는다 | 결제 미연동으로 후원 금액 입력이 잠겨도 **저장된 금액은 그대로 남고** 연동을 마치면 살아난다 — `apps/admin/src/shared/commerce/price-display.ts:54-57` |
| 재시도 | 공통 준수(자동 재시도 없음) | 목록·카테고리·상세에 각각 `다시 시도` 가 있다. **다만 '없는 프로그램'에는 주지 않는다** |
| 동시 저장 | 공통 준수(last-write-wins + 충돌 다이얼로그) | 폼은 공통 충돌 다이얼로그를 쓴다 |
| 삭제의 파급 | 지운 뒤 남는 참조를 감춰서는 안 된다 | ⚠ **프로그램을 지워도 그 프로그램을 가리키던 프로그램 문의의 `programId`·`programName` 은 남는다.** 정리하는 코드가 없다. **이미 후원이 들어온 프로그램도 지워진다**(리워드 행 삭제는 막지만 프로그램 삭제는 막지 않는다) |
| 로그 기록 | 공통 준수 | **확인하지 못했다** — 등록·수정·삭제를 관리자 로그에 남기는 배선을 찾지 못했다 |
