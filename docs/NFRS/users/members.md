**이 자원의 성격** — **개인정보가 가장 많이 모이는 화면**이다. 이름·연락처·주소·생년월일·IP 가 한자리에 있고, CSV 내보내기가 그것을 파일로 반출하며, 삭제는 되돌릴 수 없고, 알림 발송은 회원에게 실제로 나간다. 그래서 이 자원의 비기능 요구는 ① **액션별로 갈린 권한** ② **개인정보 취급** ③ **돈이 걸린 조작의 멱등성** 셋에 집중된다.

아래 각 절은 [공통 기준](../_common.md) 대비 **이 자원만의 차이**를 적는다.

## 4. 안정성

| 항목 | 요구 | 이 자원의 값 |
|---|---|---|
| 예외 처리 | 공통 준수 | 셸 안쪽 경계가 덮는다 |
| 장애 대응 | 공통 준수 | **실패가 화면을 통째로 비우지 않는다**: 목록 조회 실패는 표만 배너로 대체하고 좌측 필터는 남으며, 그룹 조회 실패는 그 축에만 재시도를 연다. 상세 조회 실패만 카드 전체를 대체한다 |
| 재시도 | 공통 준수(자동 재시도 없음) | 조회 실패 배너에 `다시 시도` 가 있고, 실패 토스트에 '다시 시도'가 붙는다. 확인 다이얼로그는 **닫지 않고** 확인 재클릭이 곧 재시도다 |
| ⚠ 404 의 복구 수단 | 공통은 '없는 것과 고장난 것을 다르게 말한다' | **상세가 문구로는 가르지만 버튼은 같다** — `회원을 찾을 수 없어요.` 에도 `다시 시도` 가 남아 있어, 이미 지워진 회원의 링크를 열면 영원히 실패하는 버튼을 보게 된다 — `MemberDetailPage.tsx:246-262`. 운영자 상세는 같은 상황에서 그 버튼을 감춘다 |
| 부분 실패 | — | 일괄 삭제·일괄 발송은 **성공/실패 건수로 말한다.** 삭제는 **전원 성공일 때만** 목록을 무효화하고 선택을 해제한다 — `queries.ts:104-114` |
| 유령 저장 금지 | 공통 준수 | 삭제가 한 건이라도 실패하면 다이얼로그가 열린 채 남고 목록을 갱신하지 않는다 |

| ⚠ 저장되지 않는 쓰기 | — | **쓰기 계열이 지금 아무것도 저장하지 않는다** — 어댑터가 resolve 만 하고 화면이 성공을 알린다. 적립금·메모·비밀번호 변경 결과는 새로고침하면 사라진다 — `data-source.ts:8-9,311,316`. 회원 그룹(`type='member'`) 생성도 마찬가지로 목록이 늘지 않는다 — `data-source.ts:198` |
