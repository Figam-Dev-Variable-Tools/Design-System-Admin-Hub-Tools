**이 자원의 성격** — **권한을 사람에게 배정하는 화면**이다. 권한 관리가 '무엇을 할 수 있는가'를 정하고, 여기가 '누가 그 역할인가'를 정한다. 그래서 비기능 요구가 **잠금(lock-out) 방지**에 집중된다: 자기 자신 삭제 · 마지막 시스템 관리자 제거 · 자기 권한 박탈 셋을 가드가 막고, 그 가드는 **화면과 어댑터 양쪽에** 있다. 그리고 운영진 그룹이 마케팅의 **발신 프로필**과 같은 실체라 삭제가 다른 자원으로 번진다.

아래 각 절은 [공통 기준](../_common.md) 대비 **이 자원만의 차이**를 적는다.

## 3. 보안

| 항목 | 요구 | 이 자원의 값 |
|---|---|---|
| 인증 | 공통 준수 | 셸 안쪽이라 세션이 없으면 렌더되지 않는다 |
| 권한 | 공통 준수 | 목록의 등록 CTA 는 `create`, 상세의 수정은 `update`, 상세의 삭제는 `remove` 다. **폼 라우트는 껍데기가 게이팅한다** — 등록은 `create`, 수정은 `update` 를 요구하며 URL 로 들어와도 403 화면이 뜬다 |
| 폼 게이팅을 껍데기가 소유하는 이유 | — | 폼이 25개고 계속 늘어난다. 화면마다 적게 하면 **다음에 추가되는 폼이 그것을 빠뜨린다** — 목록의 등록 CTA 가 실제로 12개 화면에서 빠져 있었다. 껍데기가 소유하면 새 폼은 껍데기를 쓰는 것만으로 게이팅된 채 태어난다 — `apps/admin/src/shared/crud/FormPageShell.tsx:120-127` |
| ⚠ 게이팅 누락 | — | **그룹 만들기와 그룹 삭제에 권한 판정이 없다.** 이 화면이 읽는 쓰기 권한은 `create` 하나이며 그것도 등록 CTA 에만 쓰인다 — `AdminsPage.tsx:121`. 그룹 삭제는 **메시지 템플릿의 발신 프로필까지 함께 지우는** 동작이라, 읽기만 가진 역할이 다른 자원의 발송 설정을 무너뜨릴 수 있다 |
| 잠금(lock-out) 방지 | — | **이 자원의 핵심 보안 요구다.** 셋을 막는다: ① 자기 자신 삭제 — 지우는 순간 세션의 주체가 사라지고 되돌릴 수단은 남의 계정으로 다시 만드는 것뿐이다 ② 마지막 시스템 관리자 삭제·강등 — 역할은 남지만 그것을 배정할 수 있는 사람이 없어진다 ③ 자기 역할 변경 — 저장이 끝나는 순간 이 화면의 수정 권한이 사라져 되돌릴 수 없다 — `apps/admin/src/pages/admins/guards.ts:3-17` |
| 가드가 양쪽에 있는 이유 | — | **중복이 아니라 역할 분담이다.** 화면은 누르기 전에 이유를 말하려고 가드를 보고, 어댑터(백엔드 자리)는 거절의 책임을 진다 — 확인과 실제 요청 사이에 다른 탭의 관리자가 유일한 시스템 관리자를 강등할 수 있다. **문구는 양쪽 모두 같은 곳에서 온다** — 경로에 따라 다른 문장을 말하지 않기 위해서다 — `data-source.ts:267-274` |
| 자기 자신 판정의 근거 | — | **폼 값이 아니라 서버 원본**으로 판정한다 — 폼의 계정 칸을 남의 이메일로 고쳐 적는 것만으로 잠금이 풀리면, 그 뒤 자기 역할을 마음대로 올릴 수 있다 — `AdminFormPage.tsx:132-139` |
| 자기 자신 판정의 한계 | — | 세션에 **운영자 id 가 없어** 계정(이메일)으로 맞춘다. 대소문자·앞뒤 공백을 무시한다 — 그대로 비교하면 대문자 하나로 이 가드가 통째로 빠져나간다 — `guards.ts:49-61`. 세션에 id 가 실리면 이 가정이 사라진다고 표시되어 있다 |
| 모르면 막는다 | 공통 준수(fail-closed) | 명부를 못 읽으면 삭제를 **막는다** — 모르는 것을 '막을 이유 없음'으로 읽으면 마지막 시스템 관리자가 조용히 지워진다 — `apps/admin/src/pages/admins/useAdminGuardContext.ts:9-11`. 그룹의 발신 사용처를 모를 때도 삭제를 막는다 — `data-source.ts:217-227` |
| 참조 무결성 | — | 운영자가 남아 있는 그룹, 발신 프로필로 쓰이는 그룹은 삭제가 거절된다. **'그룹 없음' 상태를 만들려고 타입을 넓히지 않는다** — 그룹이 곧 권한 묶음이라 그룹 없는 운영자는 아무것도 할 수 없는 계정이다 — `apps/admin/src/shared/domain/admin-group.ts:66-81` |
| 계정 중복 판정의 자리 | — | 폼 스키마가 아니라 **명부를 쥔 쪽**이 한다 — 화면이 든 목록으로 판정하면 다른 탭에서 방금 만들어진 계정을 통과시킨다. 422 로 돌아와 계정 칸의 인라인 오류가 된다 — `apps/admin/src/pages/admins/validation.ts:97-99` |
| 역할 목록을 복제하지 않는다 | — | 폼의 셀렉트와 목록의 필터가 **권한 스토어를 그대로 읽는다.** 복제하면 새로 만든 역할이 뜨지 않거나 **지운 역할이 남아 배정 가능해진다** — `AdminFormPage.tsx:9-10` |
