# 로그인 — 비기능 명세서

| 항목 | 내용 |
|---|---|
| 자원명 | 로그인 |
| 화면 ID | `SCR-LOGIN` |
| 메뉴 경로 | 해당 없음 — 사이드바(셸) 밖의 단독 화면 |
| 작성자 | Admin Hub 문서화 |
| 작성일 | 2026-07-22 |
| 버전 | 1.0 |
| 관련 기능 명세서 | [로그인](../FSD/login/index.md) |

**이 자원의 성격** — 인증 **이전**의 유일한 화면이다. 그래서 다른 자원 문서가 전제하는 것들(셸·권한 축·플랜 축·공통 CRUD 층)이 여기에는 하나도 걸리지 않는다. 아래 각 절은 [공통 기준](_common.md) 대비 **이 화면만의 차이**를 적는다.

## 1. 성능

| 항목 | 요구 | 이 자원의 값 | 측정 방법 |

| 첫 진입 비용 | 공통 준수(라우트 단위 코드 분할) | **일부러 분할하지 않는다.** 이 화면은 진입 화면이라, 쪼개면 첫 페인트 앞에 왕복이 하나 더 붙는다 — `apps/admin/src/App.tsx:55-58` | 코드 확인 |

## 2. 사용성

| 항목 | 요구 | 이 자원의 값 |
|---|---|---|
| UI 일관성 | 공통 준수 — 디자인 시스템 컴포넌트만 사용 | 입력·비밀번호·체크박스·배너·버튼 다섯이 전부 디자인 시스템 것이다. 이 화면에 남은 것은 **좌우 분할 레이아웃과 한국어 라벨**뿐이며, 그래서 폼 자체는 디자인 시스템으로 올리지 않기로 판정됐다(올리면 제품 카피와 '로그인 = 이메일+비밀번호+기억하기'라는 제품 결정을 디자인 시스템이 소유하게 된다) — `apps/admin/src/pages/login/components/LoginForm.tsx:12-31` |
| 접근성 — 건너뛰기 | 공통 준수 | **해당 없음** — 셸 밖이라 앞에 놓인 큰 내비게이션이 없다 |
| 접근성 — 장식 제외 | 공통 준수 | 왼쪽 비주얼 패널 전체가 보조기술에서 제외된다 — `LoginPage.tsx:379` |
| 접근성 — 오류 전달 | 공통 준수(라이브 영역) | 이 화면은 토스트를 쓰지 않는다. 대신 **오류 배너로 포커스를 옮긴다** — 키보드·스크린리더 사용자에게 즉시 전달된다 — `LoginPage.tsx:252-257` |
| 접근성 — 검증 실패 | 공통 준수(첫 위반 필드로 포커스) | 순서는 이메일 → 비밀번호다. 폼 라이브러리의 자동 포커스를 끄고 직접 옮기는데, 필드가 제어 컴포넌트라 라이브러리가 입력 참조를 갖고 있지 않기 때문이다 — `LoginPage.tsx:357-368` |
| 접근성 — 로딩 상태 | 공통 준수 | 로딩이 끝났을 때 `aria-busy="false"` 가 **관측 가능해야 한다**는 계약이 이 화면에 있어 값을 명시한다 — `LoginForm.tsx:186-193` |
| 잠금 상태 전달 | — | 상태를 색으로만 말하지 않는다: 배너 문장 + 제출 버튼 비활성 둘 다 바뀐다 |
| 입력 편의성 | 공통 준수 | 자동완성 힌트가 붙는다(이메일=계정, 비밀번호=현재 비밀번호). 비밀번호 표시 토글이 있다. **이메일 길이를 입력에서 자르지 않는다** — 초과도 문구로 안내해야 하기 때문이다 — `LoginPage.tsx:32-33` |
| 입력 편의성 — 비밀번호 공백 | — | 앞뒤 공백을 다듬지 않는다. 공백도 유효한 비밀번호 문자다 — `apps/admin/src/pages/login/validation.ts:42-43` |
| 사용자 피드백 | 공통 준수 | **이 화면에는 결과 통지 토스트가 없다.** 성공은 화면 이동으로, 실패는 폼 안 배너로 말한다 |
| 재방문 편의 | — | '이메일 저장'은 이메일만 채워 주고, '로그인 유지'는 세션을 남긴다. 두 축을 붙여 배치해 무엇이 무엇의 짝인지 읽히게 한다 — `LoginForm.tsx:172-176` |

## 8. 제약사항

| 항목 | 내용 |
|---|---|
| 외부 시스템 연계 | 없다. 소셜 로그인(OAuth)은 시스템 설정에 별도 화면이 있으나, **이 화면에는 소셜 로그인 버튼이 없다**(코드에서 확인) |
| API 제약 | 인증 API 가 아직 없다. 지금 동작하는 것은 브라우저 안 mock 이며, 실패 카운트·잠금·타임아웃 재현까지 그 안에서 이루어진다. 계정별 실패 횟수는 **새로고침하면 초기화된다** — `apps/admin/src/pages/login/api.ts:100-101` |
| 계정 제약 | 이메일 최대 254자 · 비밀번호 8~64자 — `apps/admin/src/pages/login/validation.ts:16-19`. 이 값은 서버 제약과 짝을 이루도록 정해졌다 |
| 잠금 정책 | 5회 실패 · 30분 잠금. 잠금 해제를 앞당기는 화면 수단이 없다(시간이 지나거나 시스템 관리자에게 문의) |
| 파일 용량 제한 | 해당 없음 — 이 화면에 파일 입력이 없다 |
| 업로드 가능 확장자 | 해당 없음 — 같은 이유 |
| 비밀번호 재설정 | **이 화면에 재설정 경로가 없다.** '비밀번호를 잊으셨나요' 류의 링크가 코드에 없다 — 잠금·비활성 안내가 전부 '시스템 관리자에게 문의'로 끝난다 |
| 로그아웃 | **어드민에 로그아웃 컨트롤이 없다**(코드 전수 검색 결과 사용자 대면 로그아웃 UI 0건). 세션을 버리는 경로는 401 감지 하나뿐이다 — `apps/admin/src/pages/login/session.ts:102-118` |
