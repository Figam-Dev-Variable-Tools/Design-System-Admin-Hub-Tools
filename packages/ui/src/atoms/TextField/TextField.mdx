# TextField

라벨 + 단일행 입력 + 인라인 에러 메시지를 함께 렌더하는 제어 입력 컴포넌트입니다.

## 언제 사용하나요?

- 이메일·이름·수량 등 한 줄 텍스트를 입력받을 때
- 라벨과 입력을 항상 짝으로 두어야 할 때
- 검증 실패 메시지를 입력 바로 아래에 보여줄 때
- 비밀번호·숫자 등 다양한 input type 이 필요할 때

## Props

<Controls />

| Prop          | Description                                                                       |
| ------------- | -------------------------------------------------------------------------------- |
| `label`       | 가시 라벨입니다. placeholder 로 대체하지 않습니다.                               |
| `type`        | input 종류입니다 (`text` · `email` · `password` · `number`).                     |
| `error`       | 위반 메시지입니다. 비어 있지 않으면 error 상태로 바뀝니다.                        |
| `required`    | 필수 입력 여부입니다.                                                             |
| `placeholder` | 입력 예시 텍스트입니다. 라벨을 대체하지 않습니다.                                 |
| `disabled`    | 비활성 여부입니다. 입력·포커스·`onBlur` 가 막힙니다.                              |

## Behavior

- `error` 가 비어 있지 않으면 테두리가 danger 로 바뀌고 메시지가 입력에 연결됩니다.
- `disabled` 면 입력과 포커스가 막혀 `onBlur` 가 발화하지 않습니다.

## Accessibility

- 라벨이 `htmlFor` 로 입력에 연결되어 접근 가능한 이름이 됩니다.
- 에러는 `aria-invalid` 와 `aria-describedby` 로 안내되고, 색상만이 아니라 텍스트로도 전달됩니다.
- `Tab`·`Shift+Tab` 으로 이동하고 표준 텍스트 편집 키를 지원합니다.

## Notes

- 여러 필드의 검증·표식 관리(필수 마커 등)는 상위 FormField 껍데기가 담당합니다.
