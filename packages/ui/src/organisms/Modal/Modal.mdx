# Modal

화면 위에 띄워 확인·입력을 받는 대화 상자(다이얼로그)입니다.

## 언제 사용하나요?

- 삭제·이탈처럼 되돌리기 어려운 작업을 확인받을 때
- 짧은 폼(이름 입력 등)을 흐름을 벗어나지 않고 받을 때
- 약관·상세처럼 별도 화면 없이 겹쳐 보여줄 때

## Props

<Controls />

| Prop              | Description                                                       |
| ----------------- | ---------------------------------------------------------------- |
| `title`           | 다이얼로그 제목. 접근성 이름으로도 쓰입니다.                     |
| `icon`            | 제목 왼쪽 아이콘 슬롯. 없으면 제목만 표시됩니다.                 |
| `children`        | 본문 슬롯. 확인 문구·폼 필드 등을 넣습니다.                      |
| `footer`          | 푸터 슬롯. 오른쪽 정렬된 액션 버튼(취소·확인)을 넣습니다.        |
| `onClose`         | 닫기가 요청될 때 호출됩니다(Escape·배경 클릭·닫기 버튼).         |
| `onSubmit`        | 제공하면 본문·푸터가 `<form>` 으로 감싸져 Enter 로 확인됩니다.   |
| `initialFocusRef` | 열릴 때 포커스를 받을 요소. 없으면 첫 포커스 가능 요소로 갑니다. |
| `describedBy`     | 본문 요소 id 를 연결하면 열릴 때 제목과 함께 읽힙니다.           |

## Behavior

- `Escape` · 배경(딤) 클릭 · 닫기 버튼(×) 이 `onClose` 를 호출합니다.
- 열려 있는 동안 배경 스크롤이 잠깁니다.
- 닫기에는 퇴장 애니메이션이 붙고, 부모가 언마운트하면서 실제로 사라집니다.

## Accessibility

- `role="dialog"` · `aria-modal` 로 배경을 보조기술에서 격리합니다.
- 제목은 `aria-labelledby` 로 접근성 이름이 됩니다.
- `Tab` / `Shift+Tab` 은 다이얼로그 안에서 순환합니다(포커스 트랩).
- 열릴 때 지정 요소로 포커스가 가고, 닫히면 열기 직전 요소로 복귀합니다.

## Notes

- 여러 모달을 겹칠 수 있으며, 닫기 동작은 항상 최상단 모달에만 전달됩니다.
