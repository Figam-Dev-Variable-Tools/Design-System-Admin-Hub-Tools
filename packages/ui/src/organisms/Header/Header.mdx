# Header

콘텐츠 영역 맨 위에서 화면 제목(`<h1>`)과 눈썹 문구, 우측 액션·메타 영역을 담는 상단 바입니다.

## 언제 사용하나요?

- 화면 상단에 그 화면의 제목을 보여줄 때
- 제목 위에 경로·브랜드·역할 같은 눈썹 문구를 함께 둘 때
- 우측에 오늘 날짜·로그인 계정 같은 메타 정보를 놓을 때
- 우측에 내보내기·추가 같은 화면의 주요 액션 버튼을 놓을 때

## Props

<Controls />

| Prop      | Description                                                        |
| --------- | ----------------------------------------------------------------- |
| `title`   | 화면 이름(필수). `<h1>` 으로 렌더됩니다.                          |
| `eyebrow` | 제목 위 작은 문구(경로·브랜드·역할). 빈 문자열이면 그리지 않습니다. |
| `meta`    | 우측 슬롯. 날짜·계정 같은 정보나 버튼 같은 액션을 넣습니다.        |

## Accessibility

- 네이티브 `<header>` 가 `banner` 랜드마크가 됩니다.
- 제목은 화면당 하나뿐인 `<h1>` 입니다. 화면이 자기 `<h1>` 을 또 그리지 마세요.
- 눈썹은 제목이 아니라 `<p>` 이므로 제목 계층에 끼지 않습니다.
- 헤더 자체에는 상호작용 요소가 없습니다. 탭 순서에 끼는 것은 `meta` 슬롯에 넣은 요소뿐이며, 그 키보드 계약은 넣은 쪽이 집니다.

## Notes

- 제목 문자열·날짜·로그인 계정은 앱이 정합니다. Header 는 받아서 배치만 합니다.
