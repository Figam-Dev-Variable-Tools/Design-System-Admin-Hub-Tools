# ConfirmDialog

확인/취소를 묻는 모달입니다 — 생성·수정·삭제 등 되돌리기 어려운 작업을 실행 전에 한 번 더 확인합니다.

## 언제 사용하나요?

- 삭제처럼 되돌릴 수 없는 작업을 실행하기 전에
- 새 항목을 만들거나 변경 사항을 저장하기 전에
- 저장하지 않은 내용을 두고 화면을 벗어나려 할 때
- 확인 요청을 처리하는 동안 상태(진행 중·오류)를 보여줄 때

## Props

<Controls />

| Prop           | Description                                                                     |
| -------------- | ------------------------------------------------------------------------------ |
| `intent`       | 확인의 의도(`create`·`update`·`delete`·`discard`). 톤·기본 라벨·아이콘을 함께 정합니다. |
| `title`        | 다이얼로그 제목입니다.                                                          |
| `message`      | 무엇을 확인하는지 사람이 읽는 문장입니다.                                       |
| `confirmLabel` | 확인 버튼 라벨입니다. 비우면 `intent` 의 기본 라벨을 씁니다.                    |
| `cancelLabel`  | 취소 버튼 라벨입니다. 기본값은 '취소'.                                          |
| `busy`         | 확인 처리 중이면 `true`. 확인 버튼을 잠그고 라벨을 '처리 중…' 으로 바꿉니다.    |
| `error`        | 값이 있으면 본문 아래에 오류 배너로 표시됩니다.                                 |
| `onConfirm`    | 확인 버튼을 눌렀을 때 호출됩니다.                                               |
| `onCancel`     | 취소·Escape·딤 클릭으로 닫으려 할 때 호출됩니다.                                |

`intent` 별 기본값:

| intent    | 기본 라벨 | 톤      |
| --------- | --------- | ------- |
| `create`  | 만들기    | primary |
| `update`  | 저장      | primary |
| `delete`  | 삭제      | danger  |
| `discard` | 나가기    | danger  |

## Behavior

- 확인 버튼 → `onConfirm`, 취소 버튼·Escape → `onCancel` 을 호출합니다.
- `busy` 중에는 확인 버튼이 잠기고 라벨이 '처리 중…' 으로 바뀌어 중복 클릭을 막습니다. 취소·Escape·딤은 계속 동작합니다.
- `error` 가 있어도 다이얼로그는 닫히지 않습니다 — 확인 버튼이 살아 있어 재클릭이 곧 재시도입니다.

## Accessibility

- 열리면 포커스가 다이얼로그 안으로 들어가고 배경은 격리됩니다(포커스 트랩).
- `Tab`/`Shift+Tab` 으로 순환, `Enter` 로 확인, `Escape` 로 취소할 수 있습니다.
- 열릴 때 제목과 확인 문구가 함께 안내됩니다.

## Notes

- 취소 시 '작업이 취소되었습니다' 같은 토스트는 앱에서 얹습니다 — ConfirmDialog 는 `onCancel` 을 부르기만 합니다.
