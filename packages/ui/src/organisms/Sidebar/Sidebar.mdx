# Sidebar

좌측 내비게이션 — 메뉴 그룹과 활성 항목을 보여주는 고정 사이드바입니다.

## 언제 사용하나요?

- 화면 좌측에 주 내비게이션을 고정해 둘 때
- 메뉴를 섹션 그룹으로 묶어 보여줄 때
- 하위 메뉴를 가진 항목을 펼치고 접을 때
- 지금 보고 있는 화면을 활성 항목으로 표시할 때

## Props

<Controls />

| Prop         | Description                                                              |
| ------------ | ----------------------------------------------------------------------- |
| `label`      | `<nav>` 의 접근성 이름입니다. 랜드마크가 여럿일 때 서로 구분됩니다.       |
| `sections`   | 화면에 그릴 섹션 목록입니다. `{ id, title, items }` 모양입니다.           |
| `activeHref` | 지금 켜져야 할 링크의 href 입니다. 빈 문자열이면 아무것도 켜지 않습니다.  |
| `openId`     | 펼쳐진 가지의 id 입니다. 호출부가 소유하는 제어 값입니다.                 |
| `brand`      | 상단 브랜드/로고 슬롯입니다.                                             |
| `onNavigate` | 링크 이동 의사를 알립니다. 인자로 href 를 받습니다.                      |
| `onToggle`   | 가지를 여닫으려는 의사를 알립니다. 인자로 가지 id 를 받습니다.           |

`items[]` 는 `{ id, label, href?, icon?, children? }` 모양입니다. `href` 를 가지면 잎(링크),
`children` 을 가지면 가지(펼침 버튼)입니다.

## Behavior

- **항목 이동** — 진짜 `<a href>` 를 그립니다. 가운데 클릭·새 탭 열기가 그대로 동작하고,
  수식키 없는 왼쪽 클릭에서만 `onNavigate(href)` 를 올립니다. `onNavigate` 가 없으면 평범한
  링크로 동작합니다.
- **그룹 펼침** — 가지를 누르면 `onToggle(id)` 를 올릴 뿐, 여닫는 결과는 `openId` 로 다시
  주입받는 제어 방식입니다. 접힌 하위 링크는 DOM 에 두지 않습니다.

## Accessibility

- 바깥은 보조(`complementary`) 랜드마크, 안쪽 `<nav>` 가 `label` 로 이름을 갖습니다.
- 활성 링크에는 `aria-current="page"`, 가지에는 `aria-expanded` 가 붙습니다.
- 활성 표시는 색뿐 아니라 굵기·마커로도 나타나 색만으로 의미를 전달하지 않습니다.
- 키보드 포커스 시 포커스 링이 보이고, 접힌 하위 링크는 탭 순서에 남지 않습니다.

## Notes

- 브랜드/로고는 `brand` 슬롯으로 주입합니다. Sidebar 자체는 특정 로고를 담지 않습니다.
- 라우트·권한·활성 판정은 앱의 몫입니다. Sidebar 는 걸러진 데이터를 받아 그리기만 합니다.
