# Table

정렬·행 톤·선택·행 클릭 등을 갖춘 데이터 목록 표입니다. 셀 내용은 호출부가 만든 `ReactNode` 로 들어옵니다.

## 언제 사용하나요?

- 목록 화면에서 데이터를 행/열로 보여줄 때
- 열 기준 정렬이 필요할 때
- 행을 눌러 상세로 이동하는 목록일 때
- 행 상태를 색조(성공·경고·위험 등)로 강조할 때
- 체크박스·순번·액션 같은 열을 붙여야 할 때

## Props

<Controls />

| Prop                           | Description                                                                 |
| ------------------------------ | -------------------------------------------------------------------------- |
| `caption`                      | 표의 접근 가능한 이름. 시각적으로 숨겨집니다. (필수)                        |
| `columns`                      | 데이터 열 목록. `{ id, header, align?, nowrap?, sortable? }` (필수)         |
| `rows`                         | 본문 행 목록. `{ id, cells, selected?, tone?, onActivate? }` (필수)         |
| `leadingHead` / `trailingHead` | 데이터 열 앞/뒤에 붙는 완성된 `<th>` 배열 (체크박스·순번·액션 열 등)        |
| `sortKey` / `sortDirection`    | 현재 정렬 중인 열 id 와 방향(`asc`/`desc`). 판정 결과를 그대로 받습니다.    |
| `onSortToggle`                 | 정렬 헤더를 눌렀을 때 열 id 를 넘겨 호출됩니다. 없으면 정렬 버튼이 안 생김. |
| `loading` / `skeletonRows`     | 로딩 중이면 본문을 스켈레톤 행으로 대체합니다.                             |
| `empty`                        | 행이 없을 때 표시할 내용 슬롯.                                              |

행의 `tone` 은 `danger`·`warning`·`success`·`info` 를 지원합니다.

## Behavior

- **정렬**: `sortable` 열의 헤더만 버튼이 되며, 눌러도 표가 스스로 상태를 바꾸지 않고 `onSortToggle` 로 열 id 만 올립니다. 다음 정렬 상태는 호출부가 정합니다.
- **행 클릭**: `onActivate` 가 있는 행은 어디를 눌러도 활성화됩니다. 단, 셀 안의 링크·버튼·입력 요소에서 시작한 클릭과 텍스트 드래그 중의 클릭은 활성화로 치지 않습니다.

## Accessibility

- `<caption>` 은 시각적으로 숨기되 접근성 트리에 남아 표의 이름이 됩니다.
- 정렬 상태(`aria-sort`)는 버튼이 아니라 `<th>` 가 가집니다.
- 방향 표식(▲▼↕)은 `aria-hidden` 이라 중복 낭독되지 않습니다.
- 행 자체에는 `role`·`tabIndex` 를 씌우지 않아 표의 행·열 탐색이 유지됩니다. 키보드 사용자는 행 안의 링크·버튼으로 이동합니다.
- 로딩 중에는 `aria-busy="true"` 이며 스켈레톤 칸은 낭독되지 않습니다.

## Notes

- 정적 수치 표(천 단위 구분·합계 등)가 필요하면 `Table` 대신 `DataTable` 을 사용하세요.
- 가로 스크롤은 표가 아니라 감싸는 쪽에서 처리합니다.
