# TableReorder

표 행을 위/아래로 재정렬합니다 — 드래그 + 이동 버튼.

## 언제 사용하나요?

- FAQ·배너처럼 목록의 노출 순서를 사용자가 바꿔야 할 때
- 드래그(마우스)와 버튼(키보드) 두 경로를 함께 제공하고 싶을 때
- 저장 중에 순서 변경을 잠가야 할 때

## Props

이 컴포넌트는 단일 컴포넌트가 아니라 훅 + 표 조각 묶음입니다. 표 조립에 필요한 것만 골라 씁니다.

<Controls />

| Export                                       | 역할                                                        |
| -------------------------------------------- | ----------------------------------------------------------- |
| `useReorderableRows(ids, onReorder, locked)` | 드래그 상태 훅. `{ rowProps, rowStyle, moveBy }` 를 돌려줍니다. |
| `moveArrayItem(items, from, to)`             | 한 항목을 옮긴 새 배열을 돌려주는 순수 함수.                |
| `ReorderGripHeaderCell` / `ReorderGripCell`  | 손잡이 열 헤더/셀.                                          |
| `ReorderMoveButtons`                         | 위/아래 이동 버튼 한 쌍. `onMove(index, delta)` 로 호출.    |

- `ids` — 현재 화면 순서의 id 목록
- `onReorder(orderedIds, movedId)` — 새 순서를 요청하는 콜백(저장·롤백은 호출부가 담당).
  `movedId` 는 이번에 움직인 행이라 "'A'를 3건 중 2번째로 옮겼습니다" 같은 낭독을 만들 수 있습니다.
  `(orderedIds) => …` 로만 받아도 됩니다.
- `locked` — 저장 중 드래그와 버튼을 전부 잠급니다

## Behavior

- 행을 드래그하거나 이동 버튼을 누르면 순서가 바뀝니다(둘 다 같은 연산으로 귀결).
- 첫 행은 '위로', 마지막 행은 '아래로' 가 잠깁니다.
- `locked` 이면 드래그와 모든 이동 버튼이 비활성화됩니다.

## Accessibility

- 드래그는 마우스 전용이라, 각 행에 위/아래 이동 버튼을 함께 제공합니다.
- 순서가 바뀐 사실은 **호출부가 라이브 영역으로 알립니다** — 표의 행 순서 변화만으로는 스크린리더에
  아무것도 전달되지 않습니다. `onReorder` 의 `movedId` 가 그 문장의 주어입니다.
- 이동 버튼은 대상 행 이름이 붙은 접근 가능한 이름을 갖습니다(색만으로 의미 전달하지 않음).
- 손잡이 열에는 보이지 않는 라벨이 있습니다.
- 논리 속성을 써서 RTL 에서 방향이 자연스럽게 뒤집힙니다.
