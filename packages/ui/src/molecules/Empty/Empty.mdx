# Empty

목록이 0행일 때 '왜 비었는지'와 '어떻게 벗어나는지'를 안내하는 빈 상태 컴포넌트입니다.

## 언제 사용하나요?

- 아직 등록된 항목이 없을 때 (진짜 비어있음)
- 검색 결과가 없을 때
- 필터 결과가 없을 때
- 사용자에게 다음 행동(추가·검색 지우기·필터 초기화)을 안내할 때

## Props

<Controls />

| Prop               | Description                                                                |
| ------------------ | -------------------------------------------------------------------------- |
| `label`            | 대상 명사('회원', '공지' 등). 조사('이/가')는 받침에 따라 자동 선택됩니다. |
| `createVerb`       | 진짜 비어있음 문구의 동사입니다. 기본값 `'등록'`.                          |
| `hasQuery`         | `true` 면 '검색 결과 없음'으로 분기합니다.                                 |
| `hasActiveFilters` | `true` 면 '필터 결과 없음'으로 분기합니다.                                 |
| `action`           | 진짜 비어있음 상태에서 표시할 생성 버튼 슬롯입니다.                        |
| `onClearSearch`    | 제공하면 검색 결과 없음 상태에 '검색 지우기' 버튼이 표시됩니다.            |
| `onResetFilters`   | 제공하면 필터 결과 없음 상태에 '필터 초기화' 버튼이 표시됩니다.            |

## Behavior

세 상태는 `hasQuery` / `hasActiveFilters` 조합으로 자동 분기합니다. 우선순위는 검색 > 필터 > 진짜 비어있음입니다.

| 상태           | 조건               | 복구 수단                        |
| -------------- | ------------------ | -------------------------------- |
| 진짜 비어있음  | 검색·필터 없음     | 생성 버튼 슬롯(`action`, 있으면) |
| 검색 결과 없음 | `hasQuery`         | '검색 지우기'(`onClearSearch`)   |
| 필터 결과 없음 | `hasActiveFilters` | '필터 초기화'(`onResetFilters`)  |

복구 콜백이 지정되지 않으면 해당 버튼은 그려지지 않습니다.

## Accessibility

- 빈 상태는 `role="status"` 라이브 영역으로 안내되어, 렌더 즉시 스크린리더가 사유를 읽습니다.
- 삽화 아이콘은 `aria-hidden` 으로 처리됩니다.
- 복구 버튼은 DS Button 이라 focus-visible 링·키보드 활성화를 상속합니다.
- RTL(`dir="rtl"`) 흐름을 지원합니다.
