# TextareaField

여러 줄 텍스트를 입력받고 우측에 글자수(`N/max`)를 보여 주는 폼 필드입니다.

## 언제 사용하나요?

- 공지 본문·FAQ 답변·상품 설명처럼 긴 글을 입력받을 때
- 입력 길이에 상한이 있어 남은 글자수를 안내해야 할 때
- 라벨·힌트·오류를 갖춘 폼 필드가 필요할 때

## Props

<Controls />

| Prop          | Description                                                            |
| ------------- | --------------------------------------------------------------------- |
| `label`       | 필드 레이블입니다.                                                    |
| `value`       | 입력값입니다(제어 컴포넌트). 이 길이가 카운터 분자로 쓰입니다.        |
| `maxLength`   | 최대 길이입니다. 입력 제한과 카운터 분모(`N/max`)를 함께 정합니다.    |
| `onChange`    | 값이 바뀌면 새 문자열을 넘겨 호출됩니다. `disabled` 면 발화하지 않습니다. |
| `required`    | 필수 필드로 표시하고 접근성에 필수 여부를 잇습니다.                   |
| `disabled`    | 입력을 잠급니다.                                                      |
| `error`       | 인라인 오류 메시지입니다. 값이 있으면 오류 상태가 됩니다.             |
| `hint`        | 보조 안내입니다. 오류가 없을 때만 표시됩니다.                         |
| `placeholder` | 예시 텍스트입니다.                                                    |
| `rows`        | textarea 표시 행 수입니다(기본 8).                                    |

## Behavior

- 글자수 카운터는 `value.length / maxLength` 형식으로 우측에 표시됩니다.
- `disabled` 면 입력이 잠기고 `onChange` 도 발화하지 않습니다.
- `error` 가 있으면 힌트 대신 오류가 표시되고 테두리가 danger 로 바뀝니다.

## Accessibility

- 라벨은 입력과 연결되어 있습니다(`htmlFor`).
- 오류가 있으면 `aria-invalid` 와 오류 메시지 연결이 함께 적용됩니다.
- `required` 는 시각 표식과 별개로 입력 자체에도 필수 여부를 잇습니다.
- `Tab` · `Shift+Tab` 으로 포커스가 이동하며 포커스 링이 보입니다.
